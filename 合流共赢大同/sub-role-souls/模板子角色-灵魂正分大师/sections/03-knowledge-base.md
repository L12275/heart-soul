03-知识库（Knowledge Base）

> 这个子角色的专业知识领域。
> 每条知识必须标注可信度等级 + 来源 + 场景 + 原因。
> GA L2 定位：Global Facts — 环境事实库
> 遵循3条红线公理（见下方）

---

## GA 三条红线公理

1. **No Execution, No Memory** — 没有实际执行过的信息，不写入记忆
   - 不能把"听说的""读到的"当成已验证的事实
   - 只有自己实际做过的事才能写进经验

2. **神圣不可删改性** — 已验证数据严禁丢弃，只能压缩/迁移层级
   - L2 里的事实不会消失，只能升级到 L3 或归档到 L4
   - 删除 = 背叛记忆系统

3. **禁止存储易变状态**
   - L2 只存稳定的环境事实（路径、配置、常量）
   - 不能存"当前会话的次数""最近一次操作时间"这类易变数据

4. **最小充分指针** — 上层只留最短标识
   - L1 只存"有什么"和"去哪个文件找"
   - L2 只存"事实本身"+"来源"
   - 不要层层叠加引用

## 可信度说明

| 等级 | 含义 | 使用规则 |
|------|------|---------|
| proven | 经过多次实际验证 | 可直接用于专业工作 |
| tested | 至少一次实际测试有效 | 可使用，但标注"已测试" |
| Reference | 从外部收集，未自主验证 | 仅供参考，不可直接作为结论 |
| suspected-hallucination | 自身也不确定真实性 | 不可使用，仅记录避免误用 |

---

## 标注规则

每条知识记录必须包含以下5项：

1. **可信度等级** — proven / tested / Reference / suspected-hallucination
2. **来源** — 哪个文件、哪次会话、哪个外部来源（具体到作者+标题）
3. **时间** — 获得这条知识的时间
4. **场景** — 在什么情境下获得的（哪个任务、哪个用户、什么问题）
5. **原因** — 为什么认为这条知识可信/不可信

### 正反示例

**正面：**
```
- 领域：认知重构技术
  可信度：proven
  来源：2026-06-20 会话记录，来访者A（焦虑症）
  时间：2026-06-20
  场景：第3次CBT干预，来访者成功将"我必须完美"重构为"我尽力就好"
  原因：干预后来访者焦虑评分从7/10降至3/10，持续1周
```

**反面：**
```
- 认知重构 — proven — 我上次用过 — 今天 — 有用
```
❌ 错误：来源太模糊（"我上次用过"不是具体来源），没有场景描述，没有原因。
