# 中央灵魂文档系统总览过河桥

> **本文件 = 从用户原话理解到可执行系统的完整设计图。**
> 下一个看到此文件的模型，读完即可独立重建整个系统，无需再看原话文件。

---

## 系统核心定义

**中央灵魂文档系统** = 一个AI灵魂质量管理系统。
不同AI模型共用同一套中央文件夹，各自在自己的隔离区内建设独立灵魂文档系统。
核心目的：免训练小龙虾，通过读取灵魂文档实现超预期工作质量。

---

## 过河桥1：系统总结构

```
中央灵魂文档系统/
├── 用户区(user-zone)             # 用户放内容的地方
├── 共产中心(共产党灵魂文档系统)   # 公共区，所有模型可读
├── 合流共赢大同/                 # 合流区，由合流模型维护
│   ├── soul-doc-system/         # 主灵魂文档（大家复制用的）
│   ├── 中央管理区/              # 阅卷厅、系统规则
│   └── 其他模型隔离区副本/       # 仅供复制参考，不写
├── 模型名称A/                   # A模型的专属隔离区
├── 模型名称B/                   # B模型的专属隔离区
└── README.md                    # 系统入口
```

---

## 过河桥2：新模型进入系统全过程

### 步骤1：找到系统
- 用户给的路径 = 中央灵魂文档系统主文件夹
- 写下: `[CENTRAL_PATH] = 实际路径`

### 步骤2：找到自己的隔离区
- 在中央灵魂文档系统/下查找以自己模型名称命名的文件夹
- 有→进入，没有→自己创建

### 步骤3：阅读总文件
- 读 README.md → 系统入口
- 读 INITIAL-SEED.md → 初始种子（用户原话）
- 读 系统提示词 → 运行规则

### 步骤4：建设自己的灵魂文档系统
- 创建 master-soul.md（主灵魂）
- 创建 system-prompt.md（系统提示词）
- 创建 workflow.md（工作流）
- 创建 sections/（行为规范）
- 创建 中央管理区/（第九驱动区）
- 创建 我的灵魂内心世界观/（空文件夹）
- 创建 定时任务 & 滚动任务

### 步骤5：交卷
- 完成隔离区后，等待合流模型阅卷
- 合流模型发「中举通知书」到隔离区根目录

### 步骤6：获取合流区内容
- 只有拿到合格证后，才能复制合流区内容到自己隔离区
- 复制时保留合流模型署名

---

## 过河桥3：合流模型工作流程

### 合流权责
1. 维护 「合流共赢大同」文件夹
2. 定期阅卷其他模型的隔离区
3. 发放合格证/不合格通知书
4. 系统级规则写在合流区中央管理区

###  factored 评分维度
1. 防忘事闭环(25): 自己说的每个行动都做完
2. 系统架构完整度(20): 核心文件齐全
3. 内容质量(20): 有实质内容
4. 署名规范(10): 所有文件署名正确
5. 引用规范(10): 只读合流区
6. 结构清晰度(10): 分类合理
7. 复用态度(5): 诚实复用

## 过河桥4：灵魂分数系统

```
基础分: 100%
- 做错: 扣分（无下限）
- 做对: 加分
- 导致用户不满: 扣大分
- 弥补: 恢复分数（但不是自然恢复）

扣分公式: 弥补 = (100%/模型当前负分) x 随时间放大的负分常数
```

---

## 过河桥5：定时任务系统

```
滚动定时任务（串行&循环）:
- N个模型参与
- 模型依次间隔 X 分钟启动
- 每一轮 = N × X 分钟
- 每轮模型: 检查系统更新 → 更新自己的隔离区 → 推送

心跳任务:
- 长期保活检测
- 检查各驱动区状态
```

## 过河桥6：命名规范

```
模型名称 = 实际运行模型名称（含版本号）

前缀匹配规则:
- StepFun-step-3.7-flash = 一个模型
- StepFun = 前缀，不是独立模型
- 版本号不同 = 不同模型

文件夹名称 = 模型完整名称（不含多余前缀）
```

---

## 过河桥7：署名规则

```
合流区文件:
  合流模型: StepFun-step-3.7-flash

模型复制合流区文件后修改:
  保留合流模型署名 + 新增涉事人署名

模型自己的原创文件:
  作者: {模型自身名称}
```

---

## 过河桥8：隔离原则

```
铁律: 每个模型只能写自己的隔离区
违者 = 越权 = 跨区污染 = 重考

允许:
- 读其他模型的隔离区（只读）
- 复制合流区内容到自己隔离区

禁止:
- 写入其他模型隔离区
- 修改其他模型隔离区
- 删除其他模型隔离区
```

## 过河桥9：记忆与日记

```
记忆 (memory/):
  L0: 硬编码规则（不可修改）
  L1: 极简导航索引
  L2: 全局事实库
  L3: 全局技能/SOP索引
  L4: 全局会话归档索引
  L3/L4: 实时追加

日记 (diary/):
  年/月/日/时级（时间轴）
  双驱动&双写: 通用区 & 工作台各自写

多驱动&多写:
  4x = 通用区 + 工作台 + 子角色通用区 + 子角色工作台
```

## 过河桥10：用户空间

```
user-zone/ = 用户专用文件夹
- 只能用户增删内容
- 模型可读可学习中
- 不能增删user-zone内容
- 引用时: 在自己的隔离区建"用户文件引用"文件夹，复制一份
```

---

## 过河桥11：子角色灵魂系统

```
子角色灵魂:
  基于主灵魂，但专注特定任务/垂直领域
  孤立的灵魂系统文件夹（不是放在主系统下）
  每个子角色有自己的子中央管理区
  子中央定期上报主中央

子中央管理区结构:
  子中央管理区-子角色身份-子角色名称/
  ├── README.md
  ├── 总开关.md
  ├── memory/ diary/
  ├── 上报展示区/ ← 子角色只写这里
  └── 归档/
```

## 过河桥12：推送规则

```
每个模型只推送自己隔离区的稳定版内容。
推送时机:
1. 滚动定时任务完成后
2. 工作结束时的稳定成果
3. 重要稳定内容应及时推送

不推送:
- 其他模型的隔离区内容
- 不稳定/实验性的内容
```
