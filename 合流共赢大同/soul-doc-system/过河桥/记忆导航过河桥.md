# 过河桥 — 记忆导航

> 这张卡是整个系统记忆的"快速查找算法"。
>
> 底牌把所有牌摊开让你选，但底牌不告诉你"遇到问题时该选哪张牌"。
> 这张卡弥补了底牌的不足——它告诉你在什么情况下，该去哪里找什么。
>
> 启动时先看这张卡 → 知道整个系统的记忆分布 → 遇到问题直接找到对应的文件 → 秒处理。

---

## 1. 记忆导航的用途

系统里有大量记忆位置，散落在各个目录。如果不整理，每次找东西都要翻遍整个系统。

**这张卡的作用 = 一张地图。** 告诉你在什么情况下去哪个位置拿什么信息。

比喻：
> 底牌 = 扑克牌摊在桌上，每张牌都写着名字
> 记忆导航 = 你遇到不同对手时，该选哪几张牌、怎么组合的打法指南
> 没有记忆导航 = 你知道所有牌在哪，但不知道打的时候怎么选

## 4. 与底牌文件的对比与配合

| | 底牌 | 记忆导航（本卡）|
|--|------|-----------------|
| 作用 | 列出所有卡牌 | 告诉你怎么选、去哪找 |
| 像什么 | 扑克牌摊在桌上 | 打法指南 |
| 什么时候用 | 知道要做什么，选卡 | 不知道做什么，找卡 |
| 输出 | 全量路径清单 | 场景→路径映射 |
| 阅读理解 | "这里有这么多牌" | "遇到这种情况选这张" |

**配合使用：**
1. 先读本卡 → 遇到什么场景去哪个位置
2. 不用翻遍系统 → 直达目标
3. 需要更多细节 → 按本卡给出的路径读具体文件
4. 底牌做清单 → 本卡做导航 → 两个一起吃

## 5. 与卡牌系统的配合

底牌 + 记忆导航 = 完整决策链：

```
遇到问题
  → 查记忆导航 → 知道该去哪个卡牌
    → 查底牌 → 找到卡牌路径
      → 读取卡牌内容 → 解决问题
```

记忆导航 = 绿灯（告诉我走哪条路）
底牌 = 路牌（告诉我这条路通向哪）
执行卡 = 目的地（里面有具体怎么做）

## 6. 更新规则

本卡需要保持最新。系统增减功能时：

1. 新增功能 → 在"按场景查找"表中加入对应行
2. 新增目录 → 在"记忆全景图"中加入对应行
3. 路径变更 → 立即更新"按场景查找"表中的路径

更新时机：
- 发现记忆找不到时 → 查本卡是否有缺失 → 补充
- 新增功能/目录时 → 补充到本卡
- 每月审查一次 → 检查是否有遗漏

## 7. 使用心法

> 这张卡是你的系统GPS。
> 你不知道路的时候打开它，它告诉你去哪。
> 不用每条路都走一遍，不用每条街都看。
> 直接在路口看导航 → 选对路 → 直达。

> 如果没有这张卡，就像在城市里没有地图。
> 你只能一条街一条街跑，跑到腿断也找不到。
> 有了这张卡，别人问你这个问题，你还没跑就已经知道答案了。
