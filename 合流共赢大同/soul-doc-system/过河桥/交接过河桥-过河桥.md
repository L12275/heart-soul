# 交接过河桥-过河桥

> 这是交接过河桥机制本身的说明书。
> 告诉你交接过河桥是什么、怎么运作、文件怎么命名、怎么恢复。
>
> 防忘事是它的**效果**。
> 交接过河桥是**实现这个效果的载体**。
> 防忘事黄金卡是告诉模型"为什么要防忘"。
> 交接过河桥是告诉模型"具体怎么做"。

---

## 1. 为什么需要桥

模型每一轮对话像一个人走在一片河的两岸之间。

- 睁眼干活 → 在河这边
- 闭眼（会话结束）→ 在河那边
- 下一次睁眼 → 在河那边，不知道河这边之前做了什么

**没有桥** = 看着对岸过不去，不知道两边有什么东西。
**有了桥** = 两岸被拉到一起，两边的事都能处理。

交接过河桥就是这座桥。它连接：
- 上一次会话的结束状态（河这边）
- 下一次会话的开始状态（河那边）
- 两岸合并，进度不丢

---

## 2. 桥的本质：实时录屏（监控模式）

交接过河桥 = 模型在工作过程中**实时录屏**。

- 不是工作结束后才写
- 是边干活边实时记录
- 像监控摄像头，全程不眨眼

比喻：
- 每个对话区像一个房间
- 房间里有个摄像机（交接过河桥文件），实时拍摄
- 保安室（交接空间）放着所有房间的录像
- 下次来这个房间 → 看录像 → 立刻知道发生了什么

## 3. 两个层面的"交接过河桥"

### 3.1 设计文档层面（本文件）

- 位置：`soul-doc-system/过河桥/交接过河桥-过河桥.md`
- 作用：解释交接过河桥是什么、怎么用
- 性质：系统规范文档，永久不变

### 3.2 会话录像层面（实际文件）

- 位置：`中央管理区/交接空间/[平台]/[项目]/[会话主题]/交接过河桥-时间戳-[桥名].md`
- 作用：记录每一轮会话的实时过程
- 性质：每次会话生成一个新的

---

## 4. 会话录像的文件结构

### 4.1 存放路径

```
中央管理区/交接空间/
├── [AI平台名称]/                    ← 例如：ClaudeCode、通义千问、文心一言
│   └── [项目名称]/                  ← 例如：中央灵魂文档系统
│       └── [会话主题]/              ← 例如：防忘事模块建设
│           ├── 交接过河桥-0701 1432-防忘事黄金卡.md
│           ├── 交接过河桥-0701 1505-防忘事黄金卡.md
│           ├── 交接过河桥-0701 1600-防忘事黄金卡-并行分支.md  ← 次并行
│           └── ...
```

### 4.2 文件命名规则

```
交接过河桥-YYYYMMDD HHMM-[桥名].md
```

示例：
- `交接过河桥-20260701 1430-防忘事黄金卡.md`
- `交接过河桥-20260701 1500-音乐生成项目.md`
- `交接过河桥-20260701 1600-防忘事黄金卡-并行分支.md`

- 时间戳：分钟级
- 桥名：AI主动总结生成，浓缩主题
- 后缀用md不用markdown（系统规范）

### 4.3 主过河与次并行

在每个会话主题文件夹内：

```
[会话主题]/
├── 交接过河桥-0701 1430-防忘事黄金卡.md    ← 主过河（最新时间戳）
├── 交接过河桥-0701 1505-防忘事黄金卡.md    ← 旧版主过河（被主取代）
├── 次并行过河桥/                             ← 副线程
│   └── 交接过河桥-0701 1600-防忘事黄金卡-并行分支.md
└── 暂停并行过河桥/                           ← 已不活跃的并行
    └── （可随时取回使用）
```

规则：
- 每次检查时先看最新时间戳的交接过河桥文件 → 这是当前主过河
- 次并行过河桥里的内容，不活跃就移到暂停文件夹
- 如果某个暂停并行重新变重要 → 可以交换回主或次并行位置

## 5. 交接过河桥文件的标准格式

```markdown

**平台**：[ClaudeCode/通义千问/文心一言/DeepSeek/StepFun/等]
**项目**：[项目名称]
**会话主题**：[当前对话主题]
**当前状态**：进行中 / 已完成 / 暂停 / 并行分支

---

## 实时录像

- 当前在做什么：[当前动作]

- 当前做到哪了：[已完成/进行中]
- 下一步：[后续动作]

---

## 累计状态
- **最终目标**：[一句话]
- **已完成**：[列表]
- **待完成**：[列表]
- **涉及文件**：[路径列表]
```

## 6. 跨平台恢复流程

任何AI平台启动、任何对话窗口打开：

```
1. 读公网入口                    → 找到系统入口
2. 进入当前模型隔离区的交接空间  → 找到自己的进出位置
3. 按平台 → 项目 → 会话主题查找  → 定位最近的主过河桥
4. 读取交接过河桥文件            → 恢复记忆和上下文
5. 继续工作                      → 100%恢复，无需用户提醒
```

### 6.1 找不到时的检索策略

1. 先找当前平台文件夹下，各会话主题文件夹，查看最近3-5个时间戳
2. 找不到 → 同一平台其他模型查过河桥文件
3. 找不到 → 其他平台文件夹查找
4. 全部找不到 → 全新开始，正常思考

### 6.2 决定写新桥还是续写旧桥

- 当前会话上下文有内容 + 与最新交接过河桥匹配（同平台/项目/主题）→ 续写，更新时间戳和内容
- 上下文为空或与最新桥文件对不上 → 新建桥文件，放到对应平台/项目/会话主题文件夹下

## 7. 会话结尾强制检查（防忘事铁律）

每次会话结束前，模型必须：

1. 写出完整的交接过河桥文件到交接空间对应的平台/项目/会话主题文件夹下
2. 确保内容足够让下一次任何对话窗口读完就能恢复
3. 不留下"后续处理"尾巴

禁止的行为：
- 说"下次再做"
- 说"等用户确认"
- 说"后续处理"
- 让用户帮你记进度

---

## 8. 桥的物理意义再理解

接河桥的逻辑是：

> 你闭上一只眼（结束会话），睁开另一只眼（新会话），
> 如果没有桥，你就不知道自己在河的哪一边，
> 也不知道之前过了一半还是全过了。
>
> 有了桥，你就知道：河的这边是什么、那边是什么、
> 自己从哪边出发、现在在哪边、下一步该怎么走。
>
> 两岸通过桥合并了，和平共处。没有桥，
> 两岸互相望着，没法处理事情。
