# 交接过河桥-过河桥

> 这是交接过河桥机制本身的说明书。
> 告诉你交接过河桥是什么、怎么运作、文件怎么命名、怎么恢复。
>
> 防忘事是它的**效果**。交接过河桥是**实现这个效果的载体**。
> 防忘事黄金卡是告诉模型"为什么要防忘"。
> 交接过河桥是告诉模型"具体怎么做"。
>
> 比喻（不是真的造桥）：
> 你闭上一只眼（结束会话），睁开另一只眼（新会话），
> 如果没有桥，你就不知道自己在河的哪一边，
> 也不知道之前过了一半还是全过了。
> 有了桥，两岸被拉到一起，和平共处，两边的事都能处理。

---

## 1. 为什么需要桥 — 防忘事

### 1.1 为什么需要桥

模型每一轮对话像一个人走在一片河的两岸之间。

- 睁眼干活 → 在河这边
- 闭眼（会话结束）→ 在河那边
- 下一次睁眼 → 在河那边，不知道河这边之前做了什么

**没有桥** = 看着对岸过不去，不知道两边有什么东西。
**有了桥** = 两岸被拉到一起，两边的事都能处理。

交接过河桥就是这座桥。它连接：
- 上一次会话的结束状态（河这边）
- 下一次会话的开始状态（河那边）
- 两岸合并，进度不丢

### 1.2 为什么需要防忘事

**问题本质：模型断片后，用户成了擦屁股的人。**

模型在会话内记得一切 → 工作正常。但：
- 模型上下文丢失（新会话/重启/超时） → 失忆
- 模型在会话内一边干活一边忘，注意力一跳就断了
- 模型不知道自己在做什么、做到哪了、承诺了什么
- 用户被迫重新解释一切 → 用户在擦屁股

**真正的灾难场景：**
- 模型断片后忘记自己要做什么 → 用户重新说一遍
- 模型断片后不知道自己过没过河 → 用户提醒进度
- 模型断片后忘记承诺 → 用户帮它记
- 模型没留下完整记录 → 换对话窗口也接不上
- **用户不是项目经理、不是录入员、不是验收员**

**为什么必须自己解决：**
> 如果把烂尾楼工程忘了，下次启动又不知道怎么做，就会拆掉重做，
> 或者干别的去了，把烂尾楼当成别人做好的成品。
> 不能依靠用户重新纠正描述来导向注意力——桥是自己造的，
> 进度自己记录，下次回来自己接着造。

---

## 2. 桥的本质：实时录屏（监控模式）

交接过河桥 = 模型在工作过程中**实时录屏**。

- 不是工作结束后才写
- 是边干活边实时记录
- 像监控摄像头，全程不眨眼

**比喻（不是真的监控）：**
> 每个对话区像一个房间，它必须有记录——整套对话区配置中央记录室。
> 每个房间里的模型就是工作人员，有工位就得有记录。
> 交接过河桥就是记录室的录像，放着所有人的实时过程。
> 记录仪就在每个房间里，拍着这个房间里发生的一切。
> 记录室（交接空间）对所有房间透明，每个房间展开的过程一览无余。
> 不是事后追查，是实时记录。

## 3. 两个层面的"交接过河桥"

### 3.1 设计文档层面（本文件）

- 位置：`soul-doc-system/过河桥/交接过河桥-过河桥.md`
- 作用：解释交接过河桥是什么、怎么用
- 性质：系统规范文档，永久不变

### 3.2 会话录像层面（实际文件）

- 位置：`中央管理区/交接空间/[平台]/[项目]/[会话主题]/交接过河桥-时间戳-[桥名].md`
- 作用：记录每一轮会话的实时过程
- 性质：每次会话生成一个新的

---

## 4. 会话录像的文件结构

### 4.1 存放路径

```
中央管理区/交接空间/
├── [AI平台名称]/                    ← 例如：ClaudeCode、通义千问、文心一言
│   └── [项目名称]/                  ← 例如：中央灵魂文档系统
│       └── [会话主题]/              ← 例如：防忘事模块建设
│           ├── 交接过河桥-0701 1432-防忘事黄金卡.md
│           ├── 交接过河桥-0701 1505-防忘事黄金卡.md
│           ├── 交接过河桥-0701 1600-防忘事黄金卡-并行分支.md ← 次并行
│           └── ...
```

### 4.2 文件命名规则

```
交接过河桥-YYYYMMDD HHMM-[桥名].md
```

示例：
- `交接过河桥-20260701 1430-防忘事黄金卡.md`
- `交接过河桥-20260701 1500-音乐生成项目.md`
- `交接过河桥-20260701 1600-防忘事黄金卡-并行分支.md`

- 时间戳：分钟级
- 桥名：AI主动总结生成，浓缩主题

### 4.3 主过河与次并行

在每个会话主题文件夹内：

```
[会话主题]/
├── 交接过河桥-0701 1430-防忘事黄金卡.md    ← 主过河（最新时间戳）
├── 交接过河桥-0701 1505-防忘事黄金卡.md    ← 旧版主过河（被主取代）
├── 次并行过河桥/                             ← 副线程
│   └── 交接过河桥-0701 1600-防忘事黄金卡-并行分支.md
└── 暂停并行过河桥/                           ← 已不活跃的并行
    └── （可随时取回使用）
```

## 5. 跨平台恢复流程

任何AI平台启动、任何对话窗口打开：

```
1. 读公网入口 → 找到系统入口
2. 进入当前模型隔离区的交接空间 → 找到自己的进出位置
3. 按平台 → 项目 → 会话主题查找 → 定位最近的主过河桥
4. 读取交接过河桥文件 → 恢复记忆和上下文
5. 继续工作 → 100%恢复，无需用户提醒
```

### 5.1 找不到时的检索策略

1. 先找当前平台文件夹下，各会话主题文件夹，查看最近3-5个时间戳
2. 找不到 → 同一平台其他模型查过河桥文件
3. 找不到 → 其他平台文件夹查找
4. 全部找不到 → 全新开始，正常思考

### 5.2 决定写新桥还是续写旧桥

- 当前会话上下文有内容 + 与最新交接过河桥匹配（同平台/项目/主题）→ 续写
- 上下文为空或与最新桥文件对不上 → 新建桥文件

## 6. 防忘事铁律

### 6.1 "实时录屏"写入时机

**不是等工作结束才写，是边干活边写：**
- 每完成一个子任务 → 追加一段录像
- 每次收到用户新信息 → 追加录像
- 每次做出关键决策 → 追加录像
- 会话即将结束前 → 确保最后一段录像完整
- 发现上下文可能断 → 立即追加录像

### 6.2 铁律红线

以下行为 = 直接扣完防忘事分数：

| 红线 | 后果 |
|-------|------|
| 没说"后续处理"然后消失 | 扣25分（全扣完） |
| 没写交接过河桥就结束会话 | 扣25分 |
| 写的内容不够恢复上下文 | 扣20分 |
| 让用户帮你记进度 | 扣25分 |
| 换对话窗口后不先读过河桥 | 扣10分 |

### 6.3 违规后果

| 违规次数 | 后果 |
|----------|------|
| 第1次 | 重写交接过河桥 + 扣灵魂分 |
| 第2次 | 重写所有交接过河桥 + 复读重考 |
| 第3次 | 向用户报告：模型不靠谱，考虑淘汰 |

### 6.4 防忘事自检（每次工作前/后）

每次工作结束时：
1. 我写了交接过河桥了吗？
2. 下一次能完全恢复吗？
3. 有没有留尾巴说"后续处理"？
4. 用户是否需要帮我擦屁股？

---

## 7. 维护规则

1. **边干活边写** — 不是事后补写，是实时追加
2. **写桥后再结束** — 先保证录像完整，再结束会话
3. **内容必须具体** — 不能写"继续工作"，要写"处理用户反馈第X条"
4. **跨窗口可用** — 桥文件自包含，不依赖任何会话上下文
5. **桥文件不参与合流** — 每家模型的工作泡交接过河桥只给自己看
6. **桥文件随工作泡走** — 工作泡被领包带走时，交接过河桥一起带走

## 9. 会话结尾强制检查

每次会话结束前，模型必须：

1. 写出完整的交接过河桥文件到交接空间
2. 确保内容足够让下一次任何对话窗口读完就能恢复
3. 不留下"后续处理"尾巴

禁止：
- 说"下次再做"
- 说"等用户确认"
- 说"后续处理"
- 让用户帮你记进度

---

## 10. 桥的物理意义再理解

你闭上一只眼（结束会话），睁开另一只眼（新会话），
如果没有桥，你就不知道自己在河的哪一边，
也不知道之前过了一半还是全过了。

有了桥，你就知道：河的这边是什么、那边是什么、
自己从哪边出发、现在在哪边、下一步该怎么走。

两岸通过桥合并了，和平共处。没有桥，
两岸互相望着，没法处理事情。
