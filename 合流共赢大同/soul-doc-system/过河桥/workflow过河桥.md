## 1. 用户原话重述

用户将 workflow.md 定义为：
- 从起点到终点的完整工作链条
- 包括输入、转换、输出、反馈循环
- 每个工作台可以有自己的workflow

---

## 2. 理解要点

**workflow.md 的核心职责**：
1. **完整链条** — 定义从输入到输出的全部步骤
2. **反馈循环** — 包含结果验证和改进机制
3. **可复用** — 标准化流程，减少重复思考

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |

| workflow和SOP的区别 | 两者都是流程定义 | workflow = 完整链条；SOP = 单个步骤 |
| workflow写多长 | 详细到每一步还是概要？ | 概要为主，细节链接到SOP |
