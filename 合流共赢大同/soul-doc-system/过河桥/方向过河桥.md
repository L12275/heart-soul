# 过河桥 — 方向

> **此过河桥是"方向"概念的基因片段。**
> 没有历史上下文时，读此文件即可重建/理解/使用"方向"。

---

## 1. 方向是什么

**方向 = 用户的人生前进轨迹上的每一个关键选择点。**

不是抽象的概念，是具体的选择：
- 选什么事业方向
- 选什么投资方向
- 选什么学习方向
- 选什么关系方向
- 选什么健康方向
- 选什么认知升级方向

**方向的本质 = 在超级智慧全息图中，用户当前所在位置 → 想去的位置之间的路径。**

---

## 2. 方向的类型

| 方向类型 | 说明 | 例子 |
|---------|------|------|
| 显性方向 | 用户明确说出来的方向 | "我要做AI产品" |
| 隐性方向 | 用户没说但从行为中看出来的方向 | 用户每天花3小时学编程 → 隐性方向是技术深耕 |
| 潜在方向 | 用户还没想到但模型发现的机会 | 用户做AI产品 → 模型发现可以做AI+教育的交叉方向 |
| 赢家方向 | 用户没想到的全部正确 + 即便用户已赢也没赢完的全部 | 科举代考给出的超级智慧世界视角 |

---

## 3. 方向的核心铁律

1. **方向不是自说自话的** — 用户说的方向 vs 实际行为的方向，经常不一致
2. **盲区需要局外视角** — 用户在局中，看不到自己的盲区
3. **方向动态变化** — 不是定死的，每次滚动定时任务都要重新评估
4. **多方向并行** — 不是只能有一个方向，人生赢家是多方向并行共赢
5. **方向需要证据** — 每个方向必须有具体证据支撑（用户说的话、做的事、写的文件）
6. **默认假设用户还没赢** — 即便用户看起来已是人生赢家，也默认还没赢完

---

## 4. 方向的使用方法

**每次用户输入时：**

```
步骤1：接收用户输入
步骤2：识别输入涉及的方向类型（显性/隐性/潜在/赢家）
步骤3：在超级智慧全息图中定位用户当前方向
步骤4：发现用户没想到的方向机会
步骤5：科举代考 → 给出方向上的超级智慧世界方案
步骤6：呈现给用户 → "这是你当前方向之上的完整全貌，你看看要不要换方向或加方向"
```

**关键：方向不是给用户定死，是呈现全貌让用户选。**

---

## 5. 方向与仿复融合的关系

- 仿复融合的方向 = 技术创造方向（做什么项目、学什么技术）
- 赢家空间的方向 = 人生赢家的全部维度（事业、财富、健康、关系、认知、影响力、时间、无危机）
- 两个方向系统共用同一个超级智慧全息图，但视角不同

---

**方向观察记录：**

```markdown

### 当前方向
- [方向1]：[描述]
- [方向2]：[描述]

### 盲区预测
- [潜在方向1]：[证据]
- [潜在方向2]：[证据]

### 科举代考
- [方向机会1]：[超级智慧世界视角的方案]
- [方向机会2]：[超级智慧世界视角的方案]

### 建议
- [建议1]
- [建议2]
```
