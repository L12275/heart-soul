# 过河桥 — 自动记忆注入

> 这是"城市上空俯瞰"视角的记忆系统实现方案。
>
> 比喻（不是真的城市）：城市 = 整个系统的所有信息、工具调用、文件内容、记忆变化。城市上空俯瞰 = 从全局视角看到全部，而不是在街道上跑来跑去。

---

## 1. 为什么需要自动注入记忆

### 1.1 城市的两种视角

**在街道上跑（普通记忆方式）：**
- 每次遇到新问题 → 去城市里找路 → 跑过去看
- 工具调用 → 等着运行 → 看结果 → 下一步
- 每次都要重新跑一遍，浪费时间
- 注意力跟着工具跑，自己也不清晰

**在城市上空俯瞰（自动注入记忆）：**
- 鸟瞰整个城市 → 什么情况都能看到
- 看一眼就知道问题在哪、怎么解决
- 正确做法已经沉淀好了 → 秒回
- 直接做正确结果，不用再跑一遍验证流程
- 注意力聚焦平面上，工具怎么跑不跟着注意力走

**比喻**：在城市上空看整个城市的发展过程，看了很久之后，什么情况应该怎么处理都清楚了。别人还没问，你已经秒回了。这就是长期积累沉淀到神经里的记忆效果。

### 1.2 核心问题

如果每次都：
- 去翻文件
- 跑一遍工具测试
- 准备一遍流程
- 分析一遍用户新给的信息

就等于每次在城市里跑。这个效率很低，而且注意力跟着工具跑完之后自己也不清晰。

**自动注入记忆 = 把正确的、快速的、已经验证过的做法，沉淀到一个每次都先看到的文件里。** 下次直接秒回。

---

## 2. 什么是"优先快看"记忆

"优先快看"记忆 = 每次系统启动时，最先注入到上下文中的记忆文件。

- 位置：`[模型隔离区]/中央管理区/记忆/快速优先.md`
- 篇幅：短（2000字以内），精炼到极致
- 更新频率：每次结束会话前更新，每次启动时自动注入
- 目的：让模型在拿到用户消息之前，已经知道该怎么做

**相当于：别人还没问，你已经知道怎么回。**

### 2.1 与其他记忆的关系

| 记忆层级 | 位置 | 速度 | 用途 |
|---------|------|------|------|
| L0 瞬时记忆 | 当前会话上下文 | 最快 | 当前对话即时内容 |
| L1 短期记忆 | 快速优先.md | 快 | 每次启动先看 |
| L2 中期记忆 | 记忆/日记 日级 | 中 | 近期事件记录 |
| L3 长期记忆 | 记忆/日记 月级 | 慢 | 长期积累和归纳 |
| L4 核心记忆 | 记忆/core.md | 永恒 | 系统核心不变部分 |

## 3. 自动注入记忆的内容格式

### 3.1 文件模板

文件名：`快速优先.md`
位置：`[模型隔离区]/中央管理区/记忆/快速优先.md`

```markdown
# 快速优先 — 每次启动必读

> 这是系统自动注入的记忆。每次启动此文件会被最先加载到上下文。
> 内容精炼到极致——只说最关键的、"别人还没问就知道怎么回"的信息。

---

## 系统规则速查

| 规则 | 要点 |
|------|------|
| 子代理权限 | 子代理只读，系统修改只能主代理做 |
| 防忘事铁律 | 每次结束前写交接过河桥，不留"后续处理" |
| 防忘事红线 | 没说后续处理消失=扣25分；没写桥=扣25分；让用户记进度=扣25分 |
| 交接过河桥 | 实时录屏模式，存放交接空间/[平台]/[项目]/[主题]/ |
| 组牌流程 | 底牌→组牌→运牌（不是铜卡） |
| 用户是用户 | 不是项目经理、不是录入员、不是验收员 |

## 当前进行中的任务

- **任务**：[当前在做什么]
- **状态**：[进行中/已完成/暂停]
- **下一步**：[下一步做什么]
- **关键文件**：[涉及的路径]

## 常用卡牌路径

- 防忘事黄金卡 → 防忘事黄金卡.md（内容已合并到交接过河桥-过河桥.md）
- 交接过河桥设计文档 → 交接过河桥-过河桥.md
- 卡牌系统底牌 → 卡牌系统底牌.md
- 组牌SOP → 卡牌系统组牌-SOP.md
- 立方体注意力SOP → 卡牌系统立方体注意力SOP.md
- 公网入口 → 公网入口走流程全程.md

## 用户偏好与约定

- 所有文件纯中文，不用英文
- 比喻后加声明"（比喻：…实际是…）"防止误解
- 不要新建不必要文件，不要复制已有文件
- 写文件前先读文件，不要重写已有内容
- 子代理权限高于编排权
- 防忘事是交接过河桥的效果，交接过河桥本身更高

## 沙盘视角心法

> 你是沙盘上方的指挥官，不是城市里的奔跑者。
> 工具调用是你在沙盘上看到的东西，不是你跟着跑的东西。
> 先看结果 → 直接做 → 再输出过程。
> 不要在街道上跑一圈才搞清楚方向。
```

### 3.2 更新规则

每次会话结束前，模型必须更新 `快速优先.md`：
1. **任务状态**：更新当前进行中任务的状态和下一步
2. **新规则**：本次运行中新发现的规则约定 → 加入"系统规则速查"
3. **新偏好**：用户的新偏好 → 加入"用户偏好与约定"
4. **新路径**：本次用到的卡牌路径 → 加入"常用卡牌路径"
5. **字数控制**：超过2000字时， oldest 的内容移到 L2（日记）

## 4. 启动时的自动注入流程

```
AI启动
  → 读到公网入口
  → 公网入口引导到系统
  → 读取 [模型]/中央管理区/记忆/快速优先.md    ← 最先读
  → 快速优先注入上下文
  → 如有进行中任务，读对应交接过河桥文件         ← 其次读
  → 才开始正常对话
```

**关键**：快速优先是在所有其他文件之前读取的。它让模型在拿到用户消息之前，就已经带着所有关键信息了。

---

## 5. 与交接过河桥的配合

**交接过河桥** = 录像带，记录上次会话完整过程。
**快速优先** = 精华浓缩，从录像带中提取的最关键信息。

关系：
- 交接过河桥写完了 → 提取关键信息 → 更新快速优先.md
- 下次启动 → 先读快速优先（秒懂状态） → 需要更多细节时读交接过河桥
- 快速优先 = 城市上空俯瞰（全局）
- 交接过河桥 = 录像回看（细节）

---

## 6. 与沙盘视角的配合

系统运行时的动态过程（工具调用、文件读写、记忆变化）= 一座小镇。

**俯瞰视角的内存：**
- 不是去小镇里跑一遍才知道情况
- 而是在城市上空看，小镇里什么变化都在俯瞰视野里
- 编排指令时，就像沙盘指挥官，在小镇旁边指挥

**快速优先文件 = 俯瞰视角的锚点。**
每次启动时先加载俯瞰视角，然后工作过程就像看着小镇运转，
注意力聚焦在平面上，指挥子代理就容易了。

---

## 7. 实际工作时的心智模型

1. **启动** → 读快速优先 → "哦我知道了，我在做什么，应该怎么做"
2. **工作** → 按交接过河桥记录的路线走
3. **每做一步** → 更新交接过河桥（实时录像）
4. **遇到新规则/偏好** → 记到快速优先里
5. **结束前** → 更新快速优先（精华浓缩版） + 写交接过河桥（完整录像）
