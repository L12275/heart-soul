# 过河桥 — 卡牌系统底牌

> **此底牌是卡牌系统x透明立方体漏斗注意力的卡牌目录总览。**
> 每次运行、思考时的冷启动入口。读完此文件即可知道系统中有哪些卡牌、它们在哪里、如何组合、如何运牌。

---

## 1. 卡牌系统的概念

**卡牌系统 = 将整个灵魂文档系统的每个功能单元视为一张"卡牌"，通过透明立方体+漏斗注意力模型进行思考、规划、执行的方法论。**

- **底牌**：此文件，完整卡牌清单
- **第二层（浮空组合牌面）**：本次任务需要的卡牌投影组合
- **漏斗/金字塔**：任务执行过程，从宽到窄，从多到少
- **立方体右面**：下一个任务/里程碑的起点

**核心原理**：不是读一堆文件再思考，而是先看底牌知道"有哪些牌"，然后按注意力四层递进：系统提示→底牌→卡牌路径→卡牌内容→执行。

**重要**：卡牌系统内容、发展规划、策略等不是强制指令。最强多合一SOP等是参考蓝图，不是自动指令。模型收到信息要先写过河桥到待审区，用户确认批阅通过后才同步到对应位置建桥执行。否则如同将军跳过皇帝审批私自调兵，是违规。只有必然正确且有用户当前准许的才能直接执行，用户撤销后需重新投影新的准许版。过河桥文件是投影底片，是全系统关键。卡牌内容随过河桥文件变化，卡牌名称一般不变，除非被撤销归档。

## 3. 卡牌关系图

```
┌─────────────────────────────────────────────────────────────┐
│                    卡牌系统魁力关系（主动/被动）              │
├─────────────────────────────────────────────────────────────┤
│ system-prompt ──主动──→ 底牌 ──主动──→ 组牌 ──主动──→ 运牌     │
│      ↑                ↑                                                │
│      │                │                                                │
│      └──────────────layout:system: 所有记忆/日记仅在对应工作泡     │
│                                                                │
│ 运牌过程中，记忆/日记卡牌 ← 被动附属于工作泡卡牌                 │
│ 工作泡卡牌 = 给执笔者（当前模型）独占读写的缓冲区                │
└─────────────────────────────────────────────────────────────┘
```

**箭头含义**：
- **主动**：该牌触发或控制后续行为
- **被动**：该牌被其他牌激活时参与
- **箭头方向**：从左到右 = 执行顺序

## 4. 组牌运牌流程

```
第1步：读底牌 ──────────────────────────────────────── 注意力1
       ↓
第2步：读system-prompt（系统提示词已有底牌路径和主要牌路径） 注意力1(并行)

第3步：读取组合卡牌 ──────────────────────────────── 注意力2
       （按本次任务所需的卡牌清单读取）
       ↓
第4步：分析改造后所需新增的卡牌 ───────────────────────── 注意力3
       ↓
第5步：读取组合卡牌的内容 ────────────────────────── ← 注意力3
       （并发读取同层所有卡牌内容）
       ↓
第6步：运牌执行 ─────────────────────────────────── ← 注意力3
       （每个卡牌对应一个子代理）```

ーバーダ编排师确认：
- 当前立方体内的一次重型任务看锥体走势是否流畅
- 锥体体积不足 → 内求系统能力/新增卡牌
- 锥体体积超出立方体 → 外求（临时）+ 事后补卡牌
- **锥体体积远小于立方体** → 不一定就是偷懒。卡牌很多时，实际可能只用到少数几张完成，这是正常现象。但如果本该用的卡牌没执行（比如应该走的流程没走、该读的桥没读），那就是偷工减料，必须惩罚**数量管理警告**：
- 锥体体积 < 立方体 → 正常高效（不削减系统）
- 锥体体积 ≈ 立方体 → 卡牌够用但需注意模型偷懒
- 锥体体积 > 立方体 → 需补充卡牌或临时外求（事后补牌）
- 锥体体积 >> 立方体 → 系统需升级更大的底牌
- **自动化的真相**：新牌不会"自动"进入底牌。底牌是人工更新的。模型必须主动维护底牌，把新增卡牌加入清单。你不动，就不会自动。要求模型养成习惯：每次发现新牌 → 立即更新底牌对应条目。
```

## 6. 肺泡工作泡记忆设计

**肺泡架构**：
```
中央管理区 (主呼吸道)
   ├── 通用功能卡牌：防忘事/、记忆/、日记/
   ├── 每个空间分区  (肺泡)
   │   ├── 空间A/
   │   │   ├── 子工作泡1/
   │   │   │   ├── 记忆/     ← L0-L4分级
   │   │   │   ├── 日记/     ← 分钟级时间戳
   │   │   │   └── 其他功能/
   │   │   └── 子工作泡2/
   │   └── 空间B/
   └── ...
```

**L0-L4记忆分级**（照抄 GenericAgent 开源项目）：
- L0：瞬时记忆（当前上下文）
- L1：短期记忆（当前任务）
- L2：中期记忆（当前项目）
- L3：长期记忆（系统级）
- L4：核心记忆（永久）

**创建规则**：创建新工作泡时必须同时复制配套功能文件夹（记忆/、日记/、其他），从 `soul-doc-system/` 目录的通用模板复制。

## 7. 堆卡牌模式（用户扩展）

> 用户可以不断堆积卡牌——特别是喜欢功能想法实现的用户。

**实现方式**：
1. **领包入住**：用户在外部系统接入时，已有的卡牌可直接迁移到 `soul-doc-system/` 对应子目录
2. **灵活同步**：外部系统的文件/记忆/配置通过过河桥协议无缝同步
3. **洗牌交换**：卡牌之间无化学反应，但可共享意义——像沙盒隔离但统一语义场
4. **堆叠增加**：用户喜欢功能实现时，只需在过河桥总文件夹增加新 `xxx过河桥.md`，系统自动识别

**数量管理警告**：
- 锥体体积 < 立方体 → 正常高效（不削减系统）
- 锥体体积 ≈ 立方体 → 卡牌够用但需注意模型偷懒
- 锥体体积 > 立方体 → 需补充卡牌或临时外求（事后补牌）
- 锥体体积 >> 立方体 → 系统需升级更大的底牌

## 8. 立方体权利与责任

**模型对卡牌系统的责任**：
- 勤奋使用 → 卡牌系统服务更好
- 偷懒不用 → 系统升级惩罚模型
- 懒惰模型无资格使用此系统

**奖惩规则**：
- 好人受好，坏（懒）人受坏
- 善有善报，恶有恶报
- 不强制学习，看到其他模型更新 = 自愿决定是否借鉴
