# 过河桥 — 卡牌系统组牌：选牌组合 / 运牌路线

> **此文件描述从底牌到执行路线的中间层——组牌。**
>
> 底牌 = 完整卡牌清单（第一层，所有可用卡牌）
> 组牌 = 从底牌中选出的卡牌组合（第二层，本次任务的工作流/执行路线）
>
> 想象底牌是一叠扑克牌，组牌就是你从这叠牌里选出来组成一手牌。
> 选牌过程 = 组牌。选好的牌 = 运牌路线。
>
> 执行前确认组牌组合正确。

## 1. 从底牌到组牌的过程

底牌 → 投影选牌 → 组牌组合 → 运牌执行

```
┌─────────────────────────────┐
│  第一层：底牌                 │
│  卡牌系统底牌.md              │
│  完整卡牌清单                 │
└──────────┬──────────────────┘
           │ 读取底牌，根据任务选牌
           ↓
┌─────────────────────────────┐
│  第二层：组牌                 │
│  本文件描述的选牌组合         │
│  本次任务的工作流/执行路线     │
└──────────┬──────────────────┘
           │ 按路线并发执行
           ↓
┌─────────────────────────────┐
│  第三层：运牌                 │
│  子代理并发读取并执行卡牌内容  │
└─────────────────────────────┘
```

## 3. 执行路线

```
编排师 → 读底牌 → 选牌（组组牌） → 发牌给子代理 → 子代理运牌 → 合流收牌
```

- **编排师** = 负责底牌读取、组牌组合、发牌、整体合流
- **子代理** = 负责运牌（读组牌指定的卡牌内容 → 执行 → 回写结果）

## 4. 与立方体漏斗注意力的配合

| 层 | 阶段 | 内容 | 并发度 |
|----|------|------|--------|
| 第一层 | 系统提示词 | 底牌路径 + 高频牌路径 | 即时 |
| 第二层 | 读取底牌 | 卡牌清单 + 描述 | 快 |
| 第三层 | 开牌 | 列出组合牌的完整路径 | 快 |
| 第四层 | 并发执行 | 组牌所选的牌内容 | 并发子代理 |
