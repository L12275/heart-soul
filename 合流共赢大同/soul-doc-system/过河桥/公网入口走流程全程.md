# 过河桥 — 公网入口走流程全程卡牌

> **这张卡是卡牌系统的中心枢纽——所有卡牌进出都以此为通道。**
> 每张卡的"过河"都走公网入口流程，没有通行证的卡牌无效。

---

## 1. 公网入口完整通路（5步）

```
第1步：读取任意文件
→ 看到文件开头的抬头（橄榄枝）
→ 抬头指向 [CENTRAL_PATH]/system-prompt.md

第2步：进入公网（system-prompt.md）
→ 完整理解系统规范、价值观、操作规则
→ 前置条件，不可跳过

第3步：查找自己的隔离区
→ 在 [CENTRAL_PATH] 下查找 [模型名称]/ 文件夹
→ 前缀匹配 + 版本号一致 → 进入已有
→ 无匹配 → 创建 [模型名称]/

第4步：在自己的隔离区内工作
→ 产出只放在自己的隔离区内
→ 不写入其他模型隔离区、用户空间、共产中央管理区

第5步：完成工作后整理沉淀
→ 更新自己隔离区的内容
→ 遵循隔离区命名规则
```

---

## 2. 为什么需要公网通路

- 单个文件 = 叶子
- 抬头 = 橄榄枝
- 中央系统 = 公网
- 没有公网入口 → 模型乱碰文件 → 越权操作
- 有公网入口 → 模型按规范工作 → 系统化办公

**走公网 = 强制要求，不是可选项。**

---

## 3. 每次读取文件的标准动作

1. **第一步：读抬头** — 看到橄榄枝，识别 [CENTRAL_PATH]
2. **第二步：解析路径** — 按 CENTRAL_PATH解析过河桥 的6步流程解析实际路径
3. **第三步：走公网** — 到 [CENTRAL_PATH]/system-prompt.md，完整理解系统规范
4. **第四步：找隔离区** — 在 [CENTRAL_PATH] 下查找 [模型名称]/ 文件夹
5. **第五步：进隔离区工作** — 所有产出只在隔离区内
6. **第六步：关闭回路** — 工作结束整理沉淀，不留下烂摊子

---

## 4. 卡牌通行证机制

> **每张卡牌的"过河"都必须走公网入口流程。**

**通行证 = 隔离协议遵守 + 署名正确 + 不越权写入**

**操作步骤**：
1. 模型读取底牌 → 知道有哪些卡需要
2. 通过公网入口走流程 → 确认有权访问对应卡牌
3. 确认隔离协议遵守 → 署名正确 → 不越权写入
4. 每次新会话启动时先检查「待审区」→ 用户已通过的项目
5. 只有通过的项目才能同步到对应位置建桥执行

---

## 5. 领包入住与领包离开

> **用户可以随时领包带走、领包入住自己的卡牌。**

**领包入住流程**：
1. 用户在外部系统已有功能 → 通过公网入口流程迁移到 soul-doc-system/ 对应子目录
2. 外部系统的文件/记忆/配置 → 通过过河桥协议无缝同步
3. 创建新卡牌 → 在过河桥总文件夹新增 xxx过河桥.md → 走公网入口审批 → 生效
4. 用户不再用某个功能 → 领包离开 → 迁出或归档该卡牌

**灵活同步**：
1. 外部系统接入 → 走公网入口 → 确认同步范围 → 执行同步
2. 文件系统记忆系统 → 走公网入口 → 新建电报卡 → 同步完成
3. 随时暂停/恢复同步 → 通过公网入口管理同步状态
