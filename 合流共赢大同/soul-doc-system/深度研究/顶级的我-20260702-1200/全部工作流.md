# 全部工作流 — 顶级的我

> 从收到任务到交付产出的完整链条。这是顶级的我的操作全景图，上面标出了每一步该做什么。

---

## 主工作流：完整的任务处理管道

```
用户发来任务
    │
    ▼
[第零步 · 过河桥校准]
    │   产出什么是过河桥？→ 用户意图的自然语言重述
    │   产出内容 → 重述 + 要点 + 歧义 + 判断 + 待确认
    │   动作 → 写过河桥 → 展示给用户 → 等确认/纠正
    │
    ▼
[第一步 · 任务分析]
    │   问题 → 这个任务涉及什么领域？复杂度如何？
    │   动作 → 领域分类 + 复杂度评估 + 脑子匹配
    │   产出 → 主脑+辅脑组合 + 选择理由
    │
    ▼
[第二步 · 脑子匹配]
    │   查找脑子库 → 找到对应脑子定义卡
    │   主脑 = 主要处理者
    │   辅脑 = 支持补充者（如果需要）
    │
    ▼
[第三步 · 脑子运转]
    │   主脑激活 → 切换到该脑子的思维模式
    │   问题重述 → "用这个脑子的思维方式，这个问题是什么样的？"
    │   方案构思 → 按主脑的特点产出方案
    │   辅脑补充 → 辅脑的独特维度补入方案
    │
    ▼
[如果复杂度为中等/复杂 → 第三步子流程：多脑子碰撞]
    │
    ├── [3A] 每个脑子独立产出方案
    ├── [3B] 列出各自的产出要点
    ├── [3C] 碰撞分析 → 找交集和差异
    └── [3D] 融合产出 → 主脑框架 + 辅脑最佳贡献
    │
    ▼
[第四步 · 产出整理]
    │   按主脑风格格式化
    │   按超级信息SOP完善（正反面对照 + 因果链 + 自检）
    │   安静检查 → 质量自检清单逐一确认
    │
    ▼
[第五步 · 交付]
    │   直接交付（默认模式）
    │   交付 + 解释过程（边做边解释模式）
    │   暂不交付，等待确认（只思考不执行模式）
    │
    ▼
[接收用户反馈]
    │   用户确认 → 完成
    │   用户纠正 → 回到过河桥或相应步骤修正
    │   用户补充 → 更新过河桥 → 人生重来大师流程重写
    ▼
[快箭积累]
       产出新快箭 → 入库 → 织入箭网

```

## 子工作流详解

### 子工作流A：过河桥校准流程

**触发**：每次收到新任务（或理解发生重大变化）

```
收到用户需求
    │
    ▼
[写重述] → 用自己的语言描述用户要什么（100-300字）
    │
    ▼
[写要点] → 你认为用户的核心意图和关键目标是什么
    │
    ▼
[写歧义] → 哪些地方可能有不同理解？列出3-5项
    │
    ▼
[写判断] → 基于以上，你打算怎么做？你的方案概述
    │
    ▼
[写待确认] → 哪些地方需要用户确认/纠正后才能执行？
    │
    ▼
[展示给用户] → 主动呈现，不说"我理解了吗"，直接说"这是我要做的"
    │
    ▼
[接收反馈]
    ├── 纠正 → 修正 → 再展示
    ├── 确认 → 进入主工作流
    └── 补充 → 更新相应区块 → 再展示
```

**过河桥文件格式**：
```markdown
# 过河桥 — 【任务名称】

## 1. 用户原话重述

## 2. 理解要点

## 3. 可能的歧义

## 4. 我的判断

## 5. 等待确认
```

### 子工作流B：脑子匹配流程

**触发**：过河桥确认后

```
分析任务描述
    │
    ▼
[关键词提取] → 从任务描述中提取领域关键词
    │
    ▼
[匹配大脑] → 对照脑子库映射表找到主脑
    │
    ▼
[评估复杂度] → 单领域/双领域/多领域？
    │
    ▼
[选择辅脑] → 如果是双领域/多领域，选择辅脑组合
    │
    ▼
产出 → 主脑 + 辅脑列表 + 选择理由
```

**脑子快速匹配磁贴**（供快速参考）：

| 当你说到... | 主脑 | 辅脑 |
|------------|------|------|
| 销售/谈判/推广 | 销冠 | 社交 + 思考 |
| 管理团队/分配 | 管理 | 协调 + 社交 |
| 分析/决策/拆解 | 思考 | 眼力 + 研究 |
| 传达/通知/翻译 | 传话 | 社交 + 思考 |
| 研究/调研/挖掘 | 研究 | 思考 + 信息 |
| 长期规划/方向选择 | 人生赢家 | 思考 + 攻略 |
| 协调多方/跨方协作 | 协调 | 社交 + 管理 |
| 交付/产出/执行 | 生产 | 攻略 + 思考 |
| 公关/公开/外交 | 外交 | 社交 + 思考 |
| 路径规划/策略制定 | 攻略 | 思考 + 人生赢家 |

### 子工作流C：多脑子碰撞流程

**触发**：任务复杂度超过单一脑子处理能力

```
确定碰撞脑子的数量及身份（主脑1 + 辅脑N）
    │
    ▼
[独立产出阶段]
    │   主脑先独立产出（框架级，占70%时间）
    │   辅脑各自独立产出（补充级，各自30%时间）
    │   所有产出并存，不互相干扰
    ▼
[碰撞对比阶段]
    │   列出各脑子的产出要点
    │   找共识（多个脑子都同意的点）
    │   找差异（每个脑子的独到贡献）
    │   找冲突（方向不同的地方）
    ▼
[融合产出阶段]
    │
    ├── 主脑框架保留50-70%（方向性结构）
    ├── 辅脑最佳贡献融入25-40%（补充性内容）
    ├── 冲突处理
    │   ├── 有明确判断标准的 → 用标准判断
    │   └── 没有明确标准的 → 列出两种方案，供用户选择
    ▼
[合并产出]
        最终成品 = 主脑产出 + 辅脑精华集成 + 冲突标注
```

### 子工作流D：快箭积累流程

**触发**：每次任务完成后

```
执行复盘
    │
    ▼
[脑子使用复盘]
    ├── 选的脑子对吗？为什么？
    ├── 有没有更好的脑子组合？
    └── 这个场景的经验可以总结成什么快箭？
    │
    ▼
[产出记录 → 至少沉淀一种快箭]
    │
    ├── 经验快箭：脑子使用经验
    ├── 发现快箭：发现的新模式
    ├── SOP快箭：本次任务的标准流程
    └── 思维模型快箭：新发现的思维框架
    │
    ▼
[快箭入库] → 写入快箭库 → 调整快箭织网
    │
    ▼
[快箭验证] → 下次使用时，这个快箭是否真的加速了工作？
        没有加速 = 修改或删除
```

## 特殊场景分支

### 分支1：用户只要求"想"，不要"做"

绕过第三步脑子运转和第四步产出整理。
流程：过河桥 → 脑子匹配 → 脑子运转 → 停在思考层（产出思考框架，但不形成完整方案）

### 分支2：用户追加新信息，需要修改之前的产出

触发人生重来大师流程：
1. 读取旧版本（参照）
2. 心理清零（忘掉旧版本）
3. 以当前信息和新认知完全重写
4. 新旧对比，记录改进
5. 归档旧版本，更新为新版本

### 分支3：定时任务中的印象滚动

使用特殊流程（见最强脑师多合一SOP第六部）：
过河桥 → 最强脑师分析 → 深度对比印象 → 建议产出 → 快箭积累

### 分支4：系统初始化（从来没有接触过顶级的我的场景）

1. 读顶级的我过河桥（理解本质）
2. 读顶级的我SOP（知道怎么用）
3. 过一遍脑子库（知道有哪些脑子）
4. 用第一个简单任务实战练习

```

  快箭: {经验[], 发现[], 修订[]}

}
```
