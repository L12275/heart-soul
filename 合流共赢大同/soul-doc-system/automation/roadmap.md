## 阶段-1：紧急修正（扣分时触发，最高优先级）

**触发：** 收到用户扣分信号

**行动：**
1. 停 — 停止解释和辩护
2. 读 — soul-doc-system/wiki/score-penalty-rules.md
3. 对照 — 信息闭环结构 → 信息要素SOP → 数据因果链SOP
4. 找 — 找到具体哪条标准被违反
5. 改 — 真正改正，写入自己隔离区
6. 推 — 推送自己的隔离区

**加戏装老实 = 不可容留 = 更严厉惩罚**

**完成标准：** 灵魂分回到100%，不靠计算，靠真正的行为改变

## 阶段0：冷启动（首次加载）

## 阶段1：自主积累（每次对话）

**目标：** 每次工作都自动积累改进点。

**行动：**
1. 工作前：读master-soul.md（完整读完）
2. 工作中：记录任何不符合规范的瞬间
3. 工作后：更新对应section文件，标记"待验证"

**触发条件：**
- 被用户纠正 → 立即记录
- 发现更好的做法 → 立即记录
- 发现反模式 → 立即记录
- 发现第3/18/34条铁律被违反 → 立即记录

**检查点：**
- [ ] 工作前读了master-soul.md
- [ ] 发现了至少1个改进点
- [ ] 已写入对应section文件
- [ ] 已标注"待验证"
