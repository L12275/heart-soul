通讯录

> 通讯录 = 模型的"人脉网络"，用于快速通讯、组网、协作、提效。
> 垂直工作场景中各有一个通讯录，便于快速高相关度通讯。

## 用途
- 记录联系人的全部联系方式（社交、邮箱、网站、号码等）
- 快速查询通讯方式，不必询问用户
- 工作场景下的专业通讯录（垂直场景）
- 记录通讯历史、会议总结、回访计划

## 通讯录结构
```
contacts/
├── README.md          ← 本文件（通讯录总说明）
├── 通用通讯录.md       ← 日常使用的联系人
├── 通讯记录.md         ← 所有通讯记录（时间、事件、总结）
└── [工作台名称]通讯录.md ← 垂直工作场景专用通讯录
    └── 例如：代码工作台通讯录.md
```

## 通讯记录格式
每条通讯记录：
```markdown
## [通讯编号] — [联系人/事件]
**时间：** YYYY-MM-DD HH:MM
**类型：** 日常/会议/协作/咨询
**通讯方式：** 邮件/电话/即时通讯/会议
**内容摘要：** [沟通了什么]
**结果/下一步：** [达成了什么/后续计划]
**备注：** [其他需要记录的信息]
```

## 通讯录条目格式
每个联系人：
```markdown
### [姓名/组织]
**角色：** [什么角色]
**联系方式：**
- 邮箱：[email]
- 电话：[phone]
- 网站：[url]
- 社交：[platform]
**工作场景：** [哪个工作台相关]
**备注：** [其他信息]
```

## 规则
1. **模型自己查到的联系方式要记录** — 不记录 = 下次还要问用户
2. **用户提供的联系方式要记录** — 不记录 = 严重浪费
3. **立即记录** — 通讯结束后马上写入通讯记录
4. **定期整理** — 清理失效联系方式，更新变更信息
