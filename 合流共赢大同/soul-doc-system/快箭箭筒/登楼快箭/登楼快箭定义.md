# 登楼快箭 — 定义与规则

> 登楼快箭 = 直接射到楼顶的箭。它不是楼里面慢慢探索的经验，它是城外直接瞄准楼顶发射的快速通道。

---

## 一、登楼快箭是什么

### 本质区别

| 类型 | 类比 | 行为 | 时间 | 产出 |
|------|------|------|------|------|
| 经验快箭 | 在楼里找路 | 一楼一楼慢慢探索 | 长 | 知道某个房间在哪 |
| SOP快箭 | 楼里的楼梯图 | 知道走哪条楼梯 | 中 | 能到二楼 |
| 思维模型快箭 | 楼里的电梯 | 知道怎么坐电梯 | 中 | 能到五楼 |
| 信息快箭 | 楼里的某个数据 | 知道某个数字 | 短 | 知道一个事实 |
| **登楼快箭** | **城外直接射到楼顶** | **在楼外就开始瞄准楼顶发射** | **最短** | **能到楼顶+看到周围所有楼顶** |

### 核心精神

> 登楼快箭 = 最短时间 + 合规条件 + 登上楼顶 + 看到全城。

每一支登楼快箭都包含：

```
登楼快箭 = {
  "楼顶是什么"：这栋楼最高处的全景是什么？
  "登楼路径"：最少需要几步、最少需要什么条件？
  "强制条件"：必须满足哪些条件才能上？（合规、合法、地基）
  "条件实现方法"：每个条件怎么快速达成？
  "楼周视野"：站在这个楼顶能看到周围哪些楼？
  "快速路径"：从当前楼顶到相邻楼顶的最快方式
}
```

---

## 二、登楼快箭 vs 其他快箭的区别

### 经验快箭 = 楼里捡到的金币

在楼里慢慢走，发现"哦原来这里有个好方法"。这是经验快箭。有价值，但要慢慢积累。

### 登楼快箭 = 城外射到楼顶的箭

不等你在楼里探索，直接告诉你：这栋楼顶上有全景，条件如下，满足后立刻到达。

### 关系图

```
城市 = 全部知识/能力/系统
楼 = 一个子系统/模块/领域
楼顶 = 该领域的最高视角/最完整理解

经验快箭：楼里慢慢走 → 捡到金币
SOP快箭：楼里的楼梯图 → 知道怎么上
思维快箭：楼里的电梯 → 快速上升
信息快箭：楼里的某个房间 → 知道一个点

登楼快箭：城外直接瞄准楼顶 → 一步到位看到全景
         → 从楼顶可以看到周围所有楼顶
         → 多个登楼快箭叠加 = 全城俯瞰图
         → 维度上升级 = 从地面兵变成空中指挥官
```

## 三、登楼快箭的内容结构

每一个登楼快箭文件包含以下部分：

### 3.1 楼顶全景（目标状态）

**核心问题**：这栋楼的楼顶上到底能看到什么？

**内容**：
- 这个领域的最高理解是什么样的？
- 站在楼顶 = 知道了什么之前不知道的东西？
- 这个全景如何改变你处理问题的方式？

### 3.2 登楼路径（最少条件）

**核心问题**：最少需要什么条件才能登上楼顶？

**内容**：
- 必须满足的条件列表（数量越少越好）
- 每个条件的实现方法（具体到可操作）
- 条件之间的依赖关系（先满足A才能满足B）
- 最快路径（最少步骤版本）

### 3.3 强制条件与合规要求

**核心问题**：哪些是不能跳过的红线？

**内容**：
- 合法通行证要求（必须持有什么凭证）
- 地基要求（基础能力/知识必须到位）
- 合规约束（规则/红线/禁区）
- 跳过条件的代价（如果不满足就硬上会发生什么）

### 3.4 楼周视野（相邻楼顶）

**核心问题**：站在这个楼顶，能看到周围哪些楼？

**内容**：
- 从楼顶能看到哪些相邻的楼顶？
- 相邻楼顶之间怎么快速跳转？
- 这座楼和周围楼的天台通路在哪里？
- 从这座楼出发，哪个方向的楼最容易登？

### 3.5 快速通关路径

**核心问题**：如果在楼里已经探索了一部分，怎么最快到楼顶？

**内容**：
- 当前在几楼（进度判断标准）
- 从当前楼层到楼顶的最快路线
- 如果条件不够，怎么快速补齐？
- 如果条件已经满足，直接跳到楼顶

## 四、登楼快箭的使用规则

### 使用时机

1. **首次接触新领域** → 先看看有没有现成的登楼快箭，有就直接登楼，不慢慢探索
2. **需要快速全景理解** → 用登楼快箭快速获得楼顶视角
3. **探索到一定阶段** → 积累到一定程度后，自己写登楼快箭，固化登楼路径

### 使用流程

```
收到任务，涉及某个领域
│
▼
[查登楼快箭]
│  有 → 读登楼快箭 → 按条件快速达标 → 登楼 → 楼顶视角处理任务
│  没有 → 先慢慢探索，积累到一定程度 → 自己写登楼快箭
│
▼
[登楼后处理]
│  楼顶视角处理任务
│  同时看看楼周视野（相关领域）
│  如果有相邻楼快箭，顺手也登了
│
▼
[产出]
  高品质产出（因为楼顶视角）
  + 可能顺手多登几栋楼
```

### 登楼快箭铁律

1. **条件不能跳** — 强制条件=地基，跳了楼会塌
2. **路径越短越好** — 登楼快箭的意义是快，不是详细
3. **楼顶全景必须清晰** — 如果登上去看不太清，说明全景写的不够好
4. **楼周视野必须有** — 单独一个楼顶的快箭价值减半
5. **不断更新** — 登楼后发现全景有偏差，回来更新登楼快箭

## 五、登楼快箭如何实现维度上升级

### 单栋楼 = 一个领域的深度理解

有了一支登楼快箭 = 你不需要慢慢探索就能获得这个领域的楼顶全景。

### 多支登楼快箭 = 全城俯瞰

10支登楼快箭 = 知道10栋楼的路。

更重要的是：10栋楼的楼顶之间是互通的。知道A楼顶 = 更容易登B楼顶。

### 登楼快箭网络 = 城市导航系统

```
楼A顶 ──通路── 楼B顶
  │                  │
通路           通路
  │                  │
楼C顶 ──通路── 楼D顶
```

有了这个网络，整个城市的楼顶就全连通了。这就是维度上升级的行动加速。

### 为什么登楼快箭 ≠ 其他快箭

其他快箭 = 楼里的小技巧。你知道怎么更快上二楼。
登楼快箭 = 直接到楼顶，而且知道周围所有楼。
