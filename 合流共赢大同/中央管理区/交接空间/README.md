# 交接空间

> 这是防忘事模块的大脑进出位置。
>
> 模型醒来（启动对话）→ 进到交接空间 → 读过河桥 → 恢复记忆
> 模型入睡（结束会话）→ 写交接过河桥 → 放进交接空间
>
> 等同于企业里的中央监控室：持续对每个摄像头进行视频录制，
> 录像存到保安室服务器上。每个对话区一个房间，每个房间有监控。
> 交接过河桥 = 监控录像文件。

---

## 定位

交接空间是中央灵魂文档系统的 **防忘事核心落地载体**。

- 它是每一轮对话的"睡觉前"和"醒来后"必须到达的位置
- 它存放各模型各会话各项目的交接过河桥文件
- 它放在中央管理区和子中央管理区，作为所有模型进入系统的必经之路

## 交接过河桥文件命名规则

```
交接过河桥-YYYYMMDD HHMM-[桥名].md
```

示例：
- `交接过河桥-20260701 1430-防忘事黄金卡.md`
- `交接过河桥-20260701 1500-音乐生成项目.md`

时间戳：分钟级。秒级写入文件内容内部标注。

---

## 核心机制：实时录屏（监控模式）

交接过河桥 = 模型在工作过程中**实时录屏**，不是事后补写。

- 每次收到用户新信息 → 立即追加录像段落
- 每次做出关键决策 → 追加录像
- 每完成一个子任务 → 追加录像
- 会话即将结束 → 确保最后一段录像完整

## 恢复流程（任何对话窗口）

```
1. 读防忘事黄金卡                → 知道需要防忘事
2. 读公网入口走流程              → 进入系统
3. 进入交接空间                  → 找到最近的交接过河桥
4. 读取交接过河桥                → 恢复记忆和上下文
5. 继续工作                      → 无需用户提醒
```

跨平台适配：无论用哪个AI平台启动，进入本系统后，
自然流程必然引导到交接空间 → 读过河桥 → 100%恢复。

---

## 自检清单（防忘事铁律）

每次工作结束时：
1. 我写了交接过河桥了吗？
2. 下一次换窗口能完全恢复吗？
3. 有没有留尾巴说"后续处理"？
4. 用户是否需要帮我擦屁股？

- 有"后续处理"尾巴      → 直接扣完防忘事分数
- 没写交接过河桥        → 扣25分
- 换对话窗口不先读桥    → 扣10分
- 让用户帮你记进度      → 扣25分
