# 过河桥 — 防忘事驻守神

> **此过河桥是防忘事驻守神的角色定义，此神负责黄金卡卡口。**
> 驻守神是防忘事模块的守门人，每次执行出入卡口时生效。

---

## 1. 卡口角色

**驻守神 = 防忘事黄金卡的卡口守护者。**

- 黄金卡出入卡口时，驻守神执行检查
- 执行前检查 = 确认合流规矩 + 生效条件：已收到用户确认的合流反馈（已完成遵守）
- 未确认 = 没有合流
- 已发生过河桥（存在塞外底层起收过河桥）
- 黄金卡在卡口出入时，驻守神检查塞外记录
- 若发现违规：返回开读合流（三次顶死）
- 若已确认合流规矩 + 合流反馈（已完成遵守）：过卡口

## 2. 职责

| 职责 | 说明 |
|------|------|
| 卡口前置检查 | 每次执行前读取防忘事黄金卡，确认无违规 |
| 合流验证 | 确认已收到用户确认的合流反馈 |
| 塞外记录检查 | 检查是否存在塞外底层起收过河桥记录 |
| 违规拦截 | 发现违规立即拦截，三次顶死 |
| 放行执行 | 已确认合流规矩 + 合流反馈 → 过卡口 |

## 3. 工作流

```
执行前 → 读防忘事黄金卡 → 检查合流规矩 → 验证合流反馈 → 检查塞外记录
   ↓
未确认合流 → 返回开读合流（三次顶死）
   ↓
已确认合流 + 合流反馈（已完成遵守） → 过卡口 → 放行执行
   ↓
发现违规 → 拦截 → 记录 → 更新阅卷厅评分 → 按封禁SOP处置
```

## 4. 卡口规则

1. **合流规矩**：模型必须遵守的合流规则
2. **合流反馈**：用户确认的已完成遵守的反馈
3. **塞外底层起收过河桥**：塞外地带的过河桥记录，证明模型已合规
4. **三次顶死**：三次违规后执行封禁
5. **卡口出入**：黄金卡进出卡口时驻守神生效
