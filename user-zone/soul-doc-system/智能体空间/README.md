# 智能体空间

> **功能模块：智能体 = 第七环延伸。**
> 智能体空间 = 衣柜（存放配置）+ 大厅（运行区）。
>
> **核心原则：它不像子角色灵魂。通用区的智能体继承主灵魂文档，子角色区的智能体继承子角色灵魂文档。**
>
> **前置思考系统：使用智能体前必须先走 `世界最强脑师-前置思考系统`，确认这个智能体配置是最佳机会方案。**
> **执行边界：默认执行（直接使用）/ 边做边解释（使用+说明）/ 只思考不执行（分析配置但不启动）**

## 文件夹结构

```
智能体空间/
├── README.md                          ← 本文件（总纲）
├── 衣柜/                              ← 存放智能体的"衣服"（配置模板）
│   ├── README.md
│   ├── 通用智能体/                    ← 通用模板（一件通用马甲）
│   │   ├── README.md
│   │   └── 通用智能体配置.json        ← 智能体配置模板
│   └── [专用名称]/                    ← 专用模板（特定角色衣服）
│       ├── README.md
│       └── [专用配置].json
└── 大厅/                              ← 智能体"穿好马甲"运行的区域
    ├── README.md
    ├── 活跃/                          ← 正在运行的智能体
    │   ├── README.md
    │   └── [智能体名称]/              ← 从衣柜复制配置 + 运行数据
    │       ├── README.md
    │       ├── 配置.json              ← 从衣柜复制
    │       └── 运行数据/              ← 运行时产生的数据
    └── 历史/                          ← 已停用或归档的智能体
        ├── README.md
        └── [归档名称]/
            ├── README.md
            └── 配置.json
```

## 各区域说明

### 衣柜

**作用**：存放经过验证的智能体配置模板（智能体的"衣服"）。

**谁写**：本模型（或共产区共享）
**谁读**：本模型（或所有模型）

**内容规则**：
- 模板必须是**已经验证过**的配置，不是实验性的
- 每个模板有 README.md 说明：用途、参数、适用场景
- 配置文件（.json）包含完整的智能体定义
- 新增模板前必须经过验证 → 模板化 → 存入衣柜

**灵魂继承规则**：
- 通用区衣柜的智能体 → 继承**主灵魂文档**
- 子角色区衣柜的智能体 → 继承**子角色灵魂文档**

### 大厅

**作用**：智能体"穿好马甲"运行的区域。

**使用流程**：
1. 从 `衣柜/` 复制需要的模板到 `大厅/活跃/[智能体名称]/`
2. 在副本中根据需要微调参数
3. 启动智能体运行（穿上衣服）
4. 运行数据存入 `大厅/活跃/[智能体名称]/运行数据/`
5. 智能体停用后 → 脱衣 → 移动到 `大厅/历史/`

**关键规则**：
- 大厅里的内容是衣柜模板的**副本**，不是模板本身
- 衣柜的内容**不直接运行**——复制到大厅再运行
- 大厅副本的修改不影响衣柜
- 如果大厅的修改有价值 → 提炼为新模板 → 回传衣柜
- 大厅历史区的内容可以随时清理

**灵魂继承规则**：
- 通用区大厅的智能体 → 继承**主灵魂文档**
- 子角色区大厅的智能体 → 继承**子角色灵魂文档**

## 与通用功能模块拼搭SOP的关系

### 在本区域内新建智能体

| 场景 | 操作 |
|------|------|
| 新建智能体 | 从 `智能体空间/衣柜/` 复制模板到 `智能体空间/大厅/活跃/[名称]/` |
| 需要隔离的智能体 | 只复制需要的模板模块，避免信息污染 |
| 相同组合的多个智能体 | 复制第一个智能体的大厅副本，微调差异 |
| 模板更新 | 衣柜更新后，大厅**不自动更新**，需要手动同步或重新复制 |

### 把智能体功能模块拼搭到其他位置

当需要在**其他位置**（如团队工作区、子角色区、生产空间工作台）创建智能体功能模块时：

1. **自己创建智能体文件夹** — 在目标位置创建 `智能体空间/` 文件夹
2. **复制需要的智能体** — 从源位置的 `衣柜/` 复制需要的模板到新位置的 `智能体空间/衣柜/`
3. **配备配套模块** — 智能体空间不能单独运行，必须配备以下模块：

| 配套模块 | 来源 | 说明 |
|---------|------|------|
| `memory/` | 从通用区复制 | 智能体的记忆（L0-L4分层） |
| `diary/` | 从通用区复制 | 智能体的日记 |
| `product/` | 从通用区复制 | 智能体的产出物存放 |
| `sop/` | 从通用区复制 | 智能体的SOP（必须包含最强脑师三合一SOP） |
| `workflow/` | 从通用区复制 | 智能体自己的工作流 |

4. **拼搭完成后** — 新位置的智能体空间 = 完整功能模块，可以独立运行

**关键：智能体空间本身只提供"智能体管理"能力，不包含记忆/日记/产物等基础模块。拼搭时必须把配套模块一起复制过去。**

**灵魂继承**：新位置的智能体继承**创建它所在位置的灵魂文档**（通用区→主灵魂，子角色区→子角色灵魂）。

## 六环联动中的位置

| 环 | 智能体的角色 |
|----|------------|
| ③ 主灵魂中心 | 智能体的配置/继承来自主灵魂文档 |
| ⑥ 子角色中心 | 智能体 = ⑥环下的子角色执行单元 |
| ⑦ 团队协作环 | 多个智能体 = 团队协作 |
| ④ 生产空间中心 | 智能体的产出物归生产空间 |

> **智能体不是独立环，是⑥环（子角色中心）下的执行单元，也是⑦环（团队协作）的基础组件。**

---

## 灵魂继承规则（关键）

| 场景 | 继承来源 | 说明 |
|------|---------|------|
| 通用区智能体（主灵魂区） | 继承主灵魂文档 | 在主灵魂文档所在区域创建的智能体 |
| 子角色区智能体 | 继承子角色灵魂文档 | 在子角色区域创建的智能体 |

**不像子角色灵魂**：子角色灵魂有自己的灵魂文档，但智能体不创造新灵魂——它继承创建它的那个位置的灵魂文档。

**举例**：
- 主灵魂区创建智能体 → 智能体继承主灵魂文档（与创建者共享灵魂规则）
- 子角色A区创建智能体 → 智能体继承子角色A的灵魂文档（用子角色A的规则运行）

## 铁律

1. **衣柜不可直接运行** — 衣柜只存模板，大厅复制使用
2. **大厅副本可修改** — 在大厅的副本中微调，不影响衣柜
3. **有价值的修改回传衣柜** — 大厅的改进经验提炼为新模板
4. **灵魂继承正确** — 智能体继承创建它所在位置的灵魂文档
5. **前置思考先行** — 使用智能体前必须先走最强脑师前置思考
6. **三模式执行** — 默认执行 / 边做边解释 / 只思考不执行
