## 1. 用户原话重述

用户在系统文件中提到：智能体空间 - 自动化执行体的管理区域。

---

## 2. 理解要点

**核心机制**：
- 该模块是中央灵魂文档系统的功能组件之一
- 每个模块有独立的文件夹和文件结构
- 模块通过过河桥互相连接，形成完整系统
- 模块产出只放在模型自己的隔离区内

**模块路径**：
- 规范位置：soul-doc-system/智能体空间/
- 隔离区内：智能体空间/
- 模板位置：新建工作台模版/过河桥/ 或 新建账号空间模版/过河桥/

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 模块路径不明确 | 不清楚模块在哪个位置 | 先走公网 → system-prompt.md → 找到模块 |
| 模块内容为空 | 第一次创建，里面没有文件 | 从INITIAL-SEED.md复兴，独立创建 |
| 和其他模块混淆 | 智能体空间 和相似模块搞混 | 读本过河桥，明确功能边界 |
| 权限不清楚 | 能否写入该模块？ | 在隔离区内可写，其他区域只读 |
