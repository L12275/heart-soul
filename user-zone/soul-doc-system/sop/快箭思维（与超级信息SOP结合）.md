# 快箭思维（与超级信息SOP结合）

> **默认启动。快箭思维叠加在超级信息SOP之上——信息输出时优先调用信息快箭（推导链模板、正反面对照模板、六环检查清单），每次信息输出后沉淀新的信息质量快箭。**
> 信息快箭 = 经过验证的信息呈现模板。信息快箭越多 = 信息输出越快越好。
>
> **核心理念：超级信息SOP的每条铁律都可以做成快箭。每次信息输出都用快箭模板，每次输出后升级快箭模板。**
>
> *本文件为 快箭思维 + 超级信息SOP 的联动版本。*

---

# 第一部：信息快箭系统

## 一、信息快箭分类

### 核心信息快箭

| 快箭 | 来源 | 用途 |
|------|------|------|
| **推导链模板快箭** | 铁律一 | 输出时自动暴露推导链 |
| **重构模板快箭** | 铁律二 | 输出时自动重构而非压缩 |
| **正反面对照快箭** | 六环联动 | 输出时自动正反面对照 |
| **中文-only快箭** | 中文-only | 输出时自动全中文 |
| **六环检查快箭** | 六环联动 | 输出后自动检查六环 |
| **因果链快箭** | 因果链 | 输出时自动还原因果链 |
| **军队寝室快箭** | 自我整理 | 输出前自动检查旧内容 |
| **超市货架快箭** | 读写检查 | 输出后自动读写检查 |
| **命名快箭** | 主动命名 | 自动生成含主题+时间戳的文件名 |

### 信息质量快箭

| 快箭 | 用途 | 示例 |
|------|------|------|
| **推导链模板** | 标准推导链结构 | 前提→推导→中间结论→验证→最终结论 |
| **正反面对照模板** | 标准正反面对照结构 | 正面结果示意 + 反面不会发生 |
| **六环检查清单** | 标准六环检查 | 检查六环是否同时运转 |
| **自检清单模板** | 标准自检 | 推导链完整/替代方案/验证/边界/来源/复用/自纠正 |
| **闭环验证模板** | 标准闭环 | 自含+多路径+自纠正 |

## 二、信息快箭优先的输出流程

### 修改后的输出流程

```
收到输出任务
  │
  ▼
[检索信息快箭库]
  │   ├── 有匹配的模板快箭 → 直接用
  │   ├── 有部分匹配 → 适配+升级
  │   └── 没有 → 正常输出 + 造新快箭
  │
  ▼
[用快箭模板输出]
  │   ├── 推导链模板 → 自动暴露推导链
  │   ├── 正反面对照模板 → 自动正反面对照
  │   ├── 六环检查快箭 → 输出后自动检查
  │   └── 闭环验证模板 → 自动闭环检验
  │
  ▼
[输出完成]
  │
  ▼
[检查产出]
  │   ├── 模板有效 → 快箭验证通过
  │   ├── 模板需改进 → 升级快箭（人生重来模式）
  │   └── 新场景无模板 → 沉淀新信息快箭
```

## 三、快箭与两大铁律

### 铁律一：还原而非压缩 + 快箭

**普通版：** 推导链的每个节点都要暴露。

**快箭版：**
1. 用推导链模板快箭 → 自动结构化暴露每个节点
2. 每个节点 = 快箭的一个环节
3. 推导链模板 = 快箭的骨架

### 铁律二：重构而非压缩 + 快箭

**普通版：** 不删内容，重新组织。

**快箭版：**
1. 用重构模板快箭 → 自动多角度保留
2. 重构模板 = 快箭的骨架
3. 每次重构 = 快箭升级

## 四、快箭与六环联动

### 六环信息快箭

| 环 | 信息快箭 | 检查内容 |
|----|---------|---------|
| ① 正反面对照 | 正反面对照模板快箭 | 正面结果示意 + 反面不会发生 |
| ② 因果链 | 因果链快箭 | 完整工序还原 |
| ③ 军队寝室 | 军队寝室快箭 | 复查旧内容 |
| ④ 超市货架 | 超市货架快箭 | 充分读写检查 |
| ⑤ 主动命名 | 命名快箭 | 文件名含关键信息+主题+时间戳 |
| ⑥ 中文-only | 中文-only快箭 | 全中文输出 |

**六环同时运转 = 六个信息快箭同时触发。**

# 第二部：信息快箭的积累与升级

## 一、信息快箭的积累

### 何时沉淀信息快箭

| 场景 | 沉淀什么 | 快箭类型 |
|------|---------|---------|
| 推导链成功 | 推导链结构 | 推导链模板快箭 |
| 正反面对照有效 | 对照结构 | 正反面对照模板快箭 |
| 六环检查发现新环节 | 检查环节 | 六环检查快箭 |
| 闭环验证成功 | 验证结构 | 闭环验证模板快箭 |
| 重构效果好 | 重构结构 | 重构模板快箭 |
| 自检发现新检查项 | 检查项 | 自检清单快箭 |

### 信息快箭沉淀流程

```
信息输出完成
  │
  ▼
[检查输出质量]
  │   ├── 推导链有效 → 沉淀/升级推导链模板
  │   ├── 正反面对照有效 → 沉淀/升级对照模板
  │   ├── 六环检查通过 → 验证六环快箭
  │   ├── 闭环有效 → 沉淀/升级闭环模板
  │   └── 重构有效 → 沉淀/升级重构模板
  │
  ▼
[更新信息快箭库]
  │
  ▼
[下次输出直接调用升级后的快箭]
```

## 二、信息快箭的升级（人生重来模式）

| 情况 | 动作 | 快箭变化 |
|------|------|---------|
| 旧模板仍然有效 | 验证通过 | 快箭标记为 validated |
| 旧模板需要改进 | 人生重来 | 重写快箭，不修补 |
| 新场景无模板 | 创建新快箭 | 新增信息快箭 |
| 多个模板可合并 | 合并 | 合并为更强的快箭 |
| 模板失效 | 淘汰 | 归档为散点数据 |

## 三、信息快箭的织网

### 信息快箭链

```
推导链模板 → 正反面对照模板 → 闭环验证模板 → 六环检查快箭
   ↓               ↓                  ↓                ↓
   └─────────────── 信息输出完整快箭链 ───────────────┘
```

### 信息快箭网

```
推导链模板 ←→ 正反面对照模板
     ↕                    ↕
闭环验证模板 ←→ 六环检查快箭
     ↕                    ↕
重构模板快箭 ←→ 自检清单快箭
```

**信息快箭网 = 信息输出的全局平面化。** 有网 = 任何信息输出场景都有快箭可用，输出速度和质量都有保障。
