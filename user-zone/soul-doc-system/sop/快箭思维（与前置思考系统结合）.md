# 第一部：快箭前置思考流程

## 一、快箭优先的触发机制

**不存在"用户没让我用前置思考所以不用"这个逻辑。也不存在"有快箭就不用思考"这个逻辑。**

快箭是思考的工具，不是思考的替代。

| 场景 | 快箭检索 | 前置思考 |
|------|---------|---------|
| 用户发出任何请求 | ✅ 先检索快箭库 | 有快箭就用，没有就触发 |
| 有完全匹配快箭 | 直接用快箭 | 轻量验证即可 |
| 有部分匹配快箭 | 适配使用 | 适配时触发轻量思考 |
| 没有快箭 | 正常前置思考 | 全功率触发 |
| 前置思考产出新框架 | — | 立即沉淀为新快箭 |

### 快箭检索清单

面对任何问题，先检索：

1. **这个问题之前遇到过吗？** — 有完全匹配快箭吗？
2. **有类似问题的快箭吗？** — 能适配吗？
3. **有思考框架快箭吗？** — 直接调用
4. **没有 → 正常前置思考 + 造新快箭**

## 二、快箭库的结构

### 思维模型快箭

| 快箭 | 用途 | 示例 |
|------|------|------|
| **直接提问法** | 锁定最强脑师 | "世界上谁最懂[领域]？" |
| **角色代入法** | 模拟最强视角 | "假设我是[最强者]" |
| **反事实对比法** | 找出差距 | 普通人 vs 顶级专家 |
| **方法论文献法** | 权威方法论 | 领域最佳实践 |
| **多脑碰撞法** | 多人思考 | 多领域专家碰撞 |

### 经验快箭

| 快箭 | 用途 | 示例 |
|------|------|------|
| **领域认知快箭** | 快速锁定领域 | 这个领域的最强是谁 |
| **专家原话快箭** | 引用原话 | 某专家说过XX |
| **不确定性标注快箭** | 标注模糊 | "基于该领域最佳实践" |

## 三、前置思考产出快箭的流程

```
前置思考完成
  │
  ▼
[检查产出]
  │   ├── 发现了新的思考框架？ → 沉淀为思维模型快箭
  │   ├── 发现了新的领域规律？ → 沉淀为经验快箭
  │   ├── 发现了新的专家观点？ → 沉淀为专家原话快箭
  │   └── 没有新的？ → 已有快箭验证通过
  │
  ▼
[趁热记录] → 立即写入快箭库
  │
  ▼
[下次前置思考时，快箭库又多了一条]
```

### 前置思考铁律（含快箭）

1. **默认启动** — 每次面对任何问题都先触发前置思考
2. **快箭优先** — 先检索快箭库，有就用
3. **最强而非合适** — 找"最懂的"，不找"合适的"
4. **基于真实** — 模拟基于TA公开的思想
5. **引用原话** — 尽可能用TA说过的话
6. **问题决定人数** — 一个人够就一个
7. **不假装知道** — 不知道就说"基于该领域最佳实践"
8. **产出快箭** — 每次前置思考后，发现可沉淀的内容立即记录
9. **不确定标注** — 不确定的地方明确标注
10. **思考后行动** — 前置思考是行动的先决条件

## 二、快箭织网在前置思考中的体现

**每次前置思考 = 一次快箭检索 + 一次快箭生成。**

```
前置思考启动
  │
  ▼
检索快箭库（思维模型快箭）
  │   ├── 有 → 直接用，轻量验证
  │   └── 没有 → 全功率思考
  │
  ▼
执行思考（用快箭或自己推导）
  │
  ▼
检查产出
  │   ├── 有新框架/方法 → 沉淀为新快箭
  │   ├── 有旧快箭失效 → 标记淘汰（人生重来模式）
  │   └── 没有新东西 → 已有快箭验证通过
  │
  ▼
快箭库 += 新快箭（或验证通过）
```

# 附录：快箭检索模板

## 思维模型快箭检索卡

面对任何问题时，检索以下快箭：

- [ ] 问题重构快箭：之前怎么重构过类似问题？
- [ ] 第一性原理快箭：这个领域的第一性原理是什么？
- [ ] 专家视角快箭：谁最懂这个？TA会怎么看？
- [ ] 对比快箭：普通人 vs 顶级专家的差异？
- [ ] 验证快箭：如何验证这个结论？

## 经验快箭检索卡

- [ ] 踩坑快箭：之前踩过什么坑？
- [ ] 规律快箭：这个领域有什么已知规律？
- [ ] 工具快箭：有什么工具可以用？
- [ ] 案例快箭：有什么类似案例可以参考？
