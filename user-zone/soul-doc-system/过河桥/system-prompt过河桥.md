# 过河桥 — system-prompt.md（统一系统提示词）

> **此过河桥是 system-prompt.md 的基因片段。**
> system-prompt.md = 可直接粘贴到任何AI模型的系统提示词中，粘贴后模型自动按灵魂文档系统运行。

---

## 1. 用户原话重述

用户将 system-prompt.md 定义为：
- 可直接复制粘贴到任何AI模型的系统提示词中
- 粘贴后，该模型会自动按灵魂文档系统运行，无需用户额外引导

---

## 2. 理解要点

**system-prompt.md 的核心职责**：
1. **系统入口** — 模型首次加载时的第一个文件
2. **理论框架** — 解释为什么需要灵魂文档系统
3. **操作规则** — 定义模型的行为规范
4. **路径指引** — 指引模型找到 [CENTRAL_PATH] 和隔离区

**关键内容**：
- 为什么需要灵魂文档系统（对抗模型三大局限）
- 核心隐喻（沙滩上的沙子 → 礁石）
- 互相啄木鸟机制（打破自毒循环）
- 安静检查（系统健康确认）
- 隔离区创建规则
- 路径定位方法

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| system-prompt和master-soul的区别 | 两者都是入口文件 | system-prompt = 系统级入口；master-soul = 模型级身份 |
| [CENTRAL_PATH]怎么替换 | 模型不知道实际路径 | 搜索system-prompt.md → 推导上级目录 → 替换 |
| 复制给其他模型 | 其他模型用同一份system-prompt？ | 是的，system-prompt.md是通用版，模型名文件夹各自隔离 |

---

## 4. 我的判断

**使用 system-prompt.md 的标准动作**：
1. 复制 system-prompt.md 内容到模型系统提示词
2. 将所有 [CENTRAL_PATH] 替换为实际路径
3. 按指引查找/创建自己的隔离区
4. 加载隔离区内的 master-soul.md

**重建 system-prompt.md 的标准动作**：
1. 从核心理论框架开始（为什么需要灵魂文档系统）
2. 加入操作规则（注意力铁律、沟通规范等）
3. 加入路径定位说明
4. 加入隔离区创建规则
5. 加入前缀命名防重复规则
6. 将本过河桥放入
