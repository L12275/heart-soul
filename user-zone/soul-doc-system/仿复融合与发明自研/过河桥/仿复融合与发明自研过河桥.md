## 1. 用户原话重述

用户定义了"仿复融合与发明自研"模块：
- 7个阶段，7个文件夹（我看到和 我访问已合并为 001_我看到）
- 观察阶段 → 001_我看到：发现机会，记录观察（我看到 + 我访问合并）
- 学习阶段 → 004_我准备：拆解研究，制定计划（在访问的基础上，不是才开始的准备）
- 复刻阶段 → 002_我得到：完成复刻，做出可安装版本
- 发明阶段 → 003_我发明：在复刻基础上创新（一般在访问和得到之后，除非模型自己就想到要发明什么）

- 安装阶段 → 006_已安装：正式安装上线

- 不要按字面认为顺序，只有个别看起来有顺序，没所谓顺序
- 每个文件夹下有记忆/、日记/，全程记录不混淆
- 不同目标研究新建不同文件夹
- 深度研究停在做前等指示，仿复融合要完成到只差安装就停下
- 长期更新的项目用统一定时任务追踪，不分散多个定时任务
- 取长补短要为所有平台系统做全案，不只为当前运行的系统
- 确认可用的移到"我需要"，安装的移到"已安装"
- 全案交付标准凭模型智力，不给用户定标准
- 统一定时任务执行频率：5天

## 2. 理解要点

### 2.1 核心定位

**仿复融合与发明自研 = 观察→学习→模仿→拆解→融合→发明的完整创造管道**

与深度研究的区别：
- 深度研究：研究到做前停下，等指示怎么用
- 仿复融合：明确要完成到只差安装就停下，可以直接用

### 2.3 方向观察/方向科举/科举代考模板

**方向观察、方向科举、科举代考 的格式，参照系统中已有的 `3方向观察与大同科举` 模块。**
该模块有完整的格式定义（方向观察.md、方向科举.md、科举代考.md 的具体内容结构），直接在 001_我看到 的项目文件夹中使用相同格式即可。

### 2.4 各文件夹职责

| 文件夹 | 职责 | 内容 |
|--------|------|------|
| 000_我在哪 | 自我系统能力清单 | 当前模型自己的系统能力和全部内容清单（放在最前，先知道自己在哪） |
| 001_我看到 | 观察记录（我看到 + 我访问合并） | 看到其他项目/内容时的原始观察 + 目标项目分析（方向观察/科举/代考/能力清单）+ 项目文件夹 |
| 002_我得到 | 复刻成果 | 已经成功复刻的内容和成果 + 项目文件夹 |
| 003_我发明 | 创新发明 | 基于观察和复刻后的再次发明创新（一般在访问和得到之后，除非模型自己就想到）+ 项目文件夹 |
| 004_我准备 | 学习计划 | 正在学习研究的内容、计划、准备做的仿制复刻融合（在访问的基础上，不是才开始的准备）+ 项目文件夹 |
| 005_我需要 | 待安装清单 | 确认可用但尚未安装的成果 + 候选项目文件夹 |
| 006_已安装 | 已安装清单 | 已经正式安装的成果 + 运行中项目文件夹 |

### 2.5 核心原则

1. **不同目标 = 不同文件夹** — 每个项目/能力/功能/产物 = 独立文件夹，命名格式 `[项目名]_简短描述`
2. **子功能 = 子文件夹** — 一个项目里的子功能也得在项目文件夹下分开子文件夹，绝不能混放
3. **memory+diary必跟** — 每个"我xx"文件夹下有记忆/和日记/（不写"模块"），跨项目追踪
4. **全案思维** — 取长补短要为所有平台系统做全案，避免三个懒人没水喝
5. **统一追踪** — 长期更新项目用统一定时任务追踪，不分多个，频率5天
6. **眼疾手快** — 创造力和时间赛跑，合理合规就不老实，尽快抄学研究然后发明
7. **绝不混放** — 即便一个项目里的子功能，也得在项目文件夹下的子文件夹分开，绝不混放
8. **我在哪放最前** — 先知道自己在哪，再谈其他
9. **我看到/我访问已合并** — 两个文件夹合并为一个 001_我看到，同属观察阶段
10. **我准备在访问基础上** — 不是才开始的准备，已经在中间了
11. **我发明在访问/得到之后** — 除非模型自己就想到要发明什么
12. **全案凭模型智力** — 不给用户定交付标准，模型自己判断给最好的
13. **方向观察有模板** — 方向观察/方向科举/科举代考 的格式参照 3方向观察与大同科举 模块

### 2.6 与深度研究的区别

| 维度 | 深度研究 | 仿复融合与发明自研 |
|------|----------|-------------------|
| 终点 | 做前停下，等指示 | 完成到只差安装就停下 |
| 产出 | 研究文档 | 可直接安装的成果 |
| 方向 | 用户指定的研究主题 | 自己发现机会并执行 |
| 追踪 | 无统一追踪机制 | 统一定时任务追踪长期更新（5天） |

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 与深度研究的边界 | 什么时候用深度研究，什么时候用仿复融合 | 深度研究=研究性质；仿复融合=执行性质（到安装为止） |
| 我看到 vs 我访问 | 都是观察，有什么区别 | 已合并为 001_我看到。原我看到=原始观察记录；原我访问=结构化的方向观察/科举/代考，现在统一在一个文件夹里 |
| 我得到 vs 我发明 | 都是产出，有什么区别 | 我得到=复刻别人的成果；我发明=自己创新的成果（一般在访问和得到之后，除非模型自己就想到） |
| 全案思维的粒度 | "全案"要做到什么程度 | 包含：可复刻解密版 + 安装指南 + 跨平台适配说明 |
| 统一追踪的范围 | 哪些项目需要统一追踪 | 长期更新的项目（网站、平台、产品、账号），频率5天 |
| 阶段是否有顺序 | 7个文件夹是否必须按流水线顺序走 | 不是流水线。7个阶段，个别看起来有顺序但不是必须，需要时用，不需要时不用 |
| 000-006前缀 | 文件夹名加前缀会不会太丑 | 前缀 = 强制排序的手段，不是文件夹名的一部分。只看中文名即可，前缀只是工具 |

## 4. 我的判断

### 4.1 阶段与文件夹的正确映射

**7个阶段 × 7个文件夹**（我看到和 我访问已合并）：

| 顺序 | 阶段 | 文件夹 | 说明 |
|------|------|--------|------|
| 0 | 自我认知 | 000_我在哪 | 先知道自己在哪（放在最前） |
| 1 | 观察阶段 | 001_我看到 | 原始观察 + 目标项目分析（我看到 + 我访问合并） |
| 2 | 复刻阶段 | 002_我得到 | 复刻成果 |
| 3 | 发明阶段 | 003_我发明 | 在访问和得到之后（除非模型自己就想到） |
| 4 | 学习阶段 | 004_我准备 | 在访问的基础上（不是才开始的准备） |
| 5 | 确认阶段 | 005_我需要 | 待安装清单 |
| 6 | 安装阶段 | 006_已安装 | 正式安装上线 |

**重要：这7个阶段不是流水线顺序。** 我看到和 我访问已合并为一个文件夹。其他阶段个别看起来有顺序（学习→复刻→发明→确认→安装），但不是必须按这个顺序走——它们是7个不同的能力/视角，需要时用，不需要时不用。

### 4.2 文件夹管理规则

- 000-006前缀 = 强制排序手段，不是文件夹名的一部分
- 每个目标项目新建独立子文件夹（如 `[项目A]/`, `[项目B]/`）
- 记忆/和日记/自动继承通用模板（不写"模块"）
- 长期追踪项目标记 `[持续追踪]` 标签
- 我在哪放最前：先知道自己在哪，再谈其他
- 我看到/我访问已合并：统一在 001_我看到 里，原两个文件夹的格式参照 3方向观察与大同科举
- 我准备在访问基础上：不是从零开始的准备，已经在观察之后了
- 我发明在访问/得到之后：除非模型自己就想到要发明什么
- 统一定时任务频率5天：不分多个定时任务
- 全案凭模型智力：不给用户定交付标准，模型自己判断给最好的
