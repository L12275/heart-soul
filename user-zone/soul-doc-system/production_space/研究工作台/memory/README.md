## 深度理解：四驱动&四写与五环结构（理解后全新写）

灵魂文档系统 = 主（总）灵魂文档系统 + 子灵魂文档系统，每个系统都有通用区和工作台专区。

### 四驱动&四写（四个存放位置）

四个驱动轮 = 四个独立但互联的内容存放位置：
1. 主灵魂文档系统通用区（地平面）— 主角色共享的底层记忆池
2. 主灵魂文档系统工作台专区（专用层）— 主角色按领域的专用工作空间
3. 子灵魂文档系统通用区（地平面）— 子角色共享的底层记忆池
4. 子灵魂文档系统工作台专区（专用层）— 子角色按领域的专用工作空间

结构比喻（四驱车底盘）：
- 前轴 = 主系统通用区（左轮）+ 主系统工作台（右轮），两轮互相连接但各是一驱动的动力轮
- 后轴 = 子系统通用区（左轮）+ 子系统工作台（右轮），两轮互相连接但各是一驱动的动力轮
- 中轴/地盘 = 前轴与后轴在中间连接，四轮一起驱动整套系统

### 什么时候用几个驱动

- 只在主系统内工作 = 双驱动&双写（主通用区 + 主工作台）
- 只在子系统内工作 = 双驱动&双写（子通用区 + 子工作台）
- 主系统和子系统都涉及 = 四驱动&四写（四个位置都写）
- 只在某一个位置有关系 = 只写那一位置

### 五环结构速记

上部三环（交叉）：
- 左侧环 = 主系统工作台专区内容
- 右侧环 = 子系统工作台专区内容
- 中心环 = 四驱动&四写规则和检索思维（在两个地平面环交叉处的上面交叉）

下部两环（交叉，底部支撑）：
- 左下环 = 主系统通用区内容
- 右下环 = 子系统通用区内容

五环 = 两个地平面环在底部交叉支撑上面三个环，主/子工作台在两侧，中心是规则和检索思维。

### 双驱动&双写具体规则（适用任何一对通用区+工作台）

1. 一般工作时的通用记忆和日记 => 写到该系统的根目录通用记忆和日记文件夹（地平面级）
2. 如果与该系统的某专用工作台相关 => 除了写到通用区外，再写一份到对应工作台（专用区级）
3. 如果不与该系统的任何专用工作台相关 => 只需写到通用区
4. 如果只与该系统的某专用工作台相关，不与通用区相关 => 只写到对应工作台

### 总结

- 通用区记忆/日记 = 地平面支撑层，所有工作台的底层记忆池
- 工作台记忆/日记 = 地平面之上的专用层，更精准的内容
- 随时是双驱动&双写，只需写到正确的那一区
- 将内容放到四驱动&四写的正确位置后，检索时直接去对应驱动位置找，避免全局搜索
- 四驱动都找不到 = 再用旧方法全局搜索
