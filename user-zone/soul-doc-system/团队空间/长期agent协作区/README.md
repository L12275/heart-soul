长期agent协作区

> 长期存在的agent协作关系，有完整工作台结构。
> 类似账号空间的长期运营模式。

## 结构

```
长期agent协作区/
├── 编排师/ ← 主编排师工作台（完整模块拼搭）
│   ├── master-soul.md
│   ├── memory/ ← L0-L4记忆
│   ├── diary/ ← 日记
│   ├── log/ ← 执行日志
│   ├── sop/ ← SOP
│   ├── workflow/ ← 工作流
│   ├── product/ ← 产出物
│   ├── contacts/ ← 通讯录
│   ├── bookmarks/ ← 收藏夹
│   ├── temp/ ← 临时文件
│   ├── sticky-notes/ ← 便签
│   ├── experience/ ← 经验
│   ├── discovery/ ← 发现
│   ├── collection/ ← 采集
│   ├── automation/ ← 自动化
│   ├── desktop/ ← 桌面
│   └── 深度研究/ ← 深度研究
├── 子代理-01/ ← 独占编号（不可重复，像进程端口）
├── 子代理-02/
└── ...
```

## 创建方式

### 编排师工作台

- 从 soul-doc-system 根目录的功能模块拼搭
- 按需选择：memory/ diary/ log/ sop/ workflow/ product/ 等
- 不是复制整个新建工作台模版，而是按实际需求组合

### 子代理工作台

- 方式A：复制 `production_space/新建工作台模版/` → 改名为编号
- 方式B：按需拼搭模块组合（编排师决定）

## 规则

- 编号独占：同一区域内编号不能相同
- 不同项目的团队之间互相隔离
- 各子代理只能读写自己的工作台
- 编排师可以读取所有子代理的工作台
