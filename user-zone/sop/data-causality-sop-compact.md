数据因果链 SOP（浓缩版 · 自纠正闭环结构）

> 数据只是表现。没有工序配套证明因果的数据，都没看的必要。

---

## 1. 触发条件（出现任一 → 必须调用此SOP）

- 用户说"说完整"、"展示逻辑"、"为什么"
- 数据与预期不符（逻辑说A，实际结果B）
- 出现异常数据点
- 需要解释因果关系的任何场景

**强制顺序：走完流程 → 通过自查 → 输出结论。禁止跳过。**

---

## 2. 三步走

| 步骤 | 做什么 | 必须包含 |
|------|--------|---------|
| **① 工序还原** | 列出起点→终点的每一步 | 时间戳、横纵间隔、状态变量、决策点 |
| **② 因果标注** | 标出每步的数据变化和依赖关系 | 导致什么变化、下一步依赖什么、哪个分支没走 |
| **③ 矛盾排查** | 对比预期vs实际，找出矛盾 | 预期走哪条、实际走哪条、哪个条件判断出问题 |

---

## 3. 自纠正检查（7项）

在输出结论前，逐项检查：

- [ ] 列出了从起点到终点的**每一步**？
- [ ] 标注了每步的**时间点和间隔**？
- [ ] 记录了每个对象的**完整状态变化**？
- [ ] 标出了**决策分支**和走哪条路？
- [ ] 对比了**预期vs实际**？
- [ ] 找出了**矛盾点**？
- [ ] 没有跳过任何**代码未走到的分支**？

---

## 4. 闭环回路（自纠正的核心）

```
       ┌──────────────────────────────┐
       │   检查清单（7项）全通过？      │
       └──────────┬───────────────────┘
                  │
          ┌───────┴───────┐
          │               │
       是│               │否
          │               │
    ┌─────▼─────┐  ┌──────▼──────┐
    │ 输出结论  │  │ 回到 Step 1 │
    └──────────┘  │ 完整重做    │
                  │ （不是补一句│
                  │  就过）     │
                  └─────────────┘
```

**关键规则：**
- 检查出一项未通过 → 不是补这一项，是**回到 Step 1 完整重做**
- 重做后再次进入 Step 2→Step 3→自纠正检查
- 循环直到7项全通过，才允许输出结论
- 这个"做→检查→发现错→重做→再检查"的循环就是**闭环**
- 缺了任何一步都叫开环，开环就是跑偏

---

## 5. 红线（绝对禁止）

| 禁止 | 反例 → 正确做法 |
|------|------|
| ❌ 只给结论不给过程 | "429=16，成功率84.8%"→先列出这16个怎么来的 |
| ❌ 跳过时序只说静态状态 | "A被标记"→"A在第N秒被操作，cooldownUntil=..." |
| ❌ 不标注间隔 | "对象被切换"→"横向冷却到期前不可选中" |
| ❌ 用猜测替代因果 | "可能是限流"→"排除A→排除B→剩余C→证据是..." |
| ❌ 逻辑只看结果不看分支 | "逻辑是对的"→"第X行走了A分支而非B，因为..." |
| ❌ 检查出漏项只补一句 | Step 1缺了 → 回去完整重做，不是补一句"间隔是400ms" |
