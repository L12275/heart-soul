记忆系统

> 参考GA（GenericAgent）记忆系统：https://github.com/lsdefine/GenericAgent
> 记忆是实际运行与验证确认后的正确记忆。
> 与日记的区别：日记是当天实时事件过程等记录、不一定是正确记忆、是纪实。

---

## 5层记忆架构

```
L0 硬编码核心     → system prompt中，每轮自动加载，不可修改
L1 洞察索引       → memory/context/L1.md，≤30行，只记"有什么"和"去哪找"
L2 公理层         → memory/context/L2.md，稳定知识，遵守3条公理
L3 任务技能/SOP   → sop/ 目录，可复用流程，每3次提炼一次
L4 会话归档       → log/年/月/日/，按时间轴存放已完成会话
```

## 记忆与日记的分工

| 类型 | 内容 | 准确性 | 格式 |
|------|------|--------|------|
| 记忆（memory/） | 验证确认后的正确记忆 | ✅ 已验证 | context/L0-L4.md + daily/天级快照 |
| 日记（diary/） | 当天实时事件过程 | ⚠️ 纪实，可能不完整 | 时间轴：年/月/日/日期.md |

## 天级记忆

- 路径：`memory/daily/年/月/日/日期.md`
- 每天一份记忆快照，记录当日关键记忆
- 记忆 = 实际运行与验证确认后的正确记忆

## 3条公理（必须遵守）

1. **No Execution, No Memory** — 没实际做过的事，不写进记忆
2. **不可删除** — 已验证数据只能压缩/迁移，不能删除
3. **禁易变状态** — 不存"当前次数""最近时间"这类易变数据

## 快速上手

1. **L0**：不是文件，是system prompt中的硬编码约束
2. **L1**：打开 `memory/context/L1.md`，按格式填写导航索引（≤30行）
3. **L2**：只存已验证的事实，标来源，不存易变数据
4. **L3**：先查L1找SOP，按SOP执行，3次后提炼新SOP
5. **L4**：工作结束后，整理日志，按时间轴归档到 `log/年/月/日/`
