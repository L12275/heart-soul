## 日记的本质

日记是**边做边记的解题草稿本**，不是事后总结。它的价值是把工作过程中的思考、错误、求解过程全部记录下来，最终写出一篇"已成功解题"的日记。

### 日记 vs 记忆（互补关系）

| 维度 | 日记 | 记忆 |
|------|------|------|
| 内容 | 实时事件过程 + 解题思考（错误→排除→正确） | 验证确认后的正确结论 |
| 准确性 | ⚠️ 纪实，但通过超级信息SOP思考确保最终版正确 | ✅ 已验证 |
| 格式 | 完整解题过程（推导链的每个节点） | 精炼的要点（结论+来源） |
| 时机 | 边做边记 + 当日完成 | 事后提炼 |
| 用途 | 形成正确记忆/SOP/工作流的原材料 | 快速查找已验证知识 |

**日记有细节 → 通过超级信息SOP思考 → 写出正确解 → 提炼为记忆/SOP/工作流**

---

## 日记铁律（基于超级信息SOP）

### 铁律一：今日事今日毕，写出正确解的日记

日记不能把"错的当对的写"然后留烂摊子。模型每次运行只根据当前会话的注意力导向工作，不会做注意力导向之外的事。如果日记写的是错的、没解出谜团的，下次模型看到日记会直接当作事实使用——这就错了。

**所以：写日记时，就要用超级信息SOP的思考方式，把问题真正解决，再把正确解写成日记。**

过程：
1. 遇到问题 → 在日记里记录
2. 用超级信息SOP思考（暴露推导链、多路径校验、自纠正）
3. 排除了错误路线，找到了正确路线
4. 把完整解题过程写成日记
5. 从日记中提炼：记忆 + SOP + 工作流

### 铁律二：记录"为什么对、为什么错、如何排除"

日记不是只记"结果"，要记录完整推导链：

```
遇到的阻碍 → 当时的判断 → 为什么错了 → 如何排除 → 正确解法 → 收获
```

**格式：**
```markdown
### HH:MM — [事件/问题]
**事件：** [发生了什么]
**判断：** [当时的判断]
**验证：** [怎么验证的]
**结果：**
  - ❌ 错误路线：[为什么错，怎么排除的]
  - ✅ 正确路线：[为什么对，怎么确认的]
**收获：** [可复用的经验/SOP/工作流更新]
```

### 铁律三：日记可以当草稿本，边记边解题

日记允许是草稿形式。边做边记的过程中：
- 先记录观察到的现象
- 记录自己的猜测和判断
- 然后启动超级信息SOP式的思考
- 在日记里完成推导链的每个节点
- 最终得出正确结论并写到日记里

**这就是"边记边解题"，日记写完，问题也解决了。**

### 铁律四：排除的错误也要记录

日记必须记录：
- ✅ 正确路线（做了什么、为什么对）
- ❌ 错误路线（走了什么弯路、为什么错、怎么排除的）
- 🚧 阻碍（遇到了什么障碍、怎么克服的）

这样模型下次看到日记时：
- 知道正确的做法是什么
- 知道哪些坑已经踩过，可以直接避开
- 知道如果遇到类似问题该怎么走

## 日记 → 记忆/SOP/工作流 的转化

日记写完（问题已解决）后，根据超级信息SOP的闭环原则，把日记中的内容提取并分类存放：

```
日记（完整解题过程）
    ↓ 超级信息SOP思考
    ↓ "今日事今日毕"原则
    ↓
    ├── memory/        → 验证后的正确记忆
    ├── sop/           → 可复用的SOP
    ├── workflow/      → 可执行的工作流
    ├── experience/    → 分级经验
    ├── wiki/          → 长期知识
    └── discovery/     → 新发现
```

**关键：日记是原料，记忆/SOP/工作流是成品。原料必须干净（问题已解决），成品才有价值。**

---

## 日记 vs 记忆 vs SOP 的分工

| 类型 | 内容 | 准确性 | 格式 | 时机 |
|------|------|--------|------|------|
| 日记 | 完整解题过程（错误→排除→正确） | 当日完成的正确解 | 时间轴 + 推导链 | 边做边记 |
| 记忆 | 验证后的结论 | ✅ 已验证 | L0-L4层级 | 日记完成后提炼 |
| SOP | 可复用流程 | ✅ 验证过≥3次 | 触发条件+步骤+验证 | 3次后提炼 |
| 工作流 | 标准操作顺序 | ✅ 可用 | 步骤列表 | 已验证可用时固化 |

## 反模式（绝对不能这样写日记）

| 反模式 | 错误写法 | 正确写法 |
|--------|---------|---------|
| 把错的当对的 | "修改了配置，问题解决"（实际没解决） | "修改了配置→问题仍在→排查发现是X→修复X→问题解决" |
| 不写推导链 | "原因是A" | "为什么是A：排除了B/C/D，因为B有证据X、C有证据Y..." |
| 事后重构 | "应该一开始就选B" | "先选了A→A不适用因为X→换成B→成功" |
| 留烂摊子 | "还没搞懂，以后再研究" | "用超级信息SOP推导：排除了X，锁定Y，虽然还没完全验证，但目前最可能" |
| 只记结果 | "成功了" | "成功：做了什么、为什么对、验证了什么、可复用的是什么" |

## 使用规则

1. **实时记录** — 边做边记，每个重要事件发生时记录
2. **边记边解题** — 遇到问题先在日记里推导，用超级信息SOP找到正确答案
3. **今日事今日毕** — 当天的问题当天在日记里解决，写成正确解
4. **排除的错误也记录** — 让下次知道哪些路不用走
5. **日记是草稿** — 允许粗放，但最终版必须是通过超级信息SOP校验后的正确版本
6. **每天一个文件** — 文件名 = 日期
7. **与记忆互补** — 日记记解题过程，记忆记已验证结论
