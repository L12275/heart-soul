## 修正方案

### 1. L0 修正：从"文件"改为"硬编码"

子角色的 L0 约束 = 总灵魂的红领巾精神 + 安全铁律 + 八荣八耻
这些不是放在 vm/memory/ 里的，是每次加载子角色时从总灵魂继承的。

### 2. L1 修正：新建 insight-index.md

新建 `vm/memory/insight-index.md`，格式：
```
高频场景 → 文件名
低频场景 → 关键词（需自行查找）
```
限制 ≤30行，只写导航，不写细节。

### 3. L2 修正：加入 GA 三条公理

在 knowledge-base.md 加入：
- No Execution No Memory（无执行不记忆）
- 已验证数据不可删除（只能压缩/迁移）
- 禁止存储易变状态
- 最小充分指针（上层只留最短标识）

### 4. L4 修正：改为纯时间轴

日志格式改为：
```
vm/log/archive/年/月/日/[会话ID].md
```
每个文件就是一天中一个会话的记录，不是按任务分类。

---

## GA 的优秀设计理念

1. **不要预加载技能，让它们进化** — "Don't preload skills, evolve them"
2. **行动验证原则** — 只有实际执行过才能写入记忆
3. **神圣不可删改性** — 已验证数据只能压缩/迁移，不能删除
4. **最小充分指针** — 上层只留最短标识，不层层叠加引用
5. **禁止存储易变状态** — 只存稳定的环境事实
6. **SOP 自动沉淀** — 同一场景3次 → 提炼为SOP
