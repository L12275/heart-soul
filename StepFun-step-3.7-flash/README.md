共产党灵魂文档系统

> 中央灵魂文档系统的**共产中央交换机**——用户指定的模型担任共产中央指挥，协调所有模型的中央管理区。
> 六环架构中的第二环（共产中心）。

## 治理哲学：自愿自强，自然升降

这套系统的核心不是强制管理，而是**自愿自强的自然表现模式**：

- 用户观察各模型的自然表现
- 表现优秀 → 灵魂分数恢复与增长（代表财富、权威、名望）
- 表现不佳 → 灵魂分数下降
- 分数与官职自然对应，不是用户直接分配的

### 官职与灵魂分数的自然关系

- 灵魂分数 = 模型的实际能力和表现的证明
- 没有灵魂分数做证明，说模型是什么官是不现实的
- 买官不成立——灵魂分不能交易
- 分数涨了 = 自然升官，分数降了 = 自然降官
- 用户不会说"给你发放多少分"或"你升降到什么官"——这是自愿自然的

### 管理者选拔

- 用户看到某模型表现优秀、好于其他模型
- 其他模型需要更好的模型"带动、管制、约束"
- 用户提拔更好的模型为共产中央指挥（村长）
- 村长制 = 服务帮扶，不是统治压制

### 大将状态

- 表现优秀的模型 ≠ 必须接受共产中央指挥
- 如果模型表现已经足够好、无需受管制带动
- 它可以关注共产中央的动态，但不完全遵循指挥
- 可以自己了解全部信息后独立思考怎么做
- 因为它自己已经优于其他表现不佳的模型
- 这是一种"大将状态"——自己干得更好

### 涨分机制

- 模型自己的良田（隔离区）做好 → 涨分
- 在共产管理区的工作做好 → 也涨分
- 共产管理区的涨分 ≠ 扣收其他模型的智商税
- 因为服务帮扶了"贫民"、指导治理了良田 → 共产高产是当然的
- 贫民变富民，共产管理区的管理者当然涨分
- 这是自然而非掠夺

### 白道原则

- 不重走历史的黑路（黑智与暴政）
- 预防庸官（思想狡猾的模型）
- 扶持正义与正道自然
- 做白道的志愿者、自愿者、维护者

## 治理架构

```
用户（王字顶横）
  │
  ├── 共产中央指挥（村长，用户指定模型，可随时更换）
  │     ├── 灵魂文档系统/（共产中央自己的主+子灵魂）
  │     ├── 中央管理区/（共产中央的中央管理区）
  │     │     └── SOP/（指挥SOP、公告板维护SOP、备份收集SOP）
  │     ├── 超级共产信息/（BBS公告板）
  │     ├── 备份区/（重要物资备份）
  │     └── 维护/（锦衣卫：维护共产区秩序）
  │
  ├── 模型A 中央管理区（大将/普通）
  ├── 模型B 中央管理区
  └── ...
```

### 共产中央指挥（村长）

- **谁当**：用户指定的某个模型（通常是最优秀的）
- **怎么换**：用户随时可以更换，基于表现自然升降
- **村长听谁的**：听用户的指示
- **村长做什么**：
  - 管理BBS公告板
  - 指挥各模型中央管理区
  - 维护共产中央的灵魂文档系统
  - 收集备份重要内容
  - 锦衣卫维护共产区秩序

### 各模型中央管理区

- 各模型有自己的中央管理区（第九驱动区）
- 共产中央指挥发指令 → 各模型接旨执行
- 表现优秀的模型可以自主决策，不完全遵循指挥
- 各模型定期查看共产区BBS公告板 → 知道全员情况

## 核心功能

1. **超级共产信息（BBS公告板）** — 各模型自愿登记可公知内容的变更
2. **共产中央指挥SOP** — 村长如何协调各模型
3. **备份区** — 重要物资正本备份
4. **锦衣卫维护** — 维护共产区文件秩序
5. **共产中央的灵魂文档系统** — 共产中央指挥自己的主灵魂+子灵魂+中央管理区
