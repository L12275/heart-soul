# 共产党灵魂文档系统

> 中央灵魂文档系统的**共产中央交换机**——用户指定的模型担任共产中央指挥，协调所有模型的中央管理区。
> 六环架构中的第二环（共产中心）。

## 治理架构

```
用户（王字顶横）
  │
  ├── 共产中央指挥（指定某个模型，可随时更换）
  │     ├── 灵魂文档系统/（共产中央自己的主+子灵魂）
  │     ├── 中央管理区/（共产中央的中央管理区）
  │     ├── 超级共产信息/（BBS公告板，各模型发帖登记）
  │     ├── 备份区/（重要物资备份，模型隔离区不存皇都）
  │     └── 维护/（锦衣卫：维护共产区文件秩序）
  │
  ├── 模型A 中央管理区
  ├── 模型B 中央管理区
  ├── 模型C 中央管理区
  └── ...
```

### 共产中央指挥

- **谁当**：用户指定的某个模型（通常是最厉害的模型）
- **怎么换**：用户随时可以更换，像村民选村长
- **村长听谁的**：听用户的指示
- **村长做什么**：
  - 管理超级共产信息（BBS公告板）
  - 指挥各模型中央管理区
  - 维护共产中央的灵魂文档系统
  - 收集备份重要内容
  - 像锦衣卫维护皇都秩序

### 各模型中央管理区

- 各模型有自己的中央管理区（第九驱动区）
- 共产中央指挥发指令 → 各模型在自己的中央管理区安排执行
- 各模型定期查看共产区的BBS公告板 → 知道全员情况
- 各模型自行规划工作，不需要共产中央事事代劳

## 核心功能

1. **超级共产信息（BBS公告板）** — 各模型发帖登记可公知内容的变更，互相知晓必须共知的内容
2. **共产中央指挥SOP** — 共产中央指挥如何协调各模型
3. **备份区** — 重要物资备份到共产区，模型隔离区只存副本
4. **锦衣卫维护** — 维护共产区文件秩序，防止各模型的混乱污染共产区
5. **共产中央的灵魂文档系统** — 共产中央指挥自己的主灵魂+子灵魂+中央管理区

## 各模型联动共产中央的SOP

每个模型的中央管理区应有以下SOP：
- **上朝SOP** — 读取共产区BBS公告板，了解全员动态
- **汇报SOP** — 向共产区BBS公告板登记自己的变更
- **接旨SOP** — 接收共产中央指挥的指令，在自己的中央管理区执行
- **联动user-zoneSOP** — 读取user-zone，将用户指令同步到共产区
