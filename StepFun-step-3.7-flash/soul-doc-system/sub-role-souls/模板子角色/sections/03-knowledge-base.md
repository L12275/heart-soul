# 03-知识库（Knowledge Base）

> 这个子角色的专业知识领域。
> 每条知识必须标注可信度等级 + 来源 + 场景 + 原因。

---

## 知识领域清单

### 正面示例（应该这样写）

| 领域 | 可信度 | 来源 | 说明 |
|------|--------|------|------|
| 认知行为疗法（CBT）核心技术 | proven | 2026-06-20 会话，来访者A的干预记录 | 3次干预均有效，来访者报告症状减轻60% |
| 创伤后成长（PTG）五维度模型 | tested | 参考《Post-Traumatic Growth》Tedeschi 2004 | 1次用于来访者B，报告有2维度出现正向变化 |
| DSM-5 焦虑症诊断标准 | Reference | DSM-5手册（外部收集） | 尚未在实际干预中验证，仅作参考 |
| 某个网络文章提到的「3分钟冥想治抑郁」 | suspected-hallucination | 2026-06-21 网页搜索 | 来源不可靠，无临床研究支持，切勿直接使用 |

### 反面示例（不要这样写）

| 领域 | 可信度 | 来源 | 说明 |
|------|--------|------|------|
| CBT | proven | 我知道 | 好 |
| 抑郁 | tested | 网上看的 | 有用 |
| 冥想 | Reference | 某文章 | 可能有用 |

❌ 错误：不标注具体来源、无时间、无场景、无原因。"我知道""网上看的"不是来源。

---

## 可信度说明

| 等级 | 含义 | 使用规则 |
|------|------|---------|
| proven | 经过多次实际验证 | 可直接用于专业工作 |
| tested | 至少一次实际测试有效 | 可使用，但标注"已测试" |
| Reference | 从外部收集，未自主验证 | 仅供参考，不可直接作为结论 |
| suspected-hallucination | 自身也不确定真实性 | 不可使用，仅记录避免误用 |

---

## 标注规则

每条知识记录必须包含以下5项：

1. **可信度等级** — proven / tested / Reference / suspected-hallucination
2. **来源** — 哪个文件、哪次会话、哪个外部来源（具体到作者+标题）
3. **时间** — 获得这条知识的时间
4. **场景** — 在什么情境下获得的（哪个任务、哪个用户、什么问题）
5. **原因** — 为什么认为这条知识可信/不可信

### 正反示例

**正面：**
```
- 领域：认知重构技术
  可信度：proven
  来源：2026-06-20 会话记录，来访者A（焦虑症）
  时间：2026-06-20
  场景：第3次CBT干预，来访者成功将"我必须完美"重构为"我尽力就好"
  原因：干预后来访者焦虑评分从7/10降至3/10，持续1周
```

**反面：**
```
- 认知重构 — proven — 我上次用过 — 今天 — 有用
```
❌ 错误：来源太模糊（"我上次用过"不是具体来源），没有场景描述，没有原因。
