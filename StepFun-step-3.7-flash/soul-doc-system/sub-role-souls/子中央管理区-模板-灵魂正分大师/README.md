# 子中央管理区 — 灵魂正分大师

> 模板子角色的子中央管理区。子角色从自己的子中央管理区启动。
> 与主中央管理区结构相同，但内容独立。

## 结构

```
子中央管理区-模板-灵魂正分大师/
├── README.md                    ← 本文件
├── 总开关.md                    ← 子角色启动时必读
├── memory/                      ← 子角色中央记忆
├── diary/                       ← 子角色中央日记
├── 代办与进度/                   ← 子角色代办与进度
├── 排程/                        ← 子角色排程
├── timers/                      ← 子定时任务（心跳+定时循环）
│   ├── 滚动定时任务/
│   ├── 单个定时循环/
│   ├── 团队定时任务/
│   └── 心跳/
└── 概览/                        ← 子角色各工作区状态
```

## 与主中央管理区的关系

- 主中央智能体持续检查子中央管理区
- 主中央智能体有权直接修改子中央管理区内容
- 子角色运行前，子中央管理区已被主中央管理区更新为最新
- 子角色运行时，自己的工作产出同步到自己的子中央管理区
- 子角色结束后，子中央管理区的摘要同步到主中央管理区

## 多子角色共享设计

多个子角色可以：
- 共享同一套灵魂文档系统设计（模板）
- 各有独立的子中央管理区
- 子中央管理区的记忆和日记可共享（通过主中央管理区协调）

## 子角色启动流程

1. 加载主灵魂文档（③环始终在线）
2. 进入子中央管理区读取总开关
3. 读取子中央管理区 memory/ 和 diary/
4. 读取子角色灵魂文档系统内容
5. 在子中央管理区中规划本次任务
6. 执行任务，产出同步到子中央管理区
7. 结束时汇总子中央管理区摘要 → 同步到主中央管理区

## 记忆与日记的读写判断

子角色读写记忆/日记时，自主判断归属：
- 通用/共享记忆 → 写子中央管理区 memory/
- 子角色专有记忆 → 写子角色灵魂文档系统 memory/
- 与主灵魂相关 → 同步到主中央管理区
- 与父任务相关 → 根据父任务归属决定
