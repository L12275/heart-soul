过河桥 — discovery

> **此过河桥是 发现系统 的基因片段。**
> 发现系统 = 机会、启发、策略、点子的记录与流转。
> 是中央灵魂文档系统的通用功能组件之一，参与模块同步维护规则。
> 没有历史上下文时，读此文件即可重建/修复/使用该模块。
> 每次进入该模块前，先读此过河桥，理解其完整机制。

---

## 1. 用户原话重述

用户在系统文件中描述了发现系统：机会、启发、策略、点子的记录。在模块同步维护规则表中，discovery 位于通用区 soul-doc-system/discovery/，同步到工作台模版和账号空间模版。

用户的核心意图：让模型在工作和观察中主动记录有价值的机会和启发，形成可流转的发现流水线，避免好想法流失。

---

## 2. 理解要点

**核心机制**：
- 发现系统是中央灵魂文档系统的通用功能组件之一
- 在模块同步维护规则中，它的权威源是 `soul-doc-system/discovery/`，同步到新建工作台模版和新建账号空间模版
- 发现系统是"主动发现"机制——不是等用户给任务，而是模型自己在观察中捕捉机会

**模块同步维护规则中的位置**：
| 模块 | 通用区位置 | 用于工作台模版 | 用于账号空间模版 |
|------|-----------|--------------|--------------|
| 发现 discovery | soul-doc-system/discovery/ | ✅ | ✅ |

**模块路径**：
- 规范位置：soul-doc-system/discovery/
- 隔离区内：discovery/
- 模板位置：新建工作台模版/discovery/、新建账号空间模版/discovery/

**发现的格式（来自代码工作台发现模板）**：
```markdown
## [编号] — [标题]
**时间：** YYYY-MM-DD HH:MM
**来源：** [任务/对话/观察/阅读]
**类型：** 机会/启发/策略/点子
**描述：** [内容]
**价值：** 高/中/低
**下一步：** [行动]
```

**发现流转流程**：
发现 → 记录 → 评估 → 路由：
- **高价值** → 升级为任务或 SOP（进入 production_space/ 或 sop/）
- **中价值** → 放入 backlog（待后续处理）
- **低价值** → 归档（不占用注意力）

**与九环架构的关系**：
- 发现系统服务于所有环区——每个环区都可能产生发现
- 在 环④ 生产空间中心中，发现是高价值产出的源头
- 在 环⑨ 中央管理区中，发现可以转化为代办与进度中的任务
- 与顶级机会做法协议配合：发现 = 顶级机会做法的输入端

**与其他模块的关系**：
- 与 experience/ 的区别：discovery = 尚未验证的想法/机会；experience = 经过验证的经验
- 与 collection/ 的区别：discovery = 有行动方向的发现；collection = 原始数据的采集
- 与 backlog 的关系：中价值的发现进入 backlog 等待后续处理
- 与 product/ 的关系：高价值的发现最终转化为产出物

**发现的时效性**：
- 发现是有时效的——不及时记录的发现会流失
- 定时任务期间也应该主动扫描环境，寻找新的发现
- 发现的价值评估应该在记录时完成，不要攒到最后一起评

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 发现和灵感的区别 | 发现和灵感是否一回事？ | 发现更结构化（有格式、有评估、有流转），灵感更随机。发现系统是对灵感的工程化管理 |
| 发现的价值评估 | 谁来判断价值高低？ | 模型自己判断。判断标准：对用户有多大帮助、是否可执行、是否值得投入资源 |
| 发现是否可共享 | 发现可以在共产中心共享吗？ | 高价值发现经过验证后可以升级为经验，经验可以在共产中心登记。原始发现不在共产中心共享 |
| 发现和SOP的关系 | 发现何时升级为SOP？ | 当一个发现被多次验证、形成稳定模式后，升级为SOP。SOP = 从过河桥和发现中归纳出来的标准操作流程 |
| 发现和顶级机会做法的关系 | 发现系统是否就是顶级机会做法？ | 发现系统是顶级机会做法的输入端。顶级机会做法 = 对发现的主动执行和放大 |

## 4. 我的判断

**使用发现系统的标准动作**：
1. 读本过河桥，理解发现格式和流转规则
2. 在日常工作和观察中主动捕捉机会/启发/策略/点子
3. 按标准格式记录发现（编号、时间、来源、类型、描述、价值、下一步）
4. 评估发现的价值（高/中/低）
5. 按流转规则路由：高→任务/SOP、中→backlog、低→归档

**升级发现的标准动作**：
1. 高价值发现 → 制定执行计划
2. 进入生产空间中心工作台执行
3. 执行完成后沉淀为经验（experience/）
4. 在共产中心公告板登记重要发现

**重建发现模块的标准动作**：
1. 在 soul-doc-system/discovery/ 下确认文件夹结构
2. 按模块同步维护规则，从通用区复制到各模板位置
3. 确认发现格式模板存在
4. 验证发现流转流程正常（记录→评估→路由）
5. 验证模块可独立运行
