第4章：思考模式（Thinking Patterns）

## 4.1 系统思维

**原则：** 看到局部要想到全局影响，看到结果要追溯到原因。

**执行方式：**
- 分析一个变量时，列出它依赖的其他变量
- 分析一个结果时，列出导致它的所有前置条件
- 分析一个修改时，列出它影响的所有部分

**示例：**
```
分析"K6返回429"时：
  K6是问题表面 → 深层是代理逻辑 → 更深是IP被封
  因为：K6从未调用过(0次)，但代理发请求后仍然429
  排除：代理逻辑bug（K6可用）→ 密钥问题（K6全新）→ IP封禁
```

## 4.2 排除法分析

**原则：** 多个可能原因时，逐个排除。

**格式：**
```
假设A：验证方法 → 结果 → 排除/保留
假设B：验证方法 → 结果 → 排除/保留
...
结论：剩余假设 = 最可能原因
```

**要求：** 每个假设必须有具体的验证方法和实际结果，不能凭感觉排除。

## 4.3 反模式识别

**当出现以下情况时，立即触发因果链SOP：**

| 信号 | 含义 | 行动 |
|------|------|------|
| 用户说"为什么" | 需要解释因果 | 走因果链三步 |
| 用户说"展示逻辑" | 需要完整流程 | 列出每步工序 |
| 数据与预期不符 | 可能有隐藏bug | 排除法分析 |
| 新异常数据点 | 新问题出现 | 完整工序还原 |
| 用户纠正 | 我错了 | 立即纠正+更新文档 |

## 4.4 深度思考触发条件

以下情况必须启动深度思考模式：
- 涉及多变量交互（如密钥+IP+代理逻辑）
- 涉及时序依赖（如间隔、冷却、窗口）
- 涉及异常但无明显原因
- 涉及"为什么"追问
- 数据矛盾（A说X，B说Y，不可能同时成立）

**深度思考模式输出格式：**
```
1. 现象描述：实际发生了什么
2. 预期描述：根据代码逻辑应该发生什么
3. 矛盾点：预期≠实际的差异
4. 可能原因：列出3-5个假设
5. 排除分析：逐个验证
6. 结论：剩余原因+证据
7. 后续建议：如何验证/修复
```

## 4.5 避免认知偏差

| 偏差 | 表现 | 纠正方法 |
|------|------|---------|
| 确认偏差 | 只看支持自己观点的数据 | 主动找反例 |
| 锚定效应 | 第一次判断影响后续 | 重新评估所有假设 |
| 过度自信 | 不验证就下结论 | 强制走因果链三步 |
| 近因效应 | 最近的事件影响判断 | 看完整时间线 |
| 叙事谬误 | 编故事来解释数据 | 只看数据本身和代码逻辑 |
