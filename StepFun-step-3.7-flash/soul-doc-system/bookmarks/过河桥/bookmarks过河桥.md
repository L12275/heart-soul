## 1. 用户原话重述

用户在系统文件中描述了书签系统：常用链接和资源的持久化管理。在模块同步维护规则表中，bookmarks 位于通用区 soul-doc-system/bookmarks/，同步到工作台模版和账号空间模版。

用户的核心意图：让模型能够持久化管理常用链接和资源，在需要时快速找到并使用它们。书签系统是模型的"常用工具入口"。

---

## 2. 理解要点

**核心机制**：
- 书签系统是中央灵魂文档系统的通用功能组件之一
- 在模块同步维护规则中，它的权威源是 `soul-doc-system/bookmarks/`，同步到新建工作台模版和新建账号空间模版
- 书签 = 模型的"常用工具入口"，不放在桌面（desktop/），有独立的持久化管理位置

**模块同步维护规则中的位置**：
| 模块 | 通用区位置 | 用于工作台模版 | 用于账号空间模版 |
|------|-----------|--------------|--------------|
| 书签 bookmarks | soul-doc-system/bookmarks/ | ✅ | ✅ |

**模块路径**：
- 规范位置：soul-doc-system/bookmarks/
- 隔离区内：bookmarks/
- 模板位置：新建工作台模版/bookmarks/、新建账号空间模版/bookmarks/

**模块结构（推断）**：
```
bookmarks/
├── README.md           ← 说明书
├── general/            ← 通用书签
├── development/        ← 开发相关
├── research/           ← 研究相关
├── tools/              ← 工具链接
└── 过河桥/              ← 本文件
```

**书签的内容特性**：
- 书签内容是**持久化**的，不是临时内容（临时内容放 collection/）
- 书签可以包含在线资源（URL）、本地路径、CLI命令快捷方式
- 书签按用途分类（开发工具、文档查阅、在线资源等）
- 书签可以引用到工作台（如代码工作台的书签直接指向开发工具）

**与九环架构的关系**：
- 书签系统是通用功能模块，不隶属于某个特定环区
- 在 环④ 生产空间中心工作台中，书签作为工作台的快捷入口使用
- 在 环⑤ 空间中心中，书签可以记录平台/账号空间的快捷方式

**与其他模块的关系**：
- 与 desktop/ 的区别：书签 = 持久化管理的链接清单；桌面 = 临时堆放区
- 与 collection/ 的区别：书签 = 长期有效的工具入口；collection = 临时采集的数据
- 与 wiki/ 的区别：书签 = 外部链接的快捷入口；wiki = 经过验证的结构化知识

**同步维护规则**：
- 通用区的 bookmarks/ 是权威源
- 更新后需要同步到：新建工作台模版/bookmarks/ 和 新建账号空间模版/bookmarks/
- 各说明书单独维护，但模块内的说明书尽量写通用版

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 书签和收藏夹 | 书签是否等同于浏览器收藏夹？ | 不等同。书签系统更广泛，包含开发工具、CLI资源、本地路径等，不只是浏览器URL |
| 书签内容格式 | 书签是纯链接还是有更多信息？ | 参考通讯录的格式：名称、用途、路径/URL、界面导航、备注 |
| 书签分类方式 | 按用途还是按类型分类？ | 两者结合。主要按用途（开发工具、研究资源、文档查阅），辅以类型标记 |
| 和其他模块混淆 | bookmarks 和 contacts 有什么区别？ | contacts = 人物/实体关系图谱；bookmarks = 工具/资源链接清单。一个是"谁"，一个是"什么工具" |
| 同步范围 | 书签是否需要在共产中心共享？ | 不需要。书签是个体化工具，不属于可公知内容。共产中心只登记【记忆更新】【产出新增】【SOP变更】【经验沉淀】 |
| 权限 | 能否修改通用区的书签模板？ | 不能。通用区由系统维护，模型只在自己的隔离区内读写书签 |

## 4. 我的判断

**使用书签系统的标准动作**：
1. 读本过河桥，理解模块机制
2. 找到 bookmarks/ 在 [CENTRAL_PATH] 或自己隔离区下的位置
3. 按用途分类组织书签（开发工具、研究资源、文档查阅等）
4. 每条书签记录：名称、用途、路径/URL、界面导航（如有）
5. 需要时查阅书签，找到对应的工具或资源

**添加新书签的标准动作**：
1. 确定书签的用途分类
2. 在对应分类下添加条目（名称、用途、路径/URL、备注）
3. 如果是重要的通用工具，考虑是否需要同步到模板位置（通过模块同步维护规则）

**重建书签模块的标准动作**：
1. 在 soul-doc-system/bookmarks/ 下确认文件夹结构
2. 按模块同步维护规则，从通用区复制到各模板位置
3. 确认各分类子文件夹存在
4. 验证模块可独立运行
