## 双层结构：地平面 vs 工作台

### 比喻：工厂

- **地平面级**（soul-doc-system根目录）= 工厂公共设施
  - 工厂大门、走廊、电梯、仓库、会议室
  - 所有工作台共享的通用资源
  - 不特定于某个工作台的内容，写到这里

- **工作台级**（production_space/通用/代码工作台）= 工厂里的专用车间
  - 代码车间里面有：办公室、实验室、财务室、生产区
  - 每个车间有自己独立的桌面、工作流、通讯录、收藏夹、便签、日记、记忆
  - 只记录与这个车间高相关的内容

### 双写规则

- **通用记录**（不特定于某个工作台）→ 只写到地平面级（soul-doc-system根目录）
- **专用记录**（与某工作台高相关）→ 只写到该工作台内
- **双写**（既通用又专用）→ 同时写到地平面和工作台

## 时间轴记录SOP

### 日记 vs 记忆

| 类型 | 内容 | 准确性 | 格式 | 位置 |
|------|------|--------|------|------|
| 日记（diary/） | 当天实时事件过程 | ⚠️ 纪实，可能不完整 | 时间轴：年/月/日/日期.md | 地平面 + 工作台 |
| 记忆（memory/） | 验证确认后的正确记忆 | ✅ 已验证 | context/L0-L4 + daily/天级快照 | 地平面 + 工作台 |

### 时间轴记录规则

1. **日记是实时记录** — 工作时边做边记，不是事后总结
2. **日记展开细节** — 写清楚事件过程、决策原因、中间结果
3. **记忆是精炼后的正确记忆** — 从日记中提炼，经过验证
4. **两者互补** — 日记有细节但不一定对，记忆是对的但可能不完整

### 记录时机

- **实时日记**：每个重要事件发生时立即记录
- **天级记忆**：每天结束时，从日记中提炼验证后的记忆
- **会话归档**：每次工作结束后，整理到 log/年/月/日/
