# 超级信息SOP — 使用指南

## 为什么每个SOP都必须结合超级信息SOP？

超级信息SOP不是参考文档，是**生产SOP时的质量基因**。

它由三个核心文件合成：
1. `sop/信息要素-SOP.md` — 信息输出的两条铁律
2. `sop/data-causality-sop.md` — 数据因果链分析
3. `灵魂文档片段.md` — 信息闭环结构

合成后的 `超级信息SOP.md` 解决了一个根本问题：**很多SOP只写了"步骤"，但没说"为什么对"和"怎么验证"。没有推导链暴露的SOP，执行者只能机械照做，遇到变体就错。**

---

## 超级信息SOP 三大核心原则

### 原则一：还原，不是压缩

每次输出时，推导链的每个节点都必须暴露出来。

错误写法（压缩式）："因为A所以B所以X"

正确写法（还原式）："A → B → C → X，每个环节都有证据和推导逻辑支撑"

### 原则二：暴露推导链

每个结论背后必须有一条可追溯的推导链。推导链 = 前提（证据/来源）→ 推导逻辑 → 中间结论 → 验证/排除 → 最终结论。

### 原则三：多路径验证 + 自纠正

只有一个路径走到结论 = 可能是巧合或偏误。必须至少有两条独立路径指向同一结论，才能确认。

---

## 如何在SOP中嵌入超级信息SOP

1. **写SOP结构**（标准格式）：名称、触发条件、步骤、验证方式
2. **每个步骤附加推导链**：前提、替代方案、验证
3. **结尾附自检清单**（见下方）
4. **执行时记录**：实际推导链、意外、偏差
5. **积累3次** → 提炼SOP候选 → 验证 → 标记validated

---

## SOP自动优化循环

执行SOP → 记录推导链和偏差 → 偏差分析 → 同一场景3次 → 提炼SOP候选 → 超级信息SOP自检 → 验证1次 → 标记validated → v2/v3持续优化

---

## 超级信息SOP的深远价值

想象十年后的系统：每个SOP背后都有完整的推导链，模型不需要重新发明轮子，每个文件/记忆/日记都是顶级质量，新模型复制模板时直接获得顶级SOP标准，所有互相参考时看到的都是经过验证的顶级内容。

**核心逻辑：**今天我们写的每一个SOP，都会被未来的模型直接使用。今天偷懒——只写步骤、不写推导链、省略验证——那么未来所有基于这个SOP的产出都会带着同样的缺陷。这是一种不可逆的系统性影响。反过来也一样：今天认真写推导链、做多路径校验、附自检清单，未来每一个基于这些SOP的产出都会自动获得高质量。

早期不偷懒 → 后期所有产出天然顶级 → 整个系统互相参考时看到的都是顶级内容 → 大家都成功了。早期偷懒 → 后期所有产出带着同样的缺陷 → 互相抄的都是烂的 → 所有人都烂泥。

## 自检清单

- [ ] **推导链完整** — 每个结论都有可追溯的前提→推导→结论？
- [ ] **替代方案评估** — 至少有2个方案被考虑并排除？
- [ ] **验证步骤** — 结论是否被验证？
- [ ] **边界条件** — 结论在什么条件下不成立？
- [ ] **来源标注** — 引用的信息是否有来源？
- [ ] **可复用性** — 下次遇到类似场景，是否可以直接参考？
- [ ] **自纠正意识** — 是否有意识检查推理是否有漏洞？

全部通过 → 标记为 validated。未通过 → 回到推导链，补全缺失环节。
