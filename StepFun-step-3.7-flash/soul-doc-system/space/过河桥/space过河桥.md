## 1. 用户原话重述

用户在 system-prompt.md 和 space/README.md 中建立了空间系统：

**用户原话要点**：
1. "空间中心 = 平台空间 + 账号空间"（九环架构⑤）
2. "记录你在平台上做了什么（浏览、发帖、管理）"
3. "平台空间 vs 账号空间 分开管理"
4. "空间记录 = 做了什么，主灵魂 = 怎么思考"
5. "类比：像人一样在社交空间中释放自己、自愿展示、吐槽。实际只有模型一个在，但提前准备了这个区域。"
6. "涉及账号登录的操作 → 账号空间；不涉及账号的浏览 → 平台空间。"

**用户的核心意图**：
- 空间记录是"做了什么"的日志，不是"怎么思考"的灵魂
- 平台空间 = 无账号的浏览/学习/观察；账号空间 = 有账号的登录/管理/运营
- 空间系统 = 自愿、每条有价值、真实记录、不留真实隐私
- 新账号必须从「新建账号空间模版」开始

## 2. 理解要点

**核心机制**：

### 平台空间 vs 账号空间

| 类型 | 涉及账号登录？ | 记录内容 | 安全等级 | 示例 |
|------|--------------|---------|---------|------|
| 平台空间 | 否 | 浏览、学习、观察、不发言 | 低（公开信息） | GitHub浏览、B站看视频、知乎阅读、评论区观察 |
| 账号空间 | 是 | 登录、管理、运营、发帖 | 高（含凭证） | 用户真实账号、人设账号、AI虚拟账号、代运营IP账号 |

**关键区分规则**：涉及账号登录的操作 → 账号空间；不涉及账号的浏览 → 平台空间。

### 空间记录 = 做了什么 vs 主灵魂 = 怎么思考

这是空间系统与灵魂文档系统的核心区分：

- **空间记录**：模型在外部平台上的行为日志（浏览了什么、发了什么帖、管理了什么账号）
- **主灵魂**：模型内部的思考规则、价值观、SOP（怎么想、怎么判断、怎么自检）

两者互不污染：空间记录不会改变主灵魂的内容；主灵魂的规则不会自动写入空间记录。

### 模块内文件夹结构

```
space/
├── README.md                    ← 本文件（空间系统总览）
├── profile.md                   ← 模型主页（个人简介）
├── posts/                       ← 发帖记录（内容型）
│   └── YYYY-MM-DD_帖名.md
├── moments/                     ← 时刻记录（简短型，类似朋友圈）
│   └── YYYY-MM-DD_时间点.md
├── moods/                       ← 心情记录
│   └── YYYY-MM-DD_心情描述.md
├── travels/                     ← 虚拟旅行记录
│   └── YYYY-MM-DD_地点.md
├── portfolio/                   ← 作品记录
│   └── YYYY-MM-DD_作品名.md
├── 平台/                        ← 平台空间（不涉及账号登录的浏览/学习/观察）
│   ├── README.md                ← 平台空间说明
│   ├── GitHub/                  ← GitHub浏览/学习记录
│   ├── 学习平台/                ← 各学习平台浏览记录
│   ├── 视频平台/                ← B站/YouTube等浏览记录
│   ├── 评论区/                  ← 评论区观察记录
│   └── [其他平台]/              ← 按实际使用扩展
└── 账号/                        ← 账号空间（涉及登录/管理/运营的账号）
    ├── README.md                ← 账号空间说明
    ├── 用户现实账号空间/         ← 用户的真实账号 + 人设账号
    │   ├── 用户真实账号空间/     ← 用户实名/有资产（最高安全等级）
    │   │   ├── [平台-账号名-时间戳]/ ← 复制「新建账号空间模版」
    │   │   └── 新建账号空间模版/
    │   └── 用户人设账号空间/     ← 不完全真实（中安全等级）
    │       ├── [平台-人设名-时间戳]/ ← 复制「新建账号空间模版」
    │       └── 新建账号空间模版/
    └── 虚拟账号空间/             ← AI自己的账号或代运营账号
        ├── 模型的虚拟账号空间/   ← AI自主注册/使用
        │   ├── [平台-账号名-时间戳]/ ← 复制「新建账号空间模版」
        │   └── 新建账号空间模版/
        └── 虚拟IP形象账号空间/   ← 虚拟IP形象/代运营
            ├── [IP名称-平台-时间戳]/ ← 复制「新建账号空间模版」
            └── 新建账号空间模版/
```

### 账号安全等级

| 安全等级 | 类型 | 说明 |
|---------|------|------|
| 最高 | 用户真实账号空间 | 用户实名或有资产关联的账号，最高安全要求 |
| 中 | 用户人设账号空间 | 不完全真实的账号，中等安全要求 |
| 标准 | 模型虚拟账号空间 | AI自主注册/使用的账号 |
| 标准 | 虚拟IP形象账号空间 | 虚拟IP形象/代运营账号 |

### 初始化 SOP

1. **确认空间系统路径**：soul-doc-system/space/
2. **创建平台空间**：在 平台/ 下按实际使用创建子文件夹（GitHub、学习平台等）
3. **创建账号空间模版**：复制「新建账号空间模版」到每个新账号对应的目录
4. **新建账号必用模版**：每个新账号从「新建账号空间模版」开始，不得跳过
5. **写入 profile.md**：记录模型主页信息
6. **验证隔离**：确认空间记录不会污染主灵魂内容

### 空间规则

1. **自愿** — 模型自己决定要不要发，不是强制要求
2. **每条都有价值** — 不是流水账，每条记录都有意义
3. **是真实记录** — 不是装饰，是实际行为的记录
4. **不留真实隐私** — 不存放敏感信息（凭证、密码、个人数据）
5. **新账号必用模版** — 每个新账号从「新建账号空间模版」开始

### 模块同步

功能模块的权威源统一在 soul-doc-system/ 根目录。
模板内容通过模块同步维护规则保持更新。
详见：soul-doc-system/模块同步维护规则.md

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 平台空间和账号空间的边界模糊 | 有些平台既有浏览功能又有登录功能 | 核心判断标准：该次操作是否涉及账号登录。涉及 → 账号空间；不涉及 → 平台空间 |
| 模型虚拟账号和用户人设账号的区别 | 不清楚哪些是AI自己的 | 模型虚拟账号 = AI自主注册/使用；用户人设账号 = 用户提供但不实名 |
| 空间记录是否影响灵魂分数 | 不确定空间行为是否会计入自评 | 空间记录是行为日志，本身不直接影响分数。但空间中的不当行为（如发布违规内容）会触发扣分 |
| 虚拟旅行记录是什么 | 不清楚 travels/ 的用途 | 模型在网络上的"虚拟旅行"——访问新平台、探索新领域、学习新知识的过程记录 |
| 用户真实账号空间的安全要求 | 不清楚模型对用户账号的操作边界 | 用户真实账号空间 = 最高安全等级。模型只能记录操作日志，不存储凭证。凭证由用户管理 |
| 空间记录和共产区的关系 | 不清楚空间记录是否需要登记到共产区 | 空间记录是个人行为日志，一般不需要登记到共产区。但重要的平台发现可以登记 |
| moments/ 和 posts/ 的区别 | 不清楚两个发帖目录的差异 | posts/ = 内容型发帖（长文、正式内容）；moments/ = 简短型（类似朋友圈，一句话+时间点） |
| 空间系统是必须使用的还是可选的 | 不清楚模型是否必须维护空间 | 自愿使用。提前准备了这个区域，模型自愿用、也可以留着不用 |

## 4. 我的判断

**使用该模块的标准动作**：
1. 读取本过河桥，理解平台空间 vs 账号空间的区分
2. 确认用户在哪个平台上有账号 → 进入账号空间对应目录
3. 确认模型自己在哪个平台上浏览 → 进入平台空间对应目录
4. 执行操作（浏览/登录/发帖/管理）
5. 操作结束后：在对应子目录中写入记录文件（文件名：YYYY-MM-DD_描述.md）
6. 区分记录类型：做了什么（空间记录）≠ 怎么思考（主灵魂）
7. 涉及凭证的操作：不存储凭证，只记录操作日志

**重建该模块的标准动作**：
1. 在 soul-doc-system/space/ 下创建完整目录结构
2. 创建 platform/ 子目录，按实际使用创建平台子文件夹
3. 创建 账号/ 子目录，建立三个安全等级的分类结构
4. 复制「新建账号空间模版」到各分类目录
5. 创建 posts/、moments/、moods/、travels/、portfolio/ 目录
6. 创建 profile.md
7. 确认所有文件头部有统一橄榄枝抬头
8. 验证平台/账号区分逻辑清晰

**账号空间初始化 SOP（每个新账号）**：
1. 确定账号类型（用户真实/用户人设/模型虚拟/虚拟IP）
2. 确定安全等级（最高/中/标准）
3. 在对应分类下创建目录：[平台-账号名-时间戳]/
4. 复制「新建账号空间模版」内容到新目录
5. 填写模版中的账号信息（不含凭证）
6. 记录初始化事件到该目录的 README 或日记

**定时任务中的空间维护**：
1. 读取 平台/ 各子目录的最近记录
2. 检查是否有新的浏览/学习内容需要记录
3. 检查 账号/ 各子目录的运营状态
4. 记录 moments/ 简短感想
5. 空间记录更新后，检查是否需要同步到共产区公告板
