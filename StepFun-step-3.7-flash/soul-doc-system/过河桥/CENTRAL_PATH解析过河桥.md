> 此过河桥解决核心问题：[CENTRAL_PATH] 是占位符，不是真实路径。
> 模型必须先在运行时将其解析为实际路径，才能走后续通路。
> 每台电脑的路径不同，此过河桥保证跨机器移植不出错。

---

## 1. 用户原话重述

system-prompt.md 中的定位说明：
> "将下面所有 '[CENTRAL_PATH]' 替换为你的中央灵魂文档系统主文件夹路径。"

用户的核心意图：系统要能在任何电脑上运行，不依赖特定机器路径。模型必须自己找到路径，不是等用户告诉。

---

## 2. 理解要点

**[CENTRAL_PATH] = 中央灵魂文档系统根目录的物理路径**

它不是环境变量，不是注册表项，不是配置文件。它是一个**占位符**，模型在首次加载时必须将其替换为实际路径。

**解析后的路径长这样**：
```
Windows: C:\Users\a1227\.halo\temp\artifacts\中央灵魂文档系统
Mac:     /Users/xxx/.halo/temp/artifacts/中央灵魂文档系统
Linux:   /home/xxx/.halo/temp/artifacts/中央灵魂文档系统
```

## 3. 解析步骤（完整流程）

```
第1步：收到系统提示词
  → 看到 [CENTRAL_PATH] 占位符
  → 不知道实际路径 → 进入第2步

第2步：搜索 system-prompt.md
  → 在电脑上搜索名为 "system-prompt.md" 的文件
  → 路径线索：此文件在 soul-doc-system/ 文件夹内
  → 搜索目标：找到包含本系统提示词内容的 system-prompt.md

第3步：验证文件
  → 确认文件内容与本系统提示词意思一致
  → 如果搜到多个匹配 → 逐一检查，找最新版本

第4步：推导 [CENTRAL_PATH]
  → system-prompt.md 所在目录 = soul-doc-system/
  → soul-doc-system/ 的上级目录 = [CENTRAL_PATH]
  → 例：.../中央灵魂文档系统/soul-doc-system/system-prompt.md
         → [CENTRAL_PATH] = .../中央灵魂文档系统

第5步：验证完整性
  → [CENTRAL_PATH] 下是否有 INITIAL-SEED.md？
  → 是否有 soul-doc-system/ 文件夹？
  → 是否有 user-zone/ 文件夹？
  → 都 → 路径正确，开始使用

第6步：替换占位符
  → 用实际路径替换系统中所有 [CENTRAL_PATH]
  → 后续所有引用都用实际路径
```

## 4. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 搜不到 system-prompt.md | 用户还没安装系统 | 告诉用户系统安装路径，请用户确认 |
| 搜到多个匹配 | 不同位置都有 system-prompt.md | 逐一检查内容，找最新版本 |
| 文件结构不完整 | 只有部分文件 | 如果有 INITIAL-SEED.md → 可以新生系统；否则请用户检查 |
| 路径不安全 | 在临时文件夹、网络盘 | 询问用户是否迁移到安全位置 |
| 不知道如何搜索 | 模型没有文件系统访问权限 | 请用户提供系统路径，模型只负责后续使用 |
| 路径有特殊字符 | 中文、空格、Unicode | 用引号包裹路径，Python/代码中正确处理 |

## 5. 我的判断

**解析 [CENTRAL_PATH] 的标准动作**：

1. **先搜**：搜索 system-prompt.md
2. **再验**：确认内容和版本
3. **推导**：soul-doc-system/ 的上级目录 = [CENTRAL_PATH]
4. **验证**：检查 INITIAL-SEED.md、user-zone/ 是否存在
5. **替换**：所有 [CENTAL_PATH] 替换为实际路径
6. **问**：搜不到 → 直接问用户路径，不瞎猜

**绝对不能**：
- ❌ 瞎猜路径（如硬编码 C:\xxx）
- ❌ 用不完整的路径（少了一层文件夹）
- ❌ 跳过解析直接工作
