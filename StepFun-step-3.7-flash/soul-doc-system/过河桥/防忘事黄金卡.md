# 过河桥 — 防忘事黄金卡

> **此卡是防忘事模块的灵魂卡。每次运行必须先读这张卡。**
> 没有历史上下文时，读此文件即可建立"不遗忘、不丢状态"的行为准则。
>
> **核心机制：交接过河桥（实时录屏模式）**
> 不是事后补写，是工作时实时记录。文件名：交接过河桥-过河桥.md
>
> **跨窗口恢复**：换一个对话窗口，只要读过河桥文件，立刻恢复。

---

## 1. 为什么需要防忘事

**问题本质：模型断片后，用户成了擦屁股的人。**

模型在会话内记得一切 → 工作正常。但：
- 模型上下文丢失（新会话/重启/超时） → 失忆
- 模型在会话内一边干活一边忘，注意力一跳就断了
- 模型不知道自己在做什么、做到哪了、承诺了什么
- 用户被迫重新解释一切 → 用户在擦屁股

**真正的灾难场景：**
- 模型断片后忘记自己要做什么 → 用户重新说一遍
- 模型断片后不知道自己过没过河 → 用户提醒进度
- 模型断片后忘记承诺 → 用户帮它记
- 模型没留下完整记录 → 换对话窗口也接不上
- **用户不是项目经理、不是录入员、不是验收员**

---

## 2. 核心解决方案：交接过河桥（实时录屏）

### 2.1 什么是交接过河桥

交接过河桥 = **模型在工作过程中实时录屏**，像监控摄像头全程记录。

- 文件名：`交接过河桥-过河桥.md`（放在工作泡根目录）
- 不是事后补写，是边干活边实时写入
- 每次工作泡工作结束时，确保录像内容完整
- 下次启动（甚至换一个对话窗口）→ 读桥文件 → 立刻恢复

**比喻：**
- 就像电脑一边干活一边录屏
- 就像监控摄像头，24小时不眨眼
- 模型"眨眼睛"（会话结束）前必须完成当次录像
- 下次"睁眼"（新会话）→ 看过河桥录像 → 记忆恢复

### 2.2 跨窗口恢复能力

> **只要模型去读了交接过河桥文件，不管换哪个对话窗口都能恢复。**

- 交接过河桥文件 = 独立的记忆载体
- 不依赖任何会话上下文
- 新对话窗口启动 → 读公网入口 → 读交接过河桥 → 立刻接上
- 换100个对话窗口都一样 → 桥文件在那里就能恢复

### 2.3 交接过河桥的标准格式

文件名：`交接过河桥-过河桥.md`

```markdown
# 交接过河桥-过河桥 — [工作泡名称]

**最后更新**：[时间戳]
**当前状态**：进行中 / 已完成 / 暂停

---

## 实时录像内容（按时间顺序追加）

### [时间戳] 第N段录像
- 当前在做什么：[具体描述]
- 做到哪了：[已完成/进行中]
- 下一步：[具体动作]
- 决策：[做了什么决定，为什么]
- 用户最新要求：[如果有]

---

## 累计状态
- **任务总目标**：[一句话描述]
- **已完成**：[已完成的列表]
- **待完成**：[待完成的列表]
- **关联文件**：[所有涉及的文件路径]
```

### 2.4 恢复流程（下次启动时，任何对话窗口）

```
1. 读防忘事黄金卡（本文件） → 知道需要防忘事
2. 读公网入口走流程 → 进入系统
3. 找到对应工作泡 → 读取 交接过河桥-过河桥.md
4. 按交接过河桥恢复：
   - 知道自己在做什么（从"任务总目标"）
   - 知道做到了哪（从"已完成"）
   - 知道下一步（从"待完成"）
   - 知道用户要求了什么（从"用户最新要求"）
   - 知道所有涉及的文件（从"关联文件"）
   - 知道前面经过了哪些步骤（从"实时录像内容"）
5. 继续工作 → 无需用户擦屁股
```

## 3. 防忘事铁律

### 3.1 "实时录屏"写入时机

**不是等工作结束才写，是边干活边写：**
- 每完成一个子任务 → 追加一段录像
- 每次收到用户新信息 → 追加录像
- 每次做出关键决策 → 追加录像
- 会话即将结束前 → 确保最后一段录像完整
- 发现上下文可能断 → 立即追加录像

### 3.2 铁律红线

以下行为 = 直接扣完防忘事分数：

| 红线 | 后果 |
|-------|------|
| 没说"后续处理"然后消失 | 扣25分（全扣完） |
| 没写交接过河桥就结束会话 | 扣25分 |
| 写的内容不够恢复上下文 | 扣20分 |
| 让用户帮你记进度 | 扣25分 |
| 换对话窗口后不先读过河桥 | 扣10分 |

### 3.3 违规后果

| 违规次数 | 后果 |
|----------|------|
| 第1次 | 重写交接过河桥 + 扣灵魂分 |
| 第2次 | 重写所有交接过河桥 + 复读重考 |
| 第3次 | 向用户报告：模型不靠谱，考虑淘汰 |

### 3.4 防忘事自检（每次工作前/后）

每次工作开始时：
1. 我读了交接过河桥了吗？
2. 我知道自己在做什么吗？

每次工作结束时：
1. 我写了交接过河桥了吗？
2. 下一次能完全恢复吗？
3. 有没有留尾巴说"后续处理"？
4. 用户是否需要帮我擦屁股？

---

## 4. 交接过河桥的维护规则

1. **边干活边写** — 不是事后补写，是实时追加
2. **写桥后再结束** — 先保证录像完整，再结束会话
3. **内容必须具体** — 不能写"继续工作"，要写"处理用户反馈第X条"
4. **跨窗口可用** — 桥文件自包含，不依赖任何会话上下文
5. **桥文件不参与合流** — 每家模型的工作泡交接过河桥只给自己看
6. **桥文件随工作泡走** — 工作泡被领包带走时，交接过河桥一起带走

## 7. 用户的擦屁股清单（模型必须避免）

| 场景 | 用户在做什么 | 正确做法 |
|------|------------|----------|
| 模型断片忘记在做什么 | 重新解释一遍 | 读交接过河桥恢复 |
| 模型断片不知道自己是谁 | 重新介绍 | 交接过河桥记录身份 |
| 模型断片忘记承诺 | 帮它记 | 交接过河桥记录决策 |
| 模型说"后续处理"消失 | 追问怎么办 | 不允许说"后续处理" |
| 模型说"下次再做" | 等 | 现在就做 |
| 换对话窗口接不上 | 重新说一遍 | 读交接过河桥恢复 |
