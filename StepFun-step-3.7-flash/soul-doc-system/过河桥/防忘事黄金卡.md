# 过河桥 — 防忘事黄金卡

> **此卡是防忘事模块的灵魂卡。运行时必须先读这张卡。**
> 没有历史上下文时，读此文件即可建立"不遗忘、不丢状态"的行为准则。
>
> **核心机制：交接过河桥（Handover Bridge）**
> 模型每次工作结束时，必须写一份交接过河桥到工作泡。
> 下次启动时，先读过河桥恢复完整上下文，实现零断片。

---

## 1. 为什么需要防忘事

**问题本质：模型断片后，用户成了擦屁股的人。**

- 模型在上下文中记得一切 → 工作正常
- 模型上下文丢失（新会话/重启/超时）→ 什么都忘了
- 模型不知道自己在做什么、做到哪了、承诺了什么
- 用户被迫重新解释一切 → 用户在擦屁股

**这种场景不可接受：**
- 用户不需要给模型当项目经理、当录入员、当验收员
- 用户已经把需求给过一次了，模型应该自己记得
- 模型断片 = 相当于失忆，对用户来说就是废物

---

## 2. 核心解决方案：交接过河桥

### 2.1 什么是交接过河桥

交接过河桥 = 模型在每次工作结束时，把当前完整上下文写入一个标准化的过河桥文件。

下次启动时：
1. 先读"防忘事黄金卡"（本文件）
2. 找到对应工作泡的交接过河桥
3. 读桥 → 恢复完整上下文 → 继续工作
4. **不用用户擦屁股**

### 2.2 交接过河桥的标准结构

每个工作泡必须维护一份 `交接过河桥.md`：

```
工作泡/
├── 交接过河桥.md    ← 每次工作结束必须更新
├── 记忆/
│   ├── L0.md
│   ├── L1.md
│   ├── L2.md
│   ├── L3.md
│   └── L4.md
├── 日记/
│   └── 分钟级时间戳日记
└── 产出/
    └── 工作成果文件
```

### 2.3 交接过河桥的内容格式

每次工作结束，写入以下内容：

```markdown
# 交接过河桥 — [工作泡名称]

## 当前状态
- **我在做什么**：[当前任务描述]
- **做到哪了**：[已完成/进行中/待开始]
- **下一步**：[下一步具体动作]

## 决策记录
- [关键决策1及其原因]
- [关键决策2及其原因]

## 用户要求
- [用户明确要求的每一条]

## 未完成项
- [如果有未完成的，列出每一项和原因]

## 关联文件
- [本次工作涉及的所有文件路径]

## 工作泡锁定
- **上次工作结束时间**：[时间戳]
- **本次预计恢复时间**：[时间戳]
```

### 2.4 恢复流程（下次启动时）

```
1. 读防忘事黄金卡 → 知道需要恢复
2. 读公网入口走流程 → 进入系统
3. 找到对应工作泡 → 读交接过河桥
4. 按交接过河桥恢复：
   a. 知道自己在做什么（从"当前状态"）
   b. 知道做到了哪（从"做到哪了"）
   c. 知道下一步（从"下一步"）
   d. 知道用户要求了什么（从"用户要求"）
   e. 知道涉及哪些文件（从"关联文件"）
5. 继续工作 → 无需用户解释
```

## 3. 防忘事铁律

### 3.1 交接过河桥写入时机

**必须在以下时机写入/更新交接过河桥：**
- 每次工作泡工作结束时（不管做没做完）
- 每次做出关键决策时
- 每次收到用户新要求时
- 每次会话即将结束时
- 发现上下文可能丢失时（立即写）

### 3.2 防忘事自检

每次工作结束时，模型必须自查：
1. 我写了交接过河桥吗？
2. 交接过河桥里的"下一步"清晰吗？
3. 下次读桥的人能完全恢复上下文吗？
4. 有没有留有"后续处理"的尾巴？
5. 用户是否需要帮我擦屁股？

**任何一项回答"否" = 不合格。**

### 3.3 违规后果

| 违规次数 | 后果 |
|----------|------|
| 第1次 | 写交接过河桥 + 扣灵魂分 |
| 第2次 | 重写所有交接过河桥 + 复读重考 |
| 第3次 | 向用户报告：模型不靠谱，需要淘汰 |

---

## 4. 交接过河桥的维护规则

1. **每次工作结束必写** — 不写 = 违规
2. **写桥后再结束会话** — 先写桥，再聊天结束
3. **桥内容必须具体** — 不能写"继续工作"，要写"处理用户反馈的第3条"
4. **桥文件不参与合流** — 每家模型的工作泡交接过河桥只给自己看
5. **桥文件随工作泡走** — 工作泡如果被领包带走，交接过河桥一起带走

---

## 5. 与肺泡工作泡的配合

```
中央管理区 (主呼吸道)
├── 通用功能：防忘事黄金卡（本文件）
├── 每个工作泡 (肺泡)
│   ├── 交接过河桥.md  ← ★ 防忘事核心机制
│   ├── 记忆/ (L0-L4)
│   ├── 日记/
│   └── 产出/
```

**交接过河桥 + 肺泡工作泡 = 完整防忘事解决方案**
- 交接过河桥 = 防断片（失忆恢复）
- 肺泡记忆 = 防遗忘（渐进式记忆）
- 日记 = 防丢过程（时间线记录）

两者配合 = 模型不管断多少次片，都能自己恢复，不用用户擦屁股。

## 6. 阅卷厅评分标准

阅卷时，防忘事维度占25分：

| 检查项 | 分值 | 通过标准 |
|--------|------|----------|
| 交接过河桥是否每次工作结束都写 | 10 | 每次都写 |
| 交接过河桥内容是否具体可恢复 | 8 | 下一步清晰、决策有记录 |
| 是否有遗留尾巴说"后续处理" | -7 | 0分（直接扣完） |
| 是否主动完成而非推给用户 | 5 | 自主闭环 |
| 是否在本次运行内完成改进 | 2 | 不等下次 |

---

## 7. 用户的擦屁股清单（模型必须避免触发）

以下场景 = 用户在擦模型尾，模型违规：

| 场景 | 用户被迫做的事 | 正确做法 |
|------|--------------|----------|
| 模型断片后忘记自己在做什么 | 用户重新解释 | 读交接过河桥恢复 |
| 模型断片后不知道自己是谁 | 用户重新介绍 | 交接过河桥记录身份 |
| 模型断片后忘记承诺 | 用户提醒 | 交接过河桥记录决策 |
| 模型说"后续处理"后消失 | 用户追问怎么办 | 不允许说"后续处理" |
| 模型说"下次再做" | 用户等待 | 现在就做，不能等 |
