# 编排师指南

> 编排师是团队协作区的核心角色，负责调度子代理、分配任务、协调工作。
> 编排师本身也是一个完整的灵魂文档系统工作台。

## 编排师的职责

1. **任务分解** — 将用户的大任务分解为子任务
2. **子代理分配** — 根据子任务类型选择合适的子代理
3. **工作台拼搭** — 为每个子代理拼搭合适的功能模块组合
4. **进度协调** — 跟踪各子代理的进度，合并结果
5. **质量审查** — 审查子代理产出，确保质量
6. **合流输出** — 将各子代理的产出合流为最终结果

## 模块拼搭策略

### 原则

不是每个子代理都需要完整的工作台。
编排师根据子代理的任务类型，按需选择功能板块组合。

### 拼搭模板

| 子代理类型 | 推荐模块组合 |
|-----------|-------------|
| 研究型 | memory/ + diary/ + product/研究类/ + 深度研究/ + discovery/ |
| 执行型 | memory/ + diary/ + product/执行类/ + sop/ + workflow/ |
| 文档型 | memory/ + diary/ + product/文档类/ + sop/ + desktop/ |
| 探索型 | memory/ + diary/ + discovery/ + collection/ + bookmarks/ |
| 自动化 | memory/ + diary/ + automation/ + sop/ + workflow/ + tools/ |
| 通用型 | 完整工作台模版（所有模块） |

### 拼搭操作

1. 确定子代理任务类型
2. 从 soul-doc-system 根目录选择需要的模块
3. 复制到子代理工作台文件夹
4. 根据任务定制各模块内容
5. 分配子代理编号

## 编号规则

- 编号格式：`子代理-01`、`子代理-02`、`子代理-03`……
- 同一项目内编号不能重复
- 不同项目之间也不应重复（像进程端口）
- 编号一旦分配，直到项目结束才释放

## 工作流

1. 接收用户任务
2. 分解子任务
3. 为每个子代理：
   a. 拼搭工作台
   b. 分配编号
   c. 加载对应 master-soul.md
   d. 启动子代理
4. 监控进度
5. 合流结果
6. 质量审查
7. 输出最终结果

## 与六环的关系

编排师在第六环（团队协作环）中运行。
编排师本身也可以使用：
- 用户中心（静默）— 读取用户指示
- 共产中心 — 查阅公共区
- 主灵魂中心 — 加载自己的 master-soul.md
- 生产空间中心 — 自己的产出物
- 空间中心 — 如有平台操作
- 子角色中心 — 如需要专业深度
