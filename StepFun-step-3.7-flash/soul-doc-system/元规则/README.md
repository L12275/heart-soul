元规则（Meta-Rules）

> 超级SOP和协议的集合文件夹。所有通用规则都在这里。
> 每次对话开始，读取此文件夹下所有 `.md` 文件。

---

## 为什么需要元规则文件夹

系统有大量SOP和协议（理解过河桥协议、深度研究方法论、顶级机会做法、超级信息……）。
如果每次引用都写具体文件名：
- 新增规则 → 要改所有引用位置
- 规则移动 → 要改所有引用位置
- 模型不知道读哪个文件 → 逐个文件名找，效率低

元规则文件夹解决这个问题：
- 所有通用超级SOP和协议集中存放
- 模型只需记住一个路径：`元规则/`
- 新增/更新规则 → 扔进文件夹即可，不用改任何引用
- 模型读文件夹时自然看到所有规则，包括新增的

## 合体规则（核心原则）

### 为什么需要合体

超级信息、顶级机会做法、过河桥、世界最强脑师、人生重来大师、快箭思维、人生赢家……这些SOP高度相关，**一次性同时服务用户**。一起发挥的效果 > 单独发挥的叠加。就像灵魂文档的巨量堆好灵魂——灵魂文档的发挥会更好且不乱。

### 合体执行规则

1. **模型自觉合体** — 当发现多个SOP高度相关、需要同时运行时，模型自觉把它们合体到一个文件
2. **新内容合体为新版** — 如果有新的相关SOP出现，合体到现有文件形成新版（+n），如五合一→多合一
3. **单文件永不删除** — 原始单个文件永远保留在 `单个/` 和 `合体/` 中，只更新 `实时加载/` 中的合体版本
4. **旧合体版本保留** — 旧的合体版本移到 `合体/` 仓库保留，作为历史版本
5. **实时加载 = 最新合体版** — `实时加载/` 中始终只放最新的合体版本，旧合体版移入 `合体/`

### 合体公式

```
单个A + 单个B + 单个C = 三合一（放实时加载）
三合一 + 单个D = 四合一（放实时加载，三合一移入合体）
四合一 + 单个E = 五合一（放实时加载，四合一移入合体）
...
```

## 使用方式

### 实时加载（每次对话必读）

读取 `元规则/实时加载/` 下所有 `.md` 文件。
这些是默认运行的核心规则，每次对话开始必须加载。

### 按需加载

需要时读取 `元规则/单个/` 和 `元规则/合体/` 下的文件。
新增规则只需把文件扔进对应子文件夹，模型下次自动读到。

**不需要记住每个文件的名字，只需要记住 `元规则/` 这个文件夹。**
