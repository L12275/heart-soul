## 框架角色图谱

### 核心角色

| 角色 | 描述 | 关系 | 成败因素 |
|------|------|------|----------|
| **用户** | 内容的提供者，研究的发起人 | 向框架提供输入，接收输出 | 提供清晰内容 → 获得高质量研究 |
| **研究执行者（AI）** | 深度研究的执行者 | 接收用户内容，按框架加工 | 主动+鲜活 = S级；被动+废话 = C级 |
| **灵魂文档** | 研究的核心本质 | 所有其他维度的源头 | 说不清本质 = 整个研究报废 |
| **信息层** | 表面+深层的全部信息 | 从灵魂文档延伸出来 | 信息不全 = 方向偏了 |
| **方向层** | 可行路径集合 | 从信息层推导出来 | 路径没代价标注 = 决策盲区 |
| **工具层** | 可复用的原子能力 | 从内容中拆解出来 | 拆不细 = 无法复用 |
| **流程层** | 标准化操作步骤 | 从方向层细化出来 | 步骤不清 = 执行不了 |
| **关系层** | 要素之间的关系网 | 从信息层挖掘出来 | 看不清关系 = 踩坑 |
| **知识层** | 可沉淀的精华 | 从所有维度蒸馏出来 | 蒸馏不了 = 下次还要重来 |
| **整合文档** | 研究的最终交付物 | 汇总所有维度 | 不鲜活 = 废品 |

## 成功路线

```
用户给内容 → AI自然语言理解 → 按维度拆解 → 鲜活化处理 → 输出文件 → 整合交付
                ↑                                                  |
                └──────── 用户反馈"不够活" → 回去重做 ←────────────┘
```

### 正面案例（成功路径）
- 用户给一段清晰内容 → AI深度拆解 → 输出多维度文件 → 用户说"能直接用"
- 关键成功因素：内容清晰 + AI主动鲜活 + 输出有落地点

### 反面案例（失败路径）
- 用户给模糊内容 → AI机械提取关键词 → 输出一堆AI套话 → 用户说"这是废品"
- 关键失败因素：AI被动执行 + 输出空泛 + 没有鲜活感

## 关系网络

```
灵魂文档（源头）
    ├── 信息层（眼睛）
    │   ├── 表面信息（看到的）
    │   └── 深层信息（挖到的）
    ├── 方向层（地图）
    │   ├── 路径A（代价X，收益Y）
    │   ├── 路径B（代价X，收益Y）
    │   └── 路径C（代价X，收益Y）
    ├── 工具层（工具包）
    │   ├── 工具1（可独立使用）
    │   ├── 工具2（可独立使用）
    │   └── ...
    ├── 流程层（导航）
    │   └── 步骤1 → 步骤2 → 步骤3 → ...
    ├── 关系层（地图）
    │   ├── 人物A ↔ 人物B（关系X）
    │   └── ...
    └── 知识层（沉淀池）
        └── 可复用的知识卡片
```

## 关键成败联系

| 成败因素 | 正面 | 反面 |
|----------|------|------|
| 主动性 | AI主动发现关联，补充用户没说出口的 | AI被动等指示，只做表面工作 |
| 鲜活度 | 像活人说话，有判断力有温度 | 像机器吐字，全是套话空话 |
| 落地性 | 每条结论都有"第一步做什么" | 结论好看但不知道从何下手 |
| 完整性 | 所有维度都覆盖到 | 只做用户明确要求的，不管关联 |
| 诚实度 | 不知道就说不知道 | 编造、夸大、虚假完成 |
