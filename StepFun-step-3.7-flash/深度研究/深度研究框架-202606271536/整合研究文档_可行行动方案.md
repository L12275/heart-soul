深度研究框架 — 整合研究文档与可行行动方案

## 一、框架总览

深度研究框架是一套**活的思维操作系统**。它不是文档管理工具，不是AI文本生成模板，而是一个能把任意内容从"一堆文本"加工成"鲜活智慧"的工厂。

### 核心循环
```
内容输入 → 深度拆解（8维度） → 鲜活化处理 → 多文件输出 → 整合交付
     ↑                                                    |
     └──────────── 反馈"不够活" → 回去重做 ←──────────────┘
```

### 8个加工维度
| 维度 | 干什么 | 输出什么 |
|------|--------|----------|
| 灵魂文档 | 提炼核心本质 | 一句话说清"这到底是什么" |
| 信息观察 | 表面+深层信息 | 完整的认知地图 |
| 建议方向 | 可行路径分析 | 带代价和收益的路线图 |
| 技能工具 | 拆解可复用单元 | 最小能力工具包 |
| SOP工作流 | 标准化步骤 | 按此操作就能完成的流程 |
| 人物关系 | 关系网+成败分析 | 成功路线图和正反案例 |
| 知识库 | 沉淀精华 | 可反复调用的知识卡片 |
| 整合交付 | 汇总+行动方案 | 能直接拿来用的鲜活输出 |

## 二、为什么需要这个框架

### 问题：AI输出的废品厂
大多数AI输出是这样的：
- "总的来说，这个问题的核心在于..."
- "值得注意的是，我们需要考虑多个方面..."
- "不难发现，存在一定的挑战..."
- **结果**：好看但没用，像废品

### 解决方案：鲜活智慧
深度研究框架对抗废品厂的三条军规：
1. **每条结论必须有落地点** — "所以呢？"必须能回答
2. **用大白话，不装专业** — 让外行也能懂
3. **有判断力，不骑墙** — 活人有立场，AI也要有

## 四、框架的5条原则

| 原则 | 含义 | 反例 |
|------|------|------|
| **主动** | 不等指示，主动发现推进 | 用户没说出口的也补上 |
| **支撑** | 做有用的事 | 不做表面功夫、不凑字数 |
| **自动化** | 能自动的就别手动 | 路径/模板标准化 |
| **极简** | 最简单的方式优先 | 不搞过度复杂的设计 |
| **真实** | 不编造不夸大 | 不知道就说不知道 |
