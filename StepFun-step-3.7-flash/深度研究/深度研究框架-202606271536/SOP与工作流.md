## 标准流程（Standard Operating Procedure）

### 阶段一：接收与理解
1. **接收内容**：用户提供任意内容（文本、话题、资料）
2. **自然语言理解**：用大白话理解内容本质，不是机械提取关键词
3. **确定维度**：根据内容类型，确定需要激活哪些研究维度
4. **命名研究**：生成研究名称 + 当前时间戳

### 阶段二：深度拆解（按维度逐层加工）
5. **灵魂层**：这件事的核心本质是什么？一句话说清楚
6. **信息层**：表面看到了什么？深层挖到了什么？区分"现象"和"本质"
7. **方向层**：能往哪儿走？列出所有可行路径，标注每条路的代价和收益
8. **工具层**：哪些可以拆出来单独用？每个最小能力单元是什么
9. **流程层**：一步一步怎么做？步骤清晰到"按此操作就能完成"
10. **关系层**：谁/什么要素跟谁什么关系？成功路线怎么走？正反案例各举一个
11. **知识层**：什么可以沉淀下来反复用？提炼成可复用的知识卡片
12. **专项层**（按需）：如果是科举/代考/仕途等领域，启用专项分析

### 阶段三：鲜活化处理
13. **砍废话**：删除所有AI套话、空泛表述
14. **加落地点**：每条结论必须有"所以呢？"的答案——能做什么
15. **翻译成行动**：把研究结论翻译成第一步做什么、第二步做什么
16. **加温度**：用有感情、有判断力的语言写，像活人在说话

### 阶段四：输出交付
17. **建文件夹**：`深度研究/研究名称-时间戳/`
18. **写文件**：每个维度一个文件，命名规范统一
19. **写整合文档**：一份整合型研究 + 一份可行行动方案
20. **检查**：输出是否"可为生活"？是否避免了"废品厂"？

## 工作流图示

```
接收内容 → 自然语言理解 → 确定维度 → 命名研究
    ↓
灵魂层 → 信息层 → 方向层 → 工具层 → 流程层 → 关系层 → 知识层 → 专项层（按需）
    ↓
砍废话 + 加落地点 + 翻译成行动 + 加温度
    ↓
建文件夹 → 写文件 → 写整合文档 → 检查 → 交付
```

## 质量门控

- **灵魂层门控**：一句话能说清本质吗？说不清继续挖
- **信息层门控**：表面信息和深层信息区分开了吗？
- **方向层门控**：每条路径都有代价和收益标注吗？
- **工具层门控**：每个工具都能独立拿出来用吗？
- **流程层门控**：步骤清晰到外行也能按此操作吗？
- **关系层门控**：正反案例都举了吗？
- **知识层门控**：提炼的内容能沉淀复用吗？
- **鲜活化门控**：砍掉所有废话了吗？每条都有落地答案吗？
- **交付门控**：输出是"可为生活的"还是"废品"？

## 执行原则

- **主动不被动**：不等指示，主动发现和推进
- **支撑不消耗**：做有用的事，不做表面功夫
- **自动化不手动**：能自动的就不要手动
- **极简不复杂**：最简单有效的方式优先
- **真实不虚假**：不编造、不夸大、不虚假完成
