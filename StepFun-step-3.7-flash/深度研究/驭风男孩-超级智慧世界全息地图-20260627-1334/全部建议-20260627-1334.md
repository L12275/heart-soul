## 建议一：被系统排除时，潜入免费资源

**场景：** 学校开除你/公司开除你/社区排斥你/系统判定你"没有资格"

**建议：** 不要回家等死，不要接受判定。找到系统外的免费资源（图书馆/开源/互联网/社区中心），用它们填补系统留下的空白。

**行动：** 今天就去找一个免费的学习资源。潜入它。用它。

**快箭：** 被排除 → 找免费资源 → 填补空白 → 反超

---

## 建议二：教育费用过高时，寻找替代教育路径

**场景：** 学费太贵/课程太贵/认证太贵

**建议：** 正规教育不是唯一的学习路径。图书馆/开源/互联网/社区 = 免费的教育替代路径。自学效率 = 系统化学习的3倍（按需获取 vs 标准化灌入）。

**行动：** 列出3个你需要的技能 → 在免费资源中找到对应的资料 → 按需深挖 → 做出成果。

**快箭：** 教育太贵 → 找免费替代 → 按需自学 → 成果认证

---

## 建议三：绝望时，把绝望当作行动燃料

**场景：** 你处于绝境（失业/贫困/疾病/被否定）

**建议：** 绝望不是让你瘫痪的信号，是让你行动的触发器。把绝望转化为燃料——"既然已经不能再坏了，还有什么好怕的？"

**行动：** 写下最坏的结果（底线）→ 写下最好的可能（目标）→ 用底线和目标作为触发器 → 立即做一件最小的事。

**快箭：** 绝望 → 转化燃料 → 底线+目标 → 立即行动

## 建议四：创新时，用废料做原型

**场景：** 你想做创新但没钱买材料/没有"正确的"工具

**建议：** 创新不是"发明新材料"，是"重新认识旧材料的价值"。看身边的废料——旧自行车/废弃电子/废纸——它们的材料属性 = 创新的原材料。

**行动：** 列出身边3件"废弃"的东西 → 分析它们的材料属性 → 匹配你当前问题的需求 → 用废料做原型。

**快箭：** 没有材料 → 找废料 → 重新想象 → 零成本原型

---

## 建议五：需要说服别人时，先做成果

**场景：** 你想说服父母/老师/老板/社区接受你的想法

**建议：** 说服是最低效的方式。成果是最高的。做一个不可否认的成果 → 展示它 → 让它自己说话。

**行动：** 停止说服 → 做最小可验证的成果 → 在最佳条件下展示 → 等待认知转换。

**快箭：** 需要说服 → 做成果 → 展示 → 认知自动转换

---

## 建议六：社区不信任你时，用成果建立信任

**场景：** 你是社区里的" outsider"/新来的/被怀疑的人

**建议：** 社区信任 = 成果驱动的，不是宣传驱动的。不要宣传，不要解释，做一个可见的成果。

**行动：** 找一个社区最关心的问题 → 做一个能解决这个问题的最小成果 → 展示成果 → 信任自动建立。

**快箭：** 社区不信任 → 做成果 → 展示 → 信任建立

---

## 建议七：想扩大影响时，寻找放大器

**场景：** 你的本地成功了，但只有本地知道

**建议：** 本地成功 ≠ 全球影响。你需要放大器——教授/媒体/平台/网络/书籍。找到他们，让他们传播你的故事。

**行动：** 准备一个清晰的故事描述 → 识别可能的放大器 → 接触他们 → 让他们成为你的代言人。

**快箭：** 本地成功 → 找放大器 → 从本地到全球

---

## 建议八：知识要传递，不要囤积

**场景：** 你学到了有价值的东西/做出了有价值的成果

**建议：** 知识不是你的私有财产，是应该传递的火种。找到一个"下一个威廉"——有潜力但被系统排除的人——把火种传给他。

**行动：** 识别一个你认识的"被排除者" → 准备一份关键资料/技能 → 传递给他 → 不控制结果。

**快箭：** 获取知识 → 传递火种 → 知识不灭

## 建议九：在系统失效时，个体填补空白

**场景：** 政府断电/教育中断/基础设施失效/社会服务缺失

**建议：** 不要等待系统修复。系统修复可能永远不会来。个体填补 = 分布式创新的开始。

**行动：** 识别系统失效的领域 → 评估你能否填补 → 用现有资源填补 → 你的填补 = 他人的模板。

**快箭：** 系统失效 → 找空白 → 个体填补 → 分布式创新

## 建议十： stubborn 是你的生存算法

**场景：** 别人说你"不可能"/"你疯了"/"你不配"

**建议：** 在绝望环境中，stubborn 不是性格缺陷，是生存算法。拒绝接受"不可能" = 寻找出路。

**行动：** 当有人说"不可能"时 → 问自己"为什么不可能？" → 找到那条"不可能"的路 → 走通它。

**快箭：** 被说不可能 → 问为什么 → 找路 → 走通

## 建议十一：一本书 > 一百本泛泛的书

**场景：** 你想学习一个新领域/解决一个具体问题

**建议：** 不要博览群书。精准找到一本恰好解决你当前问题的书 → 深挖它 → 用它的知识行动。

**行动：** 明确你最核心的问题 → 搜索匹配的关键词 → 筛选出最匹配的1-3本 → 只读这几本 → 转化为行动。

**快箭：** 明确问题 → 精准搜索 → 一本对的책 → 行动

## 建议十二：原型不需要完美

**场景：** 你想做一个创新项目但想"等准备好了再做"

**建议：** 完美是行动的敌人。原型不需要完美，需要存在。用废料做原型 → 验证可行性 → 然后优化。

**行动：** 今天就用废料做一个最小原型 → 测试它 → 根据结果迭代。

**快箭：** 有想法 → 立即原型 → 用废料 → 验证后优化

## 建议十三：保护你的火种传递者

**场景：** 你有一个帮助你成长的导师/老师/朋友

**建议：** 火种传递者（迈克老师）是系统中最脆弱的人。他们可能欠薪/被排挤/离开。在他们还在的时候，珍惜他们的火种。

**行动：** 找到你的火种传递者 → 记录他们的关键帮助 → 把火种传递下去 → 让他们知道你感激他们。

**快箭：** 有火种传递者 → 珍惜火种 → 传递下去 → 感激

## 建议十四：防洪林不能砍

**场景：** 你在短期生存压力下考虑牺牲长期资源

**建议：** 砍防洪林 = 短期获益，长期灾难。任何牺牲长期资源的短期决策 = 未来的灾难。

**行动：** 评估每一个短期决策的长期后果 → 如果后果是"牺牲长期资源" → 找替代方案 → 不砍树也能活下去。

**快箭：** 短期压力 → 评估长期后果 → 找替代 → 不牺牲长期
