驭风男孩 — 社区拯救工作流

> **默认启动。适用于社区危机/社会问题/集体行动。**
> 威廉没有只救自己——他用风车拯救了整个村庄。
> 个人行动 → 成果展示 → 社区信任 → 集体行动 → 系统改变。

---

## 一、触发条件

| 场景 | 是否触发 |
|------|---------|
| 社区面临危机（饥荒/灾害/疾病） | ✅ 必须触发 |
| 社区基础设施失效（断电/断水/断教育） | ✅ 必须触发 |
| 社区缺乏信任（怀疑/分裂/冷漠） | ✅ 必须触发 |
| 个人有能力但社区不知道 | ✅ 必须触发 |
| 需要集体行动但没有人带头 | ✅ 必须触发 |

---

## 二、执行流程（6步）

```
第1步：识别危机
  │   ── 社区面临什么问题？
  │   ── 问题的根源是什么？
  │   ── 谁最受影响？
  │
  ▼
第2步：个人行动（不等别人）
  │   ── 不要等"有人来解决"
  │   ── 自己先做最小可行的行动
  │   ── 用自己有的资源（不等待外部援助）
  │
  ▼
第3步：成果展示
  │   ── 做可触摸/可见/可验证的成果
  │   ── 在最佳条件下展示给社区
  │   ── 让成果自己说话
  │
  ▼
第4步：社区信任建立
  │   ── 成果 = 最强大的信任建立工具
  │   ── 从怀疑 → 见证 → 信任
  │   ── 不需要宣传，不需要解释
  │
  ▼
第5步：集体行动
  │   ── 信任建立后 → 邀请社区参与
  │   ── 从个人项目 → 社区项目
  │   ── 社区参与 = 放大个人成果
  │
  ▼
第6步：系统改变
  │   ── 社区成功 → 影响更广的系统
  │   ── 从村庄 → 全县 → 全国
  │   ── 系统改变 = 最终目标
  │
  ▼
完成：社区拯救
```

---

## 三、每步的铁律

### 第1步：识别危机

1. **不回避问题** — 正视危机的存在
2. **找到根源** — 不是表面症状，是根本原因
3. **确定影响范围** — 谁受影响？多严重？

### 第2步：个人行动

1. **不等别人** — "没有人来解决" = 我来解决
2. **用现有资源** — 不等待外部援助
3. **最小可行行动** — 不是大计划，是小行动

### 第3步：成果展示

1. **成果必须可触摸/可见/可验证**
2. **展示条件 = 最佳**
3. **让成果自己说话**

### 第4步：社区信任

1. **成果驱动信任**
2. **从怀疑到信任 = 只需要一个不可否认的成果**
3. **不需要宣传，不需要解释**

### 第5步：集体行动

1. **信任建立后邀请参与**
2. **从个人项目 → 社区项目**
3. **社区参与 = 放大效应**

### 第6步：系统改变

1. **社区成功 → 系统影响**
2. **从村庄 → 全县 → 全国**
3. **系统改变 = 最终目标**
