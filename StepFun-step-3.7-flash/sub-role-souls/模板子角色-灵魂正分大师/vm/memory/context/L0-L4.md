## L0 — Meta Rules（元规则层）

**载体：** system prompt 硬编码（不是文件）
**职责：** 最强约束——Agent 基础行为规则与系统约束
**特点：** 每轮随上下文注入，不可被绕过，不可编辑

### GA L0 包含

- 原子工具定义和约束
- Agent Loop 行为规则（感知→推理→执行→记忆→循环）
- "No Execution, No Memory" 原则
- 神圣不可删改性
- 禁止存储易变状态

### 子角色 L0

子角色的 L0 = 总灵魂的红领巾精神 + 八荣八耻 + 安全铁律
这些不是放在 vm/memory/ 里的，是每次加载子角色时从总灵魂继承的硬约束。

**L0 不可修改，不可删除，不可绕过。**

---

## L1 — Insight Index（洞察索引）

**载体：** vm/memory/insight-index.md（≤30行）
**职责：** 极简导航——只记"有什么"和"去哪找"，不记细节

### GA L1 格式（≤30行，两级映射）

```
高频场景 → [直接指向L3文件名]
低频场景 → [关键词]，需自行在L2/L3查找
```

### 反面示例（错误的L1）

```markdown
L1：认知重构 — proven — 可用于所有焦虑症来访者
```
❌ L1 不能写细节，不能写结论，不能超过30行。

---

## L2 — Global Facts（全局事实库）

**载体：** sections/03-knowledge-base.md
**职责：** 环境事实——路径/凭证/配置/常量等稳定知识

### GA L2 三条红线公理

1. **No Execution, No Memory** — 没有实际执行过的信息，不写入记忆
2. **神圣不可删改性** — 已验证数据只能压缩/迁移，不能删除
3. **禁止存储易变状态** — 不存"当前次数""最近操作时间"这类易变数据
4. **最小充分指针** — 上层只留最短标识，不层层叠加引用

### 子角色 L2

子角色的 knowledge-base.md 就是 L2。
每条知识必须标注：可信度 + 来源 + 时间 + 场景 + 原因。

---

## L3 — Task Skills/SOPs（任务技能）

**载体：** sop/ 目录
**职责：** 特定任务的可复用流程

### GA L3 自动沉淀规则

```
同一场景出现3次 → 提炼为SOP候选 → 验证1次 → 标记validated
validated SOP积累 → 触发merge到更高层级
```

### 子角色 L3

子角色的 sop/ 目录就是 L3。SOP 从实际工作中结晶，不是预先设计的。

---

## L4 — Session Archive（会话归档）

**载体：** vm/log/archive/年/月/日/[会话ID].md
**职责：** 已完成任务的归档记录

### GA L4 管理

- 通过 cron 每12小时自动归档历史会话
- 不是实时写入，是 batch 归档
- 归档后原始会话数据可压缩

### 子角色 L4

子角色的 `vm/log/archive/年/月/日/` 就是 L4。
每次子角色工作结束后 → 整理日志 → 归入对应日期的文件夹。

## 记忆流转图

```
L0 (system prompt硬编码) ← 每轮注入，不可绕过
    ↓ 快速路由
L1 Insight Index (≤30行导航) ← "有什么+去哪找"
    ↓ 查找
L2 Global Facts (稳定知识) ← "事实+来源"
    ↓ 执行
L3 Task Skills/SOPs (可复用流程) ← "怎么做"
    ↓ 完成
L4 Session Archive (归档) ← "做过什么"
```

**新任务流程：**
[感知需求] → [L1索引定位] → [L3 SOP执行] → [L2验证事实] → [完成] → [L4归档]

---

## GA 核心原则

1. **No Execution, No Memory** — 无执行不记忆
2. **神圣不可删改性** — 已验证数据不可删除，只能压缩/迁移
3. **禁止存储易变状态** — 只存稳定的环境事实
4. **最小充分指针** — 上层只留最短标识
5. **不要预加载技能，让它们进化** — Don't preload skills, evolve them
