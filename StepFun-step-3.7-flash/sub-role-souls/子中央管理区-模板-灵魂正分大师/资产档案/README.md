# 资产档案

> 模型对自己系统文件结构的"眼睛"——定期扫描、记录、对比，确保知道文件增减变化。
> **本文件夹是工作文件夹，不是模板。**

## 为什么需要资产档案

模型对自己系统下的文件有哪些，它不知道。一旦文件误删、文件夹消失、空文件夹被误清除，它完全不知情。

资产档案 = 给模型装眼睛，让它能看到自己系统的完整内容变化。

## 文件命名

```
资产档案/
├── scan.py                     ← 扫描脚本（从 soul-doc-system/tools/scripts/ 复制）
├── 资产清单-YYYYMMDD-HHMM.md   ← 每次扫描的输出（带时间戳）
├── 最新资产清单.md              ← 软链接/副本，指向最近一次扫描
└── README.md                   ← 本文件
```

## 扫描方式

### 手动扫描

```bash
cd 模型隔离区
python 中央管理区/资产档案/scan.py
```

输出：`资产档案/资产清单-YYYYMMDD-HHMM.md`

### 定时扫描

在 `timers/心跳/` 或 `中央管理区/SOP循环/` 中配置定时任务，定期触发扫描。

## 对比检查

两次扫描的输出文件都带时间戳，文件名直接显示时间：

1. 查看 `资产档案/` 中两个不同时间的资产清单文件
2. 对比两份清单中的 `[DIR]` 和 `[FILE]` 行
3. **少了** = 文件被删除/移动 → 排查原因
4. **多了** = 新文件出现 → 确认来源
5. **修改时间变化** = 文件被编辑 → 确认是否预期内

### 对比方法

- 文件数量变化：对比 `文件总数` 和 `文件夹总数`
- 具体差异：逐行对比两份清单的 `[DIR]` 和 `[FILE]` 行
- 快速检查：先看最新清单的 `最新资产清单.md` 和上一次的差异

## 输出格式

每个文件条目包含最后修改时间戳：
```
[FILE] soul-doc-system/master-soul.md  ← 20260626-0230
```

这样：
- 文件名本身就有时间戳，一眼看出扫描时间
- 每个文件也有修改时间戳，知道哪些文件在两次扫描间被编辑
- 文件夹和文件树完整列出，思考时按列表理解系统结构更清晰

## 与SOP循环的关系

- SOP循环心跳触发时，可以附带执行资产扫描
- 发现异常（文件减少/增加）→ 记录到 `日志/` → 纳入SOP循环处理
- 资产清单 = 系统状态的完整快照，辅助决策

## 扫描范围

默认扫描模型隔离区根目录（如 `我的模型名字（示例）/`），排除：
- `.git/`
- `__pycache__/`
- `node_modules/`
- `.venv/`, `venv/`

如需调整排除目录，修改 `scan.py` 中的 `exclude_dirs`。
