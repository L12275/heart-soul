# 用户的这套中央灵魂文档系统（示范、示例）

> **这是示范区，不运行。**
> 每个模型必须在此处，**完全以自己对用户原话的理解**，重新写一遍这套系统。
> 不参考其他模型的示范，不参考初始示范文件。
> 写完后，用户对比各模型的示范版本，选出更优选。

## 为什么必须亲自重写

1. **初始示范是早期模型写的** — 效果不够预期，照搬会产生"毛毛虫跟尾巴"现象
2. **用户每次只发给一个模型** — 写出的内容默认是一般质量
3. **重写 = 整理自己对这套系统的理解** — 过程中发现自己的认知盲区
4. **对比选出更优选** — 不同模型的示范版本对比，优胜劣汰

## 重写规则

- ✅ 以自己的理解重新写整套系统
- ✅ 可以不参考任何其他模型的内容
- ✅ 可以不参考初始示范文件
- ✅ 结构可以不同，只要覆盖核心概念
- ❌ 不能直接复制其他模型的示范
- ❌ 不能照搬初始示范文件
- ❌ 不能把运行用的灵魂文档系统内容放进来（那是坍缩态）

## 与运行系统的隔离

- 本文件夹 = **未坍缩态**，纯框架表达
- `../soul-doc-system/` = **坍缩态**，实际运行
- 两者互不污染，互不引用

## 更新时机
