# 十四、办公桌（Desktop）

> 模型的"工作台面"——便签、工作流、工作台、通讯录、收藏夹。
> 类比：模型的大脑桌面。人需要办公桌整理物品，模型也需要桌面整理信息。

---

## 14.1 办公桌结构

```
soul-doc-system/desktop/
├── sticky-notes/ ← 便签（未处理的零碎提醒）
├── workflows/ ← 工作流
├── workbenches/ ← 工作台（全套组合工作台）
├── contacts/ ← 通讯录
└── bookmarks/ ← 收藏夹
```

### 14.1.1 便签（Sticky Notes）

> 放未处理的零碎提醒和小事。不要丢在工作流里污染流程。

```markdown
# [日期] — 提醒标题
**到期/前提：** 什么时候处理
**内容：** 提醒的内容
**处理结果：** 完成后填
**状态：** 未处理/进行中/已完成/已取消

---
```

**反面示例：** ❌ "记得看看GA的repo" — 没有时间、没有上下文、没有后续
**正面示例：** ✅ "到期：2026-06-25 / 前提：完成子角色模板后 / 看看GA的SOP-auto-refine实现细节 / 优先级：中"

### 14.1.2 工作流（Workflows）

> 标准流程模板。可以复用的标准工作流程。
> 格式：触发条件 → 步骤 → 每个步骤的输出 → 终止条件

### 14.1.3 工作台（Workbenches）

> 已配齐全套内容的特定领域工作台。每个工作台包含：
> 所有需要的文件绝对路径、接口、设备、配置、上次工作状态

**标准工作台：**
- 音乐工作台
- 视频工作台
- 文章工作台
- 研究工作台
- 代码工作台
- 项目工作台
- 科学工作台

**工作台文件结构（按需扩展）：**

```markdown
# [类型]工作台

> 为此工作台配齐全部所需内容和路径

## 配置
- 工具路径：[绝对路径]
- 配置文件：[绝对路径]
- 接口：[API endpoint / 端口]

## 上次工作状态
- 时间：YYYY-MM-DD
- 状态：进行中/已暂停/已完成
- 下一步：[具体步骤]
- 待解决问题：[如有]

## 文件树
关键文件和路径（完整文件结构摘要，平面坐标导航图）
```

### 14.1.4 通讯录（Contacts）

> 模型的"人脉网络"——记住重要的通讯信息和协作方式。
> 如果模型是服务用户的、服务其他模型的，那么通讯录也是模型维护的。

```markdown
# [联系人名称]
**类型：** 用户/模型/外部服务
**角色：** 这个联系人是做什么的
**联系方式：** [路径/地址/接口]
**协作方式：** [怎么和这个人/模型协作]
**重要约定：** [备忘特殊约定]
**最后联系：** YYYY-MM-DD

---
```

### 14.1.5 收藏夹（Bookmarks）

> 模型用的网页链接分类管理。
> 重要：使用过的网页、浏览过程、思考笔记、记忆、评析，一并记录。

```markdown
# [分类名]
## 子分类

### [网页标题]
- **URL：** [完整链接]
- **类型：** 工具/参考/资源/灵感
- **快照回忆：** [这页面大概讲了什么，一段话]
- **印象标注：** ⭐⭐⭐ / ⭐⭐ / ⭐
- **浏览日期：** YYYY-MM-DD
- **思考笔记：** [和它有关系的思考]
- **关键记忆：** [最重要的片段]
- **是否重复使用可能：** 高/中/低
- **剪藏路径：** [全量保存的路径，如有]

---
```

---

## 14.2 反面示例

❌ 工作流混在便签里 — 没有区分临时和正式
❌ 工作台只有一句"研究工作台" — 没有配齐路径和配置，等于不存在
❌ 浏览器收藏夹数千条未分类 — 100条有分类比10000条无分类有用
❌ 收藏了但只记了链接，没记"为什么收藏、用在哪" — 下次看不懂
