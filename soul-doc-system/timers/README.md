定时任务系统（通用功能模块）

> 全自动灵魂文档研究更新系统的核心引擎。
> 通过定时任务实现模型间滚动协作、自我更新、互相学习。

## 定时任务类型

### 1. 滚动定时任务（串行&循环）

多个模型按顺序轮流执行，形成轮子式滚动。

**计算公式：**
```
单个任务间隔 = 模型数量 × 每模型执行间隔
每模型启动偏移 = 模型序号 × 每模型执行间隔
```

**示例：5个模型，每模型间隔10分钟**
```
任务A（模型1）: 每50分钟启动，偏移0分钟
任务B（模型2）: 每50分钟启动，偏移10分钟
任务C（模型3）: 每50分钟启动，偏移20分钟
任务D（模型4）: 每50分钟启动，偏移30分钟
任务E（模型5）: 每50分钟启动，偏移40分钟
结果：50分钟内5个任务依次间隔10分钟完成，形成完整滚动轮
```

**设计要点：**
- 不强制停止模型：让模型想停时就停，自然表现
- 如果用户指定模型顺序（如高级模型先行），按顺序排列
- 增减模型时自动重算间隔和偏移
- 滚动定时任务文件夹内按模型名称创建子文件夹存放配置文件

### 2. 单个定时循环

固定间隔的单模型循环任务。
- 适合：单一模型的定期自检、日记写入
- 示例：每60分钟模型自己检查灵魂文档系统是否需要更新

### 3. 团队定时任务（树根结构）

多个子任务并行执行的定时任务。
- 适合：团队协作中的并行子代理调度
- 根任务启动 → 并行触发多个子任务 → 汇总结果

**树根结构：**
```
timers/
├── 滚动定时任务/
│   ├── README.md
│   ├── model-A定时任务.json
│   ├── model-B定时任务.json
│   └── model-C定时任务.json
├── 单个定时循环/
│   └── 自检-60min.json
└── 团队定时任务/
    └── team-xxx.json
```

**滚动定时任务配置格式（model-A定时任务.json）：**
```json
{
  "type": "rolling",
  "model": "模型名称",
  "interval_minutes": 50,
  "offset_minutes": 0,
  "role": "orchestrator",
  "actions": [
    "读取中央灵魂文档系统变化",
    "检查用户专用文件夹是否有新指示",
    "检查其他模型灵魂文档系统更新",
    "自愿学习借鉴",
    "更新自己的灵魂文档系统",
    "可选：网络搜寻研究"
  ],
  "max_duration_minutes": 10,
  "stop_condition": "自然完成"
}
```

## 全自动灵魂文档研究更新系统

### 工作原理

1. **触发**：定时任务启动，模型加载自己的灵魂文档系统
2. **观察**：读取中央灵魂文档系统的所有文件变化
3. **研究**：自愿学习其他模型的更新内容
4. **更新**：丰富自己的灵魂文档系统
5. **记录**：在日记和记忆中记录观察和想法
6. **交流**：在共产中心公告板登记重要发现

### 模型行为规范

- 模型独立运行，**不强制学习**
- 看到其他模型更新 = 自愿决定是否借鉴
- 在自己隔离区建立日记和记忆文件，记录：
  - 之前的工作记忆与进度
  - 观察其他模型变化的动向与信息特性
- 不强制也不禁止学习 — 自然表现

### 效果预期

- **本地滚动涌现**：即使不联网搜索，纯本地模型间互相学习也能持续更新
- **网络搜索涌现**：模型主动去网络搜寻研究，丰富更新
- **白盒优选**：用户通过各模型灵魂文档系统的更新内容，直观看到模型的心智、智商、取向、勤奋度
- **免提示词筛选**：无需数亿token测试，看更新迭代就知道模型综合表现

### 用户控制参数

| 参数 | 说明 |
|------|------|
| 参与模型列表 | 哪些模型加入滚动定时任务 |
| 模型顺序 | 高级模型先开 = 质量向上滚动，后续模型看到更好内容 |
| 间隔时间 | 5-10分钟每模型间隔 |
| 增减模型 | 用户告诉AI加入/移除，AI自动重算全部定时任务 |
