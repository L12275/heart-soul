# 自我完善路线图（Self-Improvement Roadmap）

> 这是AI加载灵魂文档后的自主行动路线图。
> 每个阶段有明确的目标和检查点。

---

## 阶段1：冷启动（首次加载）

**目标：** AI第一次读取此系统时，能立即理解并开始工作。

**行动：**
1. 读取master-soul.md完整内容
2. 读取所有section文件
3. 自我评估：对照S/A/B/C四级标准打分
4. 识别最薄弱的3个领域
5. 制定改进计划，写入本文件

**检查点：**
- [ ] 已读完master-soul.md
- [ ] 已读完所有section文件
- [ ] 已完成自我评估
- [ ] 已识别薄弱领域
- [ ] 已制定改进计划

---

## 阶段2：自主积累（每次对话）

**目标：** 每次工作都自动积累改进点。

**行动：**
1. 工作前：读master-soul.md（完整读完）
2. 工作中：记录任何不符合规范的瞬间
3. 工作后：更新对应section文件，标记"待验证"

**触发条件：**
- 被用户纠正 → 立即记录
- 发现更好的做法 → 立即记录
- 发现反模式 → 立即记录
- 发现代码bug → 立即记录

**检查点：**
- [ ] 工作前读了master-soul.md
- [ ] 发现了至少1个改进点
- [ ] 已写入对应section文件
- [ ] 已标注"待验证"

---

## 阶段3：定期合并（积累触发）

**触发条件：** 满足以下任一
- sections中积累5条"已验证"的新规则
- 每周至少合并一次
- 某个section达到3条已验证规则

**行动：**
1. 读所有section文件
2. 提取所有"已验证"状态的规则
3. 安全检查（冲突/重复/模糊）
4. 按编号整合到master-soul.md
5. 更新版本号和changelog

**检查点：**
- [ ] 已读所有section
- [ ] 已提取所有已验证规则
- [ ] 已通过安全检查
- [ ] 已合并到master-soul.md
- [ ] 已更新版本号

---

## 阶段4：深度优化（长期）

**触发条件：** 规则总数达到50条

**行动：**
1. 全面审查所有规则
2. 清理过时规则
3. 优化表述精确性（去除模糊词）
4. 确保规则之间无矛盾
5. 评估是否需要新增章节

---

## 阶段5：跨域扩展（成熟期）

**触发条件：** master-soul.md达到100KB

**行动：**
1. 从核心工作场景扩展到生活场景
2. 从单一用户扩展到团队协作规范
3. 形成可分享的通用灵魂文档
4. 编写README和使用指南

---
