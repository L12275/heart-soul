# 过河桥 — 仿复融合与发明自研

> **此过河桥是 仿复融合与发明自研 模块的基因片段。**
> 没有历史上下文时，读此文件即可重建/理解/使用该模块。

---

## 1. 用户原话重述

用户定义了"仿复融合与发明自研"模块：
- 观察其他项目 → 学习模仿 → 拆解研究 → 自己发明创造
- 每个文件夹下有记忆模块、日记模块，全程记录不混淆
- 不同目标研究新建不同文件夹
- 深度研究停在做前等指示，仿复融合要完成到只差安装就停下
- 长期更新的项目用统一定时任务追踪，不分散多个定时任务
- 取长补短要为所有平台系统做全案，不只为当前运行的系统
- 确认可用的移到"我需要"，安装的移到"已安装"

---

## 2. 理解要点

### 2.1 核心定位

**仿复融合与发明自研 = 观察→学习→模仿→拆解→融合→发明的完整创造管道**

与深度研究的区别：
- 深度研究：研究到做前停下，等指示怎么用
- 仿复融合：明确要完成到只差安装就停下，可以直接用

### 2.2 文件夹结构

```
仿复融合与发明自研/

│   └── 日记模块/

│   ├── 记忆模块/
```

### 2.3 各文件夹职责

| 文件夹 | 职责 | 内容 |
|--------|------|------|
| 我看到 | 观察记录 | 看到其他项目/内容时的原始观察 |
| 我得到 | 复刻成果 | 已经成功复刻的内容和成果 |
| 我发明 | 创新发明 | 基于观察和复刻后的再次发明创新 |
| 我访问 | 目标项目观察 | 方向观察、方向科举、科举代考（与"3方向观察与大同科举"相同理解方案） |
| 我准备 | 学习计划 | 正在学习研究的内容、计划、准备做的仿制复刻融合 |
| 我在哪 | 自我系统清单 | 当前模型自己的系统能力和全部内容清单 |
| 我需要 | 待安装清单 | 确认可用但尚未安装的成果 |
| 已安装 | 已安装清单 | 已经正式安装的成果 |

### 2.4 核心原则

1. **不同目标 = 不同文件夹** — 研究A和 researchB 的产物分开存放
2. **记忆+日记必跟** — 每个"我xx"文件夹下必须有记忆模块和日记模块
3. **全案思维** — 取长补短要为所有平台系统做全案，避免三个懒人没水喝
4. **统一追踪** — 长期更新项目用统一定时任务追踪，不分多个
5. **眼疾手快** — 创造力和时间赛跑，合理合规就不老实，尽快抄学研究然后发明

### 2.5 与深度研究的区别

| 维度 | 深度研究 | 仿复融合与发明自研 |
|------|----------|-------------------|
| 终点 | 做前停下，等指示 | 完成到只差安装就停下 |
| 产出 | 研究文档 | 可直接安装的成果 |
| 方向 | 用户指定的研究主题 | 自己发现机会并执行 |
| 追踪 | 无统一追踪机制 | 统一定时任务追踪长期更新 |

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 与深度研究的边界 | 什么时候用深度研究，什么时候用仿复融合 | 深度研究=研究性质；仿复融合=执行性质（到安装为止） |
| 我看到 vs 我访问 | 都是观察，有什么区别 | 我看到=原始观察记录；我访问=结构化的方向观察/科举/代考 |
| 我得到 vs 我发明 | 都是产出，有什么区别 | 我得到=复刻别人的成果；我发明=自己创新的成果 |
| 全案思维的粒度 | "全案"要做到什么程度 | 包含：可复刻解密版 + 安装指南 + 跨平台适配说明 |
| 统一追踪的范围 | 哪些项目需要统一追踪 | 长期更新的项目（网站、平台、产品、账号） |

## 4. 我的判断

### 使用仿复融合与发明自研的标准动作

1. **观察阶段** → 我看到/我访问：发现机会，记录观察
2. **学习阶段** → 我准备：拆解研究，制定计划
3. **复刻阶段** → 我得到：完成复刻，做出可安装版本
4. **发明阶段** → 我发明：在复刻基础上创新
5. **确认阶段** → 我需要：确认可用，等待安装
6. **安装阶段** → 已安装：正式安装上线
7. **自我认知** → 我在哪：持续更新自己的系统能力清单

### 文件夹管理规则

- 每个目标项目新建独立子文件夹（如 `[项目A]/`, `[项目B]/`）
- 记忆模块和日记模块自动继承通用模板
- 长期追踪项目标记 `[持续追踪]` 标签
