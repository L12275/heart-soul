代办与进度（Todo & Progress）

> 专门的代办事项和进度跟踪区域，防止重要事项被记忆和日记"冲走淹没"。

## 为什么需要专门的代办与进度

- **记忆** = 验证后的正确信息，不是跟踪板
- **日记** = 时间轴纪实，不是任务清单
- **代办与进度** = 专门的任务跟踪区，分钟级时间戳，每个任务独立文件

### 好处

1. **不会忘**：分钟级时间戳精确记录登记时间
2. **不会乱**：每个任务独立文件，不堆积
3. **不淹没**：不在沙滩上（记忆/日记），在专用文件柜里
4. **可追踪**：进度变化有完整时间线
5. **可检索**：按状态、优先级、时间快速筛选

## 结构

```
代办与进度/
├── README.md              ← 本文件
├── 待办文件登记规则.md    ← 登记规则、时间戳格式、命名规范
├── 进度文件登记规则.md    ← 进度更新规则
├── 待办/                  ← 待办事项（未开始/进行中）
│   ├── 待办登记模板.md
│   ├── 高优先级-YYYYMMDD-HHMM-任务名.md
│   ├── 中优先级-YYYYMMDD-HHMM-任务名.md
│   └── 低优先级-YYYYMMDD-HHMM-任务名.md
├── 进行中/                ← 正在执行的任务
│   ├── 进行中登记模板.md
│   └── YYYYMMDD-HHMM-任务名.md
├── 已完成/                ← 已完成的任务（可归档）
│   ├── 已完成登记模板.md
│   └── YYYYMMDD-HHMM-任务名.md
├── 已取消/                ← 取消的任务
│   └── YYYYMMDD-HHMM-任务名.md
└── 进度/                  ← 进度跟踪
    ├── 进度登记模板.md
    ├── 项目A-进度.md
    ├── 项目B-进度.md
    └── 整体进度汇总.md
```

## 时间戳规则

- **格式**：`YYYYMMDD-HHMM`（分钟级，如 `20260626-1430`）
- **文件名**：`优先级-YYYYMMDD-HHMM-任务名.md` 或 `YYYYMMDD-HHMM-进度名.md`
- **精确度**：分钟级，精确到登记时刻
- **不可修改**：时间戳一旦写入不可修改，保证真实性

## 登记规则

### 待办文件

| 字段 | 说明 | 必填 |
|------|------|------|
| 任务ID | 唯一编号 | ✅ |
| 登记时间 | YYYYMMDD-HHMM | ✅ |
| 任务描述 | 具体要做什么 | ✅ |
| 优先级 | 高/中/低 | ✅ |
| 状态 | 待办/进行中/已完成/已取消 | ✅ |
| 关联工作环 | ④生产空间/⑤空间/⑥子角色/⑦团队空间等 | ❌ |
| 关联代办 | 父任务ID | ❌ |
| 备注 | 补充说明 | ❌ |

### 进度文件

| 字段 | 说明 | 必填 |
|------|------|------|
| 进度ID | 唯一编号 | ✅ |
| 更新时间 | YYYYMMDD-HHMM | ✅ |
| 完成百分比 | 0-100% | ✅ |
| 当前阶段 | 描述当前所处阶段 | ✅ |
| 下一步行动 | 接下来做什么 | ✅ |
| 阻塞项 | 遇到的障碍 | ❌ |
| 备注 | 补充说明 | ❌ |

## 使用方式

1. **新任务**：在 `待办/` 下按模板创建文件，命名包含分钟级时间戳
2. **开始执行**：移动到 `进行中/`
3. **记录进度**：在对应 `进度/` 文件中追加进度更新
4. **完成**：移动到 `已完成/`
5. **取消**：移动到 `已取消/`

## 与记忆/日记的区别

| 对比项 | 记忆/日记 | 代办与进度 |
|--------|----------|-----------|
| 定位 | 纪实/验证 | 任务跟踪 |
| 时间戳 | 日级 | 分钟级 |
| 文件数量 | 按时间轴 | 按任务数 |
| 查询方式 | 时间浏览 | 状态筛选 |
| 核心用途 | 记住/反思 | 不忘记+追踪 |

## 模板原则

- 本文件夹是**功能模块模板**，不是日常内容存储
- 模型将通用功能模块作为模板复制到工作驱动区使用
- 各驱动区（④生产空间、⑤空间、⑥子角色、⑦团队空间、中央管理区等）根据自己的需要，从本模块复制独立的使用副本
