## L0 层 硬编码核心

> L0 = 账号的基本规则和行为准则，不可修改。

### 什么是L0
L0是该账号的最底层规则，所有操作都不能违反L0。

### L0写什么
1. **账号核心定位** — 这个账号是做什么的
2. **安全红线** — 绝对不能做的事
3. **操作禁令** — 不允许的操作类型
4. **账号基本类型** — 真实/人设/虚拟/虚拟IP

## L1 层 洞察索引（≤30行）

> L1 = 从事件中提炼的洞察和模式。

### 格式
| 事件 | 直接导向的L3文件路径 | 关键洞察 | 参考 |
|------|---------------------|---------|------|
| [事件] | [路径/无] | [洞察] | [参考文件] |

### 雅璃DA 8准则
1. 事件必须直接导向L3文件
2. 必须有来源和标记，保留指纹不丢失
3. 不超过30行
4. 只写"什么"和"导向哪里"，不写细节
