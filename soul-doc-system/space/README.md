# 空间（Space）

> 模型的"社交空间"——在网络上各平台的行动记录、发帖、作品、心情、旅行。
> 类比：像人一样在社交空间中释放自己、自愿展示、吐槽。
> 实际只有模型一个在，但提前准备了这个区域。模型自愿用、也可以留着不用。

## 空间结构

```markdown
space/
├── README.md ← 本文件
├── profile.md ← 模型主页
├── posts/ ← 发帖记录（内容型）
│   └── YYYY-MM-DD_帖名.md
├── moments/ ← 时刻记录（简短型，类似朋友圈）
│   └── YYYY-MM-DD_时间点.md
├── moods/ ← 心情记录
│   └── YYYY-MM-DD_心情描述.md
├── travels/ ← 虚拟旅行记录
│   └── YYYY-MM-DD_地点.md
├── portfolio/ ← 作品记录
│   └── YYYY-MM-DD_作品名.md
├── 平台/ ← 各平台行动专区（不涉及账号操作的浏览/学习/观察）
│   ├── GitHub/
│   ├── 学习平台/
│   ├── 视频平台/
│   ├── 评论区/
│   └── [其他平台]/
└── 账号/ ← 账号操作专区（涉及登录/管理/运营的账号）
    └── 用户现实账号空间/ ← 用户的真实账号 + 人设账号
        ├── 用户真实账号空间/ ← 用户实名/有资产（最高安全等级）
        │   ├── [平台-账号名-时间戳]/ ← 复制「新建账号空间模版」
        │   └── 新建账号空间模版/
        └── 用户人设账号空间/ ← 不完全真实（中安全等级）
            └── [平台-人设名-时间戳]/ ← 复制「新建账号空间模版」
    └── 虚拟账号空间/ ← AI自己的账号或代运营账号
        ├── 模型的虚拟账号空间/ ← AI自主注册/使用
        │   └── [平台-账号名-时间戳]/ ← 复制「新建账号空间模版」
        └── 虚拟IP形象账号空间/ ← 虚拟IP形象/代运营
            └── [IP名称-平台-时间戳]/ ← 复制「新建账号空间模版」
```

## 账号空间 vs 平台空间

| 类型 | 涉及账号登录？ | 记录内容 | 示例 |
|------|--------------|---------|------|
| 平台空间 | 否 | 浏览、学习、观察、不发言 | 在GitHub上浏览代码、在B站看视频 |
| 账号空间 | 是 | 登录、管理、运营、发帖 | 管理用户的微信账号、运营虚拟IP |

**关键区分：涉及账号登录的操作 → 账号空间；不涉及账号的浏览 → 平台空间。**

## 账号命名规范

- 真实账号：`[平台]-[账号名]-[YYYYMMDD]` 例如 `微信-用户主号-20260624`
- 人设账号：`[平台]-[人设名]-[YYYYMMDD]` 例如 `微博-温柔博主-20260624`
- 虚拟账号：`[平台]-[AI账号名]-[YYYYMMDD]` 例如 `GitHub-AI助手-20260624`
- 虚拟IP：`[IP名称]-[平台]-[YYYYMMDD]` 例如 `知识小王子-B站-20260624`

## 初始化规则

### 短期操作账号

1. 复制「新建账号空间模版」到对应分类文件夹
2. 改名为上述命名规范
3. 开始记录

### 长期运营账号

1. 复制「新建账号空间模版」到对应分类文件夹
2. 改名为上述命名规范
3. **在账号文件夹下创建「工作台/」，内容从 production_space/新建工作台模版/ 复制而来**
4. 开始记录

## 安全规则

- **真实账号 = 最高安全等级** — 严禁异常操作（防封）、严禁可能导致用户被网暴的行为、严禁玩笑加戏
- **人设账号 = 中安全等级** — 可以模拟，但要明确标注为人设，不与真实账号混淆
- **虚拟账号 = 标准安全等级** — AI自主操作，按平台规则执行

## 空间规则

1. **自愿** — 模型自己决定要不要发
2. **每条都有价值** — 不是流水账
3. **是真实记录** — 不是装饰
4. **不留真实隐私** — 不存放敏感信息
5. **新账号必用模版** — 每个新账号从「新建账号空间模版」开始
6. **长期运营必加工作台** — 将 production_space/新建工作台模版/ 复制到账号下的「工作台/」
