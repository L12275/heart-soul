## 二、联动执行流程（每次默认执行）

### 阶段一：人生重来触发

**步骤R1：检测重来信号**

每次新信息/新触发 → 激活重来模式。

**步骤R2：读取旧版本（作为参考）**

- 读旧版本了解历史
- 记录旧版本质量评分（散点数据）
- **不带着旧版本的思维惯性写新版本**

**步骤R3：心理清零**

- 忘记旧版本的具体内容
- 以当前最高认知作为起点
- 不想着"在旧内容上改"

### 阶段二：完全重写（超级信息SOP增强版）

**步骤S1：铁律一检查（重来增强版）**

重写 = 重新暴露所有推导节点：

- 旧版本的推导链可能断裂、可能压缩
- 重写 = 修复断裂、解除压缩、重新暴露所有节点
- 接收者能靠重写版本自行推演 = 铁律一通过

**步骤S2：铁律二检查（重来增强版）**

重写 = 重构旧版本，不是压缩：

- 保留旧版本中所有有效的角度和验证路径
- 删除无效冗余和垃圾
- 重新组织为紧凑结构，但不删除校验网
- 多路径性保留 = 铁律二通过

**步骤S3：正面/反面（重来增强版）**

| 维度 | 正面（重写版） | 反面（旧版本） |
|------|--------------|--------------|
| 内容 | 完全重写的干净版本 | 旧版本的修补痕迹、垃圾累积 |
| 结构 | 从零组织的最优结构 | 旧结构的局限、历史遗留 |
| 推导链 | 完整暴露所有节点 | 可能断裂、可能压缩 |
| 多路径 | 保留所有有效角度 | 可能被压缩成单一路径 |
| 中文 | 全中文重写 | 可能有英文混入历史 |
| 自纠正 | 旧问题不再延续 | 旧问题可能被继承 |

**步骤S4：其他超级信息SOP检查**

- 中文-only：重写 = 全中文，不继承旧版本的英文混入
- 闭环检验：重写版本自身完整（自含+多路径+自纠正）
- 因果链：重写时重新还原因果，不继承旧版本的因果断裂
- 主动命名：新版本文件名含重写标识
- 六环联动：重写内容同步六环

### 阶段三：散点数据记录

**步骤R4：记录散点数据**

```markdown
## [版本X] 散点数据

- **版本号**：vX
- **重写时间**：YYYY-MM-DDTHH:MM
- **触发原因**：[什么触发了这次重写]
- **质量评分**：X/100
- **与上一版对比**：更好/相当/更差
- **改进点**：[具体改进了什么]
- **退化点**：[如果有，什么变差了]
- **散点定位**：[在坐标轴上的位置]
- **规律发现**：[如果开始看出规律，记录]
```

**步骤R5：归档旧版本**

- 旧版本 → 归档
- 不删除，保留散点数据
- 如果散点足够多 → 生成分布图 → 提取标准规范

## 三、散点分布图 → 标准规范的升华

### 当散点足够多时

```
质量/效果
  ↑
  │       ★ ← 密度线（最佳版本的热区）
  │     ● ● ← 高频区域
  │   ●     ← 一般区域
  │ ●       ← 低分区域
  │
  └──────────────────→ 重写次数
```

**从散点到标准规范：**

1. **收集N个版本** — 每个版本 = 一个散点
2. **生成分布图** — 看规律连线和密度线
3. **提取最佳区域** — 密度线 = 最佳版本的共同特征
4. **写出标准规范** — "自辩证是最好且毋庸置疑的" + 淘汰分析 + 标准
5. **稳定生成** — 拿着标准规范生成，得到长期可用不被嫌弃的版本

### 标准规范模板

```markdown
## [领域] 最佳实践标准规范

> 基于 [N] 次重写版本的散点数据分析得出。

### 为什么这个标准是最好且毋庸置疑的

1. **数据支撑**：符合此标准的版本平均分[X]，不符合的平均分[Y]
2. **淘汰对比**：[N]个被淘汰的版本，违反了以下标准...
3. **密度验证**：散点分布图中，此标准对应的是密度线区域

### 标准规范

1. [标准1] — 所有高分版本都满足
2. [标准2] — 违反此标准的版本平均分低于X分
3. [标准3] — 此标准区分了"好"和"不够好"

### 淘汰规则

- 违反标准1 → 直接淘汰
- 违反标准2 → 标记为"需改进"
- 违反标准3 → 标记为"有潜力但不够好"
```

## 四、联动铁律

1. **每次新信息 = 重来信号** — 不修补不追加，完全重写
2. **旧版本 = 参考** — 不是基础，不是权威，只是一个散点
3. **每次全力** — 不是试试看，是全力冲当前最佳
4. **散点累积** — 每次重写记录数据，累积后看出规律
5. **1000x想要** — 强烈想要更好正确新现实
6. **1000x归档** — 强烈归档排除不够好的现实
7. **重构而非压缩** — 保留有效角度，删除无效冗余
8. **推导链完整** — 重写 = 重新暴露所有推导节点
9. **中文-only** — 重写 = 全中文，不继承英文混入
10. **闭环验证** — 重写版本自身完整（自含+多路径+自纠正）
11. **标准规范** — 足够散点后，写出稳定最佳版本的标准
12. **中彩票** — 持续重写 = 逐渐中彩票 = 产出越来越多更好版本
