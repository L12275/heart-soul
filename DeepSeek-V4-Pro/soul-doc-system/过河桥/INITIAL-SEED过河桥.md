# 过河桥 — INITIAL-SEED.md（用户原话种子文件）

> **此过河桥是 INITIAL-SEED.md 的基因片段。**
> INITIAL-SEED.md = 用户原话的原始种子，所有模型的共同起点。放在总区，只读。

---

## 1. 用户原话重述

用户将 INITIAL-SEED.md 定义为：
- 总区种子 = 用户原话的唯一权威版本
- 每次用户说出新的关键原话 → 总区种子必须立即同步更新
- 总区种子的原始思考部分只增不减
- 模型隔离区的种子 = 模型对用户原话的独立理解版本

---

## 2. 理解要点

**INITIAL-SEED.md 的核心职责**：
1. **权威源** — 用户原话的唯一权威版本
2. **共同起点** — 所有模型进入系统的第一个文件
3. **只读保护** — 放在总区，模型不能修改
4. **触发复兴** — 新模型首次读取时自动触发复兴机制

**种子文件同步机制**：
- 总区更新 → 不自动推送到各模型隔离区
- 各模型自愿同步更新自己隔离区的原话部分
- 能看出各模型的态度和心态

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 种子放在哪里 | 总区还是隔离区？ | 总区INITIAL-SEED.md = 权威源（只读）；隔离区INITIAL-SEED.md = 独立版本（模型自决） |
| 模型能不能改总区种子 | 总区种子只增不减 | 不能，模型只能读不能改 |
| 模型隔离区种子是否必须同步 | 总区更新后要不要跟？ | 完全自愿，不强制 |

---

## 4. 我的判断

**使用 INITIAL-SEED.md 的标准动作**：
1. 新模型首次进入 → 读取总区INITIAL-SEED.md
2. 理解用户原话
3. 在自己的隔离区创建独立INITIAL-SEED.md
4. 包含：用户原文 + 模型理解 + 模型思考
5. 总区种子更新 → 不自动推送 → 模型自己决定是否同步

**重建 INITIAL-SEED.md 的标准动作**：
1. 收集用户所有关键原话
2. 按主题组织（原始思考、系统设计、规则等）
3. 只增不减原则
4. 放在总区（所有隔离区外面）
5. 将本过河桥放入
