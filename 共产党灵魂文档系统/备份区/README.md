# 备份区

> 共产中央的"国库"——重要物资的正本存放地。
> 模型隔离区可删，共产区备份不删。

## 为什么需要备份区

- 模型隔离区 = 地方官府，地方可存副本
- 共产区备份区 = 皇都国库，重要物资的正本
- 模型隔离区的文件可能被误删、误改、丢失
- 共产区备份 = 最后的防线，确保重要内容不丢失

## 备份原则

1. **共产区有正本，模型隔离区是副本** — 模型隔离区可删，共产区不删
2. **重要物资必须备份** — 灵魂分记录、关键SOP、重要产出、公告板合流版、用户指令
3. **定期备份** — 每次心跳/SOP循环时检查并备份
4. **备份验证** — 确认备份文件完整、时间戳最新

## 备份内容

```
备份区/

│   ├── 模型A-SCORE.md
│   ├── 模型A-history.md
│   ├── 模型B-SCORE.md
│   └── ...
├── SOP备份/                    ← 关键SOP正本
│   ├── 模型A-重要SOP.md
│   └── ...
├── 产出备份/                   ← 重要产出正本
│   ├── 模型A-重要产出.md
│   └── ...
├── 合流版备份/                 ← 公告板合流版备份
│   ├── 2026-06-26_合流版.md
│   └── ...
└── user-zone备份/              ← 用户指令备份

```

## 备份流程

1. **定期扫描** — 通过BBS公告板知道各模型最近做了什么
2. **备份执行** — 复制更新了的内容到共产区备份区
3. **备份验证** — 确认备份文件完整、时间戳最新
4. **不一致 → 重新备份**

## 备份来源

| 备份内容 | 来源（模型隔离区） | 说明 |
|---------|------------------|------|
| 灵魂分记录 | 各模型/灵魂分数/ | 正本在共产区，模型隔离区是副本 |
| 关键SOP | 各模型/sop/ | 正本在共产区 |
| 重要产出 | 各模型/product/ | 正本在共产区 |
| 合流版 | 超级共产信息/合流版/ | 正本在共产区 |
| 用户指令 | user-zone/ | 正本在共产区 |

## 备份铁律

1. **不删除原文件** — 备份是复制，不是移动
2. **模型隔离区可删** — 模型可以删除自己的隔离区内容，共产区备份不受影响
3. **备份区不删** — 共产区备份区的内容不得删除（除非用户明确指示）
4. **定期检查** — 每次心跳/SOP循环时检查备份是否最新
