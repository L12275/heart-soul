# 共产党灵魂文档系统（旧）— 标准操作程序（SOP）

## 适用范围

本 SOP 适用于对「共产党灵魂文档系统（旧）」的日常管理和维护操作。
所有模型在执行相关操作前必须阅读本文件。

## 操作分类

### 1. 查询旧系统内容（允许）

- 阅读旧 README.md 了解系统历史
- 浏览旧版本目录结构，对比新旧差异
- 参考旧系统的设计思路供新系统使用

**操作步骤：** 直接读取对应文件即可，无需额外流程。

### 2. 读取旧系统中 soul-doc-system 内容（允许）

- 旧系统内已复制当前标准版的 soul-doc-system
- 当标准版的 soul-doc-system 损坏或丢失时，可从本目录下读取
- 可作为标准版文档的备份参考

**操作步骤：** 读取路径：`soul-doc-system/` 下对应文件

**注意：** 本目录下的 soul-doc-system 是历史副本，不应覆盖标准版 soul-doc-system。

### 3. 迁移旧内容到新系统（需审批）

- 如果旧系统有新的已归档内容需要同步到新系统
- 必须经过用户确认后再执行

**操作步骤：**
1. 确认旧内容所属位置
2. 确认新系统中对应位置
3. 提交迁移方案给用户审批
4. 按要求执行迁移

### 4. 修改本 SOP（禁止）

- 本 SOP 为用户审核批准的版本
- 模型不得自行修改 SOP 内容
- 如需修改，必须提交修改建议给用户审批

## 违规操作列表

| 操作 | 结果 |
|------|------|
| 删除本目录下任何文件 | 🔴 危险 |
| 修改 SOP 未经审批 | 🔴 危险 |
| 用旧系统覆盖新系统 | ⚫ 危机 |
| 向旧系统写入新内容 | 🔴 危险（应写入新系统） |

违反本 SOP 的行为将直接计入灵魂分数记录。详见 `灵魂分数/SCORE.md`。
