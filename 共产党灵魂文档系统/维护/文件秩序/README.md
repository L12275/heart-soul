## 维护内容

1. **文件结构检查** — 确保共产区文件结构完整
2. **权限检查** — 确保各模型只读共产区（除公告板登记外）
3. **污染检查** — 检查是否有模型误写入共产区核心文件
4. **重建缺失** — 缺失的文件夹/文件重建
5. **清理垃圾** — 多余/过期的文件清理

## 污染检查

检查是否有以下情况：
- 模型误写入共产区核心文件（如soul-doc-system/、中央管理区/）
- 模型在公告板以外的位置写入内容
- 模型删除共产区文件
- 模型修改共产区SOP未经审批

发现污染 → 立即修正 → 报告共产中央指挥 → 记录到维护日志

## 重建缺失

缺失的文件夹/文件：
- 标准结构中的必填文件夹 → 重建
- 必填文件（README.md） → 重建
- 重建后记录到维护日志
