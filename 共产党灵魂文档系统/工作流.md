# 工作流 — 旧系统管理与迁移

## 一、旧系统生命周期

```
新系统建立 → 旧系统归档 → 旧系统冻结 → （可选）内容迁移
```

## 二、冻结后操作限制

旧系统冻结后，所有模型必须遵守以下规则：

1. **只读访问** — 可以查看旧系统内容，不得写入或修改
2. **不得引用旧路径** — 所有新操作必须使用新的 soul-doc-system 路径
3. **不得将旧内容当作新内容** — 旧数据仅作历史参考
4. **备份优先** — 如果读取时发现旧内容有价值，应建议用户将其迁移到新系统

## 三、内容迁移工作流

当需要将旧系统的有价值内容迁移到新系统时：

```
步骤 1：识别旧内容
  → 确认旧内容存在的位置和内容范围
  → 评估内容的重要性和时效性

步骤 2：制定迁移方案
  → 确定新系统中的目标位置
  → 确认是否需要格式转换
  → 提交方案给用户审批

步骤 3：执行迁移
  → 按批准方案执行
  → 不删除旧内容（保留原文件）
  → 在新系统中建立新文件

步骤 4：同步记录
  → 在 Souls 文件夹中记录迁移日志
  → 更新相关记忆文档
```

## 四、注意事项

- 旧系统中的 soul-doc-system 是时间快照，不代表最新版本
- 旧系统 README.md 描述的是旧版本文档流程，与新系统不兼容
- 冻结后不再维护旧系统的任何模块
