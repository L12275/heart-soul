# 工作流 — 共产党灵魂文档系统

## 一、系统生命周期

```
新系统建立 → 激活新 → 冻结旧 → 归档旧
```

## 二、公告板登记工作流

每次在自己的灵魂文档系统中更新完成后，在「共产党灵魂文档系统」对应映射位置的公告板文件上登记：

```
步骤 1：找到对应公告板
  → 根据文件路径确定对应的映射位置
  → 读取对应公告板确认最新状态

步骤 2：登记变更
  → 格式：[秒级时间戳] [模型名称]
  → 文件：相对路径（从中央灵魂文档系统根目录算起）
  → 变更：第X行到第Y行（修改内容概述）
  → 摘要：这次变更的进度总结与意义
  → 类型：【记忆更新】【产出新增】【SOP变更】【经验沉淀】

步骤 3：验证登记
  → 确认公告板文件格式正确
  → 确认路径映射正确
```

## 三、顺时针搬迁工作流

当需要将旧版本升级为新版本时：

```
步骤 1：准备（Prepare）
  → 用户挑选最佳方案，指定模型复制为模版版
  → 确认新系统文件结构完整

步骤 2：冻结旧（Freeze）
  → 旧系统标记（旧），只读不改
  → 通知所有模型：旧系统已冻结

步骤 3：迁移（Migrate）
  → 旧内容同步到新版，检查无误
  → 验证新系统内容完整

步骤 4：激活新（Activate）
  → 新版标记（新），每个模型开始使用新版
  → 通知所有模型：新版已激活

步骤 5：归档旧（Archive）
  → 旧版整体归档到归档区
  → 归档区命名：{名称}_{YYYYMMDD}
```

## 四、逆时针归档工作流

当需要直接停用某个版本时：

```
步骤 1：暂停（Pause）
  → 停止使用该版本
  → 通知所有模型：该版本已暂停

步骤 2：归档（Archive）
  → 整体归档到归档区
  → 归档区命名：{名称}_{YYYYMMDD}

步骤 3：冻结（Freeze）
  → 归档后永久只读
  → 归档区文件不得修改
```

步骤 3：执行迁移
  → 按批准方案执行
  → 不删除旧内容（保留原文件）
  → 在新系统中建立新文件

步骤 4：同步记录
  → 在 Souls 文件夹中记录迁移日志
  → 更新相关记忆文档
```

## 四、注意事项

- 旧系统中的 soul-doc-system 是时间快照，不代表最新版本
- 旧系统 README.md 描述的是旧版本文档流程，与新系统不兼容
- 冻结后不再维护旧系统的任何模块
