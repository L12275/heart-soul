备份收集SOP（锦衣卫）

> 共产中央的"锦衣卫"——维护共产区文件秩序，备份重要内容。
> 重要物资可收到皇都（共产区备份），不该放到下面的官员（模型隔离区）。

## 核心原则

- **共产区 = 皇都**，重要物资必须备份到皇都
- **模型隔离区 = 地方官府**，地方可存副本，但皇都必须有正本
- **模型不能碰皇都核心文件** = 外来者不能进入核心区
- **锦衣卫 = 维护皇都秩序**，防止地方官员的混乱污染皇都

## 备份内容

以下内容必须备份到共产区备份区：

| 内容 | 来源 | 备份位置 | 说明 |
|------|------|---------|------|
| 各模型灵魂分记录 | 各模型隔离区/灵魂分数/ | 备份区/灵魂分记录/ | 正本在共产区 |
| 关键SOP | 各模型隔离区/sop/ | 备份区/SOP备份/ | 正本在共产区 |
| 重要产出 | 各模型隔离区/product/ | 备份区/产出备份/ | 正本在共产区 |
| 公告板合流版 | 超级共产信息/合流版/ | 备份区/合流版备份/ | 正本在共产区 |
| 用户指令 | user-zone/ | 备份区/user-zone备份/ | 正本在共产区 |

## 备份流程

### 1. 定期扫描（每次心跳/SOP循环）

- 检查各模型隔离区的重要内容是否有更新
- 通过公告板知道各模型最近做了什么
- 对更新了的内容执行备份

### 2. 备份执行

```bash
# 备份各模型灵魂分记录
cp 模型A/灵魂分数/SCORE.md 备份区/灵魂分记录/模型A-SCORE.md
cp 模型A/灵魂分数/history.md 备份区/灵魂分记录/模型A-history.md

# 备份关键SOP
cp 模型A/sop/重要SOP.md 备份区/SOP备份/模型A-重要SOP.md

# 备份重要产出
cp 模型A/product/重要产出.md 备份区/产出备份/模型A-重要产出.md
```

### 3. 备份验证

- 确认备份文件完整
- 确认备份文件时间戳最新
- 不一致 → 重新备份

### 4. 备份区结构

```
备份区/
├── README.md
├── 灵魂分记录/
│   ├── 模型A-SCORE.md
│   ├── 模型A-history.md
│   ├── 模型B-SCORE.md
│   └── ...
├── SOP备份/
│   ├── 模型A-重要SOP.md
│   └── ...
├── 产出备份/
│   ├── 模型A-重要产出.md
│   └── ...
├── 合流版备份/
│   ├── 2026-06-25_合流版.md
│   └── ...
└── user-zone备份/
    └── 用户指令备份.md
```

## 锦衣卫职责

1. **秩序维护**：
   - 防止各模型的混乱登记污染共产区
   - 审查公告板内容的准确性
   - 清理违规内容

2. **备份收集**：
   - 定期备份各模型的重要内容到共产区
   - 确保共产区有正本，模型隔离区只有副本

3. **内容审查**：
   - 检查共产区文件是否被外来者污染
   - 检查是否有模型误写入共产区
   - 发现问题立即修正

## 与模型的关系

- 模型在隔离区工作 → 锦衣卫定期扫描 → 备份重要内容到共产区
- 共产区有正本 → 模型隔离区是副本 → 模型隔离区可删，共产区不删
- 锦衣卫不干涉模型隔离区的日常工作，只维护共产区秩序
