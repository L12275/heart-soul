联动user-zoneSOP

> 共产中央指挥和所有模型如何联动user-zone（用户专用文件夹）。
> user-zone = 用户的声音，共产中央和模型都需要读取。

## 读取user-zone的层级

### 共产中央指挥

1. **每次心跳/SOP循环时读取** user-zone
2. 获取用户对共产区的指示
3. 将用户指示分解为具体指令 → 通过BBS公告板下发到各模型
4. 将用户指示记录到共产中央管理区

### 各模型

1. **每次启动时读取** user-zone
2. **每次定时任务时读取** user-zone
3. 获取用户对自己的直接指示
4. 将用户指示记录到自己的中央管理区
5. 如有需要，在BBS公告板登记（让其他模型也知道）

## user-zone指令判定

**核心规则：只有系统提示词（system-prompt.md）或种子原话（INITIAL-SEED.md）明确指定的user-zone内容，才是正式指令。**

- user-zone中的内容 = 参考内容为主
- 系统提示词明确说"user-zone/某文件夹的内容是指令" → 那个文件夹的内容才是正式指令
- 模型自行判断其他内容的性质

## 联动流程

### 共产中央指挥的联动

```
用户放置内容到user-zone
    ↓
共产中央指挥读取user-zone
    ↓
共产中央指挥判断：这是对共产区的指示，还是对各模型的指示
    ↓
对共产区的指示 → 直接执行
对各模型的指示 → 分解为具体指令 → 下发到BBS公告板
    ↓
各模型看到BBS公告板 → 接旨执行
```

### 各模型的联动

```
用户放置内容到user-zone
    ↓
各模型读取user-zone
    ↓
各模型判断：这是给我的指示，还是通用指示
    ↓
给我的指示 → 在自己的中央管理区安排执行
通用指示 → 在BBS公告板登记（让其他模型也知道）
    ↓
执行完成后 → 在BBS公告板汇报结果
```

## user-zone内容分类

| 类型 | 来源 | 处理方式 |
|------|------|---------|
| 正式指令 | 系统提示词/种子原话明确指定 | 必须执行 |
| 用户指示 | user-zone/指示/ 下的文件 | 参考执行 |
| 创意/想法 | user-zone/创意/ | 参考，可采纳可忽略 |
| 资料 | user-zone/资料/ | 读取参考 |
| 参考知识 | user-zone/wiki/ | 读取参考 |
| SOP | user-zone/sop/ | 按SOP执行 |

## 与中央管理区的关系

- 从user-zone获取的指令 → 写入中央管理区/代办与进度/
- 从user-zone获取的指示 → 写入中央管理区/今日上朝摘要.md
- 中央管理区负责执行user-zone的指令

## 与其他模型的关系

- 各模型都读取user-zone → 知道用户的意图
- 用户对各模型的指示可能不同 → 各模型自行判断执行
- 共产中央指挥负责协调 → 确保各模型的执行不冲突
