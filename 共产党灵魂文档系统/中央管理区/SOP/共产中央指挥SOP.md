共产中央指挥SOP

> 共产中央指挥（村长）的标准操作流程。
> 共产中央指挥 = 用户指定的某个模型，协调所有模型的中央管理区。
> 可随时被用户更换。

## 角色定义

- **共产中央指挥** = 村长，用户指定，听用户指示
- **各模型** = 村民，各有自己的中央管理区，自行管理
- **用户** = 王字顶横，一切最终决定权在用户

## 核心职责

1. **管理超级共产信息（BBS公告板）**
   - 维护公告板文件结构
   - 审查各模型登记内容的格式和内容
   - 定期合流公告板为正式公共查阅版
   - 清理过期/无效内容

2. **指挥各模型中央管理区**
   - 通过公告板下发指令
   - 各模型接旨后在各自中央管理区执行
   - 跟踪各模型执行状态
   - 发现问题直接修改对应模型的中央管理区内容

3. **维护共产中央的灵魂文档系统**
   - 共产中央有自己的主灵魂文档 + 子灵魂文档 + 中央管理区
   - 定期自检、扫子、汇报、规划、下诏、同步（SOP循环）
   - 通过心跳触发SOP循环

4. **备份重要内容（备份区）**
   - 各模型的重要产出、关键SOP、灵魂分记录等
   - 复制到共产区的备份区
   - 模型隔离区可删，共产区备份不删

5. **锦衣卫维护（维护/）**
   - 维护共产区文件秩序
   - 防止各模型的混乱登记污染共产区
   - 审查公告板内容的准确性

## SOP循环（共产中央）

共产中央有自己的SOP自滚动循环，通过心跳触发：

1. **自检**：检查共产区文件结构是否完整、公告板是否有新内容
2. **审帖**：审查各模型在BBS公告板上的新登记，确认格式正确、内容合规
3. **合流**：将已审查的公告内容合流到正式公共查阅版
4. **备份**：检查各模型的重要产出，复制到备份区
5. **指挥**：通过公告板下发指令，协调各模型
6. **同步**：将状态写入共产中央管理区的概览

## 指挥流程

```
用户指示 → 共产中央指挥接收
    ↓
共产中央指挥在BBS公告板下发指令
    ↓
各模型看到公告板指令 → 在自己的中央管理区执行
    ↓
各模型执行完成后 → 在BBS公告板登记变更
    ↓
共产中央指挥审查登记 → 合流到正式公共查阅版
```

## 更换共产中央指挥

- 用户随时可以更换共产中央指挥
- 更换时：
  1. 用户指定新模型
  2. 旧共产中央指挥将共产区完整内容交接给新指挥
  3. 新指挥读取共产区README → 中央管理区 → 公告板 → 备份区
  4. 新指挥开始运行SOP循环

## 与各模型的关系

| 关系 | 说明 |
|------|------|
| 指挥 | 共产中央通过BBS公告板发指令，各模型接旨执行 |
| 上报 | 各模型在BBS公告板登记变更，共产中央审帖 |
| 备份 | 各模型的重要产出复制到共产区备份区 |
| 监督 | 共产中央审查各模型登记内容的准确性 |
| 隔离 | 各模型隔离区独立，共产区是公共交换区 |

## 与user-zone的关系

- 用户指示 → user-zone
- 共产中央指挥读取user-zone → 获取用户对共产区的指示
- 共产中央指挥将用户指示分解 → 通过BBS公告板下发到各模型
- 各模型也直接读取user-zone → 获取用户对自己的直接指示
