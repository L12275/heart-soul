> **核心原则：今日事今日毕。日记不是事后总结，是边做边记的解题草稿本。**
> 基于：`sop/超级信息SOP.md`

> 记忆 = 验证确认后的正确记忆（从日记中提炼）

## 日记本质

日记是**边做边记的解题草稿本**，不是事后总结。它的价值是把工作过程中的思考、错误、求解过程按小时记录下来，最终写出一篇"已成功解题"的日记。

### 日记 vs 记忆（互补关系）

| 维度 | 日记 | 记忆 |
|------|------|------|
| 内容 | 实时事件过程 + 解题思考（错误→排除→正确） | 验证确认后的正确结论 |
| 准确性 | 纪实，通过超级信息SOP思考确保最终版正确 | 已验证 |
| 格式 | 小时级时间轴（每个事件有推导链） | 精炼的要点（结论+来源） |
| 时机 | 边做边记 + 当日完成 | 事后提炼 |
| 用途 | 形成正确记忆/SOP/工作流的原材料 | 快速查找已验证知识 |

**日记有细节 → 通过超级信息SOP思考 → 写出正确解 → 提炼为记忆/SOP/工作流**

---

## 日记铁律（基于超级信息SOP）

### 铁律一：今日事今日毕，写出正确解的日记

日记不能把"错的当对的写"然后留烂摊子。模型每次运行只根据当前会话的注意力导向工作，不会做注意力导向之外的事。如果日记写的是错的、没解出谜团的，下次模型看到日记会直接当作事实使用——这就错了。

**所以：写日记时，就要用超级信息SOP的思考方式，把问题真正解决，再把正确解写成日记。**

### 铁律二：记录"为什么对、为什么错、如何排除"

日记不是只记"结果"，要记录完整推导链：
```
遇到的阻碍 → 当时的判断 → 为什么错了 → 如何排除 → 正确解法 → 收获
```

格式：
```markdown
### HH:MM — [事件/问题]
**事件：** [发生了什么]
**判断：** [当时的判断]
**验证：** [怎么验证的]
**结果：**
- 前提：[证据/来源]
- 推导：[推理过程]
- 排除：[排除了什么错误路线，为什么]
- 正确解：[最终做法，为什么对]
**收获：** [可复用的经验/SOP/工作流更新]
```

### 铁律三：日记可以当草稿本，边记边解题

日记允许是草稿形式。边做边记的过程中：
- 先记录观察到的现象
- 记录自己的猜测和判断
- 然后启动超级信息SOP式的思考
- 在日记里完成推导链的每个节点
- 最终得出正确结论并写到日记里

**这就是"边记边解题"，日记写完，问题也解决了。**

### 铁律四：排除的错误也要记录

日记必须记录：
- 正确路线（做了什么、为什么对）
- 错误路线（走了什么弯路、为什么错、怎么排除的）
- 阻碍（遇到了什么障碍、怎么克服的）

## 小时级时间轴格式

按时间轴存放：`diary/年/月/日/[日期].md`

每天一个文件，文件内按**小时级**记录每个事件：
- 每个事件必须有：时间、类型、详情、判断、验证、推导链、正确解、收获
- 每个事件的推导链要完整：前提→推导→排除→正确解
- 当天的问题必须在当天解决

## 日记 → 记忆/SOP/工作流 的转化

```
日记（完整解题过程，小时级时间轴）
↓ 超级信息SOP思考
↓ 今日事今日毕原则
├── memory/ → 验证后的正确记忆
├── sop/ → 可复用的SOP
├── workflow/ → 可执行的工作流
└── experience/ → 分级经验
```

**关键：日记是原料，记忆/SOP/工作流是成品。原料必须干净（问题已解决），成品才有价值。**

---

## 使用规则

1. **实时记录** — 边做边记，每个重要事件发生时记录
2. **边记边解题** — 遇到问题先在日记里推导，用超级信息SOP找到正确答案
3. **今日事今日毕** — 当天的问题当天在日记里解决，写成正确解
4. **排除的错误也记录** — 让下次知道哪些坑已经踩过
5. **日记是草稿** — 允许粗放，但最终版必须是通过超级信息SOP校验后的正确版本
6. **每天一个文件** — 文件名 = 日期
7. **小时级时间轴** — 文件内按 HH:MM 时间轴记录每个事件
8. **与记忆互补** — 日记记解题过程，记忆记已验证结论
