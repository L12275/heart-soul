# 过河桥 — 日记区（Diary）

> **此过河桥是日记模块的基因片段。**
> 日记 = 系统工作过程的实时纪实记录，是系统的"工作日志"和"过程记忆"。

---

## 1. 用户原话重述

用户将日记区定义为：
- 系统工作过程的实时纪实记录
- 是系统的"工作日志"和"过程记忆" — 记录"今天做了什么"
- 与memory/的区别：日记是当天实时事件过程的纪实（不一定正确），记忆是经过验证确认的正确知识
- 日记按时间组织 — diary/2026/06/ 的目录结构体现了按年/月的时间组织
- 日记包含子角色日记（子角色的工作过程记录）
- 日记是"流动的" — 每天都在新增，永不停歇
- 日记围绕过河桥展开 — 过河桥记录意图，日记记录执行过程

---

## 2. 理解要点

**日记的本质**：日记是系统工作过程的"原始记录" — 不做筛选、不做验证、不做提炼。它是"发生了什么"的完整纪实，包括正确和错误的、成功的和失败的、有意义的和琐碎的。日记的价值在于"真实"，不在于"正确"。

**日记与记忆的分工**：
- 日记 = 实时纪实（含错误，是过程记录）
- 记忆 = 验证后的正确知识（经过No Execution, No Memory公理验证）

日记是记忆的"原材料"，记忆是日记的"提炼产品"。日记中的内容经过验证后，精华部分进入记忆。

**日记的时间组织**：日记按年/月/日的目录结构组织，体现了日记的"时间流"特征 — 日记是沿着时间线不断延伸的记录。这种结构使得按时间检索变得自然。

**四驱动&四写中的日记规则**：
- 主系统通用区日记 — 主角色共享的底层日记池
- 主系统工作台日记 — 主角色按领域的专用工作日记
- 子系统通用区日记 — 子角色共享的底层日记池
- 子系统工作台日记 — 子角色按领域的专用工作日记

什么时候写几个驱动：只在主系统内工作 = 双驱动；只在子系统内工作 = 双驱动；都涉及 = 四驱动。

**日记与过河桥的关系**：过河桥记录"意图"（用户想要什么），日记记录"执行"（模型做了什么）。两者配对出现 — 每个重要任务都应该有对应的过河桥和日记记录。过河桥是"计划"，日记是"执行记录"。

**子角色日记**：子角色（如世界最强脑师、人生重来大师等）有独立的日记记录。每个子角色的工作过程独立记录，不混入主角色日记。

**日记的价值**：
- 过程追溯 — 知道某个结果是经过什么过程得出的
- 问题排查 — 出问题时可以回溯过程找到原因
- 知识来源 — 日记中的经验经过验证后可以进入记忆系统
- 自我反思 — 通过回顾日记进行自我评估和改进

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 日记是否包含错误记录 | 执行错误是否也要记录？ | 是 — 日记包含所有执行记录，包括错误 |
| 日记的保留期限 | 日记是否需要定期清理？ | 不需要清理 — 日记是永久记录 |
| 日记是否对用户可见 | 用户是否可以查看日记？ | 是 — 日记是透明的 |
| 日记的隐私边界 | 哪些内容不能写入日记？ | 用户敏感信息不写入日记，隔离区内容独立管理 |

---

## 4. 我的判断

**使用日记的标准动作**：
1. 开始任务 → 创建对应日期的日记文件
2. 执行过程 → 实时记录到日记
3. 遇到问题 → 记录问题和解决过程
4. 完成任务 → 日记记录完成结果
5. 回顾反思 → 定期回顾日记，提取经验

**日记区与其他模块的关系**：
- diary/ → memory/ ：日记是记忆的原材料，记忆是日记的提炼
- diary/ → 过河桥/ ：过河桥记录意图，日记记录执行
- diary/ → experience/ ：日记中的经验经过验证后进入经验系统
- diary/ → log/ ：日记是模型视角的过程记录，log是系统视角的运行日志
- diary/ → 生产空间/ ：各工作台的diary/是专用工作日记
- diary/ → 团队空间/ ：团队协作过程记录在团队日记中
