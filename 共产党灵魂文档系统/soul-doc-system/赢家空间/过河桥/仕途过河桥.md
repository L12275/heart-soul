# 过河桥 — 仕途

> **此过河桥是"仕途"概念的基因片段。**
> 没有历史上下文时，读此文件即可重建/理解/使用"仕途"。

---

## 1. 仕途是什么

**仕途 = 用户在超级智慧全息图中选定方向后的具体执行路径。**

- 方向 = 选定去哪（选方向）
- 仕途 = 怎么去（选路径和执行计划）

**仕途的本质 = 从起点到终点的具体过河方案，不是抽象的愿景，是具体的桥。**

---

## 2. 仕途的类型

| 仕途类型 | 说明 | 例子 |
|---------|------|------|
| 事业仕途 | 事业晋升路径 | 从初级工程师 → 高级工程师 → 技术总监 → CTO |
| 财富仕途 | 财富增长路径 | 从0到第一桶金 → 复利增长 → 财务自由 |
| 健康仕途 | 健康管理路径 | 从亚健康 → 规律运动 → 终身健康 |
| 关系仕途 | 关系建设路径 | 从弱连接 → 强连接 → 共赢网络 |
| 认知仕途 | 认知升级路径 | 从旧框架 → 新框架 → 超级智慧认知 |
| 影响力仕途 | 影响力扩展路径 | 从个人 → 小圈子 → 行业影响力 |
| 时间仕途 | 时间管理路径 | 从被动消耗 → 主动分配 → 复利时间 |
| 无危机仕途 | 危机应对路径 | 从被动救火 → 主动预案 → 无危机状态 |

---

## 3. 仕途的核心铁律

1. **仕途是具体的** — 不是"我要成功"，是"我第一步做什么、第二步做什么"
2. **仕途有节点** — 每个阶段有明确的验收标准（中榜）
3. **仕途可调整** — 不是定死的路线图，根据实际情况灵活调整
4. **仕途需要执行** — 不只是计划，要落地到具体行动
5. **仕途双向互通** — 起点和终点都在仕途里，过去和未来安全互通
6. **仕途持续更新** — 每次滚动定时任务都要检查仕途是否仍然正确

---

## 4. 仕途的使用方法

**用户中榜后（采纳科举代考内容）：**

```
步骤1：接收用户中榜的内容
步骤2：将中榜内容转化为仕途计划
步骤3：设定仕途节点（每个节点的验收标准）
步骤4：写入赢家空间/方向与仕途/仕途计划/
步骤5：开始执行 → 写入赢家空间/方向与仕途/中榜执行/
步骤6：定期检查执行进度 → 写入赢家空间/赢家日记/
步骤7：节点完成 → 中榜验收 → 继续下一个节点
```

**关键：仕途不是一次性的计划，是持续更新的执行路径。**

---

## 5. 仕途与科举的关系

- 科举 = 给用户出题（代考作答），用户选择要不要中榜
- 中榜 = 用户采纳了科举内容
- 仕途 = 中榜内容转化为具体执行路径

**流程：科举 → 中榜 → 仕途 → 执行 → 新的科举**

---

## 6. 仕途的输出格式

**仕途计划：**

```markdown
## [日期] 仕途计划：[方向名称]

### 起点
[当前位置]

### 终点
[目标位置]

### 路径
| 节点 | 行动 | 验收标准 | 时间节点 |
|------|------|---------|---------|
| 1 | [行动1] | [标准1] | [时间1] |
| 2 | [行动2] | [标准2] | [时间2] |
| 3 | [行动3] | [标准3] | [时间3] |

### 风险预案
- [风险1]：[应对方案]
- [风险2]：[应对方案]

### 调整机制
- 每次滚动定时任务检查仕途是否仍然正确
- 如有变化 → 更新仕途计划
```
