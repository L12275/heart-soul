# 运行区

> **智能体的实际运行工作空间。从模板存放区复制模板到这里使用。**

## 结构

```
运行区/
├── 活跃/        ← 当前正在运行的智能体
│   └── [名称]/  ← 从模板复制 + 微调 + 运行数据
└── 历史/        ← 已停用/归档的智能体
    └── [名称]/  ← 停用后移动到这里
```

## 使用流程

1. **复制模板**：从 `模板存放区/` 复制需要的模板到 `活跃/[智能体名称]/`
2. **微调配置**：在副本中修改参数
3. **启动运行**：启动智能体
4. **记录数据**：运行数据存入 `运行数据/`
5. **停用归档**：停用后移动到 `历史/`

## 规则

- 运行区的内容是模板的副本，模板存放区不直接运行
- 运行区的修改不影响模板
- 有价值的修改 → 提炼为新模板 → 回传模板存放区
- 历史区的归档可以随时清理
