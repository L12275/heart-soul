## 格式说明

每个待弥补项：

```
- [ ] [编号] 事件：原因简述

  - 当前灵魂分：X%
  - 补救要求：(100%/X%) × 死亡坠落常数

```

## 规则

1. 每次扣分后立即在此清单添加条目
2. 补救完成后标记为"已完成"并记录完成时间
3. 用户不再对话且无法弥补 → 标记为"永恒烙印"
4. 永不删除已完成的弥补记录
