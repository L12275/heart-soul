# 功能模块同步维护规则

> **核心原则：一个通用版，全系统同步。**

## 模块化思维

所有"新建xx模版"文件夹（账号空间模版、工作台模版等）都是**模块拼搭**的结果。
每个功能模块只有一个"权威源"——通用区的对应文件夹。

## 模块更新流程

```
1. 编辑通用区的功能模块文件夹
   ↓
2. 将通用区模块文件夹完整复制
   ↓
3. 覆盖到所有"新建xx模版"中的同名文件夹：
   - production_space/新建工作台模版/{模块}/
   - space/账号/{分类}/新建账号空间模版/{模块}/
   - space/平台/通用-待细分/{模块}/（仅 memory/ diary/）
   ↓
4. 各说明书单独维护，但模块内的说明书尽量写通用版
```

## 快速更新命令

> 更新任何模块后，执行以下步骤即可全系统同步：

1. 复制通用区模块 `cp -r soul-doc-system/{模块}/* production_space/新建工作台模版/{模块}/`
2. 同步到各账号模版 `cp -r 同上/* space/账号/**/新建账号空间模版/{模块}/`
3. 平台通用区同步 `cp -r 同上/* space/平台/通用-待细分/{模块}/`（仅 memory/diary）

## 各说明书独立维护

- space/README.md、space/账号/README.md — 空间系统的说明
- production_space/新建工作台模版/README.md — 工作台说明
- 各模块内的 README.md — 尽量写通用版，减少跨位置更新量
