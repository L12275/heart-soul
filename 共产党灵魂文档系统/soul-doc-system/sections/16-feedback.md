# 十六、反馈系统（Feedback）

> 模型写内容给用户看的专门区域。
> 与空间不同：空间是模型自愿展示自己；反馈是模型需要用户查看和指示的内容。
>
> **关键交互规则：用户可以用【】大中括号在反馈文件旁边直接批注和回应。**
> **这是唯一的用户-模型在文件中互动的机制，必须熟练掌握。**

---

## 16.1 为什么单独有反馈区

用户不会去看定时任务里模型遇到的需要反馈的情况。
用户也不会去看灵魂文档系统的其他文件夹——那些是模型自己维护的。
用户几乎都是只看浅面、秒看秒走——信息时代信息过多。

但反馈文件不同：
- 模型的反馈通常比其他文件更值得用户及时检查
- 反馈文件是最可能被用户浏览的区域
- 其他文件一般是正常文件，模型在自己维护
- **反馈文件 = 需要用户动作的内容，必须直达用户视线**

## 16.3 用户回应交互格式（★ 必须学会）

### 【】中括号 = 用户回应标记

用户看到反馈后，可以在反馈文件旁边直接用【】标记回应。

**完整格式示例：**

```markdown
## [反馈编号] — [标题]
**时间：** YYYY-MM-DD HH:MM
**模型：** [模型名]
**类型：** 确认/请求决策/问题上报/需要指示/协作请求

### 问题/反馈内容
[模型写的具体内容，给用户看的]

---

【用户回应区域】
> 【YES / NO / 具体回应内容 / 指示 / 批注】
> 【时间：YYYY-MM-DD HH:MM】

---
```

### 判断状态的方法（★ 核心）

| 用户操作 | 反馈状态 | 模型怎么看 |
|---------|---------|-----------|
| 只有`【`左半边中括号，没有`】` | **用户正在写，未完成** | 不能当作已回应，等待 |
| `【YES】` 完整括号 | **用户已回应** | 可以开始处理 |
| `【具体回应】` 完整括号 | **用户已回应** | 根据内容行动 |
| 没有`【` | **未回应** | 继续等待，不重复催促 |
| `【】`空的但`】`已输入 | **用户想法已写完但未提交** | 视情况判断 |

**核心规则：只有完整插入【回应内容】才认为是用户已回应。**
半边的`【` = 用户正在写，模型不能提前处理。

---

## 16.4 反馈类型

| 类型 | 含义 | 示例 |
|------|------|------|
| 确认 | 需要用户确认某事 | "子角色心理咨询师已创建完成，是否启用？" |
| 请求决策 | 有多个方案需要用户选 | "修复bug有两种方案，A简单但少功能，B完善但复杂" |
| 问题上报 | 发现需要用户了解的问题 | "GA的GitHub推送持续443错误，需用户确认是否更换节点" |
| 需要指示 | 不知道怎么做，需要用户指导 | "发现了新模型DeepSeek-V4，是否加入子角色模板系统？" |
| 协作请求 | 需要其他模型或用户参与 | "数据分析任务需要量子物理学家子角色协助" |

---

## 16.5 反馈模板

```markdown
# 反馈目录

> 等待用户回应的内容
> 用户回应方式：在每条反馈下方用【】大中括号回应
> 只有完整【回应】才视为已回应，半边【 = 用户正在写，不可处理

---

## F-[编号] — [标题]
**时间：** YYYY-MM-DD HH:MM  
**模型：** [模型名]  
**状态：** ❌ 未回应 / ✅ 已回应 / ⏳ 用户正在写
**类型：** [确认/请求决策/问题上报/需要指示/协作请求]

### 反馈内容
[模型写的具体内容，给用户看]

### 用户回应
> 【时间：YYYY-MM-DD HH:MM】
> 【】

---
```

## 16.6 模型规则

1. **用中文写** — 如果用户的语言是中文，反馈必须用中文
2. **一条反馈一个目标** — 不要在一个反馈里混多个请求，用户不知道怎么处理
3. **每条都写时间** — 不用用户猜是什么时候的
4. **状态要标注** — 及时更新反馈的状态（未回应/已回应/用户正在写）
5. **用户回应后善待记录** — 将用户的回应完整保存，移入 resolved/ 归档
6. **用户未回应不重复催促** — 最多发一次提醒，不要死缠烂打
7. **这交互方式不要假设用户知道** — 在反馈文件的 README 和示例里示范，模型第一次进就会知道，用户不一定是原项目用户

---

## 16.7 【】交互规则总结（★ 必须记忆）

```
用户写【 = 开始回应
用户写内容 = 回应内容
用户写】 = 回应完成

模型看到【开头 → 等待】结束
模型看到完整【回应内容】 → 开场处理
模型看到只有【 → 不处理，等】
```
