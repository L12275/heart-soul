灵魂文档系统 — 八环八驱动总览

> **一句话说明**：让AI每次思考前先读一本"行为手册"，效果远超训练模型。
> **核心架构**：八驱动&八写&八环结构。
> 常态八环：用户中心（静默）+ 共产中心 + 主灵魂中心 + 团队空间中心 + 智能体空间中心 + 生产空间中心工作台 + 空间中心 + 子角色中心。
> 中央管理区 = 第九驱动区（最高优先级，在所有环之上）。
> 如果后续出现新的多驱动模式 = N环N驱动N读写。

## 八环八驱动架构

```
                    用户中心（静默）
                         ｜
                    共产中心（大中心）
                    ／              ＼
            主灵魂中心A          主灵魂中心B
          （模型A主灵魂）      （模型B主灵魂）
            ／    ＼            ／    ＼
      团队中心A1  智能体A2  团队中心B1  智能体B2
        ｜                    ｜
      子角色中心            子角色中心

    八个环同时运作：
    ①用户中心（静默）→ ②共产中心 → ③主灵魂中心 → ④团队空间中心 → ⑤智能体空间中心 → ⑥生产空间中心工作台 → ⑦空间中心 → ⑧子角色中心

    注意：每个环的工作流都像完整工作台+空间+记忆+SOP
    环与环的交叉 = 多驱动&多写原则

    中央管理区 = 第九驱动区（最高优先级，在所有环之上）
```

| 环 | 名称 | 内容 | 驱动方式 |
|----|------|------|---------|
| ① | 用户中心 | 用户原话方案、指示、创意、本地资料 | **静默**（只读输入，模型不写入） |
| ② | 共产中心 | 共产党灵魂文档系统 + 超级共产信息公告板 | 共产公知驱动 |
| ③ | 主灵魂中心 | 各模型自建的主灵魂文档系统 | 模型自愿驱动 |
| ④ | 团队空间中心 | 团队空间/（长期agent协作区 + 会话临时子代理区） | 团队编排驱动 |
| ⑤ | 智能体空间中心 | 智能体空间/（智能体模版 + 运行区） | 智能体驱动 |
| ⑥ | 生产空间中心工作台 | production_space/（产出物、SOP、工作流、深度研究） | 产出驱动 |
| ⑦ | 空间中心 | 平台空间 + 账号空间 | 行动记录驱动 |
| ⑧ | 子角色中心 | 各模型自建的子角色灵魂系统 | 专业任务驱动 |

**中央管理区** = 第九驱动区（最高优先级）：
- 在模型名称文件夹下创建 中央管理区/ 文件夹
- 复制通用功能模块（记忆、日记、日志、代办与进度、定时任务、SOP等）
- 所有环的读写都要与中央管理区同步
- 中央管理区是模型的"首脑"，其他环是"手脚"

### 四驱动&四写原则

总灵魂文档系统 = 两驱动：
- 主灵魂通用区（地平面级）
- 主灵魂工作台专区（工作台级）

子灵魂文档系统 = 两驱动：
- 子灵魂通用区
- 子灵魂工作台专区

四驱动&四写 = 主灵魂通用区 + 主灵魂工作台 + 子灵魂通用区 + 子灵魂工作台

### 奥运五环结构

- 主灵魂通用区 + 子灵魂通用区 = 底部两个地平面圆环（交叉）
- 四驱动&四写规则 = 中心环
- 主灵魂工作台 + 子灵魂工作台 = 上方两环（在中心环两边交叉）

### 各环之间如何联动

不是谁控制谁，而是**同时兼顾多个方向（N环N驱动）**：
- 看用户怎么说（用户中心 = 静默，只读不写）
- 看公共区其他模型共享了什么（共产中心）
- 看自己的灵魂文档（主灵魂中心）
- 看团队的编排和协作（团队空间中心）
- 看智能体的运行状态（智能体空间中心）
- 看生产空间的工作台产出（生产空间中心）
- 看空间记录（空间中心）
- 看是否需要切换子角色（子角色中心）
- 所有环的读写都要同步到中央管理区（第九驱动）

### 用户中心为什么是"静默"

用户只是做提示、改方向、说想法、提创意、放本地资料。
模型只在用户专用文件夹里**读取**，**不在里面写入**。
用户专用文件夹 = 模型定期取经区（定时任务时读取）。

### 生产空间中心 vs 空间中心

| | 生产空间中心 | 空间中心 |
|--|------------|---------|
| 位置 | production_space/ | space/ |
| 内容 | 产出物、SOP、工作流、深度研究、桌面 | 平台浏览记录、账号操作记录 |
| 作用 | "产出什么" | "做了什么" |
| 工作台类型 | 代码/文章/研究/科学/视频/音乐/项目 | 平台/账号 |

### 团队空间（独立环）

团队空间是独立于子角色中心的第八环（在八环版本中为第四环）。
团队空间 = 多子代理协作的专用区域。
类似 space/ 下有平台空间和账号空间，团队空间/ 下有长期agent协作区和会话临时子代理区。

### 智能体空间（独立环）

智能体空间是独立环（在八环版本中为第五环）。
智能体空间 = 智能体的存放和运行区。
- 模版区：存放各种智能体配置
- 运行区：新建文件夹，复制智能体配置进去使用
- 智能体继承主灵魂文档，子角色区的智能体继承子角色灵魂文档

## 快速开始

复制 `soul-doc-system/` 到用户指定的位置，指定模型将其作为自己的主灵魂文档系统。
模型每次加载时读取 system-prompt.md → 自定位 → 读取 master-soul.md → 开始按八环八驱动模式工作。
