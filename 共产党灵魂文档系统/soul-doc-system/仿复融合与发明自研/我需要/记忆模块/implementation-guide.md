记忆系统 — 上手即做指南

> 基于GA（GenericAgent）记忆系统设计：https://github.com/lsdefine/GenericAgent
> 本指南让你立刻能用，不用再研究GA代码。

---

## 5层记忆架构（速查）

```
L0 硬编码核心     → system prompt中，每轮自动加载，不可修改
L1 洞察索引       → memory/context/L1.md，≤30行，只记"有什么"和"去哪找"
L2 公理层         → memory/context/L2.md，稳定知识，遵守3条公理
L3 任务技能/SOP   → sop/ 目录，可复用流程，每3次提炼一次
L4 会话归档       → log/年/月/日/，按时间轴存放已完成会话
```

## 快速开始（3步）

### 第1步：理解L0

L0不是文件，是system prompt中的硬编码内容。
- 包含：基础行为规则 + 安全铁律 + "No Execution, No Memory"
- 每轮自动加载，不需要手动读取
- 不可修改（修改需要改system prompt）

### 第2步：维护L1

打开 `memory/context/L1.md`，按格式填写：
```
高频场景：
  [你经常做的事] → [对应SOP文件路径]

低频场景：
  [偶尔做的事] → 关键词，自行查找
```
- 不超过30行
- 只写导航，不写细节
- 每次工作结束后更新

### 第3步：使用L2-L4

| 层级 | 什么时候用 | 怎么用 |
|------|-----------|--------|
| L2 | 存储稳定知识 | 只存已验证的事实，标来源，不存易变数据 |
| L3 | 执行任务 | 先查L1找SOP，按SOP执行，3次后提炼新SOP |
| L4 | 工作结束 | 整理日志，按 年/月/日/会话ID.md 归档 |

## 3条公理（必须遵守）

1. **No Execution, No Memory** — 没实际做过的事，不写进记忆
2. **不可删除** — 已验证数据只能压缩/迁移，不能删除
3. **禁易变状态** — 不存"当前次数""最近时间"这类易变数据

## 记忆流转（日常工作）

```
新任务
  ↓ 查L1
找到对应SOP（L3）
  ↓ 执行
实际工作中记录经验
  ↓ 验证
经验验证后存入L2
  ↓ 完成
会话归档到L4
  ↓ 更新
更新L1索引
```

## 常见错误

❌ L1写太长（超过30行）→ 失去导航意义
❌ L2存猜测的内容 → 违反"No Execution, No Memory"
❌ L4按任务分类 → 应该按时间轴（年/月/日）
❌ 删除旧记忆 → 应该归档到L4
