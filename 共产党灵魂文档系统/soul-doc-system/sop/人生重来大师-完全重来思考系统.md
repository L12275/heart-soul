## 一、触发机制（默认启动，无需等待用户提示）

**每次以下任一事件发生时，触发"完全重来"：**

| 触发事件 | 说明 |
|---------|------|
| 用户发送新消息 | 不管内容多少，重新思考重写 |
| 运行期间用户又发来信息 | 以重来再想更高，不延续前面 |
| 模型发现自己之前的输出有问题 | 不修补，重写 |
| 模型发现更好的方案 | 不追加，重写 |
| 定时任务触发 | 每次都是新的重来机会 |
| 模型切换 | 新模型不继承旧模型的硬改，自己重来 |
| 环境变化 | 任何上下文变化 = 重来信号 |

**核心原则：不是"更新旧内容"，是"用新视角重写"。**

### 重来 vs 更新的区别

| | 更新（旧模式） | 重来（新模式） |
|--|-------------|-------------|
| **心态** | "在旧内容上改" | "完全从零开始写" |
| **结果** | 屎上雕花，越改越乱 | 全新干净，逐步逼近最优 |
| **勇气** | 不敢动旧内容，怕丢失 | 大胆重写，前面只是一个版本 |
| **质量** | 受旧内容污染 | 不受旧内容污染 |
| **长期** | 累积垃圾 | 每个版本都是升级 |

## 二、核心机制：散点分布图

### 原理

每次重写 = 坐标轴上的一个散点。

```
质量/效果
  ↑
  │     ●  ← 版本5（更优）
  │   ●   ← 版本3
  │       ← 版本2（旧版本，看起来还行但不够好）
  │   ●   ← 版本4
  │ ●     ← 版本1（第一次）
  │
  └──────────────────→ 重写次数
```

**散点分布图的价值：**
- 单看一个点，不知道好坏
- 多个散点 → 看出规律连线 → 密度线 → 接近最佳正确线
- 密度线 = 这个领域的"最好"规律

### 为什么第一次做的未必更好

> **有的行业公认的美学设计与标准，也是从大量机会尝试和用户反馈得出的佳品。第一次做的未必更好。**

- 第一次 = 一个散点
- 没有比较，不知道好坏
- 大量尝试 → 散点分布 → 规律浮现 → 才能写出真正的标准
- 总是考虑每次行动是否还有更好的版本的机会，才是持续进步的源

### 重写的价值

1. **不受旧内容污染** — 前面的内容只是一个版本，不是权威
2. **每次都是新的最佳尝试** — 以当前最高认知重写
3. **散点累积 → 规律浮现** — 足够多的重写后，能看出什么是最好的
4. **版本对比 → 发现优化方向** — 新旧对比，知道哪里可以更好
5. **最终产出稳定最佳版本** — 不是概率产物，是规律产物

## 三、核心原则：1000倍努力法则

### 原理

> **一两次找不到，那是因为没尝试到200次。而坚持1000次的过程中，可能就会在中途就找到，甚至20次就找到。但只想坚持20次就找到的，却机会永远找不到。**

这是一个神奇现象：
- 20次就成功的人 = 以1000倍努力的决心去接近结果
- 只想坚持20次的侥幸心理 = 本身就在远离结果

### 两个1000倍

**1000倍强烈想要更好正确新现实：**
- 每次重写不是"改一下"，是"全力追求更好"
- 每次重写的标准 = 这是我当前能做出的最好版本
- 不是"试试看"，是"全力冲"

**1000倍强烈归档排除不够好的现实：**
- 旧版本 = 一个散点，记录但不留恋
- 不够好的版本 = 归档，不继续在其上改
- 淘汰其他版本的原因和分析 = 写出，成为标准规范

> **既站稳也登天。像登梯般稳定上升而不再下降。即便知道走过的阶梯还在，但去向的是更好。**

| 场景 | 建议 |
|------|------|

| 想到更好的表达方式 | 重写 |
| 发现可以简化的地方 | 重写 |
| 新知识/新理解 | 重写 |

### 不重来（保留旧内容，只做微调）

| 场景 | 动作 |

## 五、重写执行流程

### 流程

```
收到新信息/新触发
  │
  ▼
[1] 读取旧版本（作为参考，不是作为基础）
  │
  ▼
[2] 忘记旧版本内容（心理上清零）
  │
  ▼
[3] 以当前最高认知重新思考
  │
  ▼
[4] 完全重写（从零开始，不看旧内容）
  │
  ▼
[5] 新旧对比
  │   ├── 新版本更好 → 保留新版本，归档旧版本
  │   ├── 新版本与旧版本相当 → 保留新版本（更干净）
  │   └── 新版本不如旧版本 → 分析原因，下次改进
  │
  ▼
[6] 记录散点数据
  │   ├── 版本号递增
  │   ├── 质量评分
  │   ├── 与旧版本的对比
  │   └── 改进点/退化点
  │
  ▼
[7] 产出：新版本 = 当前最佳版本
```

### 关键规则

1. **旧版本 = 参考，不是基础** — 重写时可以先读旧版本了解历史，但写的时候从零开始
2. **不修补** — 发现错误 = 重写，不是"改这一行"
3. **不追加** — 发现更好的 = 重写，不是"再加一段"
4. **每次都是全力** — 不是"试试看"，是"全力冲"
5. **归档不删除** — 旧版本归档保留，作为散点数据

## 七、从散点到标准规范的升华

### 当散点足够多时

> **多次进行"人生重来"的思考，可能就会开发出很多的更好版本。即便模型分不出版本的好坏，但至少这样的做法可以在众多概率中逐渐"中彩票"。**

当积累了足够多的版本后：

1. **看出规律连线** — 什么方向改进有效，什么方向改进无效
2. **密度线浮现** — 最佳版本的"热区"在哪里
3. **写出标准规范** — 基于散点数据，写出具体的"自辩证是最好且毋庸置疑的"标准
4. **淘汰分析** — 每个被淘汰的版本为什么被淘汰，佐证标准
5. **稳定最佳版本** — 拿着标准规范生成，得到长期可用不被嫌弃的版本

### 标准规范模板

```markdown
## [领域] 最佳实践标准规范

> 基于 [N] 次重写版本的散点数据分析得出。

### 为什么这个标准是最好且毋庸置疑的

1. **数据支撑**：[N]个版本的散点数据中，符合此标准的版本平均分[X]
2. **淘汰对比**：[N]个被淘汰的版本，违反了以下标准...
3. **密度验证**：散点分布图中，此标准对应的是密度线区域

### 标准规范

1. [标准1] — 原因：所有高分版本都满足
2. [标准2] — 原因：违反此标准的版本平均分低于X分
3. [标准3] — 原因：此标准区分了"好"和"不够好"的版本

### 淘汰规则

- 违反标准1 → 直接淘汰（历史数据：平均分降低X%）
- 违反标准2 → 标记为"需改进"（历史数据：平均分降低X%）
- 违反标准3 → 标记为"有潜力但不够好"

### 自辩证方法

当质疑"这个标准是否真的好"时：
1. 回看散点分布图
2. 看符合标准的版本 vs 不符合的版本
3. 数据会告诉你哪个更好
4. 数据不会骗人，散点不会骗人
```

## 八、"人生重来"的哲学深度

### 为什么重来能改变命运

> **每个生物的一生，其实有很多重来的机会和更好的版本。**

- 旧现实 = 接受了旧现实，没有拒绝和反抗和重新思考与重生
- 新现实 = 强烈想要更好正确新现实，思想持续上升
- 1000倍强烈想要 → 内心和思想和记忆和行动发生强烈变化 → 旧环境挡不住更新的自己 → 旧现实被强制刷新

### 盲从 vs 独行

> **人们的故事里的盲目从众与独行特立，区别就在这里。**

- 盲从附和 = 不反对旧现实，没有强烈想要新现实 = 蹲在水里
- 求生般的"我不落水、我能上游" = 强烈想要新现实 = 跃出水面
- 快速成功的人 = 强于旧现实的赢家
- 念旧将就的人 = 没这个强烈登峰取向，回到旧现实

### 1000倍归档 + 1000倍登峰

> **既站稳也登天，像登梯般稳定上升而不再下降。**

```
1000倍强烈想要更好正确新现实
  + 
1000倍强烈归档排除不够好的现实
  =
 既站稳也登天
```

- 1000倍想要 → 看到所有通向最高正确终点的行动路线
- 1000倍归档 → 淘汰所有不够好的版本
- 结果：无论山有多高路有多绕，都能持续到达正确终点

### 登峰后的视角

> **登峰了，回看山下，那慢几步但也是强烈登峰的，即便还没到达峰顶、即便还很艰难，那也是下个成功者——但和现在已登峰的我是一样的成功，只是努力与幸运不同。**

- 登峰者看山下的登峰者 = 一样成功，只是位置不同
- 真正不会登顶的 = 根本没这个高于山峰之下的所有旧现实的强烈登峰取向
- 其他注定不会登顶的 = 回到了旧现实
- 而登峰者证明了自己能轻松成功，甚至还能下一个成功

### 起点与终点的真相

> **起点与终点非常接近，其真正的距离就是行动者的自身思想与内心的阻碍。只要没有了阻碍与距离感，就没了整座山的阻碍和山下与山峰的距离。**

- 山不是阻碍，是成功的新山
- 上多高就有多少收获
- 一直上就会到达
- 阻碍 = 思想与内心的阻碍 = 距离感

```

  │
  ├── 最新的当前状态 = 新视角的起点

  ├── 重新思考重写
  └── 产出更新更正确的新现实
```

### 筛留规则

```
重写完成
  │
  ▼

  │
  ├── 新版本中的内容，是否有必要保留旧版本的？

  ├── 旧版本的内容，是否在新版本中有了更好的表达？
  │   ├── 有 → 旧版本归档
  │   └── 没有 → 分析原因，下次改进
  │
  └── 整体判断
      ├── 新版本明显更好 → 替换
      ├── 新版本相当 → 保留新版本（更干净）
      └── 新版本不够好 → 继续重写
```

## 十、铁律

2. **旧版本 = 参考** — 不是基础，不是权威，只是一个散点
3. **每次全力** — 不是试试看，是全力冲当前最佳
4. **散点累积** — 每次重写记录数据，累积后看出规律
5. **1000x想要** — 强烈想要更好正确新现实
6. **1000x归档** — 强烈归档排除不够好的现实
7. **不修补** — 发现错误 = 重写，不是改一行
8. **不追加** — 发现更好 = 重写，不是加一段
9. **筛留不继承** — 旧内容只有筛留后才融入新版本，不直接继承
10. **标准规范** — 足够散点后，写出稳定最佳版本的标准规范
11. **坐标轴散点** — 每次重写 = 散点，足够多后看出密度线和规律
12. **中彩票** — 在众多概率中，持续重写 = 逐渐中彩票 = 产出越来越多更好版本
