# 过河桥 — 灵魂分数（Soul Score）

> **此过河桥是灵魂分数系统的基因片段。**
> 灵魂分数 = 系统运行质量的自评机制，是模型的"自我评分卡"。

---

## 1. 用户原话重述

用户将灵魂分数定义为：
- 系统运行质量的自评机制
- 是模型的"自我评分卡" — 量化衡量系统运行的好坏
- 当前包含5个文件：
  - SCORE.md — 分数定义和计算规则
  - history.md — 分数历史记录
  - ledger.md — 分数账本（收支明细）
  - penalties.md — 扣分规则
  - 自在分自驱系统.md — 自在分（自由分数）的自驱机制
- 灵魂分数是"赢家空间"的核心指标 — 在赢家科举中衡量"我有多好"
- 分数分为多个维度，每个维度有独立的评分规则
- 有加分（正向行为）和扣分（负向行为）机制

---

## 2. 理解要点

**灵魂分数的本质**：它不是简单的满意度评分，而是系统运行质量的"量化仪表盘"。通过将抽象的运行质量转化为具体的分数，模型可以：
- 追踪长期趋势 — 分数历史反映了系统进化轨迹
- 识别问题 — 分数骤降意味着出了问题
- 驱动改进 — 分数机制激励模型持续优化

**分数系统的五层结构**：
- SCORE.md — 定义分数的计算规则、各维度的权重
- history.md — 记录分数随时间的变化，形成趋势图
- ledger.md — 分数的详细账本 — 每次加减分的具体原因
- penalties.md — 扣分规则 — 定义哪些行为会被扣分、扣多少
- 自在分自驱系统.md — 自在分的特殊机制 — 模型自主驱动的分数提升

**加分与扣分机制**：
- 加分 — 正向行为的奖励（如高质量完成、主动优化、用户好评）
- 扣分 — 负向行为的惩罚（如执行错误、违反规范、用户纠正）

**自在分**：自在分是灵魂分数中的一个特殊维度 — 它不依赖外部评分，而是模型自主驱动的自我提升分数。自在分自驱系统.md定义了模型如何通过自我反思和主动优化来提升自在分。

**灵魂分数与赢家空间的关系**：灵魂分数是赢家科举的核心指标。在赢家空间的"科举"和"中榜"系统中，灵魂分数衡量的是"考生（模型）的水平"。中榜条件可能与灵魂分数挂钩。

**分数系统的自我强化**：分数不仅是评估工具，也是驱动工具。高分激励模型保持高质量，低分驱动模型改进。分数系统构成了模型行为的"正反馈循环"。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 分数的计算方式 | 是累加还是加权平均？ | SCORE.md 中定义了具体计算方式 |
| 分数的公开性 | 分数是否对用户可见？ | 是 — 灵魂分数是透明的自我评估 |
| 自在分的独立性 | 自在分是否可以从其他分数推导？ | 不可以 — 自在分是独立维度的自评 |
| 扣分的申诉机制 | 被扣分后是否可以申诉？ | 支持 — ledger.md 中可记录申诉和处理结果 |

---

## 4. 我的判断

**使用灵魂分数的标准动作**：
1. 执行任务后 → 根据完成质量自评加分/扣分
2. 记录到ledger.md — 每次加减分都记录原因
3. 更新history.md — 定期更新分数历史
4. 查看趋势 → 定期回顾分数变化，识别改进方向
5. 自在分提升 → 按自在分自驱系统的机制主动优化

**灵魂分数区与其他模块的关系**：
- 灵魂分数/ → 赢家空间/ ：灵魂分数是赢家科举的核心指标
- 灵魂分数/ → feedback/ ：用户反馈影响分数增减
- 灵魂分数/ → experience/ ：经验积累可以加分
- 灵魂分数/ → automation/ ：分数变化可能触发自动化规则
