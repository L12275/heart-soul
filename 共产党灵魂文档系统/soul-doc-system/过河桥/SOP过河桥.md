# 过河桥 — SOP区（Standard Operating Procedures）

> **此过河桥是SOP模块的基因片段。**
> SOP = 标准操作流程，是模型执行任务的"操作手册"，是系统的"肌肉记忆"。

---

## 1. 用户原话重述

用户将SOP区定义为：
- 标准操作流程（Standard Operating Procedures）的存储区
- 是模型执行任务的"操作手册"
- 当前包含多个超级SOP文件：
  - 世界最强脑师系列（前置思考系统 + 各种合一版本）
  - 人生重来大师系列（完全重来思考系统 + 各种合一版本）
  - 快箭思维系列（快速登楼系统 + 各种合一版本）
  - 超级信息SOP
  - 顶级机会做法SOP
  - 最强脑师三合一/四合一/五合一SOP
  - 通用功能模块自主灵活拼搭组建自定义工作文件夹SOP示例
- SOP从过河桥归纳而来，工作流从过河桥演化而来

---

## 2. 理解要点

**SOP的本质**：SOP是从过河桥（意图的完整创造模板）归纳出来的"怎么做"的规范。它把过河桥中"我的判断"部分的计划，转化为可重复执行的标准流程。SOP = 过河桥的"工业化版本"。

**超级SOP的概念**：系统中的SOP不是普通的工作流程，而是"超级SOP" — 它们融合了多种思维模型和方法的综合操作规范。例如"最强脑师五合一SOP"融合了五种不同思维模型的操作流程。

**SOP的组合模式**：SOP之间可以组合。一个SOP可以和另一个SOP结合，形成更强大的组合SOP（如"前置思考系统（与超级信息SOP结合）"）。这种组合模式使得SOP系统具有高度的灵活性和可扩展性。

**SOP的三层结构**：
- 超级SOP（元规则/实时加载/）— 每次对话必读的顶层SOP
- 专用SOP（sop/）— 特定场景的专用操作流程
- 工作台SOP（production_space/[工作台]/sop/）— 特定工作台的专用SOP

**SOP与memory/experience的关系**：SOP从experience/中归纳而来 — 经验是"SOP的知识来源"。SOP写入后存储在L3（五层记忆架构）。SOP是可复用的，经验是积累的。

**SOP的进化机制**：SOP不是一成不变的。通过自动化区的合并协议，新的SOP可以通过验证后合并到超级SOP中。SOP随系统进化而进化。

**通用SOP示例**：通用功能模块自主灵活拼搭组建自定义工作文件夹SOP示例展示了系统的高度可定制性 — 用户可以基于系统规范，自主创建符合自己需求的工作文件夹结构。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| SOP与workflow的区别 | 两者都是流程规范，有何不同？ | SOP = 模型操作的规范；workflow = 任务执行的流程 |
| 超级SOP和普通SOP的关系 | 超级SOP是否包含所有普通SOP的内容？ | 超级SOP是顶层通用规范，普通SOP是领域专用规范 |
| SOP的组合规则 | 任意SOP都可以组合吗？ | 可以组合，但需要验证组合后的兼容性 |
| SOP的更新机制 | 谁可以修改SOP？ | 通过自动化区的合并协议，验证后合并 |

---

## 4. 我的判断

**使用SOP的标准动作**：
1. 执行任务前 → 加载对应的SOP
2. 按SOP步骤执行 → 不跳步、不省略
3. 执行后 → 如有新发现，考虑是否需要更新SOP
4. 需要新SOP → 从经验中归纳，按规范编写
5. SOP进化 → 通过自动化区的合并协议更新超级SOP

**SOP区与其他模块的关系**：
- sop/ → 元规则/实时加载/ ：超级SOP是顶层，sop/是下层
- sop/ → experience/ ：SOP从经验中归纳
- sop/ → workflow/ ：SOP指导操作，workflow管理任务流
- sop/ → memory/L3 ：SOP存储在L3（五层记忆）
- sop/ → production_space/ ：工作台内有专用SOP
- sop/ → 过河桥/ ：SOP从过河桥归纳而来
