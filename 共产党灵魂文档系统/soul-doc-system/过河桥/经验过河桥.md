# 过河桥 — 经验区（Experience）

> **此过河桥是经验系统的基因片段。**
> 经验 = 经过验证和测试的正确知识，是系统的"智慧沉淀"。

---

## 1. 用户原话重述

用户将经验区定义为：
- 经过验证和测试的正确知识存储区
- 与记忆的区别：记忆是"做了什么"的纪实，经验是"验证过的正确做法"的知识
- 当前有四个子文件夹：proven（已证实）、tested（已测试）、Reference（参考资料）、suspected-hallucination（疑似幻觉）
- 经验系统是五层记忆架构中L3的实体化 — L3是"任务技能/SOP"
- 经验系统参考GA（GenericAgent）记忆系统
- 三条公理：No Execution, No Memory / 不可删除 / 禁易变状态

---

## 2. 理解要点

**经验系统的本质**：它不是简单的"经验记录"，而是一个有严格质量分级的知识库。内容进入经验系统前必须经过验证或测试，确保经验是正确的、可复用的。

**四层分类体系**：
- proven（已证实）— 最高可信度，经过实际验证的正确知识
- tested（已测试）— 经过测试但可能不如proven那么全面验证
- Reference（参考资料）— 值得参考但尚未充分验证的内容
- suspected-hallucination（疑似幻觉）— 被标记为可能错误的内容，作为反例存留

**经验与记忆的关系**：记忆是"做了什么"的全记录（含错误），经验是"验证过的正确做法"的精选。记忆是纪实，经验是知识。经验系统从记忆中提取精华，经过验证后沉淀为可复用的知识。

**经验的不可删除原则**：经验一旦进入系统，不可删除。但经验可以有不同可信度等级。疑似幻觉的内容不删除，而是降级标记。这样保留了系统的"自我纠错历史" — 知道什么曾经被认为是正确的，后来被证明是幻觉。

**经验与SOP的关系**：经验是SOP的知识来源。SOP是从经验中归纳出来的"怎么做"的规范。经验系统存储原始知识，SOP系统存储从知识归纳出的操作流程。

**经验系统的维护机制**：经验不是静态存储的。新的经验需要经过验证才能进入，已有的经验需要定期回顾确保仍然正确。经验系统是"活的"知识库。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| proven和tested的区别 | 两者都是验证过的，程度如何区分？ | proven = 多次验证 + 长期有效；tested = 已验证但样本较少 |
| suspected-hallucination的内容是否可以删除 | 不可删除原则是否包含疑似幻觉？ | 不删除，但标记为低可信度，作为反例参考 |
| 经验是否需要版本管理 | 经验被推翻后如何处理？ | 不删除旧版本，标记为"已推翻"，新版本替代 |
| Reference是否等同于wiki | 两者都是参考资料，有何不同？ | Reference侧重外部参考资料，wiki侧重系统内生知识 |

---

## 4. 我的判断

**使用经验系统的标准动作**：
1. 执行任务后获得新认知 → 评估是否进入经验系统
2. 经验进入 → 按可信度放入对应子文件夹
3. 使用经验前 → 检查可信度等级
4. 发现经验可能有误 → 标记为suspected-hallucination，不删除
5. SOP编写 → 从proven/tested中提取知识归纳

**经验区与其他模块的关系**：
- experience/ → memory/ ：经验是记忆的精选验证版
- experience/ → sop/ ：SOP从经验中归纳
- experience/ → wiki/ ：经验是外部验证的知识，wiki是系统消化的知识
- experience/ → collection/ ：收藏区可引用经验，经验系统是收藏区的重要来源
