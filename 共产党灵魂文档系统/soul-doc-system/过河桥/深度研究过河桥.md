# 过河桥 — 深度研究区（Deep Research）

> **此过河桥是深度研究模块的基因片段。**
> 深度研究 = 对选定主题进行系统性深度分析的能力引擎，是系统的"显微镜"。

---

## 1. 用户原话重述

用户将深度研究区定义为：
- 对选定主题进行系统性深度分析的能力引擎
- 是系统的"显微镜" — 不是广度扫描，而是深度挖掘
- 当前只有README.md
- 与discovery/的区别：discovery是广度探索（发现有什么），深度研究是深度挖掘（把某个发现研究透）
- 深度研究是"发现→研究"工作流的下游 — discovery找到候选，深度研究做深度分析
- 研究产出放在product/研究类/中
- 研究过程记录在diary/和memory/中

---

## 2. 理解要点

**深度研究的本质**：它不是简单的信息搜索，而是系统性的深度分析。从发现区找到一个值得研究的主题后，深度研究模块会投入大量计算资源进行：文献调研、数据收集、逻辑分析、交叉验证、结论提炼。这是一个完整的"研究闭环"。

**发现→研究的完整工作流**：
1. discovery/ 广度扫描 → 发现候选主题
2. 评估 → 判断候选主题的研究价值
3. 深度研究 → 系统性深度分析
4. 产出 → 研究报告（存入product/研究类/）
5. 提炼 → 研究结论中的精华知识存入experience/和wiki/

**深度研究与工作台的关系**：深度研究是一个独立的研究类型，production_space/中有"研究工作台"专门服务于深度研究。研究工作台提供了研究所需的完整工具链 — 桌面（草稿）、diary（过程）、memory（研究记忆）、product（报告）、sop（研究SOP）、workflow（研究流程）。

**深度研究的SOP**：元规则/sop/中可能有专门的研究SOP。深度研究遵循这些SOP执行，确保研究的系统性和质量。

**深度研究的质量要求**：研究产出必须经过验证才能进入经验系统。研究结论的正确性取决于：数据来源的可靠性、分析方法的严谨性、逻辑推理的有效性、结论的可验证性。

**深度研究与团队空间的关系**：复杂研究任务可能需要多智能体协作 — 一个智能体负责数据收集，一个负责分析，一个负责验证。团队空间提供了这种协作的环境。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 深度研究是否需要用户指定主题 | 还是系统自主选择研究主题？ | 两者都支持 — 用户指定或系统自主选题 |
| 研究的深度标准 | "深度"如何衡量？ | 以覆盖全面性、分析深度、结论可靠性为标准 |
| 研究产物的范围 | 研究报告是否包含原始数据？ | 报告包含分析结论，原始数据存档 |
| 研究的复用性 | 研究结论是否可以复用于其他任务？ | 可以 — 研究结论存入experience/后可被SOP引用 |

---

## 4. 我的判断

**进行深度研究的标准动作**：
1. 选题 → 从discovery/获取候选或用户指定
2. 定义研究范围 → 明确研究的边界和目标
3. 执行研究 → 按研究SOP系统执行
4. 产出报告 → 写入product/研究类/
5. 提炼知识 → 研究结论中的精华存入experience/和wiki/
6. 归档 → 研究过程记录在diary/和memory/中

**深度研究区与其他模块的关系**：
- 深度研究/ → discovery/ ：深度研究是发现的深度加工
- 深度研究/ → production_space/研究工作台/ ：研究工作台是深度研究的专用环境
- 深度研究/ → product/研究类/ ：研究产出存入产物区
- 深度研究/ → experience/ ：研究结论中的精华存入经验系统
- 深度研究/ → wiki/ ：系统化研究知识存入Wiki
- 深度研究/ → 团队空间/ ：复杂研究任务可能需要多智能体协作
