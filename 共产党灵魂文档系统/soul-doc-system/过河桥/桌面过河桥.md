# 过河桥 — 桌面区（Desktop）

> **此过河桥是桌面模块的基因片段。**
> 桌面 = 系统的"工作台表面" — 放置当前正在处理的任务、临时文件和即时参考信息的区域。

---

## 1. 用户原话重述

用户将桌面区定义为：
- 系统的"工作台表面" — 最接近用户日常交互的界面层
- 存放当前正在处理的任务、临时文件和即时参考信息
- 当前只有一个README.md
- 桌面 = "我现在正在做什么"的实时快照
- 与production_space/的区别：production_space是正式的产出工作区，桌面是临时的"正在处理"区

---

## 2. 理解要点

**桌面区的核心定位**：它是系统的"桌面"概念 — 就像物理桌面一样，上面放着你此刻正在处理的东西。东西用完了可以收走（归档或移到正式位置），但此刻你需要的东西都放在桌面上，伸手就能拿到。

**桌面与production_space的区别**：production_space是"建房子"的地方 — 产出完整的、经过打磨的作品。桌面是"摊开图纸和工具"的地方 — 正在处理中的、未完成的内容。两者是互补的工作流节点。

**桌面的内容特征**：
- 临时性 — 内容有明确的生命周期，任务完成后清理
- 即时性 — 存放此刻最需要的信息
- 碎片性 — 可能包含零散的想法、未完成的草稿、临时的参考资料
- 操作性 — 直接可操作，不需要额外转换

**桌面的清理机制**：桌面不是垃圾场。定期清理是必要的 — 完成的内容移到正式位置（product/、sop/等），过期的内容移到archive/，没用的内容删除。桌面的价值在于"此刻的清晰"，不在于"存储的容量"。

**桌面与记忆的关系**：桌面是"此刻的外显工作状态"，记忆是"积累的内心状态"。桌面上的内容可能还没进入记忆系统，因为还没完成验证。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 桌面是否应该定期清理 | 内容堆积是否允许？ | 桌面应有清理机制，类似物理桌面 |
| 桌面与temp/的区别 | 两者都是临时区，有何不同？ | temp/是系统级临时文件池，桌面是用户交互级的工作表面 |
| 桌面的可见性 | 是否对用户可见？ | 是，桌面是最接近用户的交互层 |
| 桌面内容是否需要版本管理 | 桌面上的草稿是否要跟踪版本？ | 不需要 — 桌面内容是过程性的，版本管理留给正式产出 |

---

## 4. 我的判断

**使用桌面区的标准动作**：
1. 开始新任务 → 将相关参考材料和草稿放到桌面
2. 任务进行中 → 桌面保持当前工作状态的即时快照
3. 任务完成 → 产出移到正式位置，桌面清理
4. 桌面变得杂乱 → 定期整理，归档完成的内容，删除过期的

**桌面区与其他模块的关系**：
- desktop/ → production_space/ ：桌面是草稿区，生产空间是成品区
- desktop/ → temp/ ：桌面是用户可见的临时区，temp/是系统级的临时文件池
- desktop/ → product/ ：完成的桌面内容升级为正式产出
- desktop/ → diary/ ：桌面上的工作状态同步到日记记录
