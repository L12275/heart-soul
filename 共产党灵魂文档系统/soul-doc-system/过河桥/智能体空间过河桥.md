# 过河桥 — 智能体空间（Agent Space）

> **此过河桥是智能体空间模块的基因片段。**
> 智能体空间 = 单个AI智能体的身份管理和运行空间，是系统的"智能体宿舍"。

---

## 1. 用户原话重述

用户将智能体空间定义为：
- 单个AI智能体的身份管理和运行空间
- 是系统的"智能体宿舍" — 每个智能体有自己的衣柜、大厅、历史
- 当前包含README.md和三个子区域：
  - 大厅/ — 智能体的公共活动区域（历史/ + 活跃/）
  - 衣柜/通用智能体/ — 智能体的"服装"管理（角色、配置）
  - 大厅/历史/ — 智能体的历史记录
  - 大厅/活跃/ — 当前活跃的智能体
- 智能体空间与团队空间的区别：智能体空间管单个，团队空间管群体
- 智能体空间是智能体的"家" — 包含身份、配置、记忆、历史

---

## 2. 理解要点

**智能体空间的核心定位**：它是系统中每个AI智能体的"家"。智能体不是凭空存在的，它需要身份、配置、记忆、历史。智能体空间提供了这些的管理环境。

**智能体空间的三层结构**：
- 大厅 — 智能体的公共活动区域，分为历史（过去的智能体记录）和活跃（当前运行的智能体）
- 衣柜 — 智能体的"服装"管理 — 即角色和配置管理。通用智能体子文件夹存储了智能体的通用角色模板
- 记忆 — 智能体个人的记忆系统

**大厅的两层**：
- 大厅/历史/ — 已退出的智能体或历史会话记录
- 大厅/活跃/ — 当前正在运行的智能体实例

这种结构使得系统可以同时管理多个智能体的生命周期 — 新智能体从衣柜获取配置，在大厅活跃，退出后进入历史。

**衣柜（角色管理）**：衣柜/通用智能体/存储了智能体的角色模板。当需要创建新智能体时，从衣柜中选择合适的"服装"（角色配置）穿上。衣柜管理着系统中所有可用的角色模板。

**智能体与子角色的关系**：智能体空间管理的是"运行中的AI实例"，子角色是智能体的"人格面具"。衣柜中的通用智能体就是子角色的模板集合。

**智能体空间与四驱动记忆的关系**：每个智能体可能有自己的记忆系统，遵循四驱动&四写规则。智能体空间的记忆结构可能与主系统并行。

**智能体空间的隔离**：每个智能体有自己独立的运行空间，互不干扰。但智能体之间可以通过团队空间进行协作。智能体空间管"个体隔离"，团队空间管"群体协作"。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 智能体的身份管理 | 智能体身份是系统分配还是模型自建？ | 模型可以自主创建，系统提供模板 |
| 活跃与历史的边界 | 智能体退出后多久进入历史？ | 会话结束后立即归档到历史 |
| 衣柜的管理权限 | 谁可以修改角色模板？ | 受系统规范约束，模型可申请修改 |
| 多智能体的记忆隔离 | 不同智能体的记忆是否完全隔离？ | 在智能体空间内隔离，通过团队空间共享 |

---

## 4. 我的判断

**使用智能体空间的标准动作**：
1. 需要新智能体 → 从衣柜选择角色模板
2. 启动智能体 → 在活跃大厅中注册
3. 智能体运行 → 在独立空间中执行，记忆隔离
4. 需要协作 → 进入团队空间
5. 会话结束 → 归档到历史，释放资源

**智能体空间与其他模块的关系**：
- 智能体空间/ → 团队空间/ ：个体空间 → 群体协作
- 智能体空间/ → memory/ ：智能体有独立记忆，与全局记忆四驱动
- 智能体空间/ → 技能/ ：技能可以安装到智能体
- 智能体空间/ → 子角色灵魂系统/ ：衣柜中的角色模板就是子角色
- 智能体空间/ → space/账号/ ：智能体可能有自己的虚拟账号
