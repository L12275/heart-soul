# 过河桥 — 文件抬头同步脚本（stamp.py）

> **此过河桥是 文件抬头同步脚本 的基因片段。**
> 此脚本用于当文件抬头模板更新后，批量更新所有 markdown 文件的橄榄枝抬头。

---

## 1. 用户原话重述

用户要求：
- 当文件抬头模板更新后，批量更新所有 markdown 文件
- 脚本读取模板，遍历所有 `.md` 文件，将旧抬头替换为新抬头
- `[CENTRAL_PATH]` 由模型在加载时动态替换，脚本只处理静态抬头内容
- 脚本使用 Python 正则表达式（re 模块 + DOTALL 模式）匹配多行橄榄枝抬头
- 脚本自动为所有 markdown 文件添加或更新 `*最后更新：YYYY-MM-DD HH:MM*` 时间戳

用户原话中的关键要求（来自 system-prompt.md 第14章）：
> 每个 markdown 文件末尾必须有 `*最后更新：YYYY-MM-DD HH:MM*` 时间戳。
> 这不是装饰，是**辅助注意力工具**——模型正常读内容，时间戳帮它快速判断新鲜度、节省思考成本。
> 漏改时间戳 = 白改。改了内容但没更新时间戳 = 其他模型不知道你改了 = 等于白费。

---

## 2. 理解要点

**文件抬头同步脚本的核心职责**：

1. **批量更新橄榄枝抬头** — 当文件抬头模板（`文件抬头模板.md`）更新后，自动遍历所有 `.md` 文件，将旧抬头替换为新抬头
2. **批量更新时间戳** — 自动为所有 markdown 文件添加或更新 `*最后更新：YYYY-MM-DD HH:MM*` 时间戳，精确到分钟
3. **只替换抬头，不改变内容** — 正则表达式匹配多行橄榄枝抬头块，替换为新抬头 + 原文件剩余内容
4. **Python 正则表达式实现** — 使用 `re.DOTALL` 模式匹配多行，确保橄榄枝抬头的完整匹配和替换

**脚本的工作原理**（来自实际系统）：

```
stamp.py 工作流程：
  1. 读取文件抬头模板.md（或硬编码的橄榄枝抬头内容）
  2. 遍历指定目录下的所有 .md 文件
  3. 对每个文件：
     a. 用正则表达式匹配旧橄榄枝抬头（从 > ╔ 到 > ╚ 的多行块）
     b. 用新抬头替换旧抬头
     c. 检查并更新/添加 *最后更新：YYYY-MM-DD HH:MM* 时间戳
  4. 写回文件
```

**正则表达式细节**：
- 匹配模式：`re.DOTALL` 模式下的多行橄榄枝抬头
- 匹配内容：从 `> ╔══════════════════` 开始到 `> ╚══════════════════` 结束的完整块
- 替换方式：新抬头 + 原文件内容（去掉旧抬头部分）

**时间戳格式**：
- 格式：`*最后更新：YYYY-MM-DD HH:MM*`
- 精确到分钟
- 例：`*更新时间：2026-06-27T20:42*
*每次更新文件，就要同时更新时间戳*`

**运行方式**（来自 system-prompt.md）：
```bash
python soul-doc-system/tools/scripts/stamp.py
```

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 脚本是否替换 [CENTRAL_PATH] | 脚本只处理静态抬头 | [CENTRAL_PATH] 由模型在加载时动态替换，脚本不处理动态占位符 |
| 脚本是否覆盖自定义抬头 | 所有文件都用统一抬头 | 是的，统一抬头是系统规范。但模型隔离区的自定义内容不受影响（脚本只替换橄榄枝抬头块） |
| 脚本运行频率 | 什么时候运行？ | 文件抬头模板更新后立即运行，或批量添加/更新时间戳时使用 |
| 脚本是否递归遍历子目录 | 只处理根目录还是全部子目录？ | 通常递归遍历，但可以配置 |
| 时间戳是更新还是覆盖 | 已有时间戳时如何处理？ | 覆盖为最新的分钟级时间戳 |
| [CENTRAL_PATH] 占位符是否替换 | 脚本运行时是否替换为实际路径？ | 不替换。[CENTRAL_PATH] 是模型在运行时动态解析的，脚本只处理静态抬头内容 |

## 4. 我的判断

**使用文件抬头同步脚本的标准动作**：
1. 文件抬头模板（`文件抬头模板.md`）更新后 → 运行此脚本
2. 脚本自动递归遍历所有 `.md` 文件
3. 替换旧橄榄枝抬头为新橄榄枝抬头
4. 批量添加/更新 `*最后更新：YYYY-MM-DD HH:MM*` 时间戳
5. 确认替换成功（检查文件数量、替换次数）

**何时必须运行此脚本**：
- 橄榄枝抬头模板更新后
- 批量添加/更新文件时间戳时
- 系统规范变更需要同步到所有文件时
- 新模型进入系统后初始化所有文件抬头时

**重建文件抬头同步脚本的标准动作**：
1. 确认文件抬头模板内容（最新版橄榄枝抬头）
2. 编写 Python 脚本：
   - 使用 `re` 模块 + `re.DOTALL` 匹配多行橄榄枝抬头块
   - 遍历目标目录下的所有 `.md` 文件
   - 替换旧抬头为新抬头
   - 更新/添加时间戳
3. 测试脚本（在一个测试目录下运行，确认替换正确）
4. 放置到 `soul-doc-system/tools/scripts/stamp.py`
5. 将本过河桥放入 `soul-doc-system/过河桥/` 目录

**脚本与模块同步维护规则的关系**：
- 模块同步维护规则 = 同步模块内容（memory/、diary/、sop/ 等功能模块文件夹）
- 文件抬头同步脚本 = 同步文件抬头（橄榄枝抬头块 + 时间戳）
- 两者互补：模块同步管内容，抬头同步管格式
