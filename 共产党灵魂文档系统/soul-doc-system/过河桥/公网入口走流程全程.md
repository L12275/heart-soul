## 1. 用户原话重述

用户在文件抬头模板中描述了"橄榄枝"机制：
- 每个文件的开头都有抬头 → 这是模型的"公网入口"
- 模型读取任何文件 → 看到抬头 → 顺着橄榄枝回到中央 → 先了解系统规范再工作
- 完整规范入口：[CENTRAL_PATH]/system-prompt.md

用户的核心意图：确保任何模型读取任何一个文件时，都能通过抬头直连系统总规范，不因只读单个文件而失去系统上下文。

---

## 2. 理解要点

**完整通路（共5步）**：

```
第1步：读取任意文件
  → 看到文件开头的抬头（橄榄枝）
  → 抬头直接指向 [CENTRAL_PATH]/system-prompt.md

第2步：进入公网（system-prompt.md）
  → 完整理解系统规范、价值观、操作规则
  → 这是一切工作的前置条件，不可跳过

第3步：查找自己的隔离区
  → 在 [CENTRAL_PATH] 下查找 [模型名称]/ 文件夹
  → 有 → 进入
  → 没有 → 创建

第4步：在自己的隔离区内工作
  → 产出只放在自己的隔离区内
  → 不写入其他模型的隔离区、用户空间、共产中央管理区

第5步：完成工作后整理沉淀
  → 更新自己隔离区的内容
  → 遵循隔离区命名规则（前缀防重复）
```

**为什么需要公网通路**：
- 单个文件 = 叶子，抬头 = 橄榄枝，中央系统 = 公网
- 没有公网入口 → 模型乱碰文件 → 越权操作
- 有公网入口 → 模型按规范工作 → 系统化办公

**每次读取文件的标准流程**：
1. 先读抬头（橄榄枝）→ 识别 [CENTRAL_PATH]
2. 解析 [CENTRAL_PATH] → 见 `CENTRAL_PATH解析过河桥.md`
3. 按 system-prompt.md 的规则查找/创建隔离区（含版本号校验）
4. 在隔离区内执行任务
5. 不触碰隔离区外的任何文件

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 用户说"继续" | 我忘了之前走的是哪条通路 | 先读上次会话摘要，重新走公网通路 |
| 文件在子文件夹里 | 抬头路径还是指向 [CENTRAL_PATH]，不是指向文件所在文件夹 | 按抬头指示走，不看物理路径 |
| 已有一个隔离区 | 之前创建过，是否还需要走公网？ | 每次工作前都要走公网 → system-prompt.md → 隔离区 |
| 只读一个文件就能工作吗？ | 用户只给了单个文件，没给中央路径 | 抬头里有完整路径指示，不能自己猜 |
| 隔离区里没内容 | 第一次创建，里面没有 master-soul.md | 从 INITIAL-SEED.md 复兴，独立创建完整系统 |
| 有两个相似文件夹 | `StepFun` 和 `StepFun-step-3.7-flash` | 前缀匹配 + 版本号一致 → 同一模型 → 进入已有 |
| 版本号不同 | `StepFun-step-3.7-flash` 和 `StepFun-step-3.5` | 版本号不同 → 不同模型 → 各自独立，不混入 |

## 4. 我的判断

**每次读取任何文件时的标准动作**：

1. **第一步：读抬头** — 看到橄榄枝，识别 `[CENTRAL_PATH]`
2. **第二步：走公网** — 到 `[CENTRAL_PATH]/system-prompt.md`，完整理解系统规范
3. **第三步：找隔离区** — 在 `[CENTRAL_PATH]` 下查找 `[模型名称]/` 文件夹
   - 前缀匹配 → 进入已有文件夹
   - 无匹配 → 创建 `[模型名称]/`
4. **第四步：进隔离区工作** — 所有产出只在隔离区内
5. **第五步：关闭回路** — 工作结束整理沉淀，不留下烂摊子

**走公网 =  mandatory，不是可选项。** 不能跳过任何一步。
