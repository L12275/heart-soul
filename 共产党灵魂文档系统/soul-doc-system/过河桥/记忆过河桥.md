# 过河桥 — 记忆区（Memory）

> **此过河桥是记忆系统的基因片段。**
> 记忆 = 系统运行与验证确认后的正确记忆，是系统的"大脑皮层"。

---

## 1. 用户原话重述

用户将记忆区定义为：
- 系统运行与验证确认后的正确记忆存储
- 参考GA（GenericAgent）记忆系统架构
- 与日记的区别：日记是当天实时事件过程的纪实（不一定正确），记忆是经过验证确认的正确知识
- 当前包含README.md、GA-L0-L4-analysis.md、implementation-guide.md
- 子文件夹：context/（上下文记忆）、daily/（日记式记忆，按年/月组织）
- 记忆是五层架构中L0-L4的核心实体

---

## 2. 理解要点

**记忆系统的本质**：它不是简单的"记住做过什么"，而是一个分层的、有严格质量标准的认知系统。记忆是"验证过的正确认知"，不是所有经历的原始记录。原始记录放在日记，验证后的知识放入记忆。

**五层记忆架构**（来自memory/README.md）：
- L0 硬编码核心 → system prompt中，每轮自动加载，不可修改
- L1 洞察索引 → memory/context/L1.md，不超过30行的精华索引
- L2 公理层 → memory/context/L2.md，经过验证的公理级知识
- L3 任务技能/SOP → sop/ 目录，可复用的操作技能
- L4 会话归档 → log/年/月/日/，历史会话记录

**三条公理**：
1. No Execution, No Memory — 没有执行就没有记忆，避免空想积累
2. 不可删除 — 记忆一旦写入不可删除，保留完整认知轨迹
3. 禁易变状态 — 记忆不存储临时性的、易变的状态信息

**四驱动&四写规则**：灵魂文档系统的记忆系统遵循"四驱动"原则 — 主系统通用区、主系统工作台、子系统通用区、子系统工作台，四个位置根据需要读写。双驱动（只在主系统或子系统内）时只需写两个位置。

**记忆与日记的分工**：日记是"发生了什么"的实时记录（纪实），记忆是"确认了什么"的知识沉淀（真理）。日记中的内容经过验证后，精华部分进入记忆。日记是记忆的原材料，记忆是日记的提炼产品。

**context/和daily/的区别**：context/存放结构化的上下文记忆（如L1洞察索引），daily/存放日记式的记忆记录（按时间组织）。前者是"索引"，后者是"流水账"。

**记忆的不可修改原则**：记忆写入后不可修改（禁易变状态）。如果发现记忆有误，不是修改旧记忆，而是写入新的正确记忆，旧记忆标记为"已修正"。这样保留了认知进化的完整轨迹。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 记忆和日记的边界 | 什么内容进日记，什么内容进记忆？ | 日记 = 实时纪实（含错误）；记忆 = 验证后的正确知识 |
| 记忆的写入时机 | 什么时候把日记内容提升为记忆？ | 经过验证确认后（No Execution, No Memory） |
| 记忆错误如何处理 | 发现旧记忆有误怎么办？ | 不删除，写入新记忆并标记旧记忆为"已修正" |
| 四驱动的具体操作 | 什么时候写几个驱动位置？ | 只在主系统内 = 双驱动；只在子系统内 = 双驱动；都涉及 = 四驱动 |
| memory/context/L1的30行限制 | 为什么限制30行？ | L1是索引不是仓库，保持精简才能快速加载 |

---

## 4. 我的判断

**使用记忆系统的标准动作**：
1. 任务执行 → 实时记录到diary/
2. 验证确认 → 精华内容写入memory/
3. 需要检索 → 先去对应驱动位置查找（四驱动）
4. 发现记忆有误 → 写入修正，不删除旧记忆
5. 对话开始 → 加载L0（system prompt），按需加载L1/L2

**记忆区与其他模块的关系**：
- memory/ → diary/ ：日记是记忆的原材料，记忆是日记的提炼
- memory/ → sop/ ：L3 = SOP，记忆与SOP共享技能层
- memory/ → experience/ ：记忆是"做过什么"，经验是"验证过什么"
- memory/ → log/ ：L4 = log，记忆系统包含会话归档
- memory/ → 过河桥/ ：过河桥是意图的记录，记忆是执行后的认知
