# 过河桥 — 空间区（Space）

> **此过河桥是空间模块的基因片段。**
> 空间 = 系统与外部平台的连接接口，是系统的"外部触点"和"虚拟身份层"。

---

## 1. 用户原话重述

用户将空间区定义为：
- 系统与外部平台的连接接口
- 是系统的"外部触点" — 系统通过空间区与外部世界交互
- 当前包含README.md、SKILL-空间初始化.md、空间初始化SOP.md
- 子文件夹结构：
  - 平台/ — 各类外部平台的连接配置（GitHub、学习平台、社交平台、视频平台、评论区、通用-待细分）
  - 账号/ — 各类账号的管理（模型虚拟账号空间、用户现实账号空间、虚拟账号空间）
- 空间初始化SOP定义了如何初始化空间系统

---

## 2. 理解要点

**空间系统的核心定位**：灵魂文档系统不是孤立的，它需要与外部世界交互 — 在GitHub上推送代码、在社交平台上发布内容、在学习平台上获取知识。空间区就是这些外部连接的统一管理入口。

**空间的两层结构**：
- 平台层 — 定义"连接到哪里"（GitHub、社交平台等）
- 账号层 — 定义"以什么身份连接"（模型虚拟账号、用户现实账号等）

**平台分类**：
- GitHub — 代码仓库平台
- 学习平台 — 知识获取平台
- 社交平台 — 社交互动平台
- 视频平台 — 视频内容平台
- 评论区 — 评论互动接口
- 通用-待细分 — 待细分的其他平台

**账号的三类管理**：
- 模型虚拟账号空间 — AI模型在外部平台的虚拟身份
- 用户现实账号空间 — 用户现实中的账号信息
- 虚拟账号空间 — 其他虚拟身份的账号

**空间初始化SOP**：SKILL-空间初始化.md和空间初始化SOP.md定义了如何从零开始搭建空间系统 — 包括平台配置、账号创建、连接验证等步骤。这是空间系统的"安装指南"。

**空间与安全的关系**：空间区涉及外部连接，必然涉及安全和隐私。账号信息（特别是用户现实账号）属于敏感信息，需要受系统规范约束，不应泄露到用户空间或公开区域。

**空间与production_space的关系**：生产空间是"生产内容的地方"，空间是"发布内容的地方"。生产空间产出内容，空间把内容发布到外部平台。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 空间是否包含所有外部连接 | 还是只包含已配置的平台？ | 以已配置平台为主，通用-待细分预留扩展 |
| 账号信息的敏感程度 | 模型虚拟账号和用户现实账号是否同等保护？ | 用户现实账号更高敏感度，受 stricter 保护 |
| 空间的初始化是否需要用户参与 | 还是模型可以自主完成？ | 平台配置模型可自主完成，账号创建需用户配合 |
| 空间是否支持多账号 | 一个平台是否可以有多个账号？ | 支持 — 账号管理设计为多账号结构 |

---

## 4. 我的判断

**使用空间系统的标准动作**：
1. 系统初始化 → 运行空间初始化SOP
2. 配置平台 → 在平台/下添加对应平台的连接配置
3. 管理账号 → 在账号/下维护对应平台的账号信息
4. 需要外部交互 → 通过空间区发起外部连接
5. 定期检查 → 验证连接有效性，更新过期配置

**空间区与其他模块的关系**：
- space/ → production_space/ ：生产空间产出内容，空间区发布内容
- space/ → wiki/ ：wiki中可能包含空间配置知识
- space/ → 团队空间/ ：团队空间中的协作可能涉及空间连接
- space/ → contacts/ ：通讯录可能引用空间中的联系人信息
- space/ → 智能体空间/ ：智能体可能有自己的空间身份
