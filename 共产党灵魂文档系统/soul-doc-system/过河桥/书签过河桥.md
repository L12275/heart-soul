# 过河桥 — 书签区（Bookmarks）

> **此过河桥是书签/收藏夹模块的基因片段。**
> 书签 = 系统内外部重要链接和资源的收藏夹，是知识网络的节点索引。

---

## 1. 用户原话重述

用户将书签区定义为：
- 存放系统内外部重要链接和资源的收藏夹
- 当前包含一个文件：收藏夹说明.md
- 书签是知识网络的节点索引 — 不是存储内容本身，而是存储"在哪里可以找到内容"
- 与wiki/的区别：wiki存储系统内生的知识内容，书签存储系统外部的链接资源

---

## 2. 理解要点

**书签区的核心定位**：它是系统与外部知识世界的接口。模型在互联网上发现有价值的资源时，不是把内容搬进来（那是wiki的事），而是在书签区留下一个带描述的索引，方便以后快速定位和访问。

**收藏夹说明.md的作用**：这个文件定义了书签的规范 — 如何添加书签、如何分类、如何描述、如何维护。它是书签区的元规则。

**书签的价值**：在长时间运行的系统中，模型会积累大量有价值的链接。如果没有系统化的管理，这些链接散落在各个日记和记忆中，无法有效检索。书签区把所有外部链接集中管理，形成系统的"外脑索引"。

**书签与记忆的关系**：记忆存储"知道了什么"，书签存储"在哪里可以知道更多"。两者互补 — 记忆是内部化的知识，书签是外部化的入口。

**书签与wiki的关系**：wiki把外部内容"消化"为系统内部知识，书签保留原始链接不做消化。wiki是"吃进去的"，书签是"夹在书里的"。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 书签是否需要分类体系 | 无结构地堆链接还是有分类？ | 收藏夹说明.md 定义了分类规范 |
| 书签是否同步内容 | 只存链接还是也存摘要？ | 只存链接+描述，不存完整内容（那是wiki的事） |
| 失效链接如何处理 | 链接失效后是删除还是标注？ | 标注失效，定期清理 |
| 书签是否公开 | 是模型私有还是可以共享？ | 当前为模型私有，后续可考虑共享机制 |

---

## 4. 我的判断

**使用书签区的标准动作**：
1. 发现有价值的外部链接 → 记录到书签区
2. 按收藏夹说明.md的规范添加描述和分类
3. 定期回顾 → 清理失效链接，更新分类
4. 需要深入某个主题时 → 先查书签区是否有相关入口

**书签区与其他模块的关系**：
- bookmarks/ → wiki/ ：书签是入口，wiki是消化后的内容
- bookmarks/ → memory/ ：书签补充外部知识，记忆存储内部确认
- bookmarks/ → diary/ ：日记中发现的好资源可以沉淀到书签
