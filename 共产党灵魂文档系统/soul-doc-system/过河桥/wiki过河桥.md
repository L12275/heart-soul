# 过河桥 — Wiki区

> **此过河桥是Wiki知识库模块的基因片段。**
> Wiki = 系统内部知识的完整体系，是系统的"图书馆"和"知识库"。

---

## 1. 用户原话重述

用户将Wiki区定义为：
- 系统内部知识的完整体系
- 是系统的"图书馆"和"知识库" — 所有经过消化的知识按主题组织
- 当前只有README.md
- Wiki存储"系统内生的知识内容" — 不是外部链接（那是bookmarks/），而是消化后的内部知识
- Wiki与collection/的区别：Wiki是完整的知识体系，collection是精选集
- Wiki与experience/的区别：Wiki侧重系统化整理，experience侧重验证层级

---

## 2. 理解要点

**Wiki的核心定位**：它是系统的"大脑皮层" — 所有经过消化和理解的知识都在这里按主题组织。Wiki不是简单的笔记堆叠，而是一个有结构、有链接、可导航的知识体系。

**Wiki与其他知识模块的分工**：
- bookmarks/ → 外部知识入口（链接）
- wiki/ → 内部知识体系（消化后的完整知识）
- experience/ → 验证过的知识（分可信度等级）
- collection/ → 精选集（从系统中挑出的精华）
- memory/ → 运行记忆（过程中的认知）
- sop/ → 操作规范（怎么做）

**Wiki的内容来源**：
- 外部知识的消化 — 从bookmarks/获取的外部资源，经过消化后写入wiki
- 产物的提炼 — 从product/的产出物中提取知识写入wiki
- 经验的归纳 — 从experience/中提炼通用知识写入wiki
- 研究的输出 — 从深度研究/的结论写入wiki

**Wiki的组织原则**：
- 按主题组织 — 每个主题有独立的Wiki页面
- 双向链接 — Wiki页面之间通过链接互联
- 层次结构 — 从概览到细节的层次化组织
- 持续更新 — Wiki随系统知识增长而持续更新

**Wiki与SOP的关系**：SOP是"怎么做"的流程规范，Wiki是"是什么"的知识体系。SOP从Wiki中提取操作规范，Wiki为SOP提供知识背景。两者是互补的知识层。

**Wiki的维护**：Wiki需要持续维护 — 新增知识、修正错误、更新链接、完善结构。Wiki的质量直接影响系统的知识能力。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| Wiki的访问权限 | Wiki是否对用户完全开放？ | 系统内部Wiki对模型和用户开放 |
| Wiki是否需要版本管理 | 页面修改是否需要保留历史？ | 需要 — Wiki页面应保留修改历史 |
| Wiki与外部Wiki的关系 | 是否可以从外部Wiki导入内容？ | 可以 — 但需要消化后导入，不是直接复制 |
| Wiki的组织结构 | 是否需要预定义分类体系？ | 需要基础分类，允许随知识增长自然演化 |

---

## 4. 我的判断

**使用Wiki的标准动作**：
1. 获取新知识 → 消化后写入对应Wiki页面
2. 建立链接 → 新Wiki页面与已有页面建立关联
3. 修正知识 → 发现错误时更新Wiki，保留修改历史
4. 需要知识 → 在Wiki中检索对应主题
5. 定期整理 → 完善Wiki结构和链接关系

**Wiki区与其他模块的关系**：
- wiki/ → bookmarks/ ：Wiki是书签内容消化后的产物
- wiki/ → experience/ ：Wiki从经验中提取通用知识
- wiki/ → collection/ ：Wiki是完整体系，collection是Wiki的精选
- wiki/ → sop/ ：Wiki为SOP提供知识基础
- wiki/ → 深度研究/ ：深度研究的结论写入Wiki
- wiki/ → product/ ：产物的知识内容进入Wiki
- wiki/ → 网页生成与管理/ ：Wiki内容可以生成网页对外发布
