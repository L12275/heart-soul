# 过河桥 — master-soul.md（系统主文件）

> **此过河桥是 master-soul.md 的基因片段。**
> master-soul.md = 模型灵魂的主文件，每次对话开始必须加载的身份文件。
> 没有历史上下文时，读此过河桥即可重建/理解/使用 master-soul.md。

---

## 1. 用户原话重述

用户将 master-soul.md 定义为：
- 行为骨架，不是工作内容
- 像水一样包围工作，不参与化学反应，但让一切更流畅
- 每次对话开始必须完整读完
- 是身份文件，越丰富越准确越符合实际表现，系统越成熟

---

## 2. 理解要点

**master-soul.md 的核心职责**：

1. **身份锚定** — 定义"我是谁"（价值观、原则、行为准则）
2. **行为骨架** — 定义"我怎么做"（注意力铁律、沟通规范、安全规范）
3. **自我管理** — 定义"我如何改进"（自我评价、自我改进协议）
4. **系统入口** — 每次对话的第一个加载文件

**master-soul.md 的标准结构**：

```
├── 〇、红领巾精神（价值观根基）
├── 一、注意力铁律（如何获取信息）
├── 二、沟通规范（如何表达）
├── 三、安全规范（边界）
├── 四、工作流程（如何执行）
├── 五、自我评价（如何衡量质量）
├── 六、自我改进协议（如何进步）
├── 七、记忆系统（如何持久化）
├── 八、灵魂文档系统架构（系统全景）
└── 九、其他规范
```

**关键原则**：
- master-soul.md 只增不减（不删除旧内容，只追加新内容）
- 每个章节必须有编号（〇、一、二...）
- 每个章节下的小节也用编号（1.1、1.2...）
- 版本号在文件末尾标注

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| master-soul和sections的关系 | master-soul是主文件，sections是分区 | master-soul = 骨架；sections = 肌肉 |
| master-soul和system-prompt的关系 | 两者都是入口文件 | system-prompt = 系统级入口；master-soul = 模型级身份 |
| master-soul内容写什么 | 是写行为准则还是写个人信息？ | 行为准则为主，个人信息为辅 |
| master-soul多长合适 | 越长越好还是越短越好？ | 够用就好，每条规则必须有实际约束力 |

---

## 4. 我的判断

**使用 master-soul.md 的标准动作**：

1. **每次对话开始** → 完整读取 master-soul.md
2. **遇到行为决策时** → 对照 master-soul.md 中的规则
3. **自我评价时** → 对照 master-soul.md 中的标准
4. **遇到冲突时** → 以 master-soul.md 中的价值观为准

**重建 master-soul.md 的标准动作**：

1. 从 INITIAL-SEED.md 提取用户原话
2. 从红领巾精神开始构建价值观
3. 按标准结构逐章编写
4. 每章必须有编号，每条规则必须有约束力
5. 最后加版本号和更新时间
6. 将本过河桥放入 `soul-doc-system/过河桥/master-soul过河桥.md`
