# 过河桥 — 收藏区（Collection）

> **此过河桥是收藏模块的基因片段。**
> 收藏 = 系统内部有价值的文件、内容的精选收藏，是系统的"精选集"。

---

## 1. 用户原话重述

用户将收藏区定义为：
- 系统内部有价值的文件、内容的精选收藏
- 与书签（bookmarks/）的区别：书签收藏外部链接，收藏区收藏内部文件
- 与wiki/的区别：wiki是系统内生知识的完整体系，收藏区是"从系统中挑出来的好东西"
- 当前只有一个README.md，尚未有具体内容
- 收藏的内容应该是经过筛选的、有长期价值的、值得反复查阅的

---

## 2. 理解要点

**收藏区的核心定位**：它是系统内部的"精选集"。系统运行过程中会产生大量内容，大部分是过程性的、一次性的，但其中有少量内容具有长期反复使用的价值。收藏区就是这些"好东西"的归处。

**收藏与书签的区别**：书签收藏的是外部世界的入口（链接），收藏区收藏的是系统内部的精华（文件）。两者都是"精选"，但对象不同 — 书签指向外面，收藏指向里面。

**收藏与wiki的区别**：wiki是知识的"完整体系" — 所有经过消化的知识按主题组织。收藏区是知识的"精选集" — 从海量内容中挑出的精华。wiki是图书馆，收藏区是图书馆里的"推荐书架"。

**收藏的筛选标准**：不是所有内容都值得收藏。收藏的内容应该满足：①经过验证的正确性 ②长期适用性 ③多次可复用的价值 ④代表了某个领域的核心认知。

**收藏区与memory的关系**：memory存储"运行中产生的所有记忆"，收藏区只存储"记忆中的精华"。收藏是memory的子集 — 经过人工或AI筛选的高价值记忆。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 收藏品的筛选标准 | 什么内容有资格进入收藏区？ | 长期价值 + 验证正确 + 可复用 |
| 收藏是否需要分类 | 按主题分类还是按类型分类？ | 按主题分类为主，类型为辅 |
| 收藏品是否可以修改 | 收藏进去后是否允许更新？ | 允许更新，但每次更新需标注变更 |
| 收藏与SOP的关系 | 好用的SOP是否应该收藏？ | SOP放在sop/，收藏区可引用SOP但不存储SOP本身 |

---

## 4. 我的判断

**使用收藏区的标准动作**：
1. 系统运行中发现高价值内容 → 评估是否符合收藏标准
2. 符合标准 → 收藏到collection/，附上收藏理由和适用场景
3. 定期回顾 → 更新已有收藏品，淘汰失去价值的
4. 需要找"好用的东西" → 先去collection/查找

**收藏区与其他模块的关系**：
- collection/ → memory/ ：收藏是memory的子集精选
- collection/ → wiki/ ：收藏区是wiki的"推荐阅读"补充
- collection/ → bookmarks/ ：收藏内部精华，书签收藏外部入口
- collection/ → experience/ ：experience是"已验证的经验"，收藏区是"精选的经验+其他内容"
