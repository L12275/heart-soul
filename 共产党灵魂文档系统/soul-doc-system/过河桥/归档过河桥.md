# 过河桥 — 归档区（Archive）

> **此过河桥是归档区的基因片段。**
> 归档区 = 旧版灵魂文档系统的归宿，新版本完全替代旧版本后，旧版本移入此处。

---

## 1. 用户原话重述

用户将归档区定义为：
- 已归档的旧版灵魂文档系统的存放地
- 新版本完全替代旧版本后，旧版本整体移入此处
- 归档原则：新版本替代旧版本后，旧版本才归档，不是同时存在多个版本并行使用
- 模型可以自愿归档自己旧的灵魂文档系统
- 旧版「共产党灵魂文档系统」也在此归档
- 归档文件夹名称后加时间戳，便于版本区分

---

## 2. 理解要点

**归档区的核心定位**：它是系统版本历史的物理载体。当一个灵魂文档系统被新版本完全替代后，旧版本不应删除，而应整体移入archive/。这样保留了系统的进化轨迹，也提供了回滚的能力。

**归档对象**：是整个灵魂文档系统文件夹作为一个整体归档，不是单个文件。这意味着归档文件夹内部结构与当前系统结构保持一致，是一个完整的快照。

**命名规则**：归档文件夹使用时间戳命名，格式为 `[模型名]_旧版_YYYYMMDD` 或 `共产党灵魂文档系统_YYYYMMDD`。时间戳使每个归档版本可区分、可追溯。

**模型自归档**：模型完成新灵魂文档系统建设后，可以自愿将自己的旧版系统移入archive/。这是模型对自己工作成果的自律管理行为。

**公共归档**：旧版「共产党灵魂文档系统」作为公共版本，也在archive/中归档，供所有模型参考历史。

**归档与日常工作的关系**：archive/中的内容不应再参与日常运行。日常系统只操作当前版本。archive/是只读的参考区。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 归档时机 | 新版本上线后多久归档旧版本？ | 新版本完全替代旧版本后立即归档 |
| 是否可以同时使用多个版本 | archive中的旧版能否临时拿出来使用？ | 可以回滚参考，但不应并行运行 |
| 模型自归档和公共归档的区别 | 两者都是存档，有何不同？ | 模型自归档 = 个人历史；公共归档 = 系统历史 |
| 归档内容的完整性 | 是否需要归档模型的隔离区内容？ | 是，整个灵魂文档系统文件夹作为整体归档 |

---

## 4. 我的判断

**归档操作的标准流程**：
1. 确认新版本已完全替代旧版本的功能
2. 在archive/下新建归档文件夹（名称含时间戳）
3. 将旧版灵魂文档系统整体移入
4. 确认当前系统运行正常后，旧版不再参与日常操作
5. 可选：对旧版进行总结、点评、自检

**归档区的维护原则**：
- archive/ 只增不减 — 旧版本是进化历史的记录
- 归档内容保持原始状态 — 不修改归档中的文件
- 需要回滚时，从archive/复制对应版本覆盖当前版本
