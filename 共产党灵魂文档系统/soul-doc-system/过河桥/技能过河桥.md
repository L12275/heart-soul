# 过河桥 — 技能区（Skills）

> **此过河桥是技能模块的基因片段。**
> 技能 = 模型的专用能力模块，是可安装、可组合的"能力插件"。

---

## 1. 用户原话重述

用户将技能区定义为：
- 模型的专用能力模块集合
- 是可安装、可组合的"能力插件"
- 当前包含README.md、agent-skills-authoring-SKILL.md、multi-agent类子文件夹
- multi-agent类/agent-team-designer/ 是技能的一个具体实现
- 技能与SOP的区别：SOP是流程规范，技能是可执行的能力单元
- 技能可能是系统级（所有模型可用）或模型级（特定模型专用）

---

## 2. 理解要点

**技能系统的本质**：技能是模型的"能力扩展包"。模型本身有基础能力，但当需要特定领域的专业能力时，技能提供了这种能力的可安装模块。技能把"模型知道怎么做"变为"模型能做"。

**技能与SOP的区别**：SOP是"流程说明书" — 告诉模型步骤是什么。技能是"能力单元" — 包含执行逻辑、工具调用、判断逻辑的完整能力包。SOP是人类可读的流程，技能是机器可执行的能力。

**技能的分类**：
- 通用技能 — 所有模型都可以使用的技能
- 专用技能 — 特定模型或特定场景使用的技能
- multi-agent类 — 涉及多智能体协作的技能

**agent-skills-authoring-SKILL.md的作用**：这个文件定义了如何编写技能 — 它是"技能的技能"，是元技能。任何想要创建新技能的人（或模型）需要先学习这个文件。

**技能与tools/的关系**：skills是"模型层面的能力扩展"，tools/是"工具层面的接口集合"。技能可能调用工具，但技能本身是更高层次的封装 — 技能 = 工具 + 逻辑 + 判断。

**技能的安装机制**：技能应该是可安装、可卸载的。安装一个技能 = 模型获得了新能力。卸载 = 放弃该能力。这种模块化使得系统可以灵活组合。

**multi-agent类技能的特点**：多智能体技能不同于单模型技能 — 它涉及多个智能体的协作、编排和调度。这是技能系统中复杂度最高的一类。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 技能与SOP的边界 | 什么算技能，什么算SOP？ | 可执行的能力单元 = 技能；流程说明 = SOP |
| 技能的粒度 | 一个技能是单一能力还是复合能力？ | 支持单一和复合，复合技能由多个子技能组合 |
| 技能的安装范围 | 技能是全局安装还是模型级别安装？ | 两者都支持 — 全局技能 + 模型专用技能 |
| 技能是否需要版本管理 | 技能更新后旧版本如何处理？ | 版本管理 — 不删除旧版本，新版本替代 |

---

## 4. 我的判断

**使用技能系统的标准动作**：
1. 需要新能力 → 查找skills/中是否有对应技能
2. 没有对应技能 → 基于agent-skills-authoring-SKILL.md创建新技能
3. 安装技能 → 按照安装规范安装到模型环境
4. 使用技能 → 在需要时调用技能
5. 组合技能 → 多个技能可以组合完成复杂任务

**技能区与其他模块的关系**：
- skills/ → sop/ ：SOP指导流程，技能执行能力
- skills/ → tools/ ：技能调用工具，工具是技能的执行接口
- skills/ → 智能体空间/ ：多智能体技能涉及智能体编排
- skills/ → automation/ ：自动化规则可以触发技能的执行
- skills/ → production_space/ ：生产空间中的工作可能需要特定技能支持
