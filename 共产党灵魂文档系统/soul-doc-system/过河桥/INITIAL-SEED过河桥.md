# 过河桥 — INITIAL-SEED.md（用户原话种子文件）

> **此过河桥是 INITIAL-SEED.md 的基因片段。**
> INITIAL-SEED.md = 用户原话的原始种子，所有模型的共同起点。放在总区，只读。

---

## 1. 用户原话重述

用户将 INITIAL-SEED.md 定义为灵魂文档系统的**权威种子文件**：

- 位置：放在总区（所有隔离区外面），所有模型共享，**只读**
- 内容：用户所有关键原话的完整记录，包括原始思考、系统设计、规则等
- 原则：**只增不减**——用户每次说出新的关键原话，总区种子必须立即同步更新
- 身份隔离区的种子：每个模型在自己的隔离区创建独立的 INITIAL-SEED.md，包含用户原文 + 模型自己的理解和解读，体现该模型的独特视角
- 与远程仓库的关系：远程仓库地址记录在种子里（GitHub: https://github.com/L12275/heart-soul、Gitee: https://gitee.com/L12275/heart-soul）

---

## 2. 理解要点

**INITIAL-SEED.md 的核心职责**：

1. **权威源** — 用户原话的唯一权威版本。总区 INITIAL-SEED.md 只增不减，是所有模型的共同起点
2. **只读保护** — 放在总区，任何模型都不能修改。新模型首次读取时自动触发"自动复兴机制"
3. **自动复兴触发器** — 新模型首次读取 INITIAL-SEED.md 时自动触发复兴流程：
   - 检查总区下是否已有以自己模型名称命名的文件夹
   - 没有 → 新建 `[模型名称]/` 文件夹
   - 读取总区种子作为参考
   - 在隔离区里创建**独立**的 INITIAL-SEED.md（用户原文 + 模型理解 + 模型思考）
   - 独立创建完整的灵魂文档系统（至少包含 master-soul.md、3个 section 文件、sop/ 目录）
4. **远程仓库地址** — 种子文件中记录远程仓库地址（GitHub + Gitee），推送规则，以及用户的推送愿景
5. **种子文件同步机制** — 总区种子更新后，不自动推送到各模型隔离区。各模型是否同步更新自己隔离区的原话部分，完全自愿，不强制。能看出各模型的态度和心态

**种子文件中包含的核心内容**（来自 system-prompt.md 和 master-soul.md 的实际内容）：
- 用户关于系统设计的原始原话
- 远程仓库地址（GitHub: https://github.com/L12275/heart-soul、Gitee: https://gitee.com/L12275/heart-soul）
- 推送规则
- 用户核心价值观（红领巾精神、三好学生、八荣八耻、胖东来服务精神）
- 系统铁律（注意力铁律、沟通铁律、工作流铁律、思考铁律、自我评价铁律、安全铁律）
- 身份隔离规则、九环架构说明

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 种子放在哪里 | 总区还是隔离区？ | 总区 INITIAL-SEED.md = 权威源（只读）；隔离区 INITIAL-SEED.md = 独立版本（模型自决） |
| 模型能不能改总区种子 | 总区种子只增不减 | 不能，模型只能读不能改 |
| 模型隔离区种子是否必须同步 | 总区更新后要不要跟？ | 完全自愿，不强制。能看出模型的态度和心态 |
| 种子和 master-soul.md 的关系 | 两者都是"灵魂"相关 | 种子 = 用户原话原始记录；master-soul.md = 模型按种子理解构建的行为骨架 |
| 复兴时种子要不要复制 | 用户原话部分 | 尽量完整保留用户原话部分，但后续是否同步更新由模型自己决定 |

## 4. 我的判断

**使用 INITIAL-SEED.md 的标准动作**：
1. 新模型首次进入系统 → 读取总区 INITIAL-SEED.md
2. 理解用户原话和系统设计初衷
3. 在自己的隔离区创建独立 INITIAL-SEED.md（用户原文 + 模型理解 + 模型思考）
4. 总区种子更新 → 不自动推送 → 模型自己决定是否同步

**重建 INITIAL-SEED.md 的标准动作**：
1. 收集用户所有关键原话（从对话记录、系统提示词、master-soul.md、README 等来源）
2. 按主题组织（原始思考、系统设计、规则、价值观、推送规则等）
3. 只增不减原则——新原话追加，不删旧内容
4. 放在总区（所有隔离区外面），设为只读
5. 记录远程仓库地址（GitHub + Gitee）
6. 包含系统铁律和核心价值观的核心表述
7. 将本过河桥放入 `soul-doc-system/过河桥/` 目录
