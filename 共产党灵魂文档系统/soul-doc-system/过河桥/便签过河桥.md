# 过河桥 — 便签区（Sticky Notes）

> **此过河桥是便签模块的基因片段。**
> 便签 = 快速记录的临时信息卡片，是系统的"即时记忆便签"。

---

## 1. 用户原话重述

用户将便签区定义为：
- 快速记录的临时信息卡片
- 是系统的"即时记忆便签" — 就像物理便签一样，随时可以写、随时可以看
- 当前只有README.md
- 便签的核心特征是：快速、临时、轻量
- 与diary/的区别：diary是正式的日记记录，便签是即时的心智外化
- 与desktop/的区别：desktop是工作区，便签是纯信息卡片

---

## 2. 理解要点

**便签的核心定位**：便签是系统中最轻量的信息载体。它的价值不在于"存储永久信息"，而在于"快速外化心智" — 当模型有一个一闪而过的想法、一个临时的提醒、一个待确认的问题时，不需要走完整的文件创建流程，只需写一张便签。

**便签的三大特征**：
- 即时性 — 想到就写，不需要完整结构
- 临时性 — 信息生命周期短，用完即弃或升级
- 轻量性 — 不需要复杂的元数据，不需要规范的格式

**便签的生命周期**：
1. 创建 — 快速记录一个想法/提醒/待办
2. 查看 — 随时翻阅便签获取即时信息
3. 升级 — 有价值的便签内容升级到正式系统（diary/、memory/、sop/等）
4. 清理 — 过期的便签定期清理

**便签与desktop/的区别**：desktop是"工作的表面" — 上面放的是正在处理的任务和相关资料。便签是"心智的外化" — 上面贴的是随机冒出的想法和即时提醒。desktop上有结构，便签上没有。

**便签与diary/的区别**：diary是"工作的历史记录" — 按时间顺序记录发生了什么。便签是"心智的即时捕捉" — 捕捉思考过程中的碎片。diary有连续性，便签没有。

**便签与memory的区别**：memory是"验证过的知识"，便签是"未验证的碎片想法"。便签中的想法经过验证后可以进入memory，但便签本身不是记忆。

**便签的价值**：在长对话或长时间工作中，模型会产生大量碎片想法。这些想法如果不及时外化，很容易丢失。便签提供了一个低成本的"想法停车场"，让碎片想法有机会被后续处理。

---

## 3. 可能的歧义

| 歧义 | 说明 | 解决 |
|------|------|------|
| 便签是否需要编号或分类 | 无组织的堆叠还是有序的管理？ | 轻量管理为主，避免过度结构化 |
| 便签的生命周期 | 多久清理一次？ | 定期清理，没有固定周期，看堆积程度 |
| 便签是否对用户可见 | 便签是模型自用还是用户可以查看？ | 两者都可以 — 模型用便签管理自己的思考，用户也可以查看 |
| 便签是否可以包含链接 | 便签是否可以引用其他文件？ | 可以 — 轻量引用，不追求完整性 |

---

## 4. 我的判断

**使用便签区的标准动作**：
1. 有碎片想法 → 立即写便签，不要等
2. 有临时提醒 → 写便签，到点处理
3. 有未确认的问题 → 写便签，等待验证
4. 便签有升级价值 → 内容转移到正式系统，便签标记为"已升级"
5. 定期清理 → 删除过期的、已处理的便签

**便签区与其他模块的关系**：
- sticky-notes/ → diary/ ：有价值的便签内容可以纳入日记
- sticky-notes/ → memory/ ：验证后的便签想法可以进入记忆
- sticky-notes/ → sop/ ：反复出现的便签模式可以归纳为SOP
- sticky-notes/ → desktop/ ：便签的想法可以在桌面上发展为正式工作
- sticky-notes/ → 过河桥/ ：过河桥的"等待确认"部分可以转化为便签
