# 临时文件夹清理规则

> 临时文件 = 因工作台活动而下载/临时获取的文件，尚未确定最终去向。

---

## 目录说明

| 字段 | 说明 |
|------|------|
| 文件名 | 原始文件名 |
| 来源 | 从哪里获取的（URL、来源路径等） |
| 用途 | 为什么获取这个文件，用于什么任务 |
| 关联工作 | 关联的工作台事件/日记条目 |
| 创建日期 | 何时放入临时文件夹 |
| 最后使用日期 | 最后一次打开/引用/修改的时间 |
| 状态 | 待处理 / 已归档 / 可删除 |

---

## 清理规则

### 自动删除条件（满足以下任意一条）

1. **超时未使用** — 超过 30 天没有使用（最后使用日期 > 30 天前）→ 自动删除
2. **来源已替代** — 已有更完整/更新的版本 → 删除旧版本
3. **原始来源可复得** — 文件可从原始来源重新下载 → 可删除本地备份

### 需确认后删除

4. **用途已完成** — 相关工作已结束，文件不再需要 → 标记为可删除，等待确认
5. **重复文件** — 已存在功能等价的其他文件 → 标记为可删除

### 永不移除

6. **原始创作** — 在此工作台创作的文件（代码、文章、设计等）→ 应归档到 product/ 或 wiki/，不放在 temp/

---

## 操作流程

```
文件放入 temp/
    ↓
创建条目：记录来源、用途、日期、状态
    ↓
每天检查：超时未使用的 → 自动删除
    ↓
每周检查：用途已完成的 → 提示用户确认
    ↓
最终处理：
  - 需要的 → 移入 product/ 或 wiki/
  - 不需要的 → 删除
  - 待定的 → 保持标签，下次检查
```
