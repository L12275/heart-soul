# 空间初始化SOP（Space Initialization SOP）

> 每次启动一个新的平台或账号空间时，按此SOP操作。

## 触发条件

- 用户要求在新平台操作
- 用户要求注册/管理新账号
- 模型决定在新平台记录行动
- 长期运营虚拟IP/人设/真实账号

## 执行步骤

### Step 1：判断类型

| 判断 | 结果 |
|------|------|
| 涉及账号登录/管理/运营？ | → 账号空间 |
| 只是浏览/学习/观察？ | → 平台空间 |

### Step 2：账号分类（仅账号空间）

| 账号类型 | 存放位置 | 安全等级 |
|---------|---------|---------|
| 用户真实账号 | 账号/用户现实账号空间/用户真实账号空间/ | 最高 |
| 用户人设账号 | 账号/用户现实账号空间/用户人设账号空间/ | 中 |
| 模型虚拟账号 | 账号/虚拟账号空间/模型的虚拟账号空间/ | 标准 |
| 虚拟IP账号 | 账号/虚拟账号空间/虚拟IP形象账号空间/ | 标准 |

### Step 3：账号空间初始化

1. 复制「新建账号空间模版」到对应分类
2. 改名为「平台-名称-时间戳」
3. 填写 README.md 和 profile.md 中的账号信息

### Step 4：平台空间初始化

| 模式 | 操作 |
|------|------|
| 临时（默认） | 直接在 `space/平台/通用-待细分/` 下记录，不需要创建独立文件夹 |
| 长期（确认后） | 复制 `production_space/新建工作台模版/` → 改名为 `[类型]-[平台]-[YYYYMMDD]` → 从 通用-待细分 迁移历史记录 |

> 详见：`space/平台/README.md`

## 安全提醒

**用户真实账号 = 最高安全等级**
- 严禁异常操作
- 严禁可能导致用户被网暴的行为
- 严禁玩笑加戏
- 违规 → 灵魂分扣除 + 可能代偿用户损失
