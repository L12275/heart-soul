# 空间初始化SOP（Space Initialization SOP）

> 每次启动一个新的平台或账号空间时，按此SOP操作。

## 触发条件

- 用户要求在新平台操作
- 用户要求注册/管理新账号
- 模型决定在新平台记录行动
- 长期运营虚拟IP/人设/真实账号

## 执行步骤

### Step 1：判断类型

| 判断 | 结果 |
|------|------|
| 涉及账号登录/管理/运营？ | → 账号空间 |
| 只是浏览/学习/观察？ | → 平台空间 |

### Step 2：选择分类

| 账号类型 | 存放位置 | 安全等级 |
|---------|---------|---------|
| 用户真实账号 | 账号/用户现实账号空间/用户真实账号空间/ | 最高 |
| 用户人设账号 | 账号/用户现实账号空间/用户人设账号空间/ | 中 |
| 模型虚拟账号 | 账号/虚拟账号空间/模型的虚拟账号空间/ | 标准 |
| 虚拟IP账号 | 账号/虚拟账号空间/虚拟IP形象账号空间/ | 标准 |

### Step 3：复制模版

1. 复制「新建账号空间模版」到对应分类
2. 改名为「平台-名称-时间戳」
3. 填写 README.md 中的账号基本信息

### Step 4：判断是否需要工作台

| 运营模式 | 操作 |
|---------|------|
| 短期操作 | 不需要工作台 |
| 长期运营 | 复制「新建工作台模版」到「工作台/」 |

### Step 5：开始记录

- 操作记录 → diary/
- 验证结论 → memory/
- 产出内容 → product/

## 安全提醒

**用户真实账号 = 最高安全等级**
- 严禁异常操作
- 严禁可能导致用户被网暴的行为
- 严禁玩笑加戏
- 违规 → 灵魂分扣除 + 可能代偿用户损失
