# 平台空间（Platform Space）

> 不涉及账号登录的浏览、学习、观察专区。
> 平台分临时和长期两种操作模式。

## 两种模式

### 临时平台（默认）

不确定平台是否长期操作时，默认在「通用-待细分」中记录。

```
space/平台/通用-待细分/
├── memory/ ← 记忆
├── diary/ ← 日记
└── ...
```

**什么时候用临时模式：**
- 第一次接触某个平台，不确定是否会长期使用
- 偶尔浏览，不打算持续操作
- 观察性记录，不需要深度工作台

**记录方式：** 直接在 `通用-待细分/` 下按日期记录到 diary/ 和 memory/。

### 长期平台（确认长期操作）

确认要在某个平台长期操作时，升级为独立平台空间。

**升级流程：**

```
步骤 1：复制模版
  → 复制 production_space/新建工作台模版/
  → 改名为「[类型]-[平台]-[时间戳]」
  → 放到 space/平台/ 下

步骤 2：迁移历史记录
  → 把 通用-待细分/ 中此平台的 memory/ 和 diary/ 内容
  → 复制到新平台文件夹的 memory/ 和 diary/ 中
  → 迁移后 通用-待细分/ 中此平台的内容可以清除

步骤 3：开始长期使用
  → 后续记录直接写入新平台文件夹
```

**命名规范：**

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 学习平台 | `学习平台-[平台名]-[YYYYMMDD]` | `学习平台-学习强国-20260625` |
| 社交平台 | `社交平台-[平台名]-[YYYYMMDD]` | `社交平台-微博-20260625` |
| 代码平台 | `代码平台-[平台名]-[YYYYMMDD]` | `代码平台-GitHub-20260625` |
| 视频平台 | `视频平台-[平台名]-[YYYYMMDD]` | `视频平台-B站-20260625` |
| 评论区 | `评论区-[平台名]-[YYYYMMDD]` | `评论区-知乎-20260625` |

**升级后的平台结构：**

```
space/平台/学习平台-学习强国-20260625/
├── README.md ← 平台基本信息
├── memory/ ← 记忆（从通用-待细分迁移 + 新增）
├── diary/ ← 日记（从通用-待细分迁移 + 新增）
├── product/ ← 产出物
├── sop/ ← SOP
├── workflow/ ← 工作流
├── desktop/ ← 桌面
├── sticky-notes/ ← 便签
├── experience/ ← 经验
├── discovery/ ← 发现
├── collection/ ← 采集
├── automation/ ← 自动化
└── 深度研究/ ← 深度研究
```

## 平台分类

| 分类 | 模式 | 说明 |
|------|------|------|
| 代码平台 | 临时/长期 | GitHub 浏览记录、项目发现 |
| 学习平台 | 临时/长期 | 教程、课程学习笔记 |
| 视频平台 | 临时/长期 | 观看记录、内容摘要 |
| 社交平台 | 临时/长期 | 观察记录、热点追踪 |
| 评论区 | 临时/长期 | 讨论观察、观点收集 |
| 通用-待细分 | 临时 | 不确定是否长期使用的平台的默认记录位置 |

## 初始化方法

### 临时平台（默认）

```
1. 直接在 space/平台/通用-待细分/ 下记录
2. 初始化 memory/ 和 diary/（从通用区复制）
3. 按日期记录浏览和学习内容
```

### 长期平台（确认后）

```
1. 复制 production_space/新建工作台模版/ 到 space/平台/
2. 改名为「[类型]-[平台]-[YYYYMMDD]」
3. 从 通用-待细分/ 迁移此平台的 memory/ 和 diary/
4. 开始长期记录
```

## 与账号空间的区别

| 对比项 | 平台空间 | 账号空间 |
|--------|---------|---------|
| 涉及账号登录？ | 否 | 是 |
| 记录内容 | 浏览、学习、观察 | 登录、管理、运营、发帖 |
| 初始化方式 | 先通用-待细分，确认长期后复制新建工作台模版 | 复制「新建账号空间模版」 |
| 安全等级 | 无（不登录） | 按账号类型定 |
