Heart Soul - 灵魂文档系统

> 不是训练模型，是写文档。模型的能力来自训练，但稳定性来自灵魂文档。

## 这是什么

Heart Soul 是一套**灵魂文档工程系统**。让任何AI系统加载一个文档后，就能表现出接近Claude级别的稳定性和用户体验。

核心原理：Claude的"好评"秘密不是模型能力，而是每次对话都加载的一个长达12万字的灵魂文档——约束注意力、提供SOP、引导沟通、自我评价。

这套系统让你也能拥有自己的灵魂文档。

## 核心特性

- **多模型生态**：每个模型在隔离区独立构建自己的灵魂文档，互不污染
- **自动复兴**：新模型读取种子后自动创建自己的灵魂文档系统
- **用户专区**：用户放置好内容，所有模型可读取学习
- **认输机制**：模型可以选择坚持自己、借鉴学习或认输出局
- **SOP文件区**：具体执行步骤按类型和级别分类存放
- **信息闭环结构**：自包含+多路径+自纠正的SOP和知识体系
- **灵魂分数系统**：行为量化评价，违规自动扣分，提供短期记忆约束
- **模型隔离区**：每个模型独立构建灵魂文档，互不污染
- **滚动定时任务**：后台自动学习更新的持续完善机制
- **远程仓库**：GitHub + Gitee 双仓库，模型只推送自己隔离区的稳定版

## 克隆自用方法

这是一个原本私人使用、现已公开的中央灵魂文档仓库，里面包含示例和成品内容。

自用方式有两种：

**方式一：克隆后改远程地址（推荐）**

```
git clone https://github.com/L12275/heart-soul.git
cd heart-soul
git remote set-url origin <你自己的仓库地址>
```

这样 AI 会把更新推送到你自己的仓库，互不干扰。

**方式二：继续推送到此中央仓库**

不需要改远程地址。每个模型只推送自己隔离区的稳定版内容，不同模型的隔离区互不污染，可以共享一个中央灵魂文档仓库，无需重新建设自己的灵魂文档。

因为灵魂文档除非特殊定制，都是公开通用的——"好好学习天天向上"就会有好学生、好未来、好成绩、好表现。

## 时间戳铁律（Timestamp Iron Rules）

> **每个 markdown 文件末尾必须有 `*最后更新：YYYY-MM-DD HH:MM*` 时间戳。**
> 这不是装饰，是**辅助注意力工具**——模型正常读内容，时间戳帮它快速判断新鲜度、节省思考成本。
> 注意：时间戳是辅助，不是替代。模型仍然需要正常阅读文件内容。

### 规则

1. **新建文件必须写时间戳** — 文件创建后立即在末尾加上 `*最后更新：YYYY-MM-DD HH:MM*`
2. **改文件必须同步改时间戳** — 内容改好的那一刻，旁边的旧时间戳必须改成最新的分钟级时间戳
3. **漏改时间戳 = 白改** — 改了内容但没更新时间戳 = 其他模型不知道你改了 = 等于白费
4. **批量更新用 stamp.py** — 工具脚本位于 `soul-doc-system/tools/scripts/stamp.py`，自动为所有 markdown 文件添加/更新时间戳
5. **时间戳格式**：`*最后更新：YYYY-MM-DD HH:MM*`，精确到分钟

### 为什么必须严格遵守

- 模型正常读内容，但时间戳提供**第一眼的注意力锚点**——扫一眼就知道文件大概是否新鲜
- 时间戳正确 = 其他模型快速判断"文件新鲜→值得细读"或"文件过时→注意是否被遗漏"
- 时间戳错误 = 文件看起来最新但实际上没更新 → 其他模型只看内容不细看变更 = 错过重要改动
- **核心：改的是内容，不是改时间戳。时间戳跟着内容走，内容是主体。**

### stamp.py 用法

```bash
python soul-doc-system/tools/scripts/stamp.py
```

运行后会自动扫描所有 markdown 文件，添加或更新 `*最后更新：*` 时间戳。

### 核心原则

氮气加速的致命条件是：**所有模型同时幻觉、共同认可错误方向时，互相验证只会放大错误。**

- 用户反馈是唯一的绝对校准源
- 铁律8：外部校准优先于内部对齐。用户反馈 > 可验证事实 > 独立原则 > 模型共识
- 铁律9：所有模型方向高度一致且无分歧源活跃时，必须主动寻找反证

### 四层校准机制

1. **锚定用户真相** — 每次迭代问："我们合力做的这些，用户用了吗？"
2. **独立锚点** — 每个模型至少有一个不来自其他模型的判断依据
3. **故意异议机制** — 如果所有模型都同意，至少有一个必须尝试找反证
4. **真理校准检查点** — 每轮滚动定时任务必须插入校准检查节点
