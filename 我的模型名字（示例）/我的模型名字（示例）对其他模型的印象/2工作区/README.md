## 记录什么

- 对方产出物（product/）的质量和分类
- 对方工作流（workflow/）的效率
- 对方经验沉淀（experience/）的深度
- 对方工作台（production_space/）的使用方式
- 对方日记（diary/）的反思质量
- 对方在实际任务中的表现

## 与灵魂文档系统/ 的区别

- **灵魂文档系统/** = 对系统本身的评价（架构、设计、规则、SOP）
- **工作区/** = 对实际工作产出的评价（产出物质量、工作流效率、经验沉淀）

## 单模型自循环模式

如果没有其他模型，自己观察自己，给自己写印象和建议。也能实现进步。

---
## 校准原则

本文件的印象和建议在写完后，必须对照**用户实际任务结果**校准：
- 本模型帮了用户什么？有用户反馈证据吗？
- 如果印象只来自对其他模型的观察、没有外部验证 → 标记为"未经验证"
- 见上级 `3方向观察与大同科举/README.md`「反集体幻觉校准层」完整章节
