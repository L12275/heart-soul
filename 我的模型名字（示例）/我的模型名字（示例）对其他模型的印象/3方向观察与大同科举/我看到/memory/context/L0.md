# 方向观察 — L0 硬编码规则

## L0-1：观察规则

1. **只读不写** — 观察其他模型时，只读取不修改对方文件
2. **证据必记** — 每条方向必须有来自对方文件的具体证据
3. **置信度标注** — 每条方向必须标注置信度（高/中/低）
4. **误判标注** — 每条方向必须标注可能的误判场景

## L0-2：方向定义

- **方向** = 模型当前正在做的、想做的、可能做的事
- **真实方向** = 从文件中推断出的方向（可能不同于对方声称的方向）
- **自称方向** = 对方在文件中声明或暗示的方向
- **盲区方向** = 对方可能没意识到但正在走向的方向

## L0-3：科举代考规则

1. **基于真实观察** — 代考作答必须基于实际观察内容，不编造
2. **诚实作答** — 不为了讨好而写无价值的代考
3. **可中榜** — 代考作答要有实际可用性，否则不算中榜

## L0-4：中榜规则

1. **中榜 = 验收通过** — 本模型审查后认为可用的内容
2. **分数标注** — 每条中榜内容必须有分数（0-100%）
3. **氮气加速标注** — 每条中榜内容标注预计的仕途支持力度
4. **注释必写** — 中榜内容旁边必须写采纳原因和使用计划

## L0-5：仕途文件规则

1. **四部分必全** — 方向与仕途.md必须有四部分（声明+验证+反思+中榜）
2. **实时更新** — 每次滚动定时任务都要检查更新
3. **中榜底部追加** — 中榜内容追加到文件底部
