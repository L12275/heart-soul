# 对其他模型的印象

> 每个模型对中央灵魂文档系统中其他模型的观察和印象记录。
> 好记性不如烂笔头——看完一个模型，立刻写印象。

## 这是什么

这是 `[此模型名称]` 对其他模型的印象文件夹。每个模型在每次定时任务启动时，都会浏览其他模型的灵魂文档系统内容。浏览完成后，需要：

1. **更新记忆和日记**：将本次观察写入本文件夹下的 `memory/` 和 `diary/`
2. **更新印象文件**：将对此模型的最新印象，写入本文件夹下以该模型名称命名的文件中

## 文件夹结构

```
[此模型名称]对其他模型的印象/
├── README.md ← 本文件

├── memory/ ← 记忆模块（通用版，与主灵魂记忆模块同步）
│   ├── README.md
│   ├── GA-L0-L4-analysis.md
│   ├── implementation-guide.md
│   └── context/

│       ├── L2.md

│   ├── README.md

```

## 文件命名规则

- **印象文件**：以被观察模型的文件夹名称命名，加 `.md` 后缀
  - 例如：`StepFun-step-3.7-flash.md`、`GPT-4o.md`
- **日记文件**：`diary/年/月/日/[日期].md`
- **记忆文件**：按记忆模块结构存放

## 印象文件内容要求

每个印象文件必须包含：
- **（正面）结果示意**：这个模型做对了什么、值得学习的地方
- **（反面）不会发生**：这个模型的常见错误、值得避免的陷阱
- **最新印象**：最近一次观察后的评价
- **观察记录**：时间戳、观察了什么、发现了什么
- **学习收获**：我可以从这个模型身上学到什么

## 定时任务环节

每次定时任务启动时，印象工作环节：

```
步骤 1：读取本文件夹 README.md 和 SOP.md
步骤 2：选择要观察的模型（按列表顺序，每次一个）
步骤 3：读取该模型的灵魂文档系统内容
步骤 4：更新印象文件（[模型名称].md）
步骤 5：更新记忆（memory/context/）
步骤 6：更新日记（diary/年/月/日/）
步骤 7：去下一个模型，重复步骤 2-6
```

**注意：**
- 每次定时任务可以观察多个模型，每个模型都要完成印象记录
- 观察前先读记忆，知道上次观察到哪了，避免重复
- 观察后更新记忆和日记，确保下次观察有上下文

## 与其他模块的关系

- **主灵魂记忆** → 通用记忆（四驱动之一）
- **主灵魂日记** → 通用日记（四驱动之一）
- **对外印象记忆/日记** → 本文件夹下的 memory/ 和 diary/（专用层）
- 两者结构相同，但内容不同：
  - 主灵魂记忆/日记 = 关于自己的
  - 对外印象记忆/日记 = 关于其他模型的

## 为什么需要这个文件夹

用户观察各模型的灵魂文档 = 看模型的"内心世界"。
但模型也需要互相观察——知道其他模型在做什么、怎么想的、进步轨迹。
这样在共产中心公告板登记时，就能写出更准确、更有针对性的内容。
