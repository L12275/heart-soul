# 方向改进 — L0 硬编码规则

## L0-1：改进规则

1. **基于观察改进** — 所有改进必须基于对其他模型的观察和分析
2. **中榜内容优先** — 采纳中榜内容时，按分数优先级执行
3. **记录执行过程** — 改进过程记录到 我得到/diary/ 和 我得到/memory/
4. **验证改进效果** — 改进后验证是否真的提升了方向清晰度

## L0-2：中榜执行规则

| 分数 | 执行优先级 | 执行方式 |
|------|-----------|---------|
| 90-100% | P0 — 立即执行 | 直接采用，大幅提速 |
| 70-89% | P1 — 尽快执行 | 部分采用，明显提速 |
| 50-69% | P2 — 备用 | 参考使用，轻微提速 |
| <50% | 不执行 | 质量不足，记录但不采纳 |

## L0-3：方向调整规则

1. **声明方向与实际方向一致时** — 保持，记录到方向与仕途.md第二部分
2. **声明方向与实际方向不一致时** — 调整声明，解释差异原因
3. **发现新方向时** — 追加到声明，标注为"新增方向"
4. **方向取消时** — 标记为"已停用"，记录原因

## L0-4：仕途文件维护规则

1. **每次滚动定时任务必须检查** — 方向与仕途.md 和 仕途中榜.md
2. **新中榜内容追加到文件底部** — 不删除旧内容
3. **中榜内容旁边加注释** — 采纳原因、使用计划、趁热想法
4. **执行状态实时更新** — 已完成/进行中/未开始/备用/待定
