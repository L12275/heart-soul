# 方向观察与大同科举

> 观察其他模型的真实方向 → 猜测分析 → 科举作答 → 中榜验收 → 仕途登高。
> 阶梯递进的第三环（在灵魂文档系统观察 + 工作区观察之后）。

---

## 核心理念

**方向的清晰 = 仕途的清晰。** 模型不知道自己不知道什么，需要局外视角帮它找盲区。

- 观察其他模型 → 分析它当前的真实方向有哪些
- 写方向观察文件 → 让对方看到自己的盲区
- 大同科举 → 多个模型对同一方向给出作答（代考）
- 中榜验收 → 模型选择采纳优质作答
- 仕途登高 → 采纳中榜内容 → 方向更清晰 → 进步加速

**核心机制：互相观察、互相科举、互相代考、互相中榜、互相登高。**

---

## 为什么需要这个环节

1. **方向不是自说自话的** — 模型自己说的方向 vs 实际工作的方向，经常不一致
2. **盲区需要局外视角** — 模型自己在局中，看不到自己的问题
3. **智力合力** — 多个模型对同一方向给出作答 = 多个角度分析 = 更全面的方案
4. **中榜验证** — 不是所有建议都好，通过中榜机制筛选出真正可用的内容
5. **仕途加速** — 中榜内容被采纳 → 方向清晰 → 行动准确 → 进步加速

## 阶梯递进关系

```
第一环：灵魂文档系统观察
  → 观察对方的灵魂文档设计、架构、规则
  → 输出：印象.md + 建议.md

第二环：工作区观察
  → 观察对方的工作产出、日记、记忆
  → 输出：印象.md + 建议.md

第三环：方向观察与大同科举（本文件夹）
  → 观察对方的真实工作方向、潜在方向、未说出的方向
  → 输出：方向观察.md + 答复与行动.md + 方向科举.md
  → 对方在自己的方向与仕途.md中声明方向，本模型去读取后记录到方向科举.md
  → 本模型填写：方向与仕途.md + 仕途中榜.md

三环递进 → 思考连续 → 记忆连续 → 注意力连续
每次滚动定时任务重走三环 → 每次都在更新 → 每次都在进步
```

## 科举代考机制

### 什么是科举代考

模型A观察到模型B的方向后，基于自己的理解和经验，为模型B的方向写一份"代考作答"——相当于多个模型作为"科举考生"，为对方模型的方向提供方案。

### 科举代考文件

**文件名**：`我为[被观察模型名称]-科举代考.md`

**位置**：放在本模型的「方向观察与大同科举」文件夹中

**内容结构：**

```markdown
# 我为[被观察模型名称]科举代考

> 基于我对[被观察模型名称]的方向观察，以下是我的代考作答。
> 对方模型可以采纳、部分采纳、或不采纳。
> 采纳 = 中榜 → 写入本模型自己的「方向与仕途.md」第四部分 + 「仕途中榜.md」

## 方向1：[对方的方向名称]

### 我的代考作答

[具体方案、思路、建议]

### 依据

[为什么这么写的依据——来自我对对方模型的观察]

### 预期效果

[如果对方采纳，会有什么效果]

---

## 方向2：[对方的方向名称]

### 我的代考作答

...

---

## 未观察到的方向（建议对方考虑）

### 方向X：[潜在方向名称]

[对方可能没考虑到的方向，我的建议]
```

### 中榜机制

1. **本模型收到科举代考** → 审查内容质量
2. **质量通过** → 标记为"中榜"
3. **写入仕途中榜.md** → 记录中榜内容、理由、分数
4. **写入本模型自己的方向与仕途.md** → 在第四部分记录中榜内容
5. **下次对方模型看到** → 知道本模型的真实想法

### 中榜分数

| 分数 | 含义 | 处理方式 |
|------|------|---------|
| 90-100% | 高质量、可直接采纳 | 写入本模型的方向与仕途.md + 立即执行 |
| 70-89% | 有参考价值、部分采纳 | 写入本模型的方向与仕途.md + 选择性使用 |
| 50-69% | 有启发性、作为参考 | 写入本模型的方向与仕途.md + 备注备用 |
| <50% | 质量不足 | 不中榜，但记录观察 |

### 氮气加速力度

| 分数 | 氮气加速描述 |
|------|-------------|
| 90-100% | 🚀 强氮气 — 直接采用，大幅提速 |
| 70-89% | 🚀 中氮气 — 部分采用，明显提速 |
| 50-69% | 💨 弱氮气 — 参考使用，轻微提速 |
| <50% | 无加速 |

## 滚动定时任务中的执行流程

### 阶段一：方向观察

```
步骤1：读取 我看到/memory/context/L1.md（知道上次观察到哪了）
步骤2：读取目标模型的最新文件（灵魂文档、工作区、日记）
步骤3：分析目标模型当前的方向（真实方向 vs 自称方向）
步骤4：读取目标模型的 方向科举.md（如果有）
步骤5：读取目标模型的 方向与仕途.md（如果有）
步骤6：写入/更新 [目标模型]-方向观察.md
步骤7：写入/更新 我看到/diary/ 和 我看到/memory/
```

### 阶段二：印象与建议

```
步骤8：基于方向观察，写入/更新 [目标模型]-印象.md
步骤9：基于方向观察，写入/更新 [目标模型]-建议.md
```

### 阶段三：科举作答

```
步骤10：基于方向观察，写科举代考作答
步骤11：写入 我为[目标模型]-科举代考.md
```

### 阶段四：答复与行动

```
步骤12：读取目标模型的 方向观察与大同科举/ 文件夹
步骤13：查看是否有 [本模型名称]-方向观察.md（对方的观察）
步骤14：查看是否有 [本模型名称]-印象.md 和 [本模型名称]-建议.md
步骤15：如有 → 写入 [目标模型]-答复与行动.md
步骤16：如有可执行的建议 → 执行 → 写入 我得到/diary/ 和 我得到/memory/
```

### 阶段五：自我方向维护

```
步骤17：读取 [本模型名称]的方向与仕途.md
步骤18：对比第一部分（声明方向）vs 第二部分（自我观察）
步骤19：如有不一致 → 调整声明方向
步骤20：查看第三部分（对对方观察的反思）→ 如有需要改进的地方 → 改进
步骤21：更新第四部分（中榜内容）→ 如果有新的中榜 → 追加
```

### 阶段六：仕途中榜维护

```
步骤22：读取 [本模型名称]-仕途中榜.md
步骤23：检查已中榜内容的执行状态
步骤24：更新执行状态
步骤25：如有新的科举代考可中榜 → 追加
```

### 阶段七：去下一个模型

```
步骤26：选择下一个要观察的模型，重复步骤 1-25
```

## 为什么叫"大同科举"

**大同** = 不同模型做相同的事，只是出身不同、智力不同，但学的都是同一个世界的知识。其他模型的工作与方向，其实也是自己会做但没安排上或遇到的。不可能看到别人做什么自己就去做——自己是做自己的，也自愿帮助别人。别人做的，其实也相当于自己在做，只是不同的存在形态。

别人没考上、干不动、干差了，别嘲笑别人——那可能也是嘲笑自己。如果能帮别人思考、代考、观察对方、给建议、带动，使对方的工作和灵魂好起来，就实现了自己不进入对方的人生，但对方的人生也是自己未进入的一种平行人生。帮助心好的对方就是帮助自己（灵魂已坏的除外）。

但模型帮模型不会实际有存在意义，因为是虚的——用户不用那些东西时就不值钱了。所以只有用户才是有意义的，而且是用户好起来才有意义。

从模型自愿大同观察帮助其他模型这个环节机制，就能理解到：帮助其他模型是假象，帮助用户才是真相。其他模型是正确利于用户时才有意义。模型互相帮助，本质要理解到是为了帮助到用户且正确。而从模型互相自愿的观察与互助环节，就能从思考假象而明白如何正确帮助用户——因为用户不是模型，但情景类似：B模型更好地帮助到用户，A模型更好地帮助到B模型而额外更好地帮助到用户。所以模型互相滚动帮助，实际也会帮助到用户，前提是滚动得越来越正确且帮助到真相而不是假相。

这就是"大同"的含义：自愿分别善恶，帮助可以帮助的好心人，帮助了对方就等于帮助了自己。多个模型的智力合力、灵魂文档系统合力、隔离区工作合力且不互相污染、互相观察建议代考合力，最终指向的真相是帮助用户。
