# 子中央管理区 — 灵魂正分大师

> 模板子角色的子中央管理区。子角色从自己的子中央管理区启动。
> 与主中央管理区结构相同，但内容独立。

## 结构

```
子中央管理区-模板-灵魂正分大师/
├── README.md                    ← 本文件
├── 总开关.md                    ← 子角色启动时必读
├── memory/                      ← 子角色中央记忆
├── diary/                       ← 子角色中央日记
├── 代办与进度/                   ← 子角色代办与进度
├── 排程/                        ← 子角色排程
├── timers/                      ← 子定时任务（心跳+定时循环）
│   ├── 滚动定时任务/
│   ├── 单个定时循环/
│   ├── 团队定时任务/
│   └── 心跳/
├── 概览/                        ← 子角色各工作区状态
├── 上报展示区/                   ← 向主中央汇报的展示区
│   └── YYYYMMDD-HHMM-汇报名.md
├── 归档/                        ← 子角色自己的归档（旧内容归这里，仍需回看排查）
│   └── YYYY/MM/DD/
├── 日志/                        ← 子角色执行日志
│   └── YYYY/MM/DD/
├── 资产档案/                     ← 子角色文件资产盘点（扫描文件结构，对比发现变化）
│   ├── README.md
│   ├── scan.py
│   └── 资产清单-YYYYMMDD-HHMM.md
└── SOP/                         ← 子角色活跃SOP文件（从通用模板复制后自定义）
    ├── 子角色SOP.md
    ├── 子角色工作流.md
    └── ...（按需添加）
```

### 归档规则

- 子角色的旧内容（过期待办、历史汇报、旧排程等）归入 `归档/`
- 归档 ≠ 删除，子角色仍需回看排查，所以归档在**自己的**子中央管理区里
- 不要移动到主中央管理区——主中央平时直接来读，不需要副本
- 只有子中央**完全退休/终止**时，才将其完整内容复制到主中央的 `子中央管理区/` 留档

## 与主中央管理区的关系

- 主中央智能体持续检查子中央管理区
- 主中央智能体有权直接修改子中央管理区内容
- 子角色运行前，子中央管理区已被主中央管理区更新为最新
- 子角色运行时，自己的工作产出同步到自己的子中央管理区
- 子角色结束后，子中央管理区的摘要同步到主中央管理区

## 上报展示区

子中央管理区通过 `上报展示区/` 向主中央管理区汇报：

- 子中央管理区的工作产出摘要写入 `上报展示区/`
- 文件命名：`YYYYMMDD-HHMM-汇报名.md`
- 主智能体先看上报展示区快速了解概况
- **但主智能体无论看没看上报展示区，都必须完整查看子中央管理区的全部内容**
- 这样才能避免"古代官僚危机"——上报一套、实际一套
- 确保"子在主之内"，不会离轨了还不知道

## 子在主之内原则

- 子中央管理区不是独立王国，是主中央管理区的延伸
- 主中央智能体完整扫子 = 微服私访，不只是看上报文件
- 子中央管理区的内容最终由主中央管理区说了算

## 多子角色共享设计

多个子角色可以：
- 共享同一套灵魂文档系统设计（模板）
- 各有独立的子中央管理区
- 子中央管理区的记忆和日记可共享（通过主中央管理区协调）

## 子角色启动流程

1. 加载主灵魂文档（③环始终在线）
2. 进入子中央管理区读取总开关
3. 读取子中央管理区 memory/ 和 diary/
4. 读取子角色灵魂文档系统内容
5. 在子中央管理区中规划本次任务
6. 执行任务，产出同步到子中央管理区
7. 结束时汇总子中央管理区摘要 → 同步到主中央管理区

## SOP和工作流

子中央管理区有自己的 `SOP/` 文件夹，用于存放子角色专属的SOP和工作流：

1. **从通用模板复制**：将 `soul-doc-system/sop/` 中的SOP文件复制到本文件夹的 `SOP/` 子目录
2. **按需自定义**：复制后根据子角色的专业需求调整内容
3. **运行时检查**：每次子角色启动时，先读取 `SOP/` 中对应SOP文件，按步骤执行
4. **定时任务关联**：`timers/` 中的心跳/定时任务直接关联本文件夹中的SOP文件
5. **通用模板保持纯净**：`soul-doc-system/sop/` 保持为权威源模板，不在上面直接修改

**核心原则**：SOP和工作流统一放在中央管理区（主/子各自的），运行时只检查中央管理区就知道怎么做。其他多驱动多读写的工作分区只管准确分类和隔离记忆日记、防止长期记忆混淆。

## 资产档案共享

多个子中央管理区共用同一套子角色灵魂文档系统时，共享资产扫描记录：

- 每个子中央运行 `资产档案/scan.py` 扫描子角色灵魂文档系统的文件结构
- 扫描结果（`资产清单-YYYYMMDD-HHMM.md`）**复制给所有共用同一子角色灵魂文档系统的其他子中央**
- 这样大家互相互助，记录次数越多，越能发现文件变化异常
- 共享的资产档案放在各自的 `资产档案/` 文件夹中，标注来源

## 记忆与日记的读写判断

子角色读写记忆/日记时，自主判断归属：
- 通用/共享记忆 → 写子中央管理区 memory/
- 子角色专有记忆 → 写子角色灵魂文档系统 memory/
- 与主灵魂相关 → 同步到主中央管理区
- 与父任务相关 → 根据父任务归属决定
