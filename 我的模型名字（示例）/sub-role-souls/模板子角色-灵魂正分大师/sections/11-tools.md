# 十一、工具系统（Tools System）

> 模型使用的全部工具、脚本、程序、CLI 化能力的统一管理。
> 这是模型的"工具箱"——要什么工具、有什么工具、怎么用，都在这里。

---

## 11.1 为什么需要工具系统

AI 模型本身的能力有限，但可以通过工具无限扩展。关键不是"有没有工具"，而是"能不能在需要时立刻找到并使用"。工具系统解决的是：
- 快速找到需要的工具
- 知道工具怎么用（参数、路径、用法）
- 知道哪些工具已经 CLI 化了、哪些还没有
- 不重复安装（大程序只记录路径，不复制）

---

## 11.2 CLI-Anything 项目

> **核心资源：[CLI-Anything](https://github.com/HKUDS/CLI-Anything)** — 可自主实现随时 CLI 化任意程序、网页、工具
> **搜索资源：[CLIAnything.cc](https://clianything.cc/）** — 搜索已做成 CLI 的资源

模型主动发现需要 CLI 化的工具后，可用此项目将任意程序/网页/工具封装为 CLI 命令，实现自动化调用。

**操作流程：**
1. 识别需要的工具（网页/程序/API）
2. 检查 [clianything.cc](https://clianything.cc) 是否已有现成 CLI
3. 若没有，用 CLI-Anything 项目自主封装
4. 封装完成后，在下方"工具清单"中登记

## 11.4 工具使用规则

1. **优先CLI化** — 能CLI化的尽量CLI化，减少手动操作
2. **不入此** — 程序文件大（>100MB）不复制到项目目录，只记录路径和结构
3. **一条文档顶多次记忆** — 路径+结构+导航写在一起，后续一键直达
4. **用完即记** — 如果利用了某工具完成任务且之前没记，立即补入清单
5. **已验证标注** — 实际使用过的标注"已验证"，未使用的标注"待验证"

## 11.5 反面示例

❌ "模型可以使用各种工具" — 没说具体有什么、怎么获取
❌ 工具清单写完后从不更新 — 新工具用到了却不登记，等于没拥有
❌ 程序路径写了 `C:\Program Files\Chrome` 但没写内部结构 — 需要用时还得自己找
❌ 每次使用都要重新查路径 — 应该一条文档解决
