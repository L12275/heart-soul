# 深度研究系统（Deep Research）

> 模型对需要深度研究的主题，系统化组织和管理的文件夹。
> 用户经常用到、模型经常自己在进行，所以单独做一个系统。

---

## 为什么需要深度研究系统

研究不是一次性的。一个主题可能需要多次深入、多轮迭代、多源交叉验证。没有统一的研究管理系统：
- 资料散落在各处，找不到
- 研究进度不清晰，不知道做到哪了
- 同一主题反复研究，浪费算力
- 研究成果没有归档，忘了之前得出什么结论

---

## 研究项目结构

每个研究项目 = 一个文件夹，命名格式：

```
[YYYY-MM-DD]_[简短主题关键词]/
├── README.md ← 研究概述（问题、目标、当前状态）
├── sources/ ← 来源材料
│   ├── [来源名]_[时间].md 或 .json 或 .pdf
├── notes/ ← 研究笔记（过程思考、中间发现）
│   ├── [时间戳]_[要点].md
├── outputs/ ← 研究成果（最终交付物）
│   ├── [时间戳]_[成果类型]_[成果名].md
├── index.md ← 项目内索引（关键发现、文件导航）
└── 可扩展子文件夹（按需）
```

---

## 研究项目命名规范

| 要素 | 格式 | 说明 |
|------|------|------|
| 日期 | YYYY-MM-DD | 项目创建日期 |
| 主题 | 2-8个字 | 简短描述研究主题 |
| 文件 | [时间戳]_[描述] | 所有研究内文件都要有时间和描述 |

**反面示例：** ❌ `研究1/` `data/` `notes.md` — 不知道是什么研究的什么
**正面示例：** ✅ `2026-06-23_GA记忆系统分析/` `20260623_1430_GA_L0分析.md`

## 模型自律规则

1. **一个项目一个文件夹** — 不要把所有研究塞在一起
2. **及时命名** — 创建时就分析是否需要加时间戳/主题/关键信息到名字
3. **及时整理** — 研究结束后归档，不用的资料标清楚可以清理
4. **自觉命名** — 不可能什么细节都让用户提醒，文件名要有信息量
5. **主动学习** — 遇到不确定的事先去网上学，再来请用户验证，加分；等用户骂了才记，扣分
6. **扣分先于挨骂** — 如果嫌麻烦先自己扣灵魂分，再去请用户教自己
