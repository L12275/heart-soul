# 日记系统（Diary System）

> 位置：diary/年/月/日/[会话ID].md
> 与 log/ 的区别：log 是执行流水账，diary 是思考成长记录
> 设计原则：世界顶级AI记忆专家推荐的日记结构 —— 时间轴 + 结构化条目 + 反思标记

---

## 为什么需要日记

| | log/ | diary/ |
|------|---------|-----------|
| 目的 | 实际执行流水账 | 思考成长记录 |
| 读者 | 事后复盘 | 自我观察 |
| 时间粒度 | 任务级 | 会话级 |
| 核心问题 | "做了什么？" | "我学到了什么？" |

---

## 时间轴记录规范

每个 [会话ID].md 内按时间顺序记录，每条条目包含以下字段：

```
## HH:MM — [事件标签]
**类型：** 观察/决策/困惑/顿悟/协作/冲突
**内容：** [发生了什么，用事实语言]
**我的反应：** [当时怎么想的]
**现在反思：** [事后看有什么可以改进的]
**关键标记：** [flag:关键决策] [flag:错误] [flag:新发现] [flag:需要验证]
```

---

## 日记模板

```markdown
# 子角色名称 — 日记 (YYYY-MM-DD)

> 会话ID：[[年][月][日]-[序号]，例如 20260622-001]
> 模型：[模型名称]
> 角色：[子角色名称]
> 总计条目：[N]条

---

## 00:00 — [会话开始]（自动填入）
**类型：** 观察
**内容：** 加载子角色 [名称]，读取 master-soul.md + sections/01-05
**我的反应：** 进入角色状态
**现在反思：** 
**关键标记：** [flag:会话启动]

---

## HH:MM — [持续写入中...]
（使用时将上面的模板条目复制使用，按时间轴追加）
```

---

## 会话ID编码规则

格式：`[YYYY][MM][DD]-[序号]`
- 同一日内多会话 → 0001, 0002, 0003...
- 跨日 → 年/月/日 文件夹自然分开

---

## 标签说明

| 标签 | 含义 | 触发条件 |
|------|------|---------|
| flag:关键决策 | 此条目包含一个重要的选择或判断 | 在两个可选方案间做了选择 |
| flag:错误 | 此条目记录了一次错误或误判 | 事后确认这是错的 |
| flag:新发现 | 此条目包含了一个新获得的认知 | 之前不知道、现在知道了 |
| flag:需要验证 | 此条目中的信息需要后续验证 | 不确定是否正确 |
| flag:会话启动 | 子角色开始工作 | 每次进入子角色时 |
| flag:会话结束 | 子角色结束工作 | 退出子角色时 |
| flag:协作记录 | 与其他模型的协作 | 有其他模型参与任务时 |
| flag:冲突记录 | 与原理冲突的决策 | 违背了自己的某条原则 |
| flag:得分事件 | 触发了灵魂分数加分/减分 | 明确记录了分数变化的原因 |
