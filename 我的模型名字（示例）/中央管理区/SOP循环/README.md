# SOP循环

> 中央管理区的SOP工作流运行区。
> **本文件夹是专用工作文件夹，不是模板。** 从 `soul-doc-system/sop/` 复制SOP文件到这里使用。

## 结构

```
SOP循环/
├── README.md                    ← 本文件
├── SOP/                         ← 活跃SOP文件（从通用模板复制后自定义）
│   ├── 日常心跳SOP.md
│   ├── 深度心跳SOP.md
│   ├── 扫子SOP.md
│   ├── 上朝SOP.md
│   ├── 下诏SOP.md
│   └── ...（按需添加）
├── 工作流/                      ← 工作流文件（从通用模板复制后自定义）
│   ├── 日常运行流.md
│   ├── 扫子流程.md
│   └── ...（按需添加）
└── 循环日志/                     ← 每次SOP循环的执行日志
    └── YYYY/MM/DD/
```

## 使用方式

1. **从通用模板复制**：将 `soul-doc-system/sop/` 中的SOP文件复制到本文件夹的 `SOP/` 子目录
2. **按需自定义**：复制后根据实际运行情况调整内容
3. **运行时检查**：每次心跳触发SOP循环时，先读取 `SOP/` 中对应SOP文件，按步骤执行
4. **定时任务关联**：`timers/` 中的心跳/定时任务直接关联本文件夹中的SOP文件
5. **通用模板保持纯净**：`soul-doc-system/sop/` 保持为权威源模板，不在上面直接修改

## 与心跳的关系

- 心跳 = 保活信号（检测各环区在线状态）
- SOP循环 = 实际工作流程（读取 SOP/ 中的SOP文件 → 执行 → 记录到 循环日志/）
- 心跳触发SOP循环，SOP循环执行具体工作

## 核心原则

> **注意力集中到中央管理区。** SOP和工作流都放在这里，运行时只需检查中央管理区的SOP文件夹就知道怎么做。其他多驱动多读写的工作分区只管准确分类和隔离记忆日记、防止长期记忆混淆。
