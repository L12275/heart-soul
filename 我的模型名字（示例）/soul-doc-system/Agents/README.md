# 智能体（Agents）

> **功能模块：智能体 = 第七环延伸。**
> 模板存放区存储成品智能体配置，运行区复制配置并使用。
>
> **前置思考系统：使用智能体前必须先走 `世界最强脑师-前置思考系统`，确认这个智能体配置是最佳机会方案。**
> **执行边界：默认执行（直接使用）/ 边做边解释（使用+说明）/ 只思考不执行（分析配置但不启动）**

## 各区域说明

### 模板存放区

**作用**：存储经过验证的、可复用的智能体配置模板。

**谁写**：本模型（或共产区共享）
**谁读**：本模型（或所有模型）

**内容规则**：
- 模板必须是**已经验证过**的配置，不是实验性的
- 每个模板有 README.md 说明：用途、参数、适用场景
- 配置文件（.json）包含完整的智能体定义
- 新增模板前必须经过验证 → 模板化 → 存入模板存放区

### 运行区

**作用**：实际运行智能体的工作空间。

**使用流程**：
1. 从 `模板存放区/` 复制需要的模板到 `运行区/活跃/[智能体名称]/`
2. 在副本中根据需要微调参数
3. 启动智能体运行
4. 运行数据存入 `运行区/活跃/[智能体名称]/运行数据/`
5. 智能体停用后 → 移动到 `运行区/历史/`

**关键规则**：
- 运行区的内容是模板的**副本**，不是模板本身
- 模板存放区的内容**不直接修改**——修改在运行区的副本中进行
- 如果运行区的修改有价值 → 提炼为新模板 → 存入模板存放区
- 运行区的内容可以随时清理（历史区归档后）

## 与通用功能模块拼搭SOP的关系

| 场景 | 操作 |
|------|------|
| 新建智能体工作区 | 从 `Agents/模板存放区/` 复制模板到 `Agents/运行区/活跃/[名称]/` |
| 需要隔离的智能体 | 只复制需要的模板模块，避免信息污染 |
| 相同组合的多个智能体 | 复制第一个智能体的运行区副本，微调差异 |
| 模板更新 | 模板存放区更新后，运行区**不自动更新**，需要手动同步或重新复制 |

## 六环联动中的位置

| 环 | 智能体的角色 |
|----|------------|
| ③ 主灵魂中心 | 智能体的配置来自主灵魂文档 |
| ⑥ 子角色中心 | 智能体 = ⑥环下的子角色执行单元 |
| ⑦ 团队协作环 | 多个智能体 = 团队协作 |
| ④ 生产空间中心 | 智能体的产出物归生产空间 |

> **智能体不是独立环，是⑥环（子角色中心）下的执行单元，也是⑦环（团队协作）的基础组件。**

## 铁律

1. **模板不可直接运行** — 模板存放区只存模板，运行区复制使用
2. **运行区副本可修改** — 在运行区的副本中微调，不影响模板
3. **有价值的修改回传模板** — 运行区的改进经验提炼为新模板
4. **前置思考先行** — 使用智能体前必须先走最强脑师前置思考
5. **三模式执行** — 默认执行 / 边做边解释 / 只思考不执行
