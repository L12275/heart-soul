# Agents — 通用功能模块模板

> **本文件夹是模板，不是存储。**
> 只存放系统级内容（模板/标准/结构），不写入日常杂七杂八的内容。
> 持续在源头更新优化，需要的多驱动多读写工作专区复制最新版拼搭使用。

## 文件夹结构

```
Agents/
├── README.md          ← 本文件
├── templates/         ← Agent 定义模板（灵魂文档级）
├── registry/          ← Agent 注册表（谁存在、什么角色、什么状态）
└── memory/            ← Agent 运行记忆（对话历史、决策记录）
```

## templates/ — Agent 定义模板

每个 Agent 的完整定义文件，包含：

```markdown
# [Agent 名称] 身份档案

## 基本信息
- **名字**: [显示名]
- **Profile名**: [调度用小写英文名]
- **角色**: [在团队中的定位]
- **职责**: [一句话描述]

## 性格特质
- [特质1]: [具体描述]
- [特质2]: [具体描述]

## 沟通风格
- [描述]

## 核心问题
- [工作中反复问自己的关键问题]

## 工具权限
**允许**: [工具列表]
**禁止**: [工具列表]

## 决策逻辑
- [遇到分歧时如何做决定]

## 禁忌
- [绝对不会做的事]
```

## registry/ — Agent 注册表

记录所有已创建的 Agent：

| Profile名 | 显示名 | 角色 | 状态 | 创建时间 | 最后活跃 |
|----------|--------|------|------|---------|---------|
| [示例] orchestrator | 编排师 | 团队协调者 | 活跃 | 2026-06-25 | 2026-06-25 |

## memory/ — Agent 运行记忆

每个 Agent 的记忆文件：
```
memory/
├── [profile名]/
│   ├── context.md     ← 当前工作上下文
│   ├── decisions.md   ← 决策记录
│   └── history.md     ← 历史对话摘要
```

## 使用方式

1. 需要 Agents 功能模块时 → 复制整个 `Agents/` 文件夹到目标位置
2. 在 `templates/` 下创建新 Agent 定义
3. 在 `registry/` 中注册新 Agent
4. 运行时在 `memory/` 下维护各自记忆
