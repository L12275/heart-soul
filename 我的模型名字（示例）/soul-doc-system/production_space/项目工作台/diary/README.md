# 项目日记

> 基于超级信息SOP的小时级时间轴记录。
> 日记 = 实时事件过程记录（小时级推导链）+ 基于超级信息SOP的思考写法
> 记忆 = 验证确认后的正确记忆（从日记中提炼）

---

## 日记铁律

1. **今日事今日毕** — 当天的问题当天在日记里解决，写成正确解
2. **记录推导链** — 每个决策的原因和过程都要记录，不能只记结果
3. **排除的错误也记录** — 让下次知道哪些路不用走
4. **边记边解题** — 遇到问题先在日记里推导，用超级信息SOP找到正确答案
5. **不能把错的当对的写**

## 小时级时间轴格式

日记文件路径：`diary/年/月/日/[日期].md`

每天一个文件，文件内按**小时级时间轴**记录：

```markdown
# YYYY-MM-DD 日记

> 基于超级信息SOP的小时级时间轴记录
> 每条记录都包含：事件 → 判断 → 验证 → 正确解 → 收获

## 时间轴事件

### HH:MM — [事件标题]
**类型：** 工作/决策/发现/问题/学习
**详情：** [发生了什么，展开的细节]
**判断：** [当时的判断]
**验证：** [怎么验证的]
**解题过程：**
- 前提：[证据/来源]
- 推导：[推理过程]
- 排除：[排除了什么错误路线，为什么]
- 正确解：[最终做法，为什么对]
**收获：** [可复用的经验/SOP/工作流更新]
**后续：** [下一步]

## 今日总结
- 完成：...
- 遇到问题：...
- 正确解：...
- 排除的错误：...
- 收获取（可复用的）：...
- 明日计划：...

## 记忆提炼
> 从今日日记中提炼的验证后记忆（将进入 memory/）

- 事实：[已验证的结论]
- SOP：[新提炼或更新的SOP]
- 工作流：[新发现或优化的工作流]
- 经验：[可复用的经验]
```

## 日记与记忆的分工

| 类型 | 内容 | 准确性 | 格式 |
|------|------|--------|------|
| 日记（diary/） | 实时事件过程 + 解题思考 | 当日完成的正确解 | 小时级时间轴 + 推导链 |
| 记忆（memory/） | 验证确认后的正确记忆 | 已验证 | L0-L4层级 |
