# 产物文件夹管理规则

> 产物 = 工作台的最终/阶段性产出物。
> 超迆60天未使用，可能是不用了。

---

## 目录说明

| 字段 | 说明 |
|------|------|
| 文件名 | 最终文件名 |
| 来源 | 从 temp/ 还是直接创建 |
| 所属工作 | 关联的工作台事件/项目 |
| 文件出现时间 | 产出物的日期 |
| 完成日期 | 完成的日期 |
| 版本 | 版本号或迭代标记 |
| 后续状态 | 在用 / 已归档 / 待清理 |

## 管理规则

### 必须记录

1. **必须记录** — 从 temp/ 移入 product/ 前，必须有来源和用途记录
2. **分类存放** — 按项目/任务/类型分文件夹，不要堆在根目录
3. **命名规范** — 有信息量：日期+类型+描述+版本（如 `2026-06-23_代码_用户模块_v1.py`）

### 清理规则

1. **超时检查** — 超迆 60 天未使用/引用（以文件出现时间为起计）→ 提示确认
2. **已归档** — 项目结束后 → 移到 archive/（不删除）
3. **重复版本** — 被新版本替代后 → 旧版本移至 archive/
4. **用户确认** — 用户说“不要”或者超迆 60 天无响应 → 可删除
