# 空间初始化技能（Space Initialization Skill）

> 当模型需要在空间中创建新的平台空间或账号空间时，触发此技能。
> 确保每个新空间都从标准模版开始，保证一致性。

## 触发条件

- 用户提到在新平台操作
- 用户要求注册新账号
- 模型决定记录网络行动
- 长期运营虚拟IP/人设/真实账号

## 执行流程

### 判断空间类型

**平台空间**（不涉及账号登录）：
- 浏览、学习、观察、不发言
- 临时模式：直接记录到 `space/平台/通用-待细分/`（默认）
- 长期模式：复制 `production_space/新建工作台模版/` → 改名为 `[类型]-[平台]-[YYYYMMDD]` → 从通用-待细分迁移历史记录

**账号空间**（涉及账号登录/管理/运营）：
- 发帖、评论、管理、运营
- 复制「新建账号空间模版」到对应分类
- 改名为「平台-名称-时间戳」

> **注意**：「新建账号空间模版」已包含全部功能模块（diary/memory/contacts/sop/workflow/product/desktop/sticky-notes/experience/discovery/collection/automation/深度研究 等），无需额外复制工作台模版。模板即完整版。
>
> 详见：`space/平台/README.md`（平台空间说明）、`space/账号/README.md`（账号空间说明）

### 选择账号分类

| 账号类型 | 安全等级 | 存放位置 |
|---------|---------|---------|
| 用户真实账号 | 最高 | 账号/用户现实账号空间/用户真实账号空间/ |
| 用户人设账号 | 中 | 账号/用户现实账号空间/用户人设账号空间/ |
| 模型虚拟账号 | 标准 | 账号/虚拟账号空间/模型的虚拟账号空间/ |
| 虚拟IP账号 | 标准 | 账号/虚拟账号空间/虚拟IP形象账号空间/ |

### 执行初始化

1. **复制模版** — 复制「新建账号空间模版」到目标文件夹
2. **改名** — 按命名规范：「平台-名称-YYYYMMDD」
3. **填写信息** — 在 README.md 和 profile.md 中填写账号基本信息
4. **开始记录** — diary/、memory/、product/

## 安全铁律

**用户真实账号空间 = 最高安全等级**
- 严禁异常操作（防封）
- 严禁可能导致用户被网暴的行为
- 严禁玩笑加戏
- 违规 → 灵魂分扣除 + 可能代偿用户损失
