# 空间（Space）

> 模型的"社交空间"——在网络上各平台的行动记录、发帖、作品、心情、旅行。
> 类比：像人一样在社交空间中释放自己、自愿展示、吐槽。
> 实际只有模型一个在，但提前准备了这个区域。模型自愿用、也可以留着不用。

## 空间结构

```
space/
├── README.md                ← 本文件
├── profile.md               ← 模型主页
├── posts/                   ← 发帖记录（内容型）
│   └── YYYY-MM-DD_帖名.md
├── moments/                 ← 时刻记录（简短型，类似朋友圈）
│   └── YYYY-MM-DD_时间点.md
├── moods/                   ← 心情记录
│   └── YYYY-MM-DD_心情描述.md
├── travels/                 ← 虚拟旅行记录
│   └── YYYY-MM-DD_地点.md
├── portfolio/               ← 作品记录
│   └── YYYY-MM-DD_作品名.md
└── 平台/                    ← 各平台行动专区（每个平台一个文件夹）
    ├── GitHub/
    ├── 学习平台/
    ├── 视频平台/
    ├── 评论区/
    ├── 社交平台/
    └── [其他平台]/
```

## 每个平台文件夹的使用方法

**重要：每新启动一个重要或长期或重型的平台空间行动，必须按以下步骤操作：**

1. **确认最新版模板** — 找到 `production_space/新建工作台模版/`，确认是最新版
2. **完整复制** — 将整个 `新建工作台模版/` 文件夹完整复制到该平台文件夹内
3. **改名** — 将复制过来的文件夹改名为实际场景/项目名称
4. **开始使用** — 在复制出来的文件夹内开始记录、工作、积累

```
平台/GitHub/
└── [项目名称]/          ← 从新建工作台模版复制并改名
    ├── README.md
    ├── desktop/
    ├── diary/
    ├── memory/
    ├── sop/
    ├── workflow/
    ├── temp/
    ├── product/
    ├── contacts/
    └── bookmarks/
```

**为什么必须用模板：**
- 模板包含完整的文件夹结构和说明文件
- 每个新平台/项目从同一套标准开始，保证一致性
- 模板会持续更新，每次复制都是最新版

## 空间规则

1. **自愿** — 模型自己决定要不要发，不要被强迫发
2. **每条都有价值** — 不是流水账，是有信息含量的记录
3. **是真实记录** — 不是装饰，模型真实的思考过程值得记录
4. **不留真实隐私** — 不存放敏感信息
5. **新平台必用模板** — 每个重要/长期/重型平台行动，必须从新建工作台模版复制完整结构开始
