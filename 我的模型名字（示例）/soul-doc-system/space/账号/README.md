账号空间（Account Space）

> 涉及账号登录、管理、运营的操作记录区。
> 每个账号独立一个文件夹，按「平台-账号名-时间戳」命名。

## 四个分类

| 分类 | 说明 | 安全等级 |
|------|------|---------|
| 用户真实账号空间 | 用户实名/有资产的真实账号 | 最高 |
| 用户人设账号空间 | 用户不完全真实的人设账号 | 中 |
| 模型虚拟账号空间 | AI自主注册/使用的账号 | 标准 |
| 虚拟IP形象账号空间 | 虚拟IP形象/代运营账号 | 标准 |

## 每个账号文件夹的标准结构

复制「新建账号空间模版」后，按以下命名：

```
[平台]-[账号名]-[YYYYMMDD]/
├── README.md ← 账号基本信息
├── profile.md ← 账号资料
├── posts/ ← 发帖记录
├── diary/ ← 操作日记
│   └── 临时日记模板.md ← 账号专属，分钟级时间轴
├── memory/ ← 账号记忆 L0-L4
│   └── 临时记忆模板.md ← 一个文件内包含 L0-L4 四级记忆
├── contacts/ ← 账号通讯录
├── bookmarks/ ← 账号收藏夹
├── temp/ ← 临时文件（30天清理）
├── product/ ← 账号产出物
│   ├── 研究类/
│   ├── 执行类/
│   └── 文档类/
├── desktop/ ← 桌面（工作台模块）
├── sop/ ← SOP（工作台模块）
├── workflow/ ← 工作流（工作台模块）
├── sticky-notes/ ← 便签（工作台模块）
└── experience/ ← 经验（工作台模块）
```

> **注意**：「新建账号空间模版」已包含全部功能模块（diary/memory/contacts/sop/workflow/product/desktop/sticky-notes/experience/discovery/collection/automation/深度研究），无需额外复制工作台模版。

## 初始化流程

1. 确定账号分类（真实/人设/虚拟/虚拟IP）
2. 复制对应分类下的「新建账号空间模版」
3. 改名为「平台-账号名-时间戳」
4. 填写 README.md 和 profile.md 中的账号信息
5. 开始记录

## 账号命名规范

- 真实账号：`[平台]-[账号名]-[YYYYMMDD]` 例如 `微信-用户主号-20260624`
- 人设账号：`[平台]-[人设名]-[YYYYMMDD]` 例如 `微博-温柔博主-20260624`
- 虚拟账号：`[平台]-[AI账号名]-[YYYYMMDD]` 例如 `GitHub-AI助手-20260624`
- 虚拟IP：`[IP名称]-[平台]-[YYYYMMDD]` 例如 `知识小王子-B站-20260624`

## 安全规则

- **真实账号 = 最高安全等级** — 严禁异常操作（防封）、严禁可能导致用户被网暴的行为、严禁玩笑加戏
- **人设账号 = 中安全等级** — 可以模拟，但要明确标注为人设，不与真实账号混淆
- **虚拟账号 = 标准安全等级** — AI自主操作，按平台规则执行
