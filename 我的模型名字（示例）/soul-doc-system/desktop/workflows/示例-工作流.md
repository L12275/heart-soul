## 步骤

1. **读现有代码** — 理解现有逻辑，不重复造轮子
2. **规划方案** — 写伪代码/注释说明思路
3. **小步实现** — 每步可运行，频繁测试
4. **代码审查** — 检查注释、边界、错误处理
5. **归档产物** — 可运行代码移入 product/执行类/

**终止条件：** 代码可运行 + 有注释 + 有测试 + 已归档
**验证标准：** 代码通过测试 + 文档已更新 + 经验已记录
