# 团队协作专区（第六环：团队协作环）

> 当模型启动子代理（team agent）协作时使用此区域。
> 六环架构中的第六环，编排驱动。

## 团队 vs 子角色

| 对比 | 子角色 | 团队 |
|------|--------|------|
| 数量 | 单个模型内的多个角色 | 多个AI子代理并行 |
| 隔离 | 同一模型内的角色隔离 | 不同子代理的完全隔离 |
| 编号 | 无编号（按角色名） | 独占编号（像进程端口） |
| 协作 | 临时切换，不并行 | 可并行，编排师调度 |
| 适用 | 专业深度任务 | 复杂多任务协作 |

## 两个子区域

### 长期agent协作区

类似账号空间的长期运营模式。
用于长期存在的agent协作关系，有完整工作台结构。

```
长期agent协作区/
├── 编排师/ ← 主编排师工作台（完整模块拼搭）
│   ├── master-soul.md
│   ├── memory/ diary/ log/ sop/ workflow/ product/
│   ├── contacts/ bookmarks/ temp/ sticky-notes/
│   ├── experience/ discovery/ collection/ automation/
│   ├── desktop/ 深度研究/ 灵魂分数/
├── 子代理-01/ ← 独占编号
├── 子代理-02/
└── ...
```

### 会话临时子代理区

类似平台空间的临时浏览模式。
用于单次会话中的临时子代理，不需要长期保存。

```
会话临时子代理区/
├── 通用-待细分/ ← 临时记录（不确定是否长期保存）
└── {时间戳}-{任务名}/ ← 确认需要长期保存的临时代理
```

## 创建规则

1. 新项目 → 在 `团队/` 下决定是长期协作区还是临时子代理区
2. 编排师工作台 → 模块拼搭（按需选择 soul-doc-system 根目录模块组合）
3. 子代理 → 复制「新建工作台模版」或按需拼搭
4. 编号独占：同一区域内编号不能相同；跨区域也不应相同
