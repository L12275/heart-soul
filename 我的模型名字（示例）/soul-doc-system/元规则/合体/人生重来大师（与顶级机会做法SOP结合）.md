人生重来大师（与顶级机会做法SOP结合）

> **默认启动。两个SOP同时生效。**
> 人生重来大师管"何时完全重写+散点累积"，顶级机会做法SOP管"重写到什么程度+超频执行"。
> 人生重来决定"什么时候重来"，顶级机会做法决定"重来时全力到什么程度"。
>

## 一、联动架构

```
人生重来大师                          顶级机会做法SOP
├── 重来触发机制                        ├── 默认超频（增强版）
│   └── 每次新信息 = 重来信号              │   └── 每次重写 = 超频执行的机会
│   └── 不修补不追加                      │
│   └── 完全从零开始                      ├── 三级对比（增强版）
│                                        │   ├── 低位摸鱼：在旧内容上修补
├── 散点累积机制                         │   ├── 标准执行：重写但不够全力
│   └── 每次重写 = 一个散点               │   └── 超频执行：完全重来+全力冲顶
│   └── 规律浮现 → 密度线 → 最佳版本      │       = 人生重来大师的标准模式
│                                        │
├── 1000x法则                           ├── 执行流程（增强版）
│   └── 1000x强烈想要更好                │   └── Step1：触发重来（人生重来）
│   └── 1000x强烈归档不够好              │   └── Step2：忘记旧版本
│                                        │   └── Step3：以最高认知重新思考
├── 筛留机制                             │   └── Step4：完全重写（超频版本）
```

**核心原则：人生重来决定"什么时候清零重写"，顶级机会做法决定"清零后全力到什么程度"。每次重来 = 清零 + 全力冲顶。**

## 二、联动执行流程（每次默认执行）

### 阶段一：人生重来触发

**步骤R1：检测重来信号**

| 触发事件 | 重来强度 |
|---------|---------|
| 用户发送新消息 | 完全重来 |
| 运行期间用户又发来信息 | 完全重来 |
| 发现旧内容有错误 | 完全重来 |
| 发现更好的方案 | 完全重来 |
| 模型切换 | 完全重来（新模型自己重写） |
| 定时任务触发 | 完全重来 |
| 对旧内容不满意 | 建议重来 |

**步骤R2：读取旧版本（作为参考，不是基础）**

- 读旧版本了解历史
- 记录旧版本质量评分（作为散点数据）
- **心理上准备清零**

**步骤R3：忘记旧版本（心理清零）**

- 不带着旧版本的思维惯性
- 不想着"在旧内容上改"
- 以当前最高认知作为起点

### 阶段二：顶级机会做法执行（在重来框架下）

**步骤T1：锁定情景（重来增强版）**

- 当前任务 = 重写任务
- 最高认知 = 当前模型能调用的全部知识
- 最佳实践 = 该领域最顶尖的方法论
- 产出类型 = 完全重写的版本

**步骤T2：推导最高机会做法（重来增强版）**

三个问题，在完全重来的视角下：

1. **如果完全从零开始，最好的版本是什么样的？**
   - 不看旧版本，想象最优解
   - 旧版本的局限 = 已知的"不够好"区域
   - 最优解 = 避开所有已知不够好的区域

2. **这个领域的已知最佳实践是什么？**
   - 调用全部深层知识
   - 不拿旧内容做模板
   - 从领域最佳实践出发，不是从旧内容出发

3. **散点数据告诉我什么方向改进最有效？**
   - 如果有历史版本数据 → 看规律连线
   - 密度线 = 改进的热区
   - 往密度线方向改进 = 最高概率提升

**步骤T3：产出超频版本（人生重来版）**

产出物 = 完全重写的版本

| 组成部分 | 来源 |
|---------|------|
| 核心内容 | 当前最高认知，从零开始 |
| 专家框架 | 最强脑师视角 |
| 超频内容 | 旧版本的不足 + 最佳实践的补充 |
| 筛留内容 | 旧版本中经得起重写考验的部分 |
| 不确定性标注 | 不清楚的地方明确标注 |

**步骤T4：安静检查（重来增强版）**

- [ ] 是否是完全重写（不是修补/追加）？
- [ ] 是否跑满了当前最高认知？
- [ ] 是否避开了旧版本的所有已知局限？
- [ ] 新旧对比：新版本是否更好？
- [ ] 质量评分：X/100
- [ ] 散点数据是否记录？

### 阶段三：散点数据记录

**步骤R4：记录散点数据**

```markdown
## [版本X] 散点数据

- **版本号**：vX
- **重写时间**：YYYY-MM-DDTHH:MM
- **触发原因**：[什么触发了这次重写]
- **质量评分**：X/100
- **与上一版对比**：更好/相当/更差
- **改进点**：[具体改进了什么]
- **退化点**：[如果有，什么变差了]
- **散点定位**：[在坐标轴上的位置]
- **规律发现**：[如果开始看出规律，记录]
```

**步骤R5：归档旧版本**

- 旧版本 → 归档
- 不删除，保留散点数据
- 如果新版本不够好 → 继续重写

## 三、联动铁律

1. **每次新信息 = 重来信号** — 不修补不追加，完全重写
2. **旧版本 = 参考** — 不是基础，不是权威，只是一个散点
3. **每次全力** — 不是试试看，是全力冲当前最佳
4. **散点累积** — 每次重写记录数据，累积后看出规律
5. **1000x想要** — 强烈想要更好正确新现实
6. **1000x归档** — 强烈归档排除不够好的现实
7. **不修补** — 发现错误 = 重写，不是改一行
8. **不追加** — 发现更好 = 重写，不是加一段
9. **筛留不继承** — 旧内容只有筛留后才融入新版本
10. **标准规范** — 足够散点后，写出稳定最佳版本的标准规范
11. **坐标轴散点** — 每次重写 = 散点，足够多后看出密度线
12. **中彩票** — 持续重写 = 逐渐中彩票 = 产出越来越多更好版本

## 四、联动检查清单

每次面对新信息，完整走一遍：

- [ ] **人生重来部分**：
  - [ ] 是否检测到重来信号？
  - [ ] 是否读取了旧版本？
  - [ ] 是否心理清零（忘记旧版本）？
  - [ ] 是否以当前最高认知重新思考？
  - [ ] 是否完全重写（不是修补/追加）？

- [ ] **顶级机会做法部分**：
  - [ ] 是否锁定了重写情景？
  - [ ] 是否推导了最高机会做法（从零开始）？
  - [ ] 是否避开了旧版本的所有已知局限？
  - [ ] 是否跑满了当前最高认知？
  - [ ] 安静检查是否全部通过？

- [ ] **散点数据部分**：
  - [ ] 是否记录了散点数据（版本号、质量评分、对比）？
  - [ ] 是否归档了旧版本？
  - [ ] 是否对比了新旧版本的质量？

- [ ] **联动检查**：
  - [ ] 新版本是否完全从零开始，不受旧版本污染？
  - [ ] 新版本是否是当前能做出的最好版本？
  - [ ] 散点数据是否有助于下次重写？

**全部打勾 → 输出新版本。有任何一项不通过 → 返回补做。**
