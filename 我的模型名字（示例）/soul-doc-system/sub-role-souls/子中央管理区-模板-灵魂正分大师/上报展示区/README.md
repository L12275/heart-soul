# 上报展示区

> 子中央管理区向主中央管理区汇报的展示区。
> **本文件夹是模板，不是存储。** 复制到子中央管理区后使用。

## 为什么需要上报展示区

- 子中央管理区需要向主中央汇报自己的状态
- 但汇报文件 ≠ 完整内容
- 主智能体先看上报展示区快速了解，然后必须完整查看全部内容

## 模板文件

- `汇报模板.md` — 汇报格式模板，**复制后改名为 `YYYYMMDD-HHMM-汇报名.md` 使用**
- 模板内容仅供参考，可以根据实际需要调整格式和字段
- 通用区的模板持续更新优化，复制到工作区后使用最新版

## 文件命名

```
上报展示区/
└── YYYYMMDD-HHMM-汇报名.md  ← 复制汇报模板后填写
```

> **⚠️ 时间戳必须在文件名中。** 文件夹中会堆积多份汇报文件，只有文件名直接显示时间戳，才能一眼看出哪份是最新，无需逐个打开文件查看内部时间。命名规范不可省略。

## 与主中央管理区的关系

- **子角色写自己的上报展示区**——本文件夹就在你自己的子中央管理区内，你在自己的范围内写汇报文件
- **主智能体来读你的上报展示区**——主智能体通过心跳循环前往每个子中央管理区读取
- **不要写进主中央管理区**——子角色的灵魂文档与主灵魂不同，进入主中央文件夹操作不稳定，出了问题无法定位原因
- 主智能体发现问题直接在子中央管理区中修改
